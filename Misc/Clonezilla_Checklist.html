Backing up a test machine using Clonezilla.

  • Install an Operating System:

    • If you need a different operating system than the currently installed OS, a backup image is not available, or no OS is present. Otherwise skip to the next section.
      • Ensure that a backup of the current OS is present and create an updated image if possible.
      • Boot the machine from the OS installation media.
      • Follow the on screen instructions to reformat the OS partition (typically C:, the size should be ~20 or 40 GB) and install the desired OS.

  • Update the Operating System:

    • Install the latest service pack for your OS. Service packs can be found on the network at \\jake\applications\Service Packs
    • Install the latest OS updates ie. Windows Update. Note: the previous link only works in Internet Explorer
    • Rerun Windows Update until no updates are available. Check in on the machine occasionally during updates, as some have EULA's or other boxes to click through that will hold up the entire process. Internet Explorer installation boxes like to hide behind your other windows.

  • Windows Settings:

    • Set the password for the Administrator account to the default password for test machines if it is not already configured.
    • While logged in as "Administrator:"
      • Press ctrl+alt+del and select Change password...
      • Follow the on screen instructions.
    • Right click My Computer and select Properties. Alternatively, go to Control Panel -> System
      • Under Computer Name click Change.
      • Set the radio button under Member of to "Domain" and enter "extract.local" in the edit box.
      • Switch to the System Restore/System Protection tab and disable restore points for all drives.
      • Switch to the Automatic Updates tab (XP) or click Windows Update under See Also in System, then click Change Settings (Vista). Turn updates off or set them to manual installation.
      • Under Remote, turn on remote desktop (choose "Allow users running any version of Remote Desktop" in Vista, or XP machines might not be able to connect).
    • Start Menu -> Control Panel -> Windows Firewall
      • Turn off/disable Windows Firewall.
    • Right click on the desktop and select Properties... or Personalize.
      • Under Screen Saver -> Power...
      • Disable sleep/hibernation. Disable powering off the hard drives. Set any perfomance profile option to "High performance." Power saving modes can prevent users from making remote desktop connections. Your OS my have one or more of those options.
    • Right click My Computer and select Magage.
      • Expand System Tools -> Local Users and Groups and select Groups.
      • Right click the Administrators group. Select Add to Group.... Click Add...
      • Enter "Domain Users" into the edit box. You may be prompted for your domain username and password.
    • Open Windows Explorer/My Computer
      • For each local hard drive listed (typically 'C:' and 'D:'):
      • Right click the drive and click Properties. Uncheck the drive indexing box.
    • On Vista:
      • Right click the sidebar icon in the notification area of the taskbar. Click exit. Be sure to uncheck the "start sidebar with Windows" checkbox.

  • Create backup disk image:

    • Create a "Clonezilla" folder on the machines data partition (typically 'D:'). This must be a different partition than the one containiong the OS.
    • Insert a Clonezilla disc into the machine and boot to it. Follow the onscreen instructions to create a backup of the partition containnig the operating system. Name your backup using the following format: OSbits_Date eg: Server2008x64_08-27-2009
    • See http://clonezilla.org/ for Clonezilla info/downloads.
