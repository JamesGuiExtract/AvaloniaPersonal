Value Modifier Exercises

  1. Regular Expression Value Modifier

    • Open FileNumber_VAC.rsd and run it on FileNumbers.txt.
    • Add Regular Expression value modifiers to the "Value after clue rule" value finder to accomplish the following:
      • Select only attributes that contain a zero.
        • Hint: [\s\S]*0 will capture all characters up to and including a zero.
      • Now extract only the numerical characters from each of those attributes (go ahead and keep any non-leading or non-trailing hyphen).
        • Hint: Don't delete anything, just add an additional value modifier to your list.
      • Lastly, remove any leading zeros from the found attributes.
        • Hint: [^0] will match any character other than zero.

  2. Replace Strings Value Modifier

    • Open Dates.rsd and run it on Dates.txt.
    • Add a Replace Strings value modifier to the attribute called Dates that will change Jan into JANUARY. Run the Rule Tester and confirm that JAN has been replaced with JANUARY.
    • Now try checking the Strings in the first column are Case Sensitive checkbox, and run the Rule Tester again. Note that JAN has not been replaced. Unchecking this checkbox, or changing Jan to JAN in the replacement pair will solve this.
    • Remove the Replace Strings Value Modifier which you created.
    • The regular expression symbol \W matches any non-word character, meaning any character that is not a letter, numeral, or underscore. For example, to change Foo, Bar to Foo-Bar, you could type \W+ in the Specify string to be replaced edit box and type - in the Specify string to replace edit box.
      • Add a new Replace Strings value modifier, and, using this example as a guide, change all of the dates in Dates.txt to look like:
        JANUARY/17/1984
        JAN/23/2008
        etc.
      • Now, without deleting the first replacement string pair you created, add replacement string pairs to change
        JANUARY/17/1984 to 01/17/2008
        JAN/23/2008 to 01/23/2008
        etc.
