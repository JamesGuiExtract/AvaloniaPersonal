Output Handler Exercises

  1. Remove Characters

    • Open SSN.rsd and run it on TestImage001.tif.
    • Add a Remove Characters output handler that will remove all dashes from the output.
    • Run the rules again. Notice how the text of attribute the has changed, but the spatial information has not.

  2. Output to XML File

    • Open SSN.rsd and add an Output data to XML file output handler. Set up the output handler to output Version 1 XML files in a subfolder called "XML".
    • Run the rules on TestImage001.tif to check your set up.

  3. Output to VOA file

    • Open SSN.rsd and add an Output data to VOA file output handler that outputs a VOA file to <SourceDocName>.voa. Save the file.
    • Create a new database, add a new action, and queue all of the images in RedactionDemo Images folder.
    • Process all of the documents using a Execute Rules task that runs SSN.rsd. Confirm that .voa files have been created for all of the images.
    • Set all of the documents back to pending and then process them using a Verify Redactions task. Be sure that the task is set up to create Expected Redaction .voa files. Redact all of the social security numbers in the documents.
    • Set up a .tcl and .dat files that will test SSN.rsd by comparing the output .voa files for all of the images in the Redaction Demo Image folder with the Expected Redaction .voa files that you just created.
    • Run the test. Delete the output .voa files and run the test again. Notice that it takes the test longer to run when the output .voa files are not present. This is because the Test Harness has to run the rules on each document.
