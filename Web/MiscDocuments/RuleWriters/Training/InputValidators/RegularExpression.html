Regular Expression Input Validator

Uses

  • The Regular Expression input validator is used in conjunction with the Remove Invalid Entries output handler to remove invalid data from the output of the rules.
  • Any data that is not matched by the specified regular expression is marked as "invalid".

Usage

  • To use the Regular Expression Input Validator, you first need to put some "placeholder" text in the Please specify the input type box.
  • You then need to write a regular expression that defines what data is "valid".
    • The regular expression can be made case sensitive.

  • Example:
    • Open APN.rsd and run in on APN.txt.
    • Add a Regular Expression input validator and define valid data as any string of numbers and dashes. (The regular expression [\d-]* will work.)
    • Run the rules again. Note that the output did not change. This is because we forgot to add a Remove Invalid Entries output handler. The data is being flagged as invalid, but it is not being removed!
    • Add a Remove Invalid Entries output handler to the rules. Run the rules again and notice the difference in output.
