Long Integer and Short Integer Input Validators

Uses

  • The Long Integer and Short Integer input validators are used in conjunction with the Remove Invalid Entries output handler to remove invalid integer numbers from the output of the rules.
  • The Short Integer input validator will mark any string that is not a well-formed integer as invalid.
  • The Long Integer input validator is similar to the Short Integer input validator; however, it has more configuration options.

Usage

  • The Short Integer input validator has no configuration options.
  • The Long Integer input validator has the following configuration options:
    • You have the option of setting a minimum number.
      • Numbers less than the minimum number will be flagged as "invalid".
      • You can choose whether or not the minimum number itself is valid.
    • You also have the option of setting a maximum number.
      • Numbers greater than the maximum number will be flagged as "invalid".
      • You can choose whether or not the maximum number itself is valid.
    • You can choose to make zero an invalid number.
    • You can choose to make negative numbers invalid.

Exercises

  1. Open Numbers.rsd and run it on Numbers.txt.
    • Add a Short Integer input validator and a Remove Invalid Entries output handler to the rules.
    • Run the rules again and notice the difference in output.
  2. Add a Long Integer input validator to Numbers.rsd that will remove all invalid integers and all negative numbers. Test the rules on Numbers.txt.
