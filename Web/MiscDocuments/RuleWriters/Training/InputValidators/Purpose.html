Input Validator Purpose and Basics

Uses

  • Input validators are used in conjunction with the Remove Invalid Entries output handler to remove incorrectly found data from the output of the rules.
    • For example, you can use a Regular Expression input validator to remove telephone numbers that have non-digit characters.
    • You can use a Date validator to remove invalid dates like "June 56, 2008".
  • Input validators check to make sure that the data inputted into them match a specified criteria for being "valid" data.
    • If these criteria are not meet, the input validator will flag the data as "invalid" in memory.
    • Data items that are flagged as "invalid" can later be removed by using a Remove Invalid Entries output handler.

Input Validator Basics

  • Input Validators can be added by clicking on the Commands button and selecting the Select Input Validator... option.
    • This will bring up a dialog where you can choose which type of input validator you want and give it a description.
  • You can modify an existing input validator using the Commands button.
    • Choosing the Select Input Validator... option will let you change the type of the input validator.
    • Choosing the Configure Input Validator... option will let you change the settings of the input validator.
    • Choosing the Configure description... option will let you change the description of the input validator.
  • You can remove the input validator by changing its type to <None>.
  • If you right click on the input validator, it will bring up a context menu that you can use to Cut, Copy, Paste, and Delete the input validator.
  • You can turn off the input validator by un-checking the checkbox in front of it. This can be useful for testing.
