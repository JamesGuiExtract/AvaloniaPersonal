File Action Manager and DB Administration Exercises

  1. Use the File Action Manager and DB Administration applications to create USS files for all of the images in the RedactionDemo\Images folder.

    • Create a new database.
    • Add an action called "OCR" to the database.
    • Open the File Action Manager and use a Files from folder file supplier to queue all of the files in the RedactionDemo\Images folder.
    • Process the queued files using an OCR document task.

  2. Open TestImage001.tif.uss, which was created in the last exercise, in the USS File Viewer.

    • Try highlighting some text and note how the Start Pos and End Pos values change.
    • Search for the text THE and note that every time the Find button is pressed the next instance of THE is highlighted in the USS File Viewer.
    • Make the above search case sensitive. Notice how the results change.
    • Use a regular expression to search for a social security number with the form ###-##-####.
    • Use the AND operator to confirm that the words born, child, and defendant are all present in this file.

  3. Create Expected Redaction .voa files for all of the images in the Redaction Demo folder.
    • Redact all of the social security numbers in these images and give them a type of SSN.

  4. Setup and run a Test Harness test that will test SSN.rsd on all of the images in the Redaction Demo folder. Use the Expected Redaction files that were created in the previous exercise.
    • Create the .tcl and .dat file.
    • What are the final automated statistics for SSN.rsd?
    • Look at the .uss files of the images on which there are missed redactions. What rules could you add to catch some of these missed redactions?
