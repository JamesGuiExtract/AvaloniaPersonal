File Action Manager and DB Administration

USS files

  • Extract Systems software has the capability to store the text extracted by the OCR engine in a .uss file.
  • Storing the extracted text in a .uss file allows multiple rules to be run on the same image without having to go through the lengthy process of OCR each time.
  • If you open an image file in the Rule Tester, the Rule Tester will automatically open the image's corresponding .uss file, if it exists, rather than taking the time to OCR the image.
  • The .uss file stores both the extracted text and spatial information about where text is located on the image.
  • Generally, there is one .uss file per image file, and the name of the .uss file is <image name> .uss
  • USS files can be created using the File Action Manager and DB Administration applications.

File Action Manager

  • The File Action Manager is Extract System's main application.
  • It is used to perform the same task or set of tasks on a large numbers of files
  • For example, the File Action Manager can be used to copy millions and millions of files from one directory to another.

  • The File Action Manger uses a database back-end.
  • The columns of the database, which are called "Actions", represent what task or set of tasks we want to perform on the set of images.
  • The rows of the database represent which files we want to perform the Actions on.
  • The cells of the database contain the status of the file for each Action.
  • The status of the files is always one of the following:
    • Unattempted: We do not want this file to be processed for this Action.
    • Pending: The file is waiting to be processed.
    • Processing: The file is currently being processed.
    • Complete: The file has been processed.
    • Failed: The file failed to process for some reason.

DB Administration

  • DB Administration application is used to create and maintain databases.
  • When you first open the DB Administration you will be asked to select a server machine and a database.
    • By default, your local machine is selected as the server.
    • If you want to choose another machine on the network to be the server, you can either type the name of the machine into the Database server name box or click on the drop down menu and browse for the desired machine.
    • One you have your server selected, you can choose to either log into an existing database or create a new one.
      • To create a new database, first select Create new database and then type the name of your new database into the Database Name box.
      • You will be asked to create a password for the new database.
    • The Database tab shows the server your database is on, the name of the database, and the connection status.
    • The Summary tab shows all the actions in the database and how many files are in each status for that action.
  • Actions can be added, deleted, or renamed using the Action menu.

Queuing Files

  • Files are added to the database by "queuing" them using the File Action Manager.
    • You can open the File Action Manger using the Tools menu on the DB Administration.
  • On the Action tab, select which action you want to work with.
  • To queue files, check the Queue files for this action check box.
  • You can choose which files you want to queue on the Queue Setup tab.
  • There are three different "File Suppliers" that you can use to queue files:
    • Files from folder: Queues all the files in a specific folder.
    • Files from static list: Choose individual files to queue or read in a list of files and queue each of them.
    • Files from dynamic list: Queue all of the files in a list.
  • Once you have your file suppliers setup, press the green "play" button to queue your selected files.
  • Note that the summary tab of the Database Administration now has several files that are pending for the action.

Processing files

  • To process files, check the Process files in the queue for this action check box on the Action tab.
  • You can choose which task or tasks you want to perform on the queued files on the Processing Setup tab.
  • To create uss files, you will need to add an OCR document task.
  • After you have added the task, press the green "play" button to process your files.
  • Note that the summary tab of the Database Administration will show how many files are being processed and how many have been completed.
  • Note that the .uss files appear as each document is OCRed!
