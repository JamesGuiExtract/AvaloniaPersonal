Regular Expression Rule Value Finder

Basic Regular Expression Syntax

  • Literals : With the exception of a few special characters, all characters will match their corresponding character in the searched text
    • a will find the text a.
    • 123Something123 will find the text 123Something123.

  • {} : Used to specify a certain number of characters
    • a{2} will find a string of exactly two a's. ( It will find aa, but not a or aaa. )
    • BO{3,5} will find a B followed by three, four or five O's. (It will find BOOO, BOOOO, and BOOOOO, but not BO or BOOOOOOOO. )

  • + : Used to specify "one or more" characters
    • z+ will find a string of one or more z's. ( It will find z, zz, zzz, etc. )
    • MO+M will find two M's with one or more O's between them. ( It will find MOM, MOOM, MOOOM, etc., but it will not find MM. )

  • * : Used to specify "zero or more" characters
    • AB* will find an A followed by zero or more B's. ( It will find A, AB, ABB, ABBB, etc. )

  • ? : Used to specify an optional character
    • JUDGE?MENT will find JUDGEMENT and JUDGMENT.

  • | : The OR operator
    • Joe|Mark will find the text Joe or the text Mark.

  • () : Used to group items and show the precedence of operators
    • Nov(ember)?
      • will find the text Nov or the text November.
    • H(is|er) will find the text His or the text Her.

  • If you want to find the above-mentioned characters as literals you must "escape" the character with a \.
    • why\? will find the text why?.
    • \(dog\) will find the text (dog).

Character Classes

  • [] : Used to create a "Character Class" that matches any character between the brackets
    • [02468] will find an even, single-digit number. ( It will find 2, 6, etc. but not 7. )
    • A dash is used to represent a range of characters.
      • [A-C] will match A, B, and C, but not D or Q.
    • A caret is used to indicate NOT.
      • A[^B-D]A will find A2A, AQA, and AAA, but not ABA, ACA, or ADA.
    • Different character ranges can be used within the same square brackets.
      • [0-46-9] will find any single digit number except five.

  • \d : Matches a single digit character
    • \d is the same as [0-9].
    • \d{3}-\d{4} will find any telephone number in a ###-#### format.

  • \D : Matches a single non-digit character
    • \D is the same as [^0-9].
    • \d\D\d\D finds four consecutive characters such that the first and third are digits and the second and fourth are not digits ( e.g. 2W6t ).

  • \s : Matches a single white space character
    • Matches white space characters including space (\x20), carriage return (\r) and line feed (\n).
    • To match white space without matching end of line characters (\r\n) use \x20.

  • \S : non-white space character
    • Matches any non-white space character.
    • \S is the same as [^\s].

  • . : matches any character except for \n.
    • A.{5}A will match an A followed by any five characters followed by another A.
    • If you want to match any character including \n, use [\s\S] or [\d\D].

  • \b : Matches a word boundary
    • you\b will match the you in you have but will not match the you in your.

Regular Expression Rule

  • Regular Expression Rule Value Finder is the most basic Value finder. Just type your regular expression into the box!
  • You can make your expression case sensitive.
  • You can also read your expression in from a file.

  • Example
    • Review TestImage001.tif and note that the image has two social security numbers.
    • Open a new Rule Set Editor and add an attribute named SSN.
    • Add a new Regular Expression Rule and give it an appropriate description.
    • Write a regular expression that will find a social security number in the form ###-##-####.
    • Open the Rule Tester and test the new rule on TestImage001.tif.
