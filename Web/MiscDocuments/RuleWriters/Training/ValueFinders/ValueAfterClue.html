Value After Clue Value Finder

Value After Clue Value Finder

  • The Value After Clue value finder looks for a specified clue and then captures a string of text that follows that clue.
  • Multiple clues can be defined in a single rule.
  • Clues can be added, deleted, or modified using the Add, Remove, and Modify buttons.
  • The specified clue can be either plain text or a regular expression.
  • You have to specify how much of the text following the clue that you want to capture:
    • You can capture all of the text between the clue and the end of input.
    • You can capture all of the text between the clue and next end of line character.
    • You can capture all of the text between the clue and a second clue.
      • The second clue can also be either plain text or a regular expression.
    • You can capture a certain number of lines after the clue.
      • You will need to choose the number of lines you want to capture and whether or not you want to include the line that contains the clue.
    • You can capture a specified number of words after the clue.
      • You will need to choose the number of words you want to capture.
      • You can optionally stop searching if you find a specified character.
      • You must specify what a word is by defining what characters separate words.
  • All of the strings in the Value After Clue value finder can be made case sensitive.

Example:

  • Open FileNumber_VAC.rsd.
  • Click the Configure button to edit the rule.
  • Notice the text File Number appears in the Define the clue text(s) box.
  • Test the rule on FileNumbers.txt using the Rule Tester and observe the output.
  • Check the Above defined clue text(s) will be treated as Regular Expression(s) checkbox and change the clue to File\sNumber. Test the rule. The output should be the same as when the plain text clue was tested.
  • Select Include all strings after the clue text until the end of the input. Look at the output from the Rule Tester. Notice that for every occurrence of the clue, the rules find a string containing all text after each occurrence of the clue to the end of the file.
  • Select Include all strings after the clue until the end of the line. Look at the output from the Rule Tester. Notice that for every occurrence of the clue, the rules find a string containing all the text after the clue to the end of the line.
  • Add the regular expression clue File\s\S*\s. Test this new clue. Note that this example should find all file numbers.
  • Delete the additional clue. Re-enter the File\sNumber clue if it is no longer present. Enter the same clue again as a second clue by clicking on the radio button in front of the Include all strings between the clue text and the string specified text box then type the second specified clue, File\sNumber in the box. Test the rule with the Rule Tester. Notice that this finds file numbers with some trailing return carriage and new line characters.
  • Select Up to X number of lines after the clue text and then enter the number 2 as the number of lines to be found. Run the rule in the Rule Tester and observe the output.
  • Select Include text after the clue on the line that contains the clue and run the rules in the Rule Tester. Notice the difference in the output.
  • Select Up to X number of words after the text and then enter a number of lines to be found. Run the rule in the Rule Tester and observe the output.
  • Select Define characters(s) for which to stop further searching, define some characters, and run the rule to observe the differences in output.
  • Select Define character(s) to separate each word and select a few different characters. Run the rule in the Rule Tester and observe the differences in output.
  • Check the All strings defined in this window are case-sensitive check box, change the clue to "file/snumber", and test the rule. Now un-check the case-sensitive box and re-test. Observe the difference in output.
