Query Attribute Selector

  • The Query Attribute Selector is an attribute selector that allows you select attributes using their name, type, or their position in the attribute hierarchy. You can also select attributes if they have a certain sub-attribute.
    • This selector is used by several output handlers including Run object on attributes, Modify attributes, Move and modify attributes, and Remove attributes.
  • The Query Attribute Select uses a simple query syntax to define which attributes should be selected:
    • To query an attribute by name, simply enter the name of the attribute into the query box.
      • The query SocialSecurityNumber will select all attributes with the name SocialSecurityNumber.
    • To query an attribute by type, enter the query as Attribute name@attribute type
      • The query Name@Buyer will select Name attributes only if they have a type of Buyer.
    • You can select attributes with any name by using *.
      • The query *@Person will select all attributes with a type of Person.
    • To query a sub-attribute, enter the query as Attribute name/sub-attribute name.
      • The query Name/Middle will select the Middle sub-attribute of Name attributes.
    • To query an attribute only if it has a certain sub-attribute, enter the query as Attribute name{sub-attribute name}.
      • The query PhoneNumber{AreaCode} will select the PhoneNumber attribute only if it has an AreaCode sub-attribute.
    • You can run multiple queries at once by separating them with |'s.
      • PhoneNumber|Name will select both PhoneNumber and Name attributes.
    • You can combine the above syntax to create complicated queries.
      • Name@Seller/*/First|Name@Seller{Middle/Initial}|PhoneNumber/*@Lender is a valid query.
