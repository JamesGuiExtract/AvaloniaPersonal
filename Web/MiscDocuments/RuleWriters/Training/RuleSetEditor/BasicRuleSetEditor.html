Basic Concepts and the Rule Set Editor

Optical Character Recognition

  • What is OCR?
    • OCR stands for "Optical Character Recognition."
    • It is a complex set of algorithms that examine the pixels in an image file and recognize what characters they represent.
    • Basically, an OCR engine "grabs" the text out of an image!
  • Example:
    • Review a hardcopy of TestImage002.tif.
    • Preview TestImage002.tif in the Spot Recognition Window.
    • Use the Recognize text and process button to OCR the entire image.
    • Compare the OCR output to the image.
    • Note that the OCR output is not always perfect.

Rule Set Editor

  • The Rule Set Editor is used to create and modify rules.
  • These rules are used to search through the OCR output of an image to find specific data.
  • Rules are saved into .rsd files.

Attributes

  • What is an attribute?
    • It is the piece of data that we are looking for.
    • For example, we can have an attribute for Telephone Number, Street Address, Grantor Name, Social Security Number, etc.
  • There are buttons on the Rule Set Editor that can be used to Add, Delete, and Rename Attributes.
  • You can have several attributes in one .rsd file
  • You can switch between attributes using the dropdown menu.

Introduction to the Rule Set Editor

  • Global Preprocessors
    • Used to prepare a document before you search for data.
    • For example, remove items that cause false positives.
    • Global preprocessors affect all attribute rules.
  • Attribute Rules
    • These are the "rules" that are used to find the attribute.
    • You can have multiple rules per attribute.
    • Each attribute rule has three parts:
      • Local Preprocessors
        • Used to prepare document before you search for data.
        • For example, remove items that cause false positives.
        • Local preprocessors affect only one attribute rule.
      • Value Finder
        • The value finder is the object that finds the desired data.
        • It is the "workhorse" of the attribute rule.
      • Value Modifiers
        • These objects are used to "clean up" the data found by the Value Finder.
        • For example, you can strip the $ symbol off of dollar amounts, or pull out more specific text with a regular expression.
  • Input Validators
    • These objects are used to "erase" invalid data.
    • For example, you can remove telephone numbers that contain letter characters.
  • Attribute Splitters
    • Can be used to split data into sub-attributes.
    • For example, you can split a name into the first name, last name, and middle name.
  • Output Handlers
    • Used to format data for output.
    • They are basically value modifiers that affect all attribute rules.
    • For example, you can capitalize all of the data, change "Junior" to "Jr", etc.
