ComponentData and FKBUpdates

ComponentData

  • ComponentData is collection of reusable rules.
  • Over the years, rule writers have taken individual rules that were useful on multiple projects and added them to ComponentData.
  • By using ComponentData, rule writers do not have to write the same rules over and over again.
    • For example, we know that numbers with the format ###-##-#### will always be social security numbers. As such, ComponentData contains a Regular Expression rule that contains the regular expression \d{3}-\d{2}-\d{4}.
    • By using ComponentData, we do not have to write this same rule for every single project in which we want to redact social security numbers.
  • An encrypted version of the ComponentData rules are installed as part of every ID Shield and FLEX Index installation.
  • The unencrypted rules are stored in SourceSafe under $\Engineering\ProductDevelopment\AttributeFinder\ComponentData.
  • ComponentData is sometimes called the "National Rules Set" by the sales and services departments.

FKBUpdates

  • We continuously make updates and improvements to the ComponentData rules.
  • Every so often, we package these updates and improvements into what is known as an "FKBUpdate".
  • Every FKBUpdate includes an installation file that can be used to overwrite the existing ComponentData rules on a machine with the new, improved ComponentData rules.
  • An e-mail will be sent out whenever a new FKBUpdate is available.
  • Unless otherwise instructed, when working on a customer project, you should always use the most current FKBUpdate.
  • The most current FKBUpdate is always stored at I:\Common\Engineering\ProductReleases\FlexIndex\External\FKBUpdate.
  • You can check which FKBUpdate you currently have installed on your machine by opening the Rule Tester and clicking the About Rule Tester... button.
