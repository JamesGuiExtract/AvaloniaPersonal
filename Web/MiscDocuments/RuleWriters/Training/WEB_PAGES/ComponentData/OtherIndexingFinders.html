Other Indexing Finders

Return Address Finder

  • The Return Address Finder finds, trims, and splits the return addresses on land record documents. The return address is the address that the document should be returned to after recording.
  • To use the Return Address Finder, create a Find From RSD File value finder, and configure it to call the it attribute from <ComponentDataDir>\Indexing\AddressFinders\ReturnAddressFinder.rsd.etf.

Property Address Finder

  • The Property Address Finder finds, trims, and splits the address of the property on land record documents.
  • To use the Return Address Finder, create a Find From RSD File value finder, and configure it to call the it attribute from <ComponentDataDir>\Indexing\AddressFinders\PropertyAddressFinder.rsd.etf.

Cross Reference Finder

  • The Cross Reference Finder finds reference to other documents on land records and then splits them into Book and Page and/or Instrument number.
  • To use the Cross Reference Finder, create a Find From RSD File value finder, and configure it to call the CrossReference attribute from <ComponentDataDir>\Indexing\CrossReference\CrossReference.rsd.etf.

Legal Description Finder

  • The Legal Description Finder finds legal descriptions on land record documents. Legal descriptions are formal descriptions of the location of a piece of property.
  • By default, the legal description finder does not split the legal description. As such, it is almost always used in conjunction with the Legal Description Splitter.
  • To use the Legal Description Finder, create a Find From RSD File value finder, and configure it to call the LegalDescription attribute from <ComponentDataDir>\Indexing\LegalFinder\main.rsd.etf.

Parcel Number Finder

  • The Parcel Number finder finds and trims the assessor parcel number of the property on land records.
  • If there are multiple parcel numbers on a document, each parcel number will be returned as a separate attribute.
  • To use the Parcel Number Finder, create a Find From RSD File value finder, and configure it to call the ParcelIDNumber attribute from <ComponentDataDir>\Indexing\ParcelIDNumber\ParcelIDNumber.rsd.etf.
