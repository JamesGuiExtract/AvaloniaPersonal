Split Attributes Using RSD File Attribute Splitter

Uses

  • The Split Attributes Using RSD File attribute splitter can be used to create a custom attribute splitter.
    • It does this by inputting the data found by the rules into an external "splitting" .rsd file.
    • Any data found by the splitting .rsd file will be added as sub-attribute of the original found data.
    • These splitting .rsd files are just a normal .rsd files and as such can use any rules objects.

Usage

  • To use the Split Attributes Using RSD File attribute splitter you simply need to specify a splitting .rsd file.

  • Example:
    • Open SplitPhone.rsd and run it on PhoneNumbers.txt. Note that several phone numbers are found.
    • We want to split the phone numbers to create an "AreaCode" sub-attribute and a "Number" sub-attribute for each number.
    • To do this, we must first create our splitting .rsd file.
      • Open a new .rsd file and add two new attributes, one named AreaCode and one named Number.
      • Add a Regular Expression Rule to the AreaCode attribute that contains the regular expression \(\d{3}\). This regular expression will "grab" the area code from the phone number.
      • Add a Regular Expression Rule to the Number attribute that contains the regular expression \d{3}-\d{4}. This regular expression will "grab" the number part of the phone number.
      • Save the rules as Splitter.rsd.
    • Add a Split Attributes Using RSD File attribute splitter to SplitPhone.rsd and configure it to point to Splitter.rsd.
    • Run the rules and note that the phone number now has two sub-attributes.
