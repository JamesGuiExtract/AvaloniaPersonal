Split Attribute Using Tokens

Uses

  • The Split Attribute Using Tokens attribute splitter is used to split an attributes into sub-attributes by using a specified delimiting character.

Usage

  • To use the Split Attribute Using Tokens attribute splitter, you must first specify which character will be serving as the delimiting character.
  • Next you will have to choose how to name the sub-attributes:
    • You can have the sub-attributes be named automatically.
      • All of the sub-attributes will be given the name entered into the Sub-attribute name expression box.
      • %d will be replaced with the token number. (ie. If you enter Field%d into the Sub-attribute name expression box, the first sub-attribute with be named Field1, the second will be Field2, etc.)
    • You can specify a name for each sub-attribute.
      • You will need to specify the desired name for each token.
      • %1 indicates the first token, %2 indicates the second token, etc.
      • Name/Token pairs can be added, deleted, and modified by using the Add..., Remove, and Modify... buttons.

  • Example:
    • Open Names.rsd and run it on Names.txt. Note that a Replace Strings value modifier has been used to replace all of the spaces in the found data with semicolons.
    • Add a Split Attributes Using Tokens attribute splitter with a semicolon as the delimiting character.
    • Run the rules again and note how the sub-attributes are named.
    • Modify the attribute splitter to name the first token First, the second token Middle, and the third token to Last.
    • Run the rules again and note how the sub-attribute names have changed.
