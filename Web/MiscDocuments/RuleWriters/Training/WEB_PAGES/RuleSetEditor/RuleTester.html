The Rule Tester and Finding Attributes

Rule Tester Basics

  • The Rule Tester is used to quickly test rules on a single document.
  • You can open the Rule Tester from the Rule Set Editor. It is listed under "Tools".
  • On the General Settings tab, you can choose whether to test all of the attributes in the current .rsd file or just the currently selected attribute.
  • On the Input tab, you can choose which type of input to test your rules on.
    • If you select Text from manual input, you can type the text you want to test in the edit box.
    • You can select Most recently processed text from file to load text or an image from a file.
  • Once you have your input selected, you can press the green "play" button on the toolbar to test your rules on the selected input.

  • Example:
    • Load FileNumber_VAC.rsd.
    • These rules have been setup to find a number after the text File Number:.
    • Open the Rule Tester.
    • Select Text from manual input and type File Number: 1234 into the edit box.
    • Execute the rules and examine the found data.
    • Note the different parts of the found Attribute:
      • Name
      • Value
      • Type
    • Selected Most recently processed text from file and load FileNumbers.txt.
    • Execute the rules and note how multiple attributes are found and displayed.

  • If you select Most recently processed text from file and then load an image file, the image will automatically be OCRed.
  • The results of the OCR will be displayed in a text box.
  • If you execute rules on an image and then click on the found data, the Rule Tester will bring up a window containing the image with the found data highlighted.

Other Rule Tester Functionality

  • Clear output window button: Clears the output window
  • Disable spot recognition when pressed button: Toggling this button on will prevent the image window from popping up when you click on found data
