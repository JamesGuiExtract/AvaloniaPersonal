Double and Float Input Validators

Uses

  • The Double and Float input validators are used in conjunction with the Remove Invalid Entries output handler to remove invalid real numbers from the output of the rules.
  • The Float input validator will mark any string that is a not well-formed real number as invalid.
    • Real numbers include integers and numbers with decimal points.
    • The Float input validator considers negative numbers to be valid.
  • The Double input validator is similar to the Float input validator; however, it has more configuration options.

Usage

  • The Float input validator has no configuration options.
  • The Double input validator has the following configuration options:
    • You have the option of setting a minimum number.
      • Numbers less than the minimum number will be flagged as "invalid".
      • You can choose whether or not the minimum number itself is valid.
    • You also have the option of setting a maximum number.
      • Numbers greater than the maximum number will be flagged as "invalid".
      • You can choose whether or not the maximum number itself is valid.
    • You can choose to make zero an invalid number.
    • You can choose to make negative numbers invalid.

Exercises

  1. Open Numbers.rsd and run it on Numbers.txt.
    • Add a Float input validator and a Remove Invalid Entries output handler to the rules.
    • Run the rules again and notice the difference in output.
  2. Add a Double input validator to Numbers.rsd that will remove all invalid numbers and all numbers less than 100. Test the rules on Numbers.txt.
