Box Finder

Uses

  • The Box Finder is used to redact or extract text from a specified box on a form-like document.
  • The Box Finder uses the lines on a document to locate a box that contains or is near a specified clue.

Usage

  • Clues can be added, deleted, or modified using the Add..., Remove, and Modify... buttons.
    • You can make the clues case sensitive.
    • The clues can be plain text or regular expressions.
    • You can choose to stop searching after the first box is found.
  • You can click on the grid labeled Select location of clue relative to box to specify where the clue will be in relation to the box you want to find.
  • You must select which pages of the document you want to search.
    • By default, all pages will be searched.
    • You can choose to search the first few pages of the document.
    • You can choose to search the last few pages of the document.
    • You can list specific pages of the document that you want to search.
  • You can optionally specify a minimum and maximum height and width of the box you want to find.
    • Boxes that are too big or too small will be ignored.
  • Next, you must choose what you would like the Box Finder to return:
    • If you choose the Return spatial information of found box and artificially associated the spatial area with the following text option, the Box Finder will only return the spatial information of the box.
      • This option is only useful for ID Shield redaction rules.
      • You must specify some "placeholder" text for the spatial information.
      • You can choose to exclude the clue from the output.
    • If you choose the Return text found within the box option, the Box Finder will return the text contained in the found box.
      • This option is most often used for FLEX Index indexing rules.
      • You can choose to include the clue as part of the found text.
  • If you check the Include all lines that are found in the output checkbox, when you run the Rule Tester, the Box Finder will output an attribute that contains all of the lines that were found in the document. This can be useful for debugging Box Finder rules.
