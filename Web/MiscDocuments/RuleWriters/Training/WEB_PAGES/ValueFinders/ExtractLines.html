Extract Lines Value Finder

Usage

  • The Extract Lines value finder is used to extract the contents of specified lines in a document.

Usage

  • If you select the Extract specific line(s) including line breaks: option, the value finder will return a single multi-line attribute which contains the specified lines with the new line (\n) and carriage return (\r) characters included.
  • If you select the Extract specific line(s) excluding line breaks: option, the value finder will return a single multi-line attribute which contains the specified lines with the new line (\n) and carriage return (\r) characters removed.
  • You can specify as a single line ("3"), a comma separated lists of lines ("3,5"), or a hyphenated range of lines ("3-5").
    • Ranges entered in descending order ("5-3") will be output in reverse order.
    • Blank lines do count.
  • If you select the Extract each line (excluding line breaks) as a unique value for current attribute option, the value finder will return every line in the document as a separate attribute without new line and carriage return characters.

Exercises

  1. Open the Rule Set Editor, add a new attribute, and add an Extract Lines value finder that is configured to find lines 2, 4, and 6 with line breaks.
    • Run this rule on ExtractLines.txt. Note that we get nothing but new lines. This is because lines 2, 4, and 6 are blank!
    • Change the value finder so that it extracts lines 1, 3, and 5. Run the rules again and examine the found data.
    • Change the value finder so that it extract lines 1-5 without line breaks. Run the rules again. Note that all five lines are output as one attribute with no line break characters.
    • Try extracting lines 19-13 with line breaks and note that the order is reversed.
    • Change the value finder to Extract each line and run the rules. Note that each line is now output as a separate attribute. Also note that Extract each line ignores blank lines, because they only contain new line characters.
