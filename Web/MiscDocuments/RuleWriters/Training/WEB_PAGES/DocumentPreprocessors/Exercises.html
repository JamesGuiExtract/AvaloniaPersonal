Preprocessors Exercises

  1. Sometimes the word order in the OCR result of an image is not what it should be. These exercises use a text file that simulates this condition.

    • Open Pets.rsd in the Rule Set Editor. Run this rule on Pets.txt.
    • Notice that only one of three pet names was found and that the text, ". Thank you!, was incorrectly found as the name of a pet.
    • Add an Advanced Replace String preprocessor to remove the first occurrence of Name of Pet so that, when the rule is run, ". Thank you! is no longer found as a pet's name.
    • Copy the Advanced Replace String to the clipboard; change the preprocessor to a Select Multiple Document Preprocessors object and paste the Advanced Replace String into it.
    • Add an Advanced Replace String that will make the rule find Spot as a pet's name.

  2. Open the Rule Set Editor and load the file SelectMultiple.rsd.

    • Load the file SelectMultiple.txt in the Rule Tester and execute the rules.
    • We are looking for "Sensitive #"'s, but it looks like the word "Sensitive" was not OCRed correctly.
    • The rule is looking for any string of nine digits, but this finds too many numbers.
    • Use preprocessors to eliminate the numbers we don't want.
      • Hint: Advanced replace string can replace insensitive numbers with garbage text.
      • No cheating! Don't edit the Value Finder.
