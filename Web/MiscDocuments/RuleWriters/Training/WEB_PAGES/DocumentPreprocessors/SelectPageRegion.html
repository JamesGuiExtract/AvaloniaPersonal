Select Page Region Preprocessor

Uses

  • Limits the pages passed to Value Finders for processing.
  • Can be used either as a page filter, or to re-OCR.

Usage

  • In the Select Document Preprocessor window:
  • Choose Select page region from the dropdown menu. Click Configure...
  • The dropdown menu at the top allows you to either include or exclude the region you will define.
  • 1) Select page(s).
    • All pages includes all pages, as expected.
    • Specific page(s).
      • Selects a single page. eg. "1"
      • Selects a list of pages. eg. "1,4,5"
      • Selects a range of pages. eg. "1-4"
      • Click the help balloon to the right of the field for more detailed info.
    • All/Leading/Trailing pages that contain: selects pages that contain a string.
      • The dropdown menu toggles selector between All/Leading/Trailing.
      • Enter a string into the text field.
      • Activate the checkboxes to interpret your string as a Regular Expression and/or as Case Sensitive.
  • 2) On each page in scope.
    • Checkboxes activate options to restrict pages horizontally and/or vertically.
    • Enter starting and ending percentages to define the ranges.
    • Note: You may want to enable Display X, Y percentages in Spot Recognition Window in the RDT Configuration utility.
  • 3) For each selected region.
    • Activate the checkbox to OCR the included text.
    • Enter an angle (in degrees) to rotate the image being OCR'd.

Exercises

  1. Open the RuleSet Editor, load ".\Files\TestImageSSN.rsd", open the Rule Tester, load ".\Files\TestImage004.tif" and execute the rules.
  2. Note that both SSN's found are on page one of this document. A more efficient approach would be to only process the first page.
  3. Set up a Select page region preprocessor to include only the first page.
  4. Additionally the SSN's are in a header to the document, so we can restrict the page horizontally as well. Adjust your preprocessor to do this.
  5. Now load ".\Files\BadOCR.rsd" in the RuleSet Editor and ".\Files\BadOCR.tif" in the Rule Tester. Execute the rules.
  6. They don't find anything. Let's investigate a bit. Open ".\Files\BadOCR.tif" in a Spot Recognition window and ".\Files\BadOCR.tif.uss" in the USS File Viewer.
  7. In the top right corner of the document there is a "Taxpayer number" field which contains a number that could be based on a SSN. Note that the phrase
    "Taxpayer number" is not included in the USS file. Also note the fuzzy black borders in the image. These can confuse the OCR engine and prevent text from being
    correctly identified.
  8. Create a Select page region preprocessor to allow the rule to find the SSN.
  9. Hint: exclude the borders of the image and re-OCR it.
