Verification Oversight

Purpose

  • To guarantee the highest accuracy in Expected Redactions.
  • To provide helpful feedback to verifiers regarding performance.

Procedure

  1. Prepare the database for review:
    1. Create an appropriately named action (eg. "review") in the verification database if one does not already exist.
    2. In SQL Server Management Studio Express, open and execute the files "Create Procedure 'CreateRandomSet'.sql", then "Execute 'CreateRandomSet'.sql" on the verification database.
      1. Open SQL Server Management Studio Express and connect to the machine hosting the verification database.
      2. Select File > Open > File...
      3. Browse to ".\Files\Create Procedure 'CreateRandomSet'.sql" and open it.
      4. Execute the script (F5).
      5. In the Object Explorer on the left, expand the tree for the verification database. Expand the "Tables" folder, right-click "dbo.Action", and click Open Table. Take note of the value in the "ID" column for the original verification action.
      6. Expand the "Programmability" folder, then expand "Stored Procedures". Right click "sp_CreateRandomSet" and click Execute Stored Procedure...
      7. In the Execute Procedure window, input appropriate values for each parameter:
        • @ActionID is the "ID" value we found in Step 1.2.5 above.
        • @startDate is the beginning of the window to review. Format: month/day/year hour:minute (24 hour time. eg. 11/18/2008 14:53)
        • @endDate is the end of the window to review formatted as above.
        • @ActionName is the name of the review action you are going to use (eg. "review").
        • @PercentOfFiles is the chance that a given document will be reviewed. It should be 5.
      8. Click OK
      9. See comments in "Execute 'CreateRandomSet'.sql" for more help with parameters.
      10. Procedure output displays how many documents were selected for review for each verifier (rows labeled 'P') and how many documents each verifier verified in the time window specified (rows labeled 'U').
    3. The review action in the verification database should now have an appropriate number of documents pending.
  2. Verify the files selected for review.
  3. Check for discrepancies:
    1. Copy the reviewed documents and their uss files into a seperate folder.
    2. Set up a .dat and a .tcl file to compare the ExpectedRedaction .voa's to the Review .voa's.
      • See ".\Files\review.tcl" and ".\Files\review.dat" for examples.
    3. Run the test harness.
  4. When the test is complete, send an email to your supervisor outlining your results.
    • Example:

         Project: OH - Muskingum - Cott Day Forward
         Verifier Name: Some Verifier Person
         Time Range: 9/23/2008 11:00am - 9/24/2008 9:00am
         Files Verified: 839
         Files Re-verified: 39
         Sensitive Items, Verifier: 5
         Sensitive Items, Rule Writer: 5

  5. If you find that a verifier missed a redaction, determine whose mistake it was (check the owner of the file) and alert either your supervisor or bring constructive criticism to the verifier directly.
