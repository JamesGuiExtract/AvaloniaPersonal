Advanced Replace String Document Preprocessor

Uses

  • The Advanced Replace String preprocessor is used to change or remove text from a document before a value finder is run on that text.
  • It is most often used to prevent erroneous redactions.
  • For example, if you were trying to redact social security numbers you would probably write a Regular Expression Rule that would catch all numbers with a format of \d{3}-\d{2}-\d{4}. However, in Louisiana, state project numbers also have this format. (The top of the document often says something like, "State Project No. 033-03-0034".) You regular expression rule will also catch the state project number, even though it is not a social security number. This can be avoided by using an Advanced Replace String preprocessors to remove the all strings with a format of STATE\sPROJECT\sNO\.\s\d{3}-\d{2}-\d{4}.

Usage

  • To use the Advanced Replace String preprocessor you must specify the text you want to find and the text you want to replace the found text with.
    • These two items can be either plain text or a regular expression.
    • They can also be made case sensitive.
    • You can remove a certain piece of text by leaving the Replace with: box blank.
    • You can also read these items from .txt files.
  • You can also choose which occurrences of the specified text you want to replace:
    • All occurrences
    • The first occurrence
    • The last occurrence
    • A certain occurrence

  • Example:
    • Open ARS_StateProjectNumber.rsd. This file finds numbers with a ###-##-#### format.
    • Run this rules file on StateProjectNumber.txt. Note that the state project number is erroneously found.
    • Add an Advanced Replace String preprocessor that removes STATE\sPROJECT\sNO\.\s\d{3}-\d{2}-\d{4}.
    • Run the rules again. Note that now only the valid social security number is found.
