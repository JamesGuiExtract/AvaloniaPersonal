Document Preprocessor Purpose and Basics

Uses

  • Document preprocessors are generally used to remove text that could cause false positives.
    • I once used an Advanced Replace Strings preprocessor to remove "State Project No: 333-11-2222" so that it would not be caught by my social security number rules.
    • When you are looking for dates of birth, a Replace Strings preprocessor can be used to remove other dates that are not dates of birth.

Preprocessor Basics

  • There are two types of preprocessors:
    • Global preprocessors affect all attribute rules in an .rsd file.
    • Local preprocessors only affect the attribute rule that they are in.
    • Other than scope, the two types of preprocessors function the same.
  • Preprocessors can be added by clicking on the Commands button and selecting the Select Rule-Specific Document Preprocessor... option.
    • This will bring up a dialog where you can choose which type of preprocessor you want and give it a description.
  • You can modify an existing preprocessor using the Commands button.
    • Choosing the Select Document Preprocessor... option will let you change the type of the preprocessor.
    • Choosing the Configure Document Preprocessor... option will let you change the settings of the preprocessor.
    • Choosing the Configure description... option will let you change the description of the preprocessor.
  • You can remove the preprocessor by changing its type to <None>.
  • If you right click on the preprocessor, it will bring up a context menu that you can use to Cut, Copy, Paste, and Delete the preprocessor.
  • You can turn off the preprocessor by un-checking the checkbox in front of it. This can be useful for testing.
