ID Shield Rules Delivery Checklist

Date: _____________________________

Customer: _________________________

Partner: ___________________________

Task

Comments

Add rules to SourceGear Vault

  1. Ensure that all rules related .rsd and .dat files have been checked into SourceGear Vault under the Rules sub-folder of the main project folder.
  2. Ensure that all testing related .tcl and .dat files have been checked into SourceGear Vault under the TestingFiles sub-folder of the main project folder.
Add keys and counters to Master.rsd
  1. Check out Master.rsd:

    1. Set the appropriate counter.
    2. Set the appropriate USB key numbers.
    3. Make sure the file is NOT internal.
    4. Add VOA or XML output handlers as required by the SOW.

  2. Check in Master.rsd.
Quality check all .rsd files
  1. Confirm all references are to .etf files and NOT .rsd or .dat files.
  2. Confirm that all files, except Master.rsd, are internal use only.
  3. Confirm that Master.rsd is the only file that has keys or counters.
  4. Confirm that Master.rsd is the only file with a document classifier.
  5. Confirm that Master.rsd is the only file that is creating .voa or .xml files.
  6. Confirm that there are no incorrect references or missing files.
  7. Check to see if any rules objects have been accidentally turned off. Rules that are turned off on purpose should be well labeled to avoid future confusion.
  8. Confirm that the proper counter and USB key numbers are set in Master.rsd and that Master.rsd is NOT internal use only.
  9. Confirm that the proper types of output files are being created by Master.rsd.
  10. In SourceGear Vault, label the Rules sub-folder with the date, using the format YYYY-MM-DD.
Create delivery test
  1. Confirm that rule writers have added appropriate examples to the Images folder of the delivery test.
  2. Create a Rules folder and .tcl and .dat files for the delivery test if not already present.
  3. Create an EAV folder and make an .eav file for each image in the delivery test.
  4. Perform a "first pass" run of the delivery test using the YYYY-MM-DD labeled rules.
  5. Look for and report any usual or undesirable behavior including:
    1. Unexpected errors in the exception log.
    2. Missed data or clues that should obviously be caught.
    3. False positives that could easily be fixed.
    4. Unnecessary double redactions.
Create delivery
  1. Create a YYYY-MM-DD sub-folder under the Deliveries folder.
  2. Get the rules from the YYYY-MM-DD label in SourceGear Vault and place them in the YYYY-MM-DD folder.
  3. Encrypt all .rsd and .dat files.
  4. Delete the unencrypted files.
  5. Create a readme.txt file retaining the history of previous deliveries that includes the following information:
    1. Date of the delivery.
    2. Fields the rules are redacting.
    3. Version, patch, and FKBUpdate used by the rules.
    4. Counter.
    5. USB keys.
    6. Types of files output by the rules.
  6. Build a .zip file in the YYYY-MM-DD folder and name it with the format
    YYYY-MM-DD <Project Name> Redaction.zip
Execute delivery test
  1. Ghost a test machine and install the current public build.
  2. Install the current patch, if appropriate.
  3. Install the current FKBUpdate, if needed.
  4. Copy the delivery test folder to the ghosted machine.
  5. Copy the rules from the .zip file.
  6. Execute the delivery test.
  7. Confirm that all items in the delivery test pass.
  8. Confirm that their are no unexpected errors in the exception log.
  9. Confirm that the proper output files are created.
Run final capture rate test
  1. Prepare the LogProcessStats utility to monitor the final capture rate test.
  2. Confirm that the only license file in use matches the customer-deployed license file. Testing will require a Disable USB Key Serial Numbers license.
  3. Run the final capture rate test using the rules from the .zip file.
  4. Look at the results from the LogProcessStats utility and confirm that neither memory leaks nor resource leaks are present.
  5. Confirm that their are no unexpected errors in the exception log.
  6. Record the results of the final capture test in the statistics spreadsheet and include the delivery date in the comments section.
Send delivery
  1. Add an external statistics spreadsheet into the YYYY-MM-DD folder. The templates for the external spreadsheets are located in SourceGear Vault under: $\Engineering\Web\MiscDocuments\RuleWriters. For hybrid projects, use IDShieldStatisticsTemplateExternal (Hybrid).xls. For all other projects, use IDShieldStatisticsTemplateExternal (Classic).xls.
  2. Fill in the spreadsheet, using the statistics from the final capture rate test.
  3. Place the spreadsheet inside of the .zip file.
  4. Create an appropriate e-mail and attach the .zip file. Be sure to mention which workflow the customer should use in the e-mail.
  5. Send the e-mail to all support personnel, Wayne, and Arvind.
