EmailFile Utility - Testing Checklist

  1. General

    • Confirm that EmailFile requires a suitable license file.
    • Confirm that EmailFile displays a usage message when run with the /? flag
    • Confirm that EmailFile can be configured by running with the /c flag
      • Confirm that the Email Settings dialog appears.

      • Confirm that there is an enabled edit box for the outgoing SMTP server.
      • Confirm that there is an enabled edit box for Port and that it defaults to 25.
      • Confirm that there is a checkbox to require authentication to send, unchecked by default.
      • Confirm that when that box is checked:
        • the User name edit box is enabled.
        • the Password edit box is enabled.
        • the Use SSL check box is enabled.

      • Confirm that there is an editable Sender display name edit box.
      • Confirm that there is an editable Sender email address edit box.
      • Confirm that there is an editable Email signature edit box.

      • With Require authentication unchecked, confirm that you are required to specify an outgoing server, a sender display name, and a sender email address.
      • With Require authentication checked, confirm that you are required to specify an outgoing server, a sender display name, a sender email address, a user name, and a password.
      • Provide an email address in an invalid format. Confirm that you are prompted to confirm before hitting OK or sending a test email.
      • Confirm that email settings are saved to a .config file under the common ProgramData folder.
      • Confirm that deleting the .config file clears the settings.

  2. EmailFile - Interactive testing

    • Set up some email settings.
    • Confirm that you can send an email by running with a single argument (the target email address).
    • Confirm that you can specify a subject line with the /subject flag.
    • Specify a .txt file with the /body flag and confirm that the contents of the .txt file are the body of the email.
    • Confirm that it fails gracefully if you specify a non-existent file.
    • Confirm that reasonable things happen if you specify a binary file.
    • Confirm that the /client flag fails gracefully on a machine with no standalone email client installed.
    • Confirm that the /client flag works correctly.
    • Confirm that if a file is specified, it is sent correctly.
    • Confirm that the /z flag zips the specified attached file.
    • Confirm that the /ef flag works as expected.

  3. Fixed SCRs

    • [LegacyRC #5230]
      • Try to send a file that doesn't exist.
      • Confirm that an error is displayed and no email is sent.

    • [LegacyRC #6093]
      • Clear any existing email settings.
      • Run EmailFile with the /client flag.
      • Confirm that the client appears and you are not prompted to configure email settings.
