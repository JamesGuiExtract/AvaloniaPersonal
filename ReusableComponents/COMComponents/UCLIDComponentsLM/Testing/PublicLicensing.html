Public Licensing Testing Checklist

NOTE: Items prefaced with an asterisk indicate functionality that is not yet complete.

  1. ExtractTRP2 - Functionality

    • Execute the following test cases from a command window. Use Task Manager to verify presence / absence of the application.
    • Confirm that ExtractTRP2.exe can be started without arguments.
    • Attempt to start multiple instances. Confirm that only one instance is active.
    • Confirm that ExtractTRP2 /? provides a message box with usage instructions.
    • Confirm that ExtractTRP2 /exit closes the application.
    • Confirm that unsupported command-line arguments such as ExtractTRP2 /abc provide a message box with usage instructions.

    • [LegacyRC #4915] Inconsistent behavior with both full license and unexpired temp license
      • Create a temp license file a_temp.lic and a full license full.lic and put both in your license file directory.
      • Start processing files.
      • Roll the clock back.
      • Confirm that ExtractTRP2 is not running.
      • Confirm that the license state is not corrupted.
    • [LegacyRC #4934] NOT FIXED ExtractTRP2.exe and Unlock files
      • Create an evaluation license.
      • Confirm that the FAM opens and runs.
      • Close the FAM.
      • Set test machine time back one month.
      • Confirm that the FAM reports a license corruption error.
      • Create a suitable Unlock file and copy it to the LicenseFiles folder.
      • Confirm that the Unlock file works even though the ExtractTRP2.exe process is not killed before restarting the FAM.
    • With no instance of ExtractTRP2.exe running, confirm that entering ExtractTRP2 /exit at the command line starts a new instance and immediately closes it.
    • [LegacyRC #6073] ExtractTRP2 and UAC
      • With a fresh install on a UAC-enabled machine, create a temp license and log in as an unprivileged user.
      • Confirm that Extract software runs correctly.
      • Confirm that ExtractTRP2.exe runs.
    • [LegacyRC #6112] Behavior with Corrupt Temporary License and Permanent License
      • Create a temporary license.
      • Start processing something.
      • Roll back the date.
      • After the license state is corrupt, drop a permanent license in the LicenseFiles directory.
      • Reboot.
        • Confirm that you can open the FAM without error.

  2. User Key Generator - Functionality

    • Step 1
      • Confirm that the initial caption is User License - Step 1 of 3.
      • Confirm that the user license key is generated, displayed and non-selectable when the application starts.
      • Confirm that the tab order is appropriate for the Name, Company Name, Phone Number and Email edit boxes.
      • Confirm that appropriate strings are presented for Product and Version.
      • Confirm that for FLEX Index or ID Shield or LabDE, both Client and Server are presented for Type.
      • Confirm that the Next button is disabled until all information is provided.
      • Confirm that there is an Open License Folder button that opens an Explorer window to the appropriate folder.

      • [LegacyRC #6113]
        • Test this fix on 32-bit Vista or 32-bit 2008 Server.
        • Confirm the the Open license folder button in the User License utility works properly.
    • Step 2
      • Click the Next button.
      • Confirm that the caption is User License - Step 2 of 3.
      • Confirm that Select email system radio buttons are presented for desktop email and web-based email.
      • Confirm that the default is desktop email.
    • Step 3
      • Desktop email
        • Select Desktop email and click Next.
        • Confirm that the caption is User License - Step 3 of 3.
        • Confirm that a three-item list of instructions is presented.
        • Confirm that a here button is provided to automatically generate a license request email.
        • Confirm that a Show manual license request instructions button is provided.

        • Click the here button on a system that has email configured.
        • Confirm that the license request email is created.
        • Click the here button on a system that does not have email configured.
        • Confirm that an Internet Connection Wizard appears to try and configure email settings.

        • Click the Show manual license request instructions button.
        • Confirm that a six-item list of instructions is presented.
        • Confirm that an enabled Click here button is provided to copy the email body to the clipboard.
        • Confirm that the other two Click here buttons are disabled.
        • Click the Click here button.
        • Confirm that appropriate email text has been copied to the clipboard.
        • Confirm that the next Click here button is now enabled.
        • Click the second Click here button.
        • Confirm that appropriate email subject text has been copied to the clipboard.
        • Confirm that the next Click here button is now enabled.
        • Click the third Click here button.
        • Confirm that appropriate recipient address text has been copied to the clipboard.
        • Confirm that the Finish button is now enabled.
      • Web-based email
        • Select web based email and click Next.
        • Confirm that the caption is User License - Step 3 of 3.
        • Confirm that a six-item list of instructions is presented.
        • Confirm that an enabled Click here button is provided to copy the email body to the clipboard.
        • Confirm that the other two Click here buttons are disabled.
        • Click the Click here button.
        • Confirm that appropriate email text has been copied to the clipboard.
        • Confirm that the next Click here button is now enabled.
        • Click the second Click here button.
        • Confirm that appropriate email subject text has been copied to the clipboard.
        • Confirm that the next Click here button is now enabled.
        • Click the third Click here button.
        • Confirm that appropriate recipient address text has been copied to the clipboard.
        • Confirm that the Finish button is now enabled.
