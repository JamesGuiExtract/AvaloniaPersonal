Report Viewer - General

Command-line Use

  • ReportViewer.exe exists in the CommonComponents folder.
  • ReportViewer.exe can be executed with a full license
  • ReportViewer.exe can be executed with an evaluation license

  • [DotNET #719] New command-line option for report viewer
    • Report viewer supports command-line options that allow for the subject of an email to be specified and for the sender (configured in the e-mail settings) to be overwritten.
    • Test the <ReportName> tag the subject line; it should turn into the name of the report.
    • Confirm that the <ReportName> tag works with both standard and saved reports.

  • [LegacyRC #5261] "/prompt" option
    • Run ReportViewer.exe, specifying a server, database, report file, and /prompt.
    • Fill in data for the Parameter Entry Form prompt and click OK.
    • The desired report is generated.
    • Confirm that there is no exception.

Open Report dialog

  • An Open Report dialog appears over the Report Viewer window.
  • The Standard reports tab is available.
  • The Saved reports tab is available.
  • A Report Viewer thumbnail image is available for each installed report.
  • Selecting a report and clicking Cancel does not change the displayed report.
  • Selecting a report and clicking OK displays the selected report.
  • Selecting a report and pressing <Enter> displays the selected report.
  • Double-clicking a report displays the selected report.
  • The Delete selected report button is disabled for each Standard report and enabled for each Saved report.
  • The preview image resizes smoothly when the window is resized.
  • Rename a thumbnail image; selecting the associated report changes the preview image into a meaningful text message.
    • Resize the window; the replacement text remains centered.

Report Viewer user interface

  • The window title without a report is Report Viewer.
  • Select a non-parameterized report such as List of all actions in the project.
  • A Parameter Entry Form dialog does not appear.
  • A marquee-style loading bar appears while the report is generated.
  • The report appears in the report viewer window.
  • The window title is now Report Viewer - <report name>.

  • [LegacyRC #5652] Support for PgUp, PgDn
    • Open a multi-page report.
    • PgUp and PgDn can be used to navigate pages.

  • [DotNet #380] Report Viewer
    • Generate a report with meaningful output.
    • Save a report template.
    • Rename the saved template's .JPG report thumbnail image.
    • A new report template can still be saved.

  • File menu
    • Open report... displays the associated dialog and allows the user to select another report.
    • Refresh refreshes the data in the currently open report.
      • The F5 shortcut performs the same function.
    • Save report template... displays the associated dialog and allows the user to provide a report name.
      • The saved report appears in the Saved reports tab.
      • [LegacyRC #5178] The name of the saved report cannot match an existing Standard report.

    • Export to PDF... displays a Save As dialog for a PDF file.
    • [LegacyRC #5158] Export... displays a Save As dialog allowing the user to save the report in various formats.
    • An Export completed message is displayed when saving to each of the following file formats:
      • Crystal reports (*.rpt)
      • PDF (*.pdf)
      • Microsoft Excel (97-2003) (*.xls)
      • Microsoft Excel (97-2003) Data Only (*.xls)
      • Microsoft Word (97-2003) (*.doc)
      • Microsoft Word (97-2003) - Editable (*.rtf)
      • Rich Text Format (*.rtf)
      • XML (*.xml)
    • Exit closes the Report Viewer.

  • Help menu
    • About Report Viewer... displays the About dialog window.
    • [LegacyRC #5161] The About box contains an appropriate image and copyright information.

  • Toolbar
    • All toolbar buttons are disabled if no report is loaded.
    • Open a report.
    • The Export Report button displays the appropriate dialog.
    • The Print Report button displays the appropriate dialog.

    • The Go To First Page button navigates to the first page.
    • The Go To First Page button is enabled when displaying a non-first page of a multi-page report.

    • The Go To Previous Page button navigates to the previous page.
    • The Go To Previous Page button is enabled when displaying a non-first page of a multi-page report.

    • The Go To Next Page button navigates to the next page.
    • The Go To Next Page button is enabled when displaying the non-last page of a multi-page report.

    • The Go To Last Page button navigates to the last page.
    • The Go To Last Page button is enabled when displaying the non-last page of a multi-page report.

    • The / <pagecount> label exists after the Go To Page edit box.
    • The Go To Page edit box is enabled when displaying a multi-page report.
    • The Go To Page edit box navigates to the specified page.
    • The Go To Page edit box navigates to the last page if the specified page number is beyond the page count.

    • The Go To buttons and Go To Page edit box are disabled for a single-page report.
    • The Go To First Page and Go To Previous Page buttons are disabled when displaying the first page of a multi-page report.
    • The Go To Next Page and Go To Last Page buttons are disabled when displaying a last page of a multi-page report.

    • The Find Text button can successfully locate text in each column of each table of the report.
    • The Find Text button cannot find text located on a previous page of the report.

    • The Zoom button displays a drop-down menu with Page Width, Whole Page, Customize..., and various percentages.
    • Selecting the Customize zoom setting displays a Zooming dialog allowing the user to specify a custom zoom level.
    • The minimum custom zoom factor is 25 percent.
    • The maximum custom zoom factor is 400 percent.
    • Selecting one of the Zoom settings modifies the zoom level in the displayed report.

  • Status Bar
    • Current Page No.:, Total Page No.:, and Zoom Factor: are accurately displayed on the status bar.

Report specific features

  1. Database settings change history
    • Prompted for ReportingPeriod parameter.
    • Jump to the Parameter Entry Form section and perform the checklist for the parameter(s) used by this report.
    • Database Settings Changes table and contents are formatted properly at 400% zoom.
    • Table title is displayed in white text on black.
    • These columns are generated: Time, Setting Name, Old Value, New Value, User, Machine.
    • The column headings have a distinct background color.

    • Database with meaningful data
      • One row is provided for each event and the information is accurate.

    • Empty Database
      • Database Settings Changes table has no rows.

  2. Detailed report for specific document
    • Prompted for DocumentName parameter.
    • Jump to the Parameter Entry Form section and perform the checklist for the parameter(s) used by this report.
    • A meaningful report is presented when a fully-qualified path to a processed file is provided.

    • Database with meaningful data (file queued and processed in database)
      • Document Information table and contents are formatted properly at 400% zoom.
      • Table title is displayed in white text on black.
      • Document name, File size (bytes), Number of pages, and File priority rows exist and contain accurate information.

      • For each table below, also confirm the following:
        • The table and contents are formatted properly at 400% zoom.
        • Table titles are displayed in white text on black.
        • The column headings have a distinct background color.
        • Rows are provided for relevant activity and information is accurate.

      • Summary of Status table. These columns are generated: Action, Status.
      • Queuing Events table. These columns are generated: Event time stamp, Event type, File modify time, File size(bytes), Machine, User.
      • Status Change Events table. These columns are generated: Time stamp, Action, From status, To status, Machine, User.
      • Tags Applied to Document table. These columns are generated: Tag Name. Note: This table may not be present if no tags were applied for the specified document.
      • Document Tag Change History table. The columns are generated: Time, Tag name, Added?, Machine, User. Note: This table may not be present if no tags were applied for the specified document.
      • File Action Comments table. These columns are generated: User, Action, Time Stamp, Comment. Note: This table may not be present if no comments were applied for the specified document.

    • Empty database (or nonexistent filename)
      • Document name contains File not found: <filename>.
      • File size (bytes), Number of pages, and File priority are empty.
      • There are none of the following tables: Summary of status, Queuing events, Status change events, Tags applied to document, Document Tag Change History, File action comments.

  3. Detailed usage report by specific user for specific day
    • Prompted for UserName and ReportDate parameters.
    • Jump to the Parameter Entry Form section and perform the checklist for the parameter(s) used by this report.
    • Report Parameters table and contents are formatted properly at 400% zoom.
    • Table title is displayed in white text on black.
    • User and Date information are accurate.

    • Database with meaningful data
      • The user activity table and contents are formatted properly at 400% zoom.
      • The table has no title.
      • The top row of the table represent minutes within the hour ranging (vertically) between 0 and 59.
      • The leftmost column displays specific hours starting with 6:00 AM and ending with 6:00 PM.
      • Some cells in the table have a white background to indicate no input event activity during the specific minute.
      • Remaining cells in the table have a colored background to indicate input event activity during the specific minute.

      • Total minutes with input event activity is under the Hours and minutes table.
      • The total is accurate based on the shaded cells in the table.

    • Empty database
      • The user activity table and contents are formatted properly at 400% zoom.
      • The table has no title.
      • The top row of the table represent minutes within the hour ranging (vertically) between 0 and 59.
      • The leftmost column displays specific hours starting with 6:00 AM and ending with 6:00 PM.
      • The remaining cells in the table have a white background to indicate no input event activity during the specific minute.

      • Total minutes with input event activity is under the Hours and minutes table.
      • The total is 0.

Parameter Entry Form

  • Verify Parameter Entry Form behavior using an appropriate standard report.
  • Cancel closes the dialog window and does not load a report.

  1. DocumentName
    • A meaningful report is presented when a fully-qualified path to a processed file is provided.

  2. DownloadOrUpload
