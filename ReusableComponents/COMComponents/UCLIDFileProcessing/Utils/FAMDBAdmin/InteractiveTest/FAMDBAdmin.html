. FAM DB Admin Testing Checklist

File Action Manager Database Administration

Licensing

  • File Action Manager Database Administration can be opened when using a full license
  • File Action Manager Database Administration can be opened when using an evaluation license

Database Selection - Part 1

  • A Database server name combo box is presented.
  • If both a Server and a Database name have been previously selected, confirm that both names are preloaded into the combo boxes.
  • Confirm that the combo box includes (local) as a option if SQL Server has been installed or not if only the native client was installed.
    • If SQL Server Express 2005/2008/2008 R2 is not installed on the test machine, confirm that (local) is not included in the combo box.
    • If SQL Server Express 2005/2008/2008 R2 is installed on the test machine, confirm that (local) is included in the combo box.
    • If SQL Server Express 2005/2008/2008 R2 is installed on the test machine with a named instance specified, confirm that (local)\<instanceName> is included in the combo box.
  • A <Browse...> item is available in the combo box.
  • Selection of the Browse item changes the cursor to a wait cursor.
  • After the wait cursor is replaced with the normal cursor, the combo box has no selection.
  • After Browse, the combo box is populated with an alphabetical collection of machine names or machine names\instance name.

  • Exit the application and restart.
  • A Database name combo box is presented.
  • If a database has previously been selected, the name is preloaded into the combo box.
  • Selection of a different database server does not clear the database name combo box.
  • A <Browse...> item is available in the combo box.
  • Selection of the Browse item changes the cursor to a wait cursor.
  • After the wait cursor is replaced with the normal cursor, the combo box has no selection.
  • After Browse, the combo box is populated with an alphabetical collection of database names.
  • Confirm that the two radio buttons are mutually exclusive.

  • Select Allison02 or FNP2 as the database server.
  • Confirm that browsing for databases on this machine provokes a reasonable error.
  • Confirm that Exception details include the ConnectionString text.

Database Creation

  • Select (local) as the database server name.
  • Browse for existing database names on the local machine.
  • Type an existing name into the Database name combo box, select Create new database and click the Next button. Confirm that a meaningful error is presented.
  • Type a new name into the Database name combo box.
  • Select Create new database and click the Next button.
  • Confirm that a Set Admin Password message box is presented.
  • After OK, confirm that a Set Admin Password dialog is presented.
    • Confirm that an error message appears if no passwords are provided.
    • Confirm that an error message appears if the passwords do not match.
    • Confirm that an error message appears if the passwords match but have different case.
    • Confirm that after providing matching passwords, the main File Action Manager Database Administration window appears.

  • Close File Action Manager Database Administration and launch a new instance.
  • Select (local) as the database server name.
  • Type a new name into the Database name combo box.
  • Select Create new database and click the Next button.
  • Confirm that a Set Admin Password message box is presented.
  • After OK, confirm that a Set Admin Password dialog is presented.
  • Confirm that after providing matching passwords, the main File Action Manager Database Administration window appears.
  • Confirm that no exceptions were logged.

    • Confirm that an error message is displayed about the incorrect schema version.
    • Confirm that you are prompted to update the schema and confirm that regardless of your response, the UI is still in a good state.
    • Confirm that an application trace is logged when you update the schema.

    • Login to the test machine as a member of the local Administrators group.
    • Confirm that DB Admin can login to an otherwise empty database created by SQL Management Studio.
    • Confirm that a confirmation dialog is presented.
    • Confirm that DB Admin can create a new database with the same name as an empty database created by SQL Management Studio.
    • Confirm that a confirmation dialog is presented.

    • Create two more empty databases with SQL Management Studio.
    • Create two SQL logins using SQL Management Studio.
      • User A - granted db_owner privileges to the new databases.
      • User B - granted db_datareader + db_datawriter privileges to the new databases.
      • Neither user should be a member of the Administrators group.

    • Login to the test machine as User B.
      • Confirm that the user can add actions and counters to one of the new databases.
      • Confirm that the user cannot clear the new database.
