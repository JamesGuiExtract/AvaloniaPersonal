Convert FAM DB Utility

  1. Database Setup

    • Get a local copy of ID_Shield.bak from I:\Common\Testing\Product Testing\FLEX_Index\5_0_Database.
    • Create a 5.0 database named ID_Shield using SQL Server Management Studio.
    • Restore the database from the above backup file. Note that you must select Overwrite existing database from the Restore options.

  2. Basic Operation and Error Conditions

    • [LegacyRCAndUtils #4901, #5277] Confirm that ConvertFAMDB requires a license file (else error message displayed at startup).
    • [LegacyRCAndUtils #4798] Confirm that the application uses a non-MFC icon.

    • Confirm that an error message is displayed if only an input Server is defined.
    • Confirm that an error message is displayed if a non-existent input database is defined.
    • Confirm that an error message is displayed if the defined input database is not from 5.0.

    • Confirm that the output Server defaults to the selected input Server.
    • Confirm that the output Database defaults to the selected input Database with an appended _7_0.

    • Confirm that clicking Close exits without converting the database.
    • Confirm that clicking Start displays a confirmation dialog that presents a benchmark conversion time for a large database.
      • Confirm that clicking No returns to the main window.
      • Confirm that clicking Yes converts the database.
      • Confirm that a success message is presented after the conversion has finished.
