Run FPS File - Testing Checklist

NOTE: This utility is designed to exercise an FPS file with a specified document.

  1. Operation and Basic Errors

    • Confirm that RunFPSFile.exe requires a license file.
    • Confirm that the utility supports the following syntax:
      RunFPSFile.exe <FPSFile> <SourceDocName> [/ignoreDB] [/queue] [/priority numericvalue] [/process] [/forceprocessing] [/ef <filename>]
    • Confirm that an error message is displayed if only an input FPS file is defined.
    • Confirm that an error message is displayed if only a source image is defined.
    • Confirm that an error message is displayed if no switches are defined.
    • Confirm that a non-default icon is provided.

    • [LegacyRC #5755] Confirm that RunFPSFile properly handles relative paths to the source document.

    • /ignoreDB switch
      • Confirm that an error message is displayed if the /process switch is also used.
      • Confirm that an error message is displayed if the /queue switch is also used.
      • Confirm that an error message is displayed if the /forceprocessing switch is also used.
      • Confirm that an error message is displayed if the /priority switch is also used.
    • /process switch
      • Confirm that an error message is displayed if the /ignoreDB switch is also used.
      • Confirm that an error message is displayed if the /forceprocessing switch is also used (without the /queue switch).
      • Confirm that an error message is displayed if the /priority switch is also used (without the /queue switch).
    • /queue switch
      • Confirm that an error message is displayed if the /ignoreDB switch is also used.
    • /forceprocessing switch
      • Confirm that an error message is displayed if the /ignoreDB switch is also used.
      • Confirm that an error message is displayed if the /queue switch is not used.
    • /priority switch
      • Confirm that an error message is displayed if the /ignoreDB switch is also used.
      • Confirm that an error message is displayed if the /queue switch is not used.
      • Confirm that an error message is displayed if the /priority setting is less than 1 or greater than 5.

  2. Optional Arguments and Usage

    • Confirm that /ef <filename> is a supported option and redirects any exceptions to the specified filename.
    • Confirm that /? is a supported option and displays a Usage message box.

  3. /ignoreDB Functionality

    • Confirm that no database updates are made.
    • Confirm that FPS files with the following tasks execute without error
      • Core: Add watermark
      • Core: Archive or restore associated file
      • Core: Cleanup image
      • Core: Conditionally execute task(s)
      • Core: Convert to searchable PDF
      • Core: Convert VOA to XML
      • Core: Copy, move or delete file
      • Core: Execute rules
      • Core: Launch application
      • Core: OCR document
      • DataEntry: Verify extracted data
      • Redaction: Create metadata xml
      • Redaction: Create redacted image
      • Redaction: Extend redactions to surround context
      • Redaction: Filter ID Shield data file
      • Redaction: Verify sensitive data
    • Confirm that FPS files with the following tasks fail and provide a meaningful error message
      • Core: Apply Bates number
      • Core: Manage tags
      • Core: Set file-action status in database

  4. /queue Functionality

    • Confirm that the document specified by <SourceDocName> is supplied to the database for the action in the FPS file if it was not already in the database.
    • Confirm that if <SourceDocName> is already in the database for the action in the FPS file, then its status is not changed.
    • Confirm that if <SourceDocName> is already in the database for the action in the FPS file, then its status is changed to Pending if the /forceprocessing switch is included.

    • Confirm that a document supplied with /priority 1 is added to the database with Low priority.
    • Confirm that a document supplied with /priority 2 is added to the database with Below normal priority.
    • Confirm that a document supplied with /priority 3 is added to the database with Normal priority.
    • Confirm that a document supplied with /priority 4 is added to the database with Above normal priority.
    • Confirm that a document supplied with /priority 5 is added to the database with High priority.
    • Confirm that a document supplied without a /priority setting is added to the database with Normal priority.

  5. /process Functionality

    • Confirm that if <SourceDocName> is already in the database and Pending for the action in the FPS file, then its status is changed to Completed (assuming no processing errors).
    • Exercise the following scenarios for <SourceDocName> already in the database and not Pending for the action in the FPS file
      • just /process switch: - exception displayed that <SourceDocName> not in Pending status
      • add /queue switch: - exception displayed that <SourceDocName> not in Pending status
      • add /queue and /forceprocessing switches: - status changes to Completed
    • Exercise the following scenarios for <SourceDocName> not already in the database
      • just /process switch: - exception displayed that <SourceDocName> not found in database
      • add /queue switch: - <SourceDocName> added to database, processed, ends with Completed status
      • add /queue and /forceprocessing switches: - <SourceDocName> added to database, processed, ends with Completed status

  6. Miscellaneous

    • [LegacyRC #5697] Confirm that processing a file ignores any scheduled processing settings in the associated FPS file.
