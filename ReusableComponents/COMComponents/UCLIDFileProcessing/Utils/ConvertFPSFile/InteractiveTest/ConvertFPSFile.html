Convert FPS File Utility - Testing Checklist

NOTE: This utility is designed to convert pre-5.0 FPS files to the current standard.

NOTE: Referenced FPS files are located at .\UCLIDFileProcessing\Utils\ConvertFPSFile\InteractiveTest.

  1. Basic Operation and Error Conditions

    • Confirm that ConvertFPSFile requires a license file.

    • Confirm that an error message is displayed if only an input FPS file is defined.
    • Confirm that an error message is displayed if only an output FPS file is defined.
    • Confirm that a warning message is displayed if the output FPS file already exists.
    • Confirm that a second warning message is displayed if the output FPS file is the same as the input file.
    • Confirm that an error message is displayed if the input FPS file is newer than version 10.

  2. Convert Version 2 FPS File

    • Confirm that an error message is displayed when opening this FPS file in the FAM.
    • Confirm that V2_OCR_Images.fps converts successfully.
    • Confirm that the new version does not define an Action.
    • Confirm that the new version includes a Folder file supplier providing TIF Images from C:\FlexDataEntry\Images.
    • Confirm that the new version includes one OCR document file processor.
    • [LegacyRC #5288] Confirm that the application closes without error.

  3. Convert Version 5 FPS File

    • Confirm that the new version includes one Verify Redactions file processor.

  4. Convert Version 6 FPS Files

    • Confirm that an error message is displayed when opening this FPS file in the FAM.

    • Confirm that the new version does not define an Action.

  8. Convert Version 10 FPS File

    • Confirm that an error message is displayed when opening this FPS file in the FAM.
    • Confirm that V10_3-files.fps converts successfully.
    • Confirm that the new version uses ActionName = test. Note that if test is not defined in your database, no Action will be selected.

    • Confirm that the new version includes one OCR document file processor.

  9. FLEX Index 6.0 FPS File

    • Create a 6_0_Test database on the test machine.
    • Add a and b as actions.
    • Confirm that no error message is displayed when opening FLEX_60.fps in the FAM.
    • Confirm that the FPS file uses the a action in the 6_0_Test database.
    • Confirm that the FPS file does not includes a file supplier.
    • Confirm that the FPS file includes the following processing tasks
      • OCR document - for all pages and using a cleaned image
      • Execute rules - for I:\users\Art_Jones\Public\a.rsd and using the USS file
      • Copy, move or delete file - to copy <SourceDocName> to <FPSFileDir>\<SourceDocName>.copied
      • Set file-action status in database - to set b to Failed
      • Verify redactions - with the following settings
        • Include all pages in redaction review
        • Require users to specify redaction type
        • Enable collection of redaction accuracy feedback for all verified documents, saving the original document and with $DirOf(<SourceDocName>)\ExpectedRedactions as the feedback data folder
        • Default settings for the ID Shield data file location
        • Create output image only for documents that contain redactions
        • Create metadata for all documents
        • Redaction text and color settings use <ExemptionCode> with 8pt Times New Roman plus Black border and fill

  10. ID Shield 7.0 FPS File

    • Confirm that the standard Demo_IDShield database exists on the test machine.
    • Confirm that no error message is displayed when opening IDShield_70.fps in the FAM.
    • Confirm that the FPS file uses the Compute action in the Demo_IDShield database.
    • Confirm that the FPS file includes a Files from folder file supplier providing TIF images from C:\Demo_IDShield\Input.
    • Confirm that the FPS file includes a Match based upon file name pattern skip condition to skip files containing redaction.
    • Confirm that the FPS file includes the following processing tasks
      • OCR document - for all pages and using a cleaned image
      • Execute rules - for C:\Demo_IDShield\Rules\Master.rsd.etf and using the USS file
      • Filter ID Shield data file - to select only SSN items and save to <SourceDocName>.ssn.voa
      • Launch application - to open notepad.exe
      • Set file-action status in database - to set Verify to Pending
      • Archive or restore associated file - to archive and then delete <SourceDocName>.voa
