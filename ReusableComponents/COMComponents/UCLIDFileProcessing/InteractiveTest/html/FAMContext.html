FAM Context Testing Checklist

Goals

  • Exercise various aspects of context definition and use with File Action Manager.
  • Contain test cases for bug fixes associated with FAM context.

User Interface

  • Create Context Dialog
    • Confirm that a Create context dialog appears after selecting Tools - Edit Custom Tags... from the FAM.
    • Confirm that a Create context dialog appears after selecting Ctrl+E from the FAM.
    • Confirm that the dialog includes a Context name edit box.
    • Confirm that the dialog includes an FPSFileDir edit box.
    • Confirm that the dialog includes balloon help for FPSFileDir.
      • Confirm that the balloon help discusses when the context will be active and that the SDF file must be copied to folders associated with other contexts.
    • [ISSUE-12899] Confirm that a UNC path recommended warning message is presented if FPSFileDir is not a UNC path.

  • Save Configuration
    • Confirm that a Save Configuration? message appears after clicking Use current context's database from a new FAM.
    • Confirm that a Save Configuration? message appears after selecting <DatabaseServer> from the Database Server browse button on a new FAM.
      • Confirm that clicking No sets the Database Server to <DatabaseServer>.
    • Confirm that a Save Configuration? message appears after selecting <DatabaseName> from the Database Name browse button on a new FAM.
      • Confirm that clicking No sets the Database Name to <DatabaseName>.
    • Confirm that clicking Yes brings up a Save As dialog defaulting to the last folder in which an FPS file was saved.
    • Confirm that after saving the FPS file, a Create context dialog appears.
      • Confirm that FPSFileDir is defaulted to the recently saved FPS file's folder.
      • Confirm that a CustomTags.sdf file is also open for editing.
      • Confirm that the prompt dialog is modal to the SQLCDBEditor window.

  • CustomTags.sdf
    • Confirm that the following menu items are available.
      • File
        • Save
        • Exit
      • Tools
        • Update to current schema
      • Help
        • About
    • Confirm that a Save button is available on the toolbar.
    • Confirm that the only tab is CustomTags.
    • Confirm that the tab includes an Edit Contexts button.
    • Confirm that the tab includes an Add Database Tags button.
    • Confirm that a Tag name column exists.
    • Confirm that one column for each defined context exists.
    • Confirm that cells are fully editable.
    • Confirm that full rows cannot be cut, copied, pasted or dragged.
    • Confirm that angle brackets are automatically added, if necessary.
    • Confirm that duplicate Tag name values are not allowed.

    • Edit Contexts
      • Confirm that clicking the button displays an Edit contexts dialog.
        • Confirm that the dialog includes a table with the following columns.
          • Name
          • FPSFileDir
        • Confirm that cells are fully editable.
        • Confirm that full rows cannot be cut, copied, pasted or dragged.
        • Confirm that clicking the OK button allows the changes to be reflected in the CustomTags tab.
        • Confirm that clicking the Cancel button discards the changes and that no changes appear in the CustomTags tab.
      • Confirm that duplicate FPSFileDir values are not allowed.
      • Confirm that if, after editing, any FPSFileDir values are non-UNC paths, a UNC path recommended warning message is presented.
      • Confirm that the window is modal to the SQLCDBEditor window.
      • Confirm that an FPSFileDir with a trailing backslash is saved without the backslash.
      • Confirmed that an empty dialog can be closed without error.

    • Add Database Tags
      • Confirm that clicking the button adds <DatabaseServer> and <DatabaseName> to the Tag name column.
      • Confirm that clicking the button again displays a Database tags already exist message box.

    • Confirm that expandable tag functions can be used as tag values.
    • Confirm that after closing the tag editor, all of the context tag information (including context name) and the database connection are refreshed properly.
    • Confirm that auto-column sizing in SQLCDBEditor takes max data length for table columns into account.
    • Confirm that an exception is displayed when the user tries to create an empty ContextTags database.
    • Confirm that a warning dialog is presented when saving database tags that are defined but not used.
    • Confirm CustomTags.sdf can be saved with:
      • Multiple contexts without tags.
      • Multiple tags without contexts.
    • Confirm that the user is prompted to save changes upon close.

Miscellaneous

  • Confirm that the active context is displayed in the FAM title bar.
  • Confirm that closing the Create context dialog without providing a context name still allows CustomTags.sdf to be edited and saved.
    • Confirm that the Current Context on the FAM Database tab is No context defined! and is red in color.
    • Confirm that the FAM Connection Status remains Not Connected.
  • Confirm that a FAM cannot be run if tag values are missing for the current context.
  • Confirm that an FPSFileDir with a trailing backslash is saved without the backslash.

  • [ISSUE-12905] FPSFileDir
    • Open a CustomTags.sdf file for editing (either directly or via an "Edit Custom Tags" option in the FAM).
    • Click the Edit Contexts button to open the context editing dialog.
    • Add a new row by entering a name, but do not add an FPSFileDir.
    • Click OK.
    • Confirm that a message box is displayed notifying that an FPSFileDir is needed.
    • Confirm that an exception is not thrown after closing the message box.

  • [ISSUE-12918] Confirm that database tags remain in the FPS file after running then saving.

  • [ISSUE-12935] Confirm that FPS files can now be loaded and run when the corresponding CustomTags.sdf file is read-only.

  • [ISSUE-12977] Context Tags Plugin
    • Confirm that invalid data does not throw an exception in SQLCDBEditor.
    • Confirm that a glyph is shown in the row header if an error is present.
    • Confirm that only a single row at a time is allowed to have an unresolved error.
