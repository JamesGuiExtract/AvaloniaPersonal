Standard LabDE
Load & Save Testing Checklist

  • Prompt For Save : stand-alone mode

    • Open an image.
    • Select File - Close. Confirm that the document is closed without a prompt.

    • Open an image.
    • Select File - Exit. Confirm that the application is closed without a prompt.

    • Open an image.
    • Change a character in a single field.
    • Select File - Close. Confirm that a Data Not Saved prompt is displayed.
    • Confirm that Cancel returns to the document and that data is retained.
    • Select File - Close. Confirm that a Data Not Saved prompt is displayed.
    • Confirm that No closes the document and does not update the VOA file.
    • Re-open the image.
    • Change a character in a single field.
    • Select File - Close. Confirm that a Data Not Saved prompt is displayed.
    • Confirm that Yes closes the document and does update the VOA file.

    • Open an image.
    • Swipe text into the Lab ID field.
    • Select File - Close. Confirm that a Data Not Saved prompt is displayed.
    • Confirm that Cancel returns to the document.
    • Select File - Close. Confirm that a Data Not Saved prompt is displayed.
    • Confirm that No closes the document and does not update the VOA file.
    • Re-open the image.
    • Swipe text into the Lab ID field.
    • Select File - Close. Confirm that a Data Not Saved prompt is displayed.
    • Confirm that Yes closes the document and does update the VOA file.

  • Prompt For Save : FAM mode

    • Start processing.
    • Select File - Stop processing. Confirm that the FAM is closed without a prompt.
    • Confirm that the document returns to Pending status.

    • Start processing.
    • Select Stop processing from the FAM toolbar. Confirm that the FAM is closed without a prompt.
    • Confirm that the document returns to Pending status.

    • Start processing.
    • Change a character in a single field.
    • Select File - Stop processing. Confirm that a Data Not Saved prompt is displayed.
    • Confirm that Cancel returns to the document and that data is retained.
    • Select File - Stop processing. Confirm that a Data Not Saved prompt is displayed.
    • Confirm that No closes the FAM and does not update the VOA file.
    • Start processing and confirm that the same image is displayed.
    • Change a character in a single field.
    • Select File - Stop processing. Confirm that a Data Not Saved prompt is displayed.
    • Confirm that Yes closes the FAM and does update the VOA file.
    • Start processing and confirm that the next image is displayed.

    • Start processing.
    • Swipe text into the Lab ID field.
    • Select File - Stop processing. Confirm that a Data Not Saved prompt is displayed.
    • Confirm that Cancel returns to the document.
    • Select File - Stop processing. Confirm that a Data Not Saved prompt is displayed.
    • Confirm that No closes the FAM and does not update the VOA file.
    • Start processing and confirm that the same image is displayed.
    • Swipe text into the Lab ID field.
    • Select File - Stop processing. Confirm that a Data Not Saved prompt is displayed.
    • Confirm that Yes closes the FAM and does update the VOA file.
    • Start processing and confirm that the next image is displayed.

    • [DataEntry #291] Crashes when stopped with unsaved changes
      • Start processing.
      • Change data such that the contents are all valid.
      • Stop processing from the FAM.
      • Confirm that a Data Not Saved prompt is displayed.
      • Confirm that Cancel returns to the document without closing the FAM and that data is retained.
      • Run this test again and Confirm that Ok saves the data and closes the FAM.

      • Start processing.
      • Change data such that some data are invalid.
      • Stop processing from the FAM.
      • Confirm that an Invalid value exception is displayed.
      • Close the exception and confirm that the FAM closes.

  • Load & Save : stand-alone mode

    • Select an image for verification. Delete or rename any associated VOA file.
    • Open the image.
    • Enter data into a field.
    • Close the application and save data.
    • Confirm that a VOA file is created.
    • Open the VOA file in the VOA Viewer
      • Confirm that the entered data is present.
      • Confirm that empty attributes are present for other fields available in the Data Entry Application.
    • Re-open the image.
    • Confirm that the entered data is still present.

    • Select an image for verification. Confirm that an associated VOA file is present.
    • Open the image.
    • Confirm that data is appropriately presented in various fields.
    • Select a field that contains spatial data (solid color highlight appears when selected).
    • Swipe different text into the field.
    • Close the application and save data.
    • Open the VOA file in the VOA Viewer and confirm that the swiped data is present.
    • Re-open the image.
    • Confirm that the swiped data is still present and the new highlight is provided.

  • Load & Save : FAM mode

    • Delete or rename any VOA file associated with images to be verified.
    • Start processing.
    • Enter data into a field.
    • Save data.
    • Confirm that a VOA file is created.
    • Open the VOA file in the VOA Viewer
      • Confirm that the entered data is present.
      • Confirm that empty attributes are present for other fields available in the Data Entry Application.
    • Reset verification status to Pending.
    • Re-verify the image.
    • Confirm that the entered data is still present.

    • Confirm that VOA files are present.
    • Start processing.
    • Confirm that data is appropriately presented in various fields.
    • Select a field that contains spatial data (solid color highlight appears when selected).
    • Swipe different text into the field.
    • Close the application and save data.
    • Open the VOA file in the VOA Viewer and confirm that the swiped data is present.
    • Reset verification status to Pending.
    • Re-verify the image.
    • Confirm that the swiped data is still present and the new highlight is provided.

  • Handling Invalid / Unviewed Data : stand-alone mode

    • Exercise the Demo_LabDE images.
    • Open 003.tif.
    • [DataEntry #327] Confirm that the Patient Name field is initially selected and displays as viewed (non-bold font).
    • Confirm that no data values are invalid.
    • Click the Save button.
    • Confirm that no prompt about unviewed data is presented.
    • Open 151.tif.
    • Confirm that only the Date of Birth value is invalid.
    • Click the Save button.
    • Confirm that an Invalid value error is presented.
    • Modify the Date of Birth field to provide a valid date.
    • Confirm that some other data is still presented in bold, indicating that it is still unviewed.
    • Click the Save button, confirm that no prompt about unviewed data is presented.
    • Confirm that the changed data is saved with the new date in the VOA file.

  • Handling Invalid / Unviewed Data : FAM mode

    • Exercise the Demo_LabDE images.
    • Open 003.tif.
    • [DataEntry #327] Confirm that the Patient Name field is initially selected and displays as viewed (non-bold font).
    • Confirm that no data values are invalid.
    • Click the Save button.
    • Confirm that no prompt about unviewed data is presented.
    • Advance through other documents until opening 151.tif.
    • Confirm that only the Date of Birth value is invalid.
    • Click the Save button.
    • Confirm that an Invalid value error is presented.
    • Modify the Date of Birth field to provide a valid date.
    • Confirm that some other data is still presented in bold, indicating that it is still unviewed.
    • Click the Save button, confirm that no prompt about unviewed data is presented.

  • Miscellaneous

    • [DataEntry #259] Hints turn in to spatial data after save
      • Open image 003.tif from Demo_LabDE.
      • Select the Ref. Range field of one of the eGFR rows.
      • Confirm that a red hint is provided.
      • Type some text into the cell and save the results.
      • Reopen image 003.tif.
      • Select the same cell.
      • Confirm that the displayed tooltip matches the manually entered text.
      • Confirm that the hint is green.
