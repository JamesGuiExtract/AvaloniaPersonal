LabDE Verification Testing

NOTE: Test notes should be archived under I:\Common\Testing\Product Testing\LabDE_10\VerificationTests

  1. Server Preparation

    • Select a server test machine ( i.e. SuperTester or UltraTester or a quad-core ).
    • Copy suitable image files for processing to the test machine. The Endurance Test checklist provides suitable image folders.
    • Confirm that SQL Server is installed on the server. Either the Standard edition or the Express edition of SQL 2005 or SQL 2008 is acceptable.
    • Create a V: drive mapped to the root folder containing the local test images.
    • Use USB License Key Manager to connect to a USB key.
    • Install Corepoint.
      • The default options can be used throughout
      • Don't install the samples.
      • At the completion of the install, an evaluation license for Corepoint will need to be created.
      • Open Start | Programs | Corepoint Health | Corepoint Integration Engine | Corepoint Integration Engine - Configuration
      • From the file menu, select "Import..."
      • Browse to the Demo_LabDE\Solution\Corepoint Integration and select the AdvancedHealth.nix file.
      • Confirm all prompts to import all contained objects.
      • Close the Corepoint Configuration.
      • Open Start | Programs | Corepoint Health | Corepoint Integration Engine | Corepoint Integration Engine - Administration
      • Confirm that Connections "PT_LABDE_XML_ORU" and "PT_AdvancedHealth_ORU" are running. "OB_AdvancedHealth_ORU" will not be running-- that is okay.

  2. Database Preparation

    • Copy the Demo_LabDE folder to the root of the C drive on the server machine.
    • Create a network share for the Demo_LabDE folder. The current account needs write access to the share. Remote machines need only read access.
    • Run Demo_LabDE\DemoFiles\Installs\LongDemo\Install.bat.
    • Run Demo_LabDE\Clean.bat to create a Demo_LabDE database.

  3. File Preparation

    • Modify the Demo_LabDE\FPS\ExtractData.fps file supplier to supply the images from drive V:.
    • Re-configure Verify.fps to use the full UNC path to the StandardLabDE_AH_Phase1.config file. (eg. "\\machinename\Demo_LabDE\Solution\StandardLabDE_AH_Phase1.config")
    • Re-configure MatchToPatientAndOrder.fps to use the full UNC path to the StandardLabDE_AH_Phase2.config file. (eg. "\\machinename\Demo_LabDE\Solution\StandardLabDE_AH_Phase2.config")

  4. Second Processing Machine Preparation

    • Install the chosen build of LabDE.
    • Add a license file for LabDE Server.
    • Create a V: drive mapped to the root folder of the image server machine.
    • Use USB License Key Manager to connect to a USB key.
