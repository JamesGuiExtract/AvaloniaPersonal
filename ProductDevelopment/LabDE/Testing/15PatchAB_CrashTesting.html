LabDE 1.5 Patch A and B
Crash Testing

Purpose and Details

  • Purpose: Exercise various scenarios that had caused reliable or sporadic FAM crashes with LabDE 1.5
  • Details: Recently added SCRs generally do not have official test cases present within the collected checklists. One part of validating a LabDE 1.5 Patch B release candidate is to explicitly exercise these otherwise untested recent changes.

Scenarios

  1. Phase 1 Setup
    • Exercise ExtractData.fps from Demo_LabDE.
    • Enable gflags for ProcessFiles via gflags /p /enable ProcessFiles.exe /full.
    • gflags can be found at I:\users\Jeff_Shergalis\Public\WinDebugx86\gflags.exe.
    • Rename a few of the first VOA files so that data is not loaded and presented during verification.

  2. Test A - Medical Record Number
    • Open a Demo_LabDE sample image.
    • Right-click in the Medical Record # field.
    • Confirm that LabDE does not crash.

  3. Test B - Gender Combo Box
    • Open a Demo_LabDE sample image.
    • Click on the edit box portion of the Gender combo box.
    • Confirm that LabDE does not crash.

  4. Phase 2 Setup
    • Correct the filenames for the renamed VOA files from Phase I.

  5. Test C - Ordering Physician table
    • Open a Demo_LabDE sample image.
    • Click in the Code field of the Ordering Physician table.
    • Press Shift+Space to select the whole row.
    • Confirm that LabDE does not crash.

  6. Test D - Dragging test rows
    • Open a Demo_LabDE sample image.
    • Drag multiple test rows from one order on top of the row header for a new order.
    • Confirm that LabDE does not crash.
    • Copy and paste text into a different cell after dropping the test rows.
    • Confirm that the open document can be saved or skipped.
    • Confirm that LabDE does not crash.

  7. Test E - Copying data from another application
    • Open a text editor and type text for a sample Patient name, Medical Record #, and Order Number.
    • Open a Demo_LabDE sample image.
    • Copy text from the text editor and paste it into the appropriate fields.
    • Use shortcut keys and the context menu.
    • Exercise several sample images.
    • Confirm that LabDE does not crash.

  8. Test F - Pasting into Medical Record number field
    • Open a Demo_LabDE sample image.
    • Add a number to the Medical Record # field.
    • Select the text.
    • Click in the Image Viewer to change focus.
    • Switch to another application and copy numeric text to the Clipboard.
    • Click on the LabDE title bar to restore focus.
    • Press Ctrl+V to paste the number into the field.
    • Confirm that LabDE does not crash.

  9. Test G - Gender Combo Box
    • Open a Demo_LabDE sample image.
    • Click on the drop-down arrow of the Gender combo box.
    • Confirm that LabDE does not crash.

  10. Test H - Typing in the new row
    • Open a Demo_LabDE sample image.
    • Select the first cell of the new row of the Order table.
    • Type crea as fast as possible.
    • Confirm that LabDE does not crash.

  11. Test I - Lab Name field
    • Open a Demo_LabDE sample image.
    • Right-click in the Lab Name field.
    • Confirm that LabDE does not crash.

Enhancements and Bug Fixes

  • Confirm that gflags is enabled.

  • [DataEntry #934] Edit mode
    • Open a Demo_LabDE sample image.
    • Select a Flag cell in the Test Details table.
    • Change the flag to ensure edit mode.
    • Click on the row header to select the entire row.
    • Press the Down arrow.
    • Confirm that the next row is selected.
    • Confirm that the flag value is not changed again.

  • [DataEntry #935] Case-insensitive smart tags
    • Open a Demo_LabDE sample image.
    • Select and clear an edit box.
    • Type a period character.
    • Confirm that a drop-down list appears at the cursor location.
    • Confirm that the initial list contains all of the defined smart tags.
    • Confirm that as characters are typed by the user, the list of available smart tags automatically adjusts to retain just the tags that begin with the typed text.
    • Confirm that typing a space character after the smart tags list is provided automatically expands the selected tag into the associated text.

    • Confirm that typing text that matches the beginning of a smart tag except for differences in case works properly.
    • Confirm that text is correctly expanded even though the case is different than the database string.

  • [DataEntry #936] Smart tags in a bad state
    • Open a Demo_LabDE sample image in a new FAM instance.
    • Type .comments followed by a space in the Comments field.
    • Confirm that the text is expanded.
    • Skip the document without saving changes.
    • Type .comments followed by a space in the Comments field for the second document.
    • Confirm that the text is expanded.
    • Confirm that subsequent text is completely visible as it is being typed.

  • [DataEntry #937] Unexpected exception
    • Open a Demo_LabDE sample image in a new FAM instance.
    • Right-click in the Medical Record # field.
    • Close the verification UI without saving changes.
    • Start processing again.
    • Right-click in the Medical Record # field.
    • Close the verification UI without saving changes.
    • Confirm that no exception is thrown and that the FAM remains in a good state.

    • Repeat the above tests by clicking in the Order Name cell and typing CMP before closing without saving.
    • Confirm that no exception is thrown and that the FAM remains in a good state.

  • [DataEntry #941] Crash on Alt+Tab
    • Open a Demo_LabDE sample image in a new FAM instance.
    • Drop-down the Gender combo box.
    • While the combo box is dropped down, navigate back to the FAM via Alt+Tab.
    • Confirm that the FAM does not crash.
    • Navigate back and forth between the FAM and the verification UI via Alt+Tab.
    • Confirm that the FAM does not crash.

  • [DataEntry #954] Gender changes on click
    • Begin testing the Demo_LabDE images with fresh VOA files.
    • Skip A003.tif.
    • Verify B221.tif.
    • Click in the gender combo box.
    • Confirm that Gender does not change to F.

  • [DataEntry #958] Auto-complete list in wrong place after Shift+Space
    • Verify an image.
    • Click in a populated cell that contains an auto-complete list. An example is the Units column in the Test table.
    • Confirm that all the text in the cell is selected.
    • Press Shift+Space to select the entire row.
    • Confirm that an auto-complete list is not displayed near the upper left corner of the display.

  • [DataEntry #385] Up/down arrow in table cells with auto-complete corrupts memory.
    • Open image A003.tif from Demo_LabDE.
    • Select the Units cell from one of the GFR tests.
    • Click F2 to enter edit mode.
    • Click space once and backspace once.
    • Press the down arrow a few times.
    • Confirm no exceptions or crashes occur.
