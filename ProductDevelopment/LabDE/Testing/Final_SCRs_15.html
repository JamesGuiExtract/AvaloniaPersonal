LabDE 1.5 Final SCR Testing

Purpose and Details

  • Purpose: Exercise recent fixed SCRs
  • Details: Recently added SCRs generally do not have official test cases present within the collected checklists. One part of validating a 1.5 release candidate is to explicitly exercise these otherwise untested recent changes.

SCRs

  1. [DataEntry #723] Resizing the DEP
    • Open a Demo_LabDE sample image.
    • Resize the DEP so that it is very wide.
    • Close LabDE.
    • Reopen LabDE.
    • Confirm that all DEP controls are properly visible.

  2. [DataEntry #754] Tabbing by row
    • Open a Demo_LabDE sample image.
    • Select a table that supports tabbing by row.
    • Manually select a single cell in the table.
    • Press TAB several times.
    • Confirm that individual cells are selected until reaching the end of the row.
    • Confirm that the next TAB selects the entire following row.
    • Manually select a single cell in the table.
    • Press SHIFT+TAB several times.
    • Confirm that individual cells are selected until reaching the beginning of the row.
    • Confirm that the next SHIFT+TAB selects the entire previous row.

  3. [DataEntry #762] Focus issues
    • Open a Demo_LabDE sample image.
    • Separate the DEP and the image window.
    • Select a cell for input in the DEP.
    • Put focus in the image window.
    • Use a shortcut key for the image window such as F10.
    • Confirm that the DEP is not brought into focus.
    • Perform regular keyboard input for the active cell.
    • Confirm that the DEP is brought into focus.

  4. [DataEntry #763, #786] SQL Compact Importer
    • Merge two similar databases using options to insert all valid rows and to skip any problem rows.
    • Confirm that a message is presented describing any problem rows.
    • Confirm that the LabOrderTest table is successfully merged.

  5. [DataEntry #766] Incorrect highlight height
    • Copy the Z151.tif.voa file from .\PVCS_Testing\DataEntry\766 to the appropriate local folder.
    • Open the Z151.tif Demo_LabDE sample image.
    • Examine the highlight for the Anion Gap test in the Unknown Order.
    • Confirm that the highlight is an appropriate height.

  6. [DataEntry #805] Save / commit without validation
    • Confirm that the .config file supports a SkipValidationIfDocTaggedAs setting.
    • Define an appropriate tag.
    • Open an image.
    • Modify the data so that it is not completely valid.
    • Apply the special tag to the document.
    • Confirm that the document can be saved without being valid.
    • Confirm that the FAM considers the document successfully processed.

  7. [DataEntry #814, #815, #816] Medical Record # and Order Number
    • Confirm that the medical record number is valid with or without a leading m or M or z or Z.
    • Confirm that the medical record number is valid with or without a trailing space character.
    • Confirm that the Order Number is valid only for digits.
    • Confirm that the Order Number is valid with or without a trailing space character.

  8. [DataEntry #818, LegacyRC #5624, #5625, #5630] Input event tracking
    • Prepare a verification FPS file with Enable input event tracking.
    • Confirm that the EnableInputEventTracking field in the DBInfo table is set to 1.
    • Start verifying documents during a specific (noted) minute. Make sure to perform some input events during the minute.
    • During another minute, only make input events by scrolling the scroll bar.
    • During another minute, only make input events by clicking the mouse in the scroll bar.
    • During another minute, only make input events by moving the mousewheel.
    • During another minute, do not make any input events.
    • During another minute, only make input events by typing text from the keyboard.
    • During another minute, only make input events by tabbing between cells and rows.
    • Stop verifying documents near the beginning of a specific (noted) minute.
    • Check input event tracking results
      • Run the Detailed usage report for specific user for specific day and select the appropriate user and day.
      • Confirm that input events were noted for each of the following
        • The minute during which processing started
        • The minute during which scrolling the scroll bar events were made
        • The minute during which mouse clicks in the scroll bar events were made
        • The minute during which mousewheel events were made
        • The minute during which keyboard events were made
        • The minute during which tabbing events were made
        • The minute during which processing stopped
      • Confirm that input events are not noted for the minute(s) in which events were not made.

  9. [DataEntry #820] Malfunction with v key
    • Open an image.
    • Copy row or cell data to the clipboard.
    • Select a different cell (without an associated auto-complete feature) and move the cursor to the end of the text.
    • Type a v character.
    • Confirm that the contents of the cell are not deleted.
    • Navigate to a cell with an associated auto-complete feature and move the cursor to the end of the text.
    • Type a v character.
    • Confirm that the contents of the cell are not selected.

  11. [DataEntry #824] Apply tags button

  12. [DataEntry #829] Result status field

  13. [DataEntry #831] Swipe append

  14. [DataEntry #833] Order mapper

  15. [DataEntry #835] Original Name column in Test table

  16. [DataEntry #845] Auto-zoom and fit mode
    • Open an image that contains many tests.
    • Click the Fit to page button.
    • Set the auto-zoom option to Zoom out if necessary to fit current selection.
    • Tab through cells until about halfway through the Tests.
    • Use the scrollbar to scroll the image window so that the highlighted test is all the way to the bottom of the image window.
    • Tab to the next test.
    • Confirm that the selected test is scrolled into visibility.

  17. [LegacyRC #5618] FAM allows action name expressions
    • Select Action dialog
      • Confirm that the Select Action dialog includes a combo box populated with each defined action.
      • Confirm that the Select Action dialog includes a Tag Selector button allowing the user to choose an action including a $ function.
      • Confirm that such an action can be saved and reloaded.
    • Functionality
      • Create a new database.
      • Confirm that the AutoCreateActions field in the DBInfo table is set to 0.
      • Add a Test_<user_name> action with your user name inserted in the appropriate location.
      • Add a Test action to the database.
      • Create and save a UserName.fps file that uses Test_$UserName() as the action.
      • Create and save a Test.fps file that uses Test as the action.
      • Confirm that files can be supplied and processed using each FPS file.
      • Delete or rename the Test action.
      • Confirm that opening Test.fps displays an error about the undefined action name.
      • Delete or rename the Test_<user_name> action.
      • Exercise UserName.fps
        • Confirm that opening the FPS file does not display an error about the undefined action name.
        • Confirm that exercising the FPS file displays an exception indicating that the specified action name is invalid.
        • Close the FPS file.
        • Set the AutoCreateActions field in the DBInfo table to 1.
        • Exercise the FPS file.
        • Confirm that the action is automatically created and that processing works without error.

  18. [LegacyRC #5627] Set File Action Status allows action name expressions
    • Manually Set Action Status dialog

      • Confirm that the Manually Set Action Status dialog includes a Tag Selector button allowing the user to choose an action including a $ function.
      • Confirm that status can be modified for an action that uses a $ function.
      • Confirm that trying to modify status for an action that uses a $ function that does not actually exist in the database displays a meaningful error.
