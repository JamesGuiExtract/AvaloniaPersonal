LabDE 1.0 Beta 3 Testing

Purpose and Details

  • Purpose: Exercise recent fixed SCRs
  • Details: Recently added SCRs generally do not have official test cases present within the collected checklists. One part of validating a Beta 2 release candidate is to explicitly exercise these recent changes.

SCRs

  1. [DataEntry #179, #564] Help Menu and About Box
    • Open a Demo_LabDE sample image.
    • Confirm that a Help menu exists.
    • Confirm that Help - LabDE Help displays the LabDE help file.
    • Confirm that Help - About LabDE displays the LabDE About box.
    • Confirm that the About box includes the framework version.
    • Confirm that the About box includes the application version.

  2. [DataEntry #254] Save Without Commit
    • Open a Demo_LabDE sample image.
    • Confirm that data in one or more fields are invalid.
    • Modify some data.
    • Confirm that Save and Commit requires all data to be valid.
    • Confirm that Save accepts invalid data.
    • Confirm that the output VOA file includes one or more invalid items or is lacking one or more missing but required items.

  4. [DataEntry #416] Code Fields Are Not Tab Stops
    • Perform Phase 1 verification of a test image.
    • Confirm that tabbing through fields in the DEP skips over the following
      • Ordering Physician Code
      • Orders - Epic Code
      • Selected Order Details - Lab Code
      • Test Code
    • Confirm that tabbing through fields in the DEP does not skip over the following fields if they previously had content
      • Comments (for operator's use)
      • Selected Order Details - Comment
      • Test Comment

  5. [DataEntry #487] Table Cells Are Marked Read When Viewed By Row
    • Perform Phase 1 verification of a test image.
    • Confirm that selecting a cell within a row marks that cell as read.
    • Confirm that selecting the entire row marks each cell in the row as read.

  6. [DataEntry #493] Result Date/Time Fields Now Have Highlights
    • Load a test image for which Result Date and Result Time are available.
    • Confirm that each item has an appropriate highlight.

  7. [DataEntry #505, #559] DataEntry and LabDE Have New Licenses
    • Confirm that processing of images with LabDE rules will not work with a FLEX Index Server license.
    • Confirm that processing of images with LabDE rules will work with a LabDE Server license.

  8. [DataEntry #528, #529] Validation Is Disabled For Ordering Physician In Phase 1
    • Perform Phase 1 verification of a test image.
    • Confirm that the Ordering Physician field is not validated.
    • Perform Phase 2 verification of a test image.
    • Confirm that the Ordering Physician field is validated.

  9. [DataEntry #530] Comments Can Be Added For Documents
    • Confirm that the DEP includes a Comments edit box.
    • Confirm that Comments provided during one verification session are retained and displayed during a subsequent verification session.

  10. [DataEntry #531] Documents Can Be Skipped
    • Confirm that a File - Skip document menu item is available.
    • Confirm that selecting the Skip document menu item saves current settings without validation and advances to the next document.
    • Confirm that skipping the last document saves current settings and stops processing.

    • Exercise DB Admin with respect to skipped files
      • Confirm that the Summary tab includes a column for Skipped files.
      • Confirm that the Summary tab is updated properly with changes in file status.
      • Confirm that Manually set action status includes options for Skipped files.
      • Confirm that when updating status on Skipped files, the combo box for By is enabled.
      • Confirm that the By combo box includes <any user> as the single option.

    • Exercise Processing Setup with respect to skipped files
      • Confirm that a File processing scope group box exists.
      • Confirm that the default option is to Process all files in the queue (in order of priority).
      • Confirm that the alternate option is to Process all files that I skipped.
      • Confirm that the associated combo box includes options for I and Anyone.
      • Confirm that processing files skipped by the current user works as expected.
      • Confirm that processing files skipped by any user works as expected.

  11. [DataEntry #533, #534, #562] Image Path Edit Box and Copy Button
    • Confirm that the DEP includes a read-only edit box containing the fully qualified path to the image.
    • Confirm that the DEP includes a Copy button that allows the image path to be copied to the Clipboard.
    • Confirm that the Copy button is disabled during Phase 1 verification.
    • Confirm that the Copy button is enabled during Phase 2 verification.

  12. [DataEntry #550] Tools - Options Menu
    • Confirm that selecting Tools - Options displays an Options dialog.
    • Confirm that three options are presented for Center on selected text and
      • Don't change zoom level
      • Zoom out if necessary to fit current selection
      • Automatically adjust zoom to area around current selection
    • Confirm that the slider bar is only enabled with Automatically adjust zoom to area around current selection.
    • Confirm that Don't change zoom level works as expected.
    • Confirm that Zoom out if necessary to fit current selection works as expected.
    • Confirm that Automatically adjust zoom to area around current selection works as expected. Exercise different levels of context around the selected data.

  13. Laboratory Name Field Has Highlights
    • Load a test image for which Laboratory information is available.
    • Select the Laboratory Name field.
    • Confirm that appropriate highlights are displayed.
