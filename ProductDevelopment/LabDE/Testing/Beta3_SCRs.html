LabDE 1.0 Beta 3 Testing

Purpose and Details

  • Purpose: Exercise recent fixed SCRs
  • Details: Recently added SCRs generally do not have official test cases present within the collected checklists. One part of validating a Beta 2 release candidate is to explicitly exercise these recent changes.

SCRs

  1. [DataEntry #179, #564] Help Menu and About Box
    • Open a Demo_LabDE sample image.
    • Confirm that a Help menu exists.
    • Confirm that Help - LabDE Help displays the LabDE help file.
    • Confirm that Help - About LabDE displays the LabDE About box.
    • Confirm that the About box includes the framework version.
    • Confirm that the About box includes the application version.

  2. [DataEntry #254] Save Without Commit
    • Open a Demo_LabDE sample image.
    • Confirm that data in one or more fields are invalid.
    • Modify some data.
    • Confirm that Save and Commit requires all data to be valid.
    • Confirm that Save accepts invalid data.
    • Confirm that the output VOA file includes one or more invalid items or is lacking one or more missing but required items.

      • Add 2345 to the end of each cell; re-select the row and re-check the tooltips (some tooltips should now be positioned beneath the text to allow room).
      • Go to page 2, rotate the image so that it is vertical, and repeat the previous steps.
      • Repeat the previous steps for page 3.
      • Repeat the previous steps for page 4.

      • Re-select the row and confirm that Jean is now positioned below the text to make room for the lengthened last name tooltip.
      • Repeat the previous steps for page 3.

      • On page 2, use the angular highlight tool to swipe ACCESSION #: A003388367 PATIENT ID #: into the Medical Record field.
