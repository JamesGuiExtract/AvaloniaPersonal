LabDE Pre-Code Freeze Testing

  1. Purpose

    • Confirm proper project settings in the code baseline.
    • Basic testing of core features to find problems before it is too late to fix them.
    • Performance testing of new features and enhancements to confirm expected behavior.
    • Review current state of documentation to confirm that contents reflect the expected behavior of the new release.
    • Testing of the LabDE demo to ensure effective operation after installation.

  2. Engineering Project Settings - All Projects

    • Confirm that Optimizations are set to Disabled (/Od) via C/C++ - Optimization.
    • Confirm that C++ Exceptions are set to Yes With SEH Exceptions (/EHa) via C/C++ - Code Generation.
    • Confirm that Generate Debug Info is set to Yes (/DEBUG) in Release mode via Linker - Debugging.
    • Confirm that all .NET projects' platform target is x86.

  3. Core Features - Exercise Testing Checklists

    • Sanity Tests - run on each supported operating system
    • Stress Testing - run on each supported operating system

    • Other Base Checklists - once each
    • Recently Fixed SCRs - basic testing

  4. Performance Testing

    • New Rule Objects
      • Create comprehensive set of automated tests.
      • Create Memory Leak test(s).
      • Exercise basic functionality with large set of images.

      • Order Mapper - Output Handler

    • New Task Objects
      • Create comprehensive set of automated tests.
      • Create Memory Leak test(s).
      • Exercise basic functionality with large set of images.

      • Convert VOA To XML - Task
      • Verify Extract Data - Task

  5. Documentation Review

    • Review LabDE.chm checking accuracy and completeness.
    • Review LabDEinstall.chm checking completeness of installation instructions.

  6. Demo Application

    • Confirm that all Demo_LabDE images are suitably sanitized.
    • Confirm that installation and basic testing of Demo_LabDE works on a clean machine.
