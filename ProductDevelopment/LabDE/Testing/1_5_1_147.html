LabDE 1.5.1.147 (1.5.1 RC13) Testing

Purpose and Details

  • Purpose: Exercise recent fixed SCRs
  • Details: Recently added SCRs generally do not have official test cases present within the collected checklists.

Fixes for Patch C thru RC12

  1. [DataEntry #240, 891] Font should be Verdana
    • Confirm that all data is displayed in Verdana.

  2. [DataEntry #702] Repeated pasting sometimes clears text
    • Copy the plain text of a name.
    • Select the Epic Care patient name row.
    • Paste into it.
    • Continue pasting ad nauseum.
    • Confirm that pasting does not clear the row.

  3. [DataEntry #908] Run with no files pending
    • Create an .fps file with a Verify extracted data task.
    • Ensure that there are no files pending in the queue and no file suppliers are enabled.
    • Run the .fps file.
    • Confirm no exceptions are seen.

  4. [DataEntry #841] Inserting rows in name tables
    • Open a document with a populated .voa file.
    • For the ordering physician table and each of the patient name tables:
      • Select the row in the table.
      • Press ctrl+i.
      • Confirm that no row is inserted.
    • Confirm that the ctrl+i shortcut can be used to insert a row in a multi-row table.

  5. [DataEntry #856] Tab order
    • Open a document with at least one test.
    • Confirm that tabbing by row is enabled.
    • Select a test name cell.
    • Press tab.
    • Confirm that selection advances by a single cell.

  6. [DataEntry #861, #932, #957] Validation
    • Open the image in I:\Common\Testing\PVCS_Testing\DataEntry\932 for verification.
    • In the COMPREHENSIVE PANEL order, select and cut the CREATININE row of the test table.
    • Select the C REACTIVE PROTEIN order.
    • Right-click in the row header for the C-Reactive Protein row in the test table and Insert copied row(s).
    • Select the C REACTIVE PROTEIN cell in the order table and type Creatinine, then press tab.
    • Confirm that the test name and code cells for the Creatinine row of the test table are marked valid.

    • Open a document with a valid ordering physician and a different valid copy-to physician.
    • Drag and drop the copy-to physician onto the ordering physician.
    • Confirm that the ordering physician is not flagged as invalid.

    • Open an image for verification that contains the name of a valid physician.
    • Manually enter a different valid physician into the ordering physician table.
    • Select that row of the table and swipe the valid physician name from the document.
    • Confirm that the newly-swiped physician is not flagged as invalid.

  7. [DataEntry #870] Object reference not set to an instance of an object
    • Open an document with an associated .voa file for verification.
    • Select an empty cell.
    • Switch focus to the FAM and press the stop button.
    • Confirm that no exception is seen.
      • If an exception was seen, close the exception window and press play again.
      • Note whether or not further exceptions are seen.

  8. [DataEntry #874] Smart tags in lab name/code/units fields

    • Type . to activate the smart tag drop down.
    • Press the down arrow.
    • Confirm that the first item in the list is not immediately expanded.
    • Repeat test in the lab code field.
    • Repeat test in the units field.

  9. [DataEntry #882] Order mapper mapping tests w/ no AKAs

    • Put the .rsd file from I:\Common\Testing\PVCS_Testing\DataEntry\882 in the same directory as the database.
    • Exercise that .rsd file in the rule tester.
    • Confirm that a Manual Diff order is found.

  10. [DataEntry #885] Confirm that the font size for tooltips is automatically reduced to ensure that the entire text of the tooltip will be visible when zoomed fit-to-page.

  11. [DataEntry #899] For rows in multi-row tables, confirm that selecting an entire row and pressing the delete key removes the row from the table.

  12. [DataEntry #933] F2 entering edit mode in wrong cell
    • Select a cell in a table and enter edit mode.
    • Select a populated cell in a different table such that it does not enter edit mode.
    • Press F2.
    • Confirm that edit mode is activated in the currently selected cell.

  13. [DataEntry #961] Ctrl+a clears autocomplete field in edit mode

    • Enter edit mode in that cell.
    • Press ctrl+a, then ctrl+a again.

  15. [DataEntry #975] Flag table column is not properly updated when copying/pasting entire rows.

  17. [DataEntry:959] Shift + space clears active cell

  18. [DataEntry:963] RCW exceptions from table operations.
    • In FAM mode, queue 2 documents with multiple test and order rows
    • Dragging
      • Drag one order row to a different position in the table.
      • Tab and ensure the first cell of the new row of both the order and test tables are in the tab order.
      • Drag one test component row to a different position in the same table.
      • Tab and ensure the first cell of the new row of the test table is in the tab order.
      • Drag one test component row into a different order.
      • Tab and ensure the first cell of the new row of both the order and test table are in the tab order.
    • Swiping
      • Select an entire order row.
      • Re-swipe the order's tests.
      • Ensure the tests are parsed correctly and that no tooltip is displayed indicating an error.
      • Tab and ensure the first cell of the new row of both the order and test table are in the tab order.
    • Save the document, and skip/commit to proceed to the next document.
    • Repeat the Dragging and Swiping tests above.

  19. [DataEntry:837] Image viewer exceptions when minimized or very small
    • In FAM mode, queue at least 3 documents for verification and start verifying.
    • Press F11 to move the image viewer to a separate window.
    • Minimize the image viewer window.
    • Minimize the image window.
    • Skip the current document and ensure the next document's data is loaded without exception.
    • Use both F7 and F8 (zoom in/out) and ensure no exceptions are thrown.
    • Press Ctrl + R to rotate the open image clockwise.
    • Press Alt + W to set fit-to-width mode.
    • Restore the image window and ensure the image is rotated clockwise and fit-to-width is set.
    • Resize the image window as small as possible (so only a small part of the title bar is visible).
    • Skip the current document and ensure the next document's data is loaded without exception.
    • Use both F7 and F8 (zoom in/out) and ensure no exceptions are thrown.
    • Press Ctrl + Shift + R to rotate the open image counter-clockwise.
    • Press Alt + P to set fit-to-page mode.
    • Restore the image window to a reasonable size and ensure the image is rotated counter-clockwise and fit-to-page is set.
    • Resize the image window as small as possible (so only a small part of the title bar is visible).
    • Press the Stop button in the FAM to stop processing.
    • Press the Start button to restart verification.
    • Ensure the document data is loaded without exception.
    • Restore the image window to a reasonable size and ensure the image is loaded.
