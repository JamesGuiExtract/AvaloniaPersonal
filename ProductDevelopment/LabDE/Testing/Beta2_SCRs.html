LabDE 1.0 Beta 2 Testing

Purpose and Details

  • Purpose: Exercise recent fixed SCRs
  • Details: Recently added SCRs generally do not have official test cases present within the collected checklists. One part of validating a Beta 2 release candidate is to explicitly exercise these recent changes.

SCRs

  1. [DataEntry #305, #490, #491, #492] Drag and Drop of Table Rows
    • Open a Demo_LabDE sample image.
    • Confirm that one or more test rows can be dragged to a different location within the same table.
    • Confirm that one or more test rows can be dragged to a compatible table (i.e. between Physician tables).
    • Confirm that one or more test rows can be dragged to a different parent attribute (i.e. across Orders).
    • Confirm that tooltips are not displayed during a drag & drop operation.
    • Examine the tooltips of the affected rows after drag & drop and confirm that the correct tooltips are associated with each element.
    • Examine tab order of cells after drag & drop and confirm that the tab order is correct.

  2. [DataEntry #451] Order Mapper and Multi-threaded FAM
    • Exercise the Demo_LabDE rules for multiple images within a multithreaded FAM instance.
    • Confirm that Order Mapper results are as expected with no unexpected exceptions.

  3. [DataEntry #471] Medical Record Number is Required
    • Exercise second-stage verification.
    • Confirm that an output VOA file cannot be saved without a valid Medical Record Number.

  4. [DataEntry #473] LabDE Help File
    • Confirm that a Help file is presented to the user when accessed from the Start menu.

  5. [DataEntry #474] Delete RSD and DAT Files
    • Review the Demo_LabDE Rules folder.
    • Confirm that all relevant files are encrypted.

  6. [DataEntry #482] FAM Auto-close After Last Document
    • Verify several documents.
    • Confirm that the FAM closes automatically after results are saved for the last document.

  7. [DataEntry #494] Collection Date and Collection Time
    • Verify several documents.
    • Confirm that Collection Date results are available from the rules.
    • Confirm that Collection Time results are available from the rules.
