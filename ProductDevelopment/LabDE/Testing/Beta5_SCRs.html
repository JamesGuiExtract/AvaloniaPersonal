LabDE 1.0 Beta 5 Testing

Purpose and Details

  • Purpose: Exercise recent fixed SCRs
  • Details: Recently added SCRs generally do not have official test cases present within the collected checklists. One part of validating a Beta 5 release candidate is to explicitly exercise these recent changes.

SCRs

  1. [DataEntry #173] Trim Whitespace on Load
    • Open a Demo_LabDE sample image.
    • Confirm that leading whitespace is not seen for any attributes.
    • Confirm that trailing whitespace is not seen for any attributes.

  2. [DataEntry #211] Case Sensitivity in Validation
    • Open a Demo_LabDE sample image.
    • Modify a Units value using a different case than provided in the validation list.
    • Confirm that the new casing is accepted as valid.

  3. [DataEntry #273, #594] Carriage Returns Removed From Fields
    • Open a Demo_LabDE sample image.
    • Swipe the name of a lab that spans two or more lines.
    • Confirm that the string result replaces any carriage return characters with a single space.
    • Swipe other text spanning two or more lines into a multi-line Comment field.
    • Confirm that the string result retains carriage return characters and displays the result on multiple lines.

  4. [DataEntry #311] Limits on Zoom
    • Open a Demo_LabDE sample image.
    • Zoom in as far as you can go or until the image disappears.
    • Click on a populated field in the DEP.
    • Confirm that no ELI22431: Cannot be less than 1. exceptions are seen.
    • Zoom out a long ways.
    • Confirm that no ELI21133: Unable to draw image. exceptions are seen.

  5. [DataEntry #312] Escape During Pan
    • Open a Demo_LabDE sample image.
    • Select the Pan tool and navigate around the image.
    • Click the Escape key during a Pan operation.
    • Confirm that the Pan operation ends and that the cursor returns to normal.
    • Zoom out a long ways.
    • Confirm that no ELI21133: Unable to draw image. exceptions are seen.

  6. [DataEntry #316] Keyboard Navigation During Swipe
    • Open a Demo_LabDE sample image.
    • Start swiping some text.
    • Confirm that keyboard navigation of the menu is not supported during a swipe.

  7. [DataEntry #326] VOA Viewer
    • Open a VOA file created from a Demo_LabDE sample image.
    • Confirm that changes can be saved as an EAV file.
    • Confirm that changes can be saved as a VOA file.

  8. [DataEntry #399] Support For Remote Database
    • Prepare a remote testing machine to host the LabDE Solution and associated database file. Note the size of the remote database file.
    • Update one or more local .config files to reference the remote machine via UNC path for the LocalDataSource setting.
    • Open a local Demo_LabDE sample image.
    • Confirm that a tmp file is created in the local temp folder that is the same size as the remote database file.
    • Confirm that changes can be made to the local VOA file.

  9. [DataEntry #415] Tab Key Behavior
    • Open a Demo_LabDE sample image.
    • Select the starred new row of the Order table.
    • Swipe one entire order.
    • Click F2 to enter edit mode in the order name cell.
    • Backspace to remove the name found by the rules.
    • Start typing a valid order name.
    • Press Tab once auto-complete has supplied the end of the order name.
    • Confirm that focus moves to the next cell in the row instead of to the cell below the order name.

  10. [DataEntry #431] Validation of Date Time Fields
    • Open a Demo_LabDE sample image.
    • Select a Result Date field.
    • Confirm that obviously incorrect dates such as 42/01/2009 cannot be entered.
    • Select a Result Time field.
    • Confirm that obviously incorrect times such as 42:73 cannot be entered.
    • Select a Date of Birth field.
    • Confirm that obviously incorrect dates such as 42/01/2009 cannot be entered.

  11. [DataEntry #450] Unwanted Enter Key Behavior
    • Open a Demo_LabDE sample image.
    • Select any cell but the first in any row of a table.
    • Do something to cause a dialog to appear such as clicking F4 when there are no unviewed items.
    • Click the Enter key.
    • Confirm that cell selection is unchanged.

  12. [DataEntry #461] Document Printing Without Highlight, Tooltip or Error Items
    • Open a Demo_LabDE sample image.
    • Select some data so that highlights are visible on the document.
    • Confirm that highlights are not included on printed output.
    • Select some data so that tooltips are visible on the document.
    • Confirm that tooltips are not included on printed output.
    • Select some data so that an error icon is visible on the document.
    • Confirm that the error icon is not included on printed output.

  13. [DataEntry #464] Pasting Text
    • Exercise the 003.tif image from Demo_LabDE.
    • Click in the Test Name field in the starred row.
    • Paste plain text from the Clipboard into the cell and press Enter.
    • Repeat several times.
    • Confirm that no error is seen.

  14. [DataEntry #467] Missing Highlights
    • Exercise the image and VOA file located at I:\Common\Testing\PVCS_Testing\DataEntry\467.
    • Select the first Reference Range cell for the first order.
    • Confirm that a highlight / tooltip is displayed.

  15. [DataEntry #470] MessageSequenceNumber.txt
    • Confirm that the MessageSequenceNumber.txt file is located in the Solution folder.
    • Process a Demo_LabDE sample image for SendToEMR.
    • Confirm that the MessageSequenceNumber.txt file is updated.

  16. [DataEntry #472] Enforce Field Length Constraints
    • Process a Demo_LabDE sample image for MatchToPatientAndOrders.
    • Add a Copy To: Physician longer than 24 characters.
    • Modify the Lab Name field to be longer than 24 characters.
    • Save the VOA file.
    • Process the same image for SendToEMR.
    • Confirm that the Copy To: Physician field is limited to 24 characters.
    • Confirm that the Lab Name field is limited to 24 characters.

  17. [DataEntry #556] Table Colors For Disabled & Inactive
    • Open a Demo_LabDE sample image.
    • Confirm that the background color used for a disabled table row such as Ordering Physician is different than the color used for a selected but inactive row.

  18. [DataEntry #613, #618] Lab Name Control
    • Open a Demo_LabDE sample image.
    • Confirm that the Lab Name data no longer is initially displayed as already viewed.
    • Confirm that the Lab Name control is an edit box instead of a combo box.

  19. [DataEntry #619] Lab Code Updates Lab Name
    • Exercise the 003.tif image from Demo_LabDE.
    • Click in the Lab Code field and enter acl.
    • Confirm that the text is modified to uppoer case.
    • Confirm that the Lab Name field is updated.

  20. [DataEntry #623] Toolbar Button Text
    • Open a Demo_LabDE sample image.
    • Confirm that the Save and commit toolbar button is properly in sentence case.

  21. [DataEntry #624, #632] Application Switching During Swipe
    • Remove the USB key from the local machine or remove the configuration to a remote machine.
    • Open a Demo_LabDE sample image.
    • Swipe a Test into a Test row.
    • Confirm that LabDE is waiting because a USB key is not available.
    • Use USB License Key Manager to assign a key.
    • Confirm that the rules finish running and results are properly added to the specified row.

  22. [DataEntry #626] Dragging Whole Order
    • Exercise the 151.tif image from Demo_LabDE.
    • Select all rows in the Order table.
    • Scroll down until the Test table is visible.
    • Click on a row header in the Order table and drag the row around the table.
    • Confirm that the selected row is dragged over a different row, the Selected Order Details table does not update based on the different row.

  23. [DataEntry #627] Physician Name
    • Open a Demo_LabDE sample image.
    • Confirm that the Physician Name tables now include a column for Middle.
    • Confirm that the Physician Name tables are now ordered as Last, First, Middle.

  24. [DataEntry #629, #631, #646] LabDE Splash Screen and Installation
    • Confirm that the LabDE installation splash screen does not reference FLEX Index.
    • Confirm that the installation no longer asks for a destination directory.
    • Confirm that SqlCompactExporter and SqlCompactImporter utilities are included with the installation.

  25. [DataEntry #633] Rule Execution Stack Is Empty
    • Open a Demo_LabDE sample image.
    • Swipe an Order into an Order row.
    • Confirm that the rules finish running and results are properly added to the specified row.
    • Confirm that no ELI11649: Rule execution stack is empty. exceptions are seen.

  26. [DataEntry #634] Tab Stops For Middle Name & Suffix
    • Open a Demo_LabDE sample image.
    • Confirm that the Middle Name and Suffix cells are tab stops only if they contain text.

  27. [DataEntry #641] Cell Selection And Delete
    • Open a Demo_LabDE sample image.
    • Select one or more cells of a row in the Test table.
    • Confirm that Delete removes the highlights and text for the selected cells.
    • Confirm that the highlights and text for the non-selected cells are not affected.

  28. [DataEntry #642] Deleting Spatial Info In Dependent Controls
    • Open a Demo_LabDE sample image.
    • Select a row in the Order table.
    • Type Ctrl+D to remove the highlights for the order.
    • Confirm that the highlights for the rows and cells in the Test table are retained.

  29. [DataEntry #643] Support For Order Code And Test Code
    • Open a Demo_LabDE sample image.
    • Confirm that NA (for SODIUM) is accepted as an entry in the Test Name cell.
    • Confirm that BMP1 (for BASIC METABOLIC PANEL) is accepted as an entry in the Order Name cell.
