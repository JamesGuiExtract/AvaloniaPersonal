LabDE Post-Release Activities

  1. Source Code Backup - On the Build Machine

    • Rename the D:\temp folder on the build machine to temp a.b.c.d.
    • As Vault and/or domain administrator.
      • Undo current checkouts.
      • Copy the current source folder to an appropriately named folder under I:\Common\Engineering\VSSReleaseBuilds.

  2. Documentation Backup - From I:\Common\Tech Comm

    • Zip the I:\Common\Tech Comm\RoboHELP\File Action Manager folder and store it in I:\Common\Tech Comm\RoboHELP\Archives.

  3. Release Notes and Licensing

    • Create an email to company employees describing in fair detail each of the new features and enhancements associated with the new release.
    • Update the Notable LabDE Versions.rtf file located at I:\Common\Engineering\ProductReleases\LabDE.
    • Update the I:\Common\Engineering\Tools\SecureClients\COMLicense folder used to create license files for the new release.
    • Confirm that the new licensing folder cannot be used to create the following license packages:
      • Disable USB Key Serial Numbers
      • Rule Development Toolkit
      • Simple Rule Writing

  4. Post-Release Testing Activities

    • Finish exercising any remaining testing checklists.
    • Finish creating and exercising any missing automated tests.
    • Finish creating and exercising any missing Memory Leak tests.
    • Update testing checklists per comments and notes from testers.

    • Review and test the Untested SCRs in DataEntry.
    • Review and test the Untested SCRs in DotNetRCAndUtils.
    • Review and test the Untested SCRs in FlexIDSCore.
    • Review and test the Untested SCRs in LegacyRCAndUtils.
