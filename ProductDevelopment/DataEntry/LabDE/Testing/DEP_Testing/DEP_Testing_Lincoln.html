DEP Testing - NM - Lincoln

  1. DEP Layout

    • Confirm that the DEP includes a Book edit box.
    • Confirm that the DEP includes a Page edit box.
    • Confirm that the DEP includes a Date Filed edit box.
    • Confirm that the DEP includes a Date of Instrument edit box.
    • Confirm that the DEP includes a Document Type drop-down combo box with a large number of entries available.
      • Confirm that Lincoln.sdf includes a DocumentType table with the associated entries.
    • Confirm that the DEP includes a Grantor group box.
      • a table with column headings for:
        • Name (Last First Middle Suffix)
        • Deceased?
    • Confirm that the DEP includes a Grantee group box.
      • a table with column headings for:
        • Name (Last First Middle Suffix)
        • Deceased?
    • Confirm that the DEP includes a Subdivision group box.
      • a table with column headings for:
        • Block
        • Lot
        • Other
        • Subdivision
        • Unit
      • Confirm that the Subdivision cell is a drop-down combo box.
      • Confirm that Lincoln.sdf includes a Subdivision table with the associated entries.
    • Confirm that the DEP includes a Town Range Section group box.
      • a table with column headings for:
        • Town Range Section
        • Description

  2. Validation

    • Confirm that the following validations exist when verifying the rules results from a blank file.
      • The rules find Document Type as VOID.
      • The Book field must be empty.
      • The Page field must be empty.
      • The Date Filed field must be empty.
      • The Date of Instrument field must be empty.
      • The Grantor field must be empty.
      • The Grantee field must be empty.
      • The Subdivision/Block field must be empty.
      • The Subdivision/Lot field must be empty.
      • The Subdivision/Other field must be empty.
      • The Subdivision/Unit field must be empty.
      • The Town Range Section field must be empty.
      • The Description field must be empty.
    • Confirm that the following validations exist when verifying the rules results from a non-blank file.
      • The Subdivision/Block + Lot + Unit field is limited to 30 characters.
      • The Town Range Section field cannot be specified if Subdivision/Block or Lot or Other or Unit is defined.
      • The Description field cannot be specified if Subdivision/Block or Lot or Other or Unit is defined.
      • At least one Grantor must be present.
      • At least one Grantee must be present.
      • The Grantor field is limited to 32 characters if Deceased? is false.
      • The Grantor field is limited to 26 characters if Deceased? is true.
      • The Grantee field is limited to 32 characters if Deceased? is false.
      • The Grantee field is limited to 26 characters if Deceased? is true.

  3. Auto-Update Queries

    • VOID Document Types and Grantor / Grantee
      • Open a document with VOID doc type.
      • Confirm that no warnings are displayed when the other fields are properly blank.
      • Add some Grantor text.
      • Confirm that a warning icon is displayed.
      • Delete the Grantor text and confirm that the warning is automatically cleared.
      • Repeat the test and confirm that Undo of the changes clears the warning.
      • Add some Grantee text.
      • Confirm that a warning icon is displayed.
      • Delete the Grantee text and confirm that the warning is automatically cleared.
      • Repeat the test and confirm that Undo of the changes clears the warning.
