LabDE 1.0.3.189 Testing

Purpose and Details

  • Purpose: Exercise recent fixed SCRs
  • Details: Recently added SCRs generally do not have official test cases present within the collected checklists.

SCRs

  1. [DataEntry #727, #728] MultiMatchSelectionMode
    • Open the VOA file from I:\Common\Testing\PVCS_Testing\DataEntry\727.
    • Confirm that several Test components include multiple Units subattributes.
    • Open the image from I:\Common\Testing\PVCS_Testing\DataEntry\727.
    • Confirm that no exception is thrown.
    • Confirm that for the Test components with multiple Units subattributes, the displayed Units are empty.

  2. [DataEntry #729] Order Mapper and temporary database
    • Process an image.
    • Make a modification to the original database.
    • Process another image.
    • Confirm that the temporary database has been updated to reflect the recent change.

  3. [DataEntry #740] Filename and action name in window caption
    • Create an FPS file to verify images.
    • Exercise the FPS file without queuing any files.
    • Confirm that the window caption is Waiting - LabDE (<action_name>).
    • Queue some files and exercise the FPS file again.
    • Confirm that the window caption is <SourceDocName> - LabDE (<action_name>).

  4. [DataEntry #742] Validation of medical record number
    • Exercise second-stage verification.
    • Confirm that the medical record number is valid with or without a leading M or Z.

  5. [DataEntry #743] No Tab stop for Result Status
    • Process an image.
    • Confirm that the Result Status field is no longer a tab stop.

  6. [DataEntry #744] Default value for order-level comment
    • Process several documents.
    • Confirm that the default value for the order-level comment is Order entered by $FullUserName() on $Now().
    • Confirm that the order-level comment field can be modified.

  7. [DataEntry #745] Left and right arrows in edit mode
    • Process an image.
    • Confirm that while editing a cell, the right arrow key does not change cell selection.
    • Confirm that while editing a cell, the left arrow key does not change cell selection.

  8. [DataEntry #746] Multi-cell support for copy and paste
    • Process several documents.
    • Confirm that the collection date and time values can be copied from one order and pasted into another order.
    • Confirm that the units value for a single test can be pasted into multiple tests.

  9. [DataEntry #747] Shift+Space shortcut
    • Process an image.
    • Select a cell in a table and press Shift+Space.
    • Confirm that the whole row is selected and that the user can review whole rows by navigating with the up and down arrows.
    • Select another cell and begin an edit operation.
    • Press Shift+Space.
    • Confirm that the cell editing operation is completed and the whole row is selected.
    • Confirm that the user can review whole rows by navigating with the up and down arrows.
