LabDE 1.0 Beta 6 Testing

Purpose and Details

  • Purpose: Exercise recent fixed SCRs
  • Details: Recently added SCRs generally do not have official test cases present within the collected checklists. One part of validating a Beta 6 / 1.0 release candidate is to explicitly exercise these otherwise untested recent changes.

SCRs

  1. [DataEntry #583] Tooltips And Image Areas
    • Supply and Verify the two images located at I:\Common\Testing\PVCS_Testing\DataEntry\583.
    • Select the Lab Name field for 001.tif.
    • Confirm that the tooltip is properly placed relative to the highlight.
    • Select the Lab Name field for 008.tif.
    • Confirm that the tooltip is properly placed relative to the highlight.

  2. [DataEntry #645] Highlights With Mixed Row And Cell Selection
    • Open a Demo_LabDE sample image that contains an order with multiple tests.
    • Click in the row header of the first test to select the first test row.
    • Scroll to the bottom of the test table.
    • Holding the Shift key, click in the comment cell of the last test row.
    • Confirm that the first row is highlighted in the image.
    • Confirm that the other selected cells are also highlighted in the image.

  3. [DataEntry #653] Clicking On Last Visible Order Row
    • Open a Demo_LabDE sample image such as 413.tif that contains multiple orders.
    • Click on the last visible Order in the Order table.
    • Confirm that the chosen Order is properly selected.

  4. [DataEntry #654] Ordering Physician Table And DEP Resize
    • Open a Demo_LabDE sample image.
    • Resize the DEP smaller.
    • Confirm that the Ordering Physician table never gets smaller such that information is inappropriately hidden.

  5. [DataEntry #659] Enabling Of Swiping Tools
    • Open a Demo_LabDE sample image.
    • Select the Lab Name field.
    • Confirm that the swiping tools are disabled in the toolbar.
    • Select the Zoom, Pan or Review tool.
    • Confirm that the swiping tools are still disabled in the toolbar.

  6. [DataEntry #660] Empty Lab Fields Not Persistent
    • Open a Demo_LabDE sample image that contains Lab Name information.
    • Clear the Lab Name fields.
    • Stop processing and save changes.
    • Re-open the image.
    • Confirm that the Lab Name fields remain cleared.

  7. [DataEntry #661] Missing Tooltip
    • Open 352.tif and click F10 to show all highlights.
    • Select an item on page 1.
    • Confirm that the appropriate tooltip is shown.
    • Navigate to page 2.
    • Click F10 twice so that show all highlights is on again.
    • Navigate back to page 1.
    • Confirm that the original tooltip is still visible.

  8. [DataEntry #662] Swiping Backwards
    • Open a Demo_LabDE sample image.
    • Use the angular swiping tool to swipe a test row backwards.
    • Confirm that the test information is properly formatted.
    • Confirm that ELI25004: Nullable object must have a value. exceptions are not seen.

  9. [DataEntry #663] Installation Of Message Queuing
    • Open Add or Remove Programs from the COntrol Panel.
    • Click on Add/Remove Windows Components.
    • Confirm that Message Queuing is installed.

  10. [DataEntry #677] Hint In Wrong Location
    • Open Demo_LabDE sample image 003.tif.
    • Add the comment paragraph starting with 10/13/2008... as the comment to the GFR African American test.
    • Click on the Ref Range column for that test.
    • Confirm that the hint is appropriately placed.

  11. [DataEntry #679] Show All Highlights And New Swipes
    • Select a Demo_LabDE sample image that contains a lot of tests.
    • Delete or rename the VOA file.
    • Open the image.
    • Select the new row in the Order table.
    • Swipe as many tests as possible with a single swipe.
    • Click F10 to show all highlights.
    • Confirm that highlights are shown for the recent swipe.

  12. [DataEntry #680] Highlighting Error
    • Open a Demo_LabDE sample image that contains a lot of tests.
    • Change the name of each order to B12 or something else such that all of the tests are invalid.
    • Activate the Review and select tool.
    • Click Esc to hide any visible tooltips.
    • Hover the mouse over a test name such that the tooltip will overlap with an error icon in the image.
    • Confirm that the error icon is not on top of the tooltip.
