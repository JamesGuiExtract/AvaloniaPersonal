Demo_LabDE Testing Checklist

NOTE: See the Bidirectional Workflow checklist for additional test cases.

  1. Instructions and Installation

    • Confirm that the latest Demo_LabDE folder can be copied from the current build folder to C:\.
    • Confirm that all files under the Rules folder are encrypted.
    • Confirm that DataEntry: Verify extracted data requires only a LabDE Client license.
    • Confirm that the blank database associated with the demo has the current schema version.

  2. Demo Images

    • Exercise the installed rules and images.
    • Review the following items after finishing the verification
      • Confirm that VOA files were updated for each file in which changes were made.

  3. FPS Files and Context

    • Confirm that the FPS folder contains CustomTags.sdf.
    • Confirm that each FPS file uses <DatabaseServer> and <DatabaseName> tags in a Demo context.
    • Confirm that the following tags are defined.
      • DatabaseServer
      • DatabaseName
      • InputFolder
      • RulesFile
      • VerifyConfigFile

  4. User Interface

    • General
      • Confirm that unviewed text is displayed in Bold font.
      • Confirm that viewed text is displayed in normal font.
      • Confirm that presence and of an Undo command. Test its behavior:
        • in different controls (party table, address table, text boxes...)
        • after saving
        • editing multiple attributes at once
        • using the button, keyboard shortcut, and menu item
        • any other situations that come to mind
      • [ISSUE-12810] Confirm that the toolbar includes a toggle button for showing and hiding the Thumbnails pane.

    • Comments (for operator's use)
      • Confirm that a multi-line edit box exists for Comments.
      • Confirm that text can be swiped into the edit box.

    • Duplicate Documents
      • Confirm that a Duplicate document(s) button is available.
      • Confirm that the number preceding the text on the button changes as appropriate.
        • It should be non-zero when processing the Extended folder of images.

    • Refresh Database Data
      • Confirm that a Refresh database data button is available.

    • Patient Information
      • Confirm that a single-row table is present with the following column headings.
        • First Name
        • Middle Name
        • Last Name
        • Suffix
      • Confirm that a Patient name can be swiped into the selected row.
      • Confirm that a Date of Birth edit box is present.
      • Confirm that a Gender combo box is present with the following options.
        • M
        • F
        • U
      • Confirm that a Medical Record # edit box is present.

    • Physicians
      • Confirm that a single-row table for Ordering Physician is present with the following column headings.
        • Last Name
        • First Name
        • Middle Name
        • Code
      • Confirm that a Physician name can be swiped into the selected row.
        • Confirm that Code is automatically updated for a valid Physician.
      • Confirm that a multiple-row table for CopyTo Physician is present with the following column headings.
        • Last Name
        • First Name
        • Middle Name
        • Code
      • Confirm that a Physician name can be swiped into the selected row.
        • Confirm that Code is automatically updated for a valid Physician.

    • Orders
      • Confirm that columns exist for
        • Order Num
        • Order Picker (button, without a header label)
        • Order Name
        • Collection Date
        • Collection Time
      • Confirm that an Order can be swiped into the selected row.
      • Confirm that text can be swiped into the following individual cells.
        • Order Name
        • Collection Date
        • Collection Time

    • Selected Order Details
      • Confirm that Lab Filter edit boxes exist for
        • Name
        • City
        • State
      • Confirm that a single-line edit box is provided for Lab Name.
      • Confirm that the displayed Lab Name is a concatenation of Lab Name, Street address, and City.
      • Confirm that a Result Date edit box is available.
      • Confirm that a Result Time edit box is available.
      • Confirm that a multiple-line edit box is provided for order-level Comment.
        • Confirm that the default text is Verified by $UserName() on MM/DD/YYYY.
      • Confirm that an unlabeled multiple-line table is provided for Test Components.
        • Test Name
        • Value
        • Units
        • Ref. Range
        • Flag
          • [BLANK]
          • A
          • H
          • HH
          • I
          • L
          • LL
          • N
          • R
          • S
        • Status
          • C
          • F
          • P
      • Confirm that a multiple-line edit box is provided for test-level Comment.

  5. Validation

    • Patient
    • Physician
    • Order
    • Selected Order Details

  6. Enable Logging

    • Confirm that the test machine includes a utility license for Data Entry Verification Logging.
    • Prepare files for verification.
    • Modify the .CONFIG file to include the following text within the Extract.DataEntry.Utilities.DataEntryApplication.Properties.Settings section.
      • <setting name="EnableLogging" serializeAs="String">
      •   <value>True</value>
      • </setting>
    • Run the I:\users\Steve_Kurth\Public\Utilities\DbgView.exe utility that will capture and display real-time logging results.
    • Verify an image. Swipe some text, exercise Undo and Redo functionality, and delete an attribute.
    • Confirm that DbgView displays trace information as:
      • Category [control name] (attribute name): Trace text...
    • Confirm that the following Category information is presented by default.
      • AutoUpdateResult
      • DataLoad
      • DataSave
      • AttributeInitialized
      • AttributeUpdated
      • AttributeDeleted
      • SwipedText
      • TooltipNotification
        • Swipe empty text into an Order or Test field
      • Undo
      • Redo
      • Focus
      • InputEvent
      • FormattingRuleResult
    • Exercise filtering with the following syntax. Note that * is replaced by a comma-delimited list of categories.
      • <setting name="LogFilter" serializeAs="String">
      •   <value>*</value>
      • </setting>
    • Confirm that the following categories can also be logged - via filtering.
      • AutoUpdateQuery
      • ValidationQuery
      • ValidationResult

  7. Miscellaneous

    • [ISSUE-13135] Confirm that a meaningful exception is displayed and that verification halts if logging is enabled but not licensed.
