LabDE 1.0 Final SCR Testing

Purpose and Details

  • Purpose: Exercise recent fixed SCRs
  • Details: Recently added SCRs generally do not have official test cases present within the collected checklists. One part of validating a 1.0 release candidate is to explicitly exercise these otherwise untested recent changes.

SCRs

  1. Stress Testing with various images, builds, rules, O/S
    • Review the various LogProcessStats / Stress Test results executed on post-Beta 5 builds.
    • Confirm that no unusual memory leaks are present.
      • Compare results for the same image sets, rules and build exercised on different operating systems.
      • Compare results for different image sets but the same rules, build and operating systems.
      • Compare results for the same image sets, build and operating system exercised on different rules.
      • Compare results for the same image sets, rules and operating system exercised on different builds.

  2. Miscellaneous
    • Confirm that all table columns are now fully resizable.
    • Confirm that Result date is a required field for first-stage verification.

  3. [LegacyRC #5460] Unstoppable FAM
    • Configure an FPS file to supply and process while the file supplier is listening.
    • After queuing and processing has started confirm that processing can be stopped without error.

  4. [DataEntry #693] Verify Extracted Data Memory Leak Test #1
    • Confirm that a recent memory leak test has been run and that no unexpected results are seen.

  5. [DataEntry #695] User License Shows LabDE Version Number
    • Run the User License utility.
    • Confirm that Extract Systems LabDE is available as the Product.
    • Confirm that the proper version number is displayed.

  6. [DataEntry #696] Deleting Spatial Info And Indirect Hints
    • Open a Demo_LabDE sample image.
    • Select a Test - Comment cell.
    • Confirm that the indirect hint shows dotted-line boxes around other items in the row.
    • Select a populated cell in the same row and press Delete.
    • Select the Test - Comment cell again.
    • Confirm that the indirect hint does not show a dotted-line boxes around the information from the deleted cell.

  7. [DataEntry #697] Deleting All Orders
    • Open a Demo_LabDE sample image containing multiple Orders.
    • Select all the Orders and press Delete.
    • Confirm that all of the Orders are deleted without an exception.
