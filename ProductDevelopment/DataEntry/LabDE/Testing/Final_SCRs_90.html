LabDE 9.0 Final SCR Testing

Purpose and Details

  • Purpose: Exercise recent fixed SCRs
  • Details: Recently added SCRs generally do not have official test cases present within the collected checklists. One part of validating a 9.0 release candidate is to explicitly exercise these otherwise untested recent changes.

SCRs

  1. Descriptions of DataEntry Changes

    • [DataEntry #1082] Confirm that the LabDE splash screen does not contain any patent information.

    • [DataEntry #695, #1014, #1091] LabDE Version Number
      • Confirm that User License provides the correct version number for a LabDE license request.
      • Confirm that the LabDE RESX file has the correct version number.
      • Confirm that the LabDE About box provides correct version numbers for both Application and Framework.
      • Confirm that the Demo_FLEXIndex About box provides the correct version number.

    • [DataEntry #1080] SupportsSwiping Disabled
      • Confirm that the Word highlight tool is properly disabled when the other swiping tools are disabled.

    • [DataEntry #1081] Support For Multiple DEPs
      • Configure a multi-DEP scenatio where each DEP uses a different SQL CE database.
      • Begin verification. After a document is displayed, change the document type to load the document in a different DEP.
      • Confirm that the DEP change is not unusually slow.
      • Confirm that no exceptions are seen.

    • [DataEntry #1087] LabDE Start Menu Shortcuts
      • Confirm that Configure FAM Service opens the expected SDF file.
      • Confirm that View Exception Log opens the expected UEX file.

    • [DataEntry #1089] Confirm that DataEntryApplication.LabDE.resources is installed in the CommonComponents folder.

    • [DataEntry #1092] Data Entry Exception
      • Exercise verification for Demo_LabDE.
      • Verify Z151.tif and validate everything except the Basic Metabolic Panel order.
      • Add a BUN test with appropriate information.
      • Select the next empty row in the Tests table and add a K for the missing POTASSIUM test.
      • Confirm that an unhandled exception prompt is not displayed.
      • Confirm that the remaining missing tests can be added and the verification can be completed.

    • [DataEntry #1094] DEP Versions
      • Confirm that the FLEX Index related DEPs have a proper version number.
      • Confirm that the LabDE related DEPs have a proper version number.

    • [DataEntry #1095, #1099, #1100] LabDE Reports
      • Confirm that the two standard LabDE reports are installed correctly and available via DB Admin.
      • Confirm that preview images are available for each LabDE report.
      • Confirm that the LabDE Data Entry Rates By User report works correctly and displays meaningful data.
      • Confirm that the LabDE High Effort Files By User report works correctly and displays meaningful data.

    • [DataEntry #1097] Confirm that the Other Products section of the Demo Shield install no longer includes a reference to IcoMap.
      • Confirm the same for the FLEX Index Demo Shield install.
      • Confirm the same for the ID Shield Demo Shield install.

    • [DataEntry #1104] Validation Queries
      • Confirm that the H, HH or A are acceptable Flag values for results that are above the reference range.
      • Confirm that the L, LL or A are acceptable Flag values for results that are below the reference range.
      • Confirm that the above results hold for both the LabDE and the HealthNetworkLabs DEPs.

    • [DataEntry #1108] Confirm that Commit without saving is enabled even when the PreventSave option is used.
