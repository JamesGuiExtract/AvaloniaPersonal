UEXViewer Utility - Testing Checklist

  1. Size Limit on .Uex Files \ Creation of Multiple .Uex Files
    • Install a FLEX Index or ID Shield build that is not the newest build.
    • Use the following steps to safely generate a large number of errors:
      1. Open the USB License Key Manager and change the License Server to point to an invalid machine.
      2. Clear the current database and then add to it an action called "GenerateErrors".
      3. Delete any existing .Uex files in C:\Documents and Settings\All Users\Application Data\Extract Systems\LogFiles\Misc
      4. Open and run "Error_Generator.fps" which is located in \Engineering\ProductDevelopment\Utils\UEXViewer\InteractiveTest (Running this FPS will cause 5029 files to fail.)
    • Confirm that multiple .Uex files have been created.
    • Confirm that all .Uex files are approximately 2 MB. (between 1.9 and 2.1 MB is acceptable)
    • Confirm that all .Uex files, except the most current, are stamped with both the date and the time.
    • Confirm that the most current .uex file is named ExtractException.uex.
    • Open the UEXViewer via View Exception Log.
      • Confirm that the most recent .uex file is loaded by default.
      • Confirm that the complete filename without path is shown on the title bar.
      • Confirm that the "Exception log files in the current folder" combo box list displays the complete filename without path.
      • Confirm that the above-mentioned combo box list contains a list of all the .Uex files in the same directory as the loaded .Uex file, listed in order by name.
      • Confirm that clicking on a .Uex filename in the combo box list loads that file.
    • Uninstall FLEX Index or ID Shield.
    • [FlexIDSCore #2458] Install a newer FLEX Index or ID Shield build and confirm that the previously existing ExtractException.uex has been renamed with the date and time as part of the filename.

  2. Menus
    • File Menu
      • Open:
        • Confirm that a "File->Open" opens a dialog that allows for selection of a .Uex file.
        • Copy the .Uex files created above into a randomly named directory on the desktop. Rename some of the .Uex files and then open one of the renamed files.
        • Confirm that the complete filename without path is shown on the title bar.
        • Confirm that the complete filename without path is shown in the drop down menu labeled "Exception log files in current folder:"
        • Confirm that "CTRL + O" performs the same function as "File->Open"
      • Open previous log file:
        • Confirm that "File->Open previous log file" loads the previous log file in the combo box list entitled "Exception log files in current folder:".
        • Confirm that "File->Open previous log file" is disabled when the first log file in the combo box list is being viewed.
        • Confirm that the complete filename without path is shown on the title bar.
        • Confirm that the complete filename without path is shown in the combo box list labeled "Exception log files in current folder:"
        • Confirm that "CTRL + Left" performs the same function as "File->Open previous log file"
        • Confirm that the < button on the lower right of the interface performs the same function as "File->Open previous log file"
      • Open next log file:
        • Confirm that "File->Open next log file" loads the next log file in the combo box list entitled "Exception log files in current folder:"
        • Confirm that "File->Open next log file" is disabled when the last log file in the combo box list is being viewed.
        • Confirm that the complete filename without path is shown on the title bar.
        • Confirm that the complete filename without path is shown in the combo box list labeled "Exception log files in current folder:"
        • Confirm that "CTRL + Right" performs the same function as "File->Open previous log file"
        • Confirm that the > button on the lower right of the interface performs the same function as "File->Open previous log file"
      • Save as:
        • Confirm that "File->Save as" opens a dialog that allows for saving the listed exceptions to a .Uex file.
        • Save some exceptions to a separate file and confirm that the .Uex file can be opened in the UEXViewer.
        • Confirm that the desired exceptions are present in the .Uex file.
        • Confirm that the desired exceptions are ordered most recent first.
      • Exit:
        • Confirm that "File->Exit" exits the UEX Viewer

    • Edit Menu
      • Paste:
        • View a selected exception. Click the Details button. Click the Copy to Clipboard button. Close the Extract Exception dialog. Confirm that Selecting Edit->Paste adds the copied exception to the top of the list.
        • Confirm that the Pasted exception only has column entries for Top ELI Code and Top Exception.
        • Confirm that "CTRL + V" performs the same function as "Edit->Paste"
      • Find:
        • Confirm that "Edit->Find" brings up a modeless Find dialog.
        • Confirm that "CTRL + F" performs the same function as "Edit->Find"
      • Delete selection:
        • Select a large subset of the loaded exceptions. Confirm that selecting "Edit->Delete selection" removes them from the display.
        • Confirm that a wait cursor is displayed during a lengthy delete operation.
        • Confirm that "Edit->Delete selection" is disabled when no exceptions are selected.
        • Confirm that "Del" performs the same function as "Edit->Delete selection"
      • Invert Selection:
        • Select a subset of the loaded exceptions. Confirm that clicking Invert Selection selects the other loaded exceptions.
        • Confirm that clicking Invert Selection again changes selection back to the original subset.
      • Clear:
        • Confirm that the exceptions loaded above are cleared from the display.
        • Confirm that the title bar reverts to UEX Viewer.
        • Confirm that the "Exception log files in the current folder" combo box list, the < button, and the > button are disabled.

    • Exception Menu
      • View Details:
        • Reload the ExtractException.uex file. Confirm that selecting an exception and clicking Exception->View details displays the Extract Exception dialog.
        • Confirm that Viewing an exception does NOT add the item to the ExtractException.uex file.
        • Confirm that Exception->View details is disabled when no exceptions are selected.
        • Confirm that "Exception->View details" is disabled when more than one exception is selected.
        • Confirm that double-clicking on an exception performs the same function as Exception->View details.
        • Confirm that "CTRL + D" performs the same function as Exception->View details.

    • Help Menu
      • About UEX Viewer:
        • Confirm that selecting "Help->About UEX Viewer" opens the UEXViewer information window.

  3. Find Dialog
    • User Interface.
      • Confirm that an edit box is provided to specify search text.
      • Confirm that a message is displayed if no matches are found.
      • Confirm that a group box is provided allowing specification of All, Next or Previous exceptions.
      • Confirm that an error message is displayed if the search text is empty.
    • Find All.
      • Confirm that each exception containing the search text is selected.
    • Find Next.
      • Select "next matching exception". Confirm that only the first exception (after the first currently selected exception) containing the search text is selected.
      • Confirm that subsequent Find operations advance forwards through the list.
      • Confirm that a message is displayed when no more matches are found.
    • Find Previous.
      • Select "next matching exception". Confirm that only the first exception (before the first currently selected exception) containing the search text is selected.
      • Confirm that subsequent Find operations advance backwards through the list.
      • Confirm that a message is displayed when no more matches are found.

  4. Command-Line Options
    • Unregistration:
      • Confirm that UEXViewer.exe /u unregisters the .uex file association.
    • Registration.
      • Confirm that UEXViewer.exe /r registers the .uex file association.
      • Confirm that double-clicking on a file containing multiple exceptions loads the entire list into the UEXViewer.
      • Confirm that double-clicking on a .Uex file containing a single exception displays the exception in the Extract Exception dialog.
      • Confirm that double-clicking on a .Uex file containing a single exception does NOT add the item to the ExtractException.uex file.
    • Filename
      • Confirm that UEXViewer.exe <filename> opens the specified file.
      • Confirm that if <filename> contains multiple exceptions, the entire list is loaded.
      • Confirm that if <filename> contains only one exception, the exception is displayed in the Extract Exception dialog.
    • Usage:
      • Confirm that UEXViewer.exe /? displays a Usage message-box.

  5. Miscellaneous Items
    • Drag and drop a .Uex file onto the UEXViewer. Confirm that the new list replaces the old list.
    • While holding CTRL, drag and drop a .Uex file onto the UEXViewer. Confirm that the new list is appended to the old list.
    • Confirm that UEXViewer documentation is available in the FLEX Index Help file.
    • Confirm that UEXViewer documentation is available in the ID Shield Help file.
    • Confirm that UEXViewer documentation includes either a Start menu path or a fully-qualified path to the EXE.

    • [LegacyRCAndUtils #4642] Encrypted Debug values
      • Confirm that a mapped drive exists to \\fnp2\internal - typically drive I:\.
      • Open a local copy of Application_Trace.uex attached to the PVCS entry.
      • Confirm that the values associated with Item 1 and Item 2 are meaningful numbers.
      • Close the UEX Viewer.
      • Disconnect the mapped drive to \\fnp2\internal. If \\Jake\internal is also mapped, disconnect this as well.
      • Reopen Application_Trace.uex attached to the PVCS entry.
      • Confirm that the values associated with Item 1 and Item 2 are <Encrypted>.

    • [LegacyRCAndUtils #4108] Navigation shortcut keys don't work after loading from combo box list
      • Open ExtractException.uex.
      • Using the combo box, select an earlier exception log.
      • Confirm that the < and > buttons work.
      • Using the combo box, select an earlier exception log.
      • Confirm that the "<CTRL> + Left/Right" keyboard shortcuts work.
      • Using the < and > buttons, select a different exception log.
      • Confirm that the "<CTRL> + Left/Right" keyboard shortcuts work.

    • [LegacyRCAndUtils #5148] Renamed exception log
      • Confirm that exceptions are now logged to ExtractException.uex
      • Confirm that the UEX Viewer successfully opens both old and new UEX files.
      • Confirm that the UEX Viewer successfully navigates between old and new UEX files.
