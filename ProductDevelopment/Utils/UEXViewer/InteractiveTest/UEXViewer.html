UEXViewer Utility - Testing Checklist

  1. Size Limit on .Uex Files \ Creation of Multiple .Uex Files
    • Install FLEX Index or ID Shield.
    • Use the following steps to safely generate a large number of errors:
      1. Create and clear a (local) database called Test.
      2. Add a GenerateErrors action.
      3. Delete or rename any existing .Uex files in C:\Documents and Settings\All Users\Application Data\Extract Systems\LogFiles.
      4. Confirm that a drive mapping to K: exists for \\jake\internal.
      5. Open and run Error_Generator.fps which is located in \Engineering\ProductDevelopment\Utils\UEXViewer\InteractiveTest (Running this FPS file to completion will cause approximately 2500 files to fail.)
      6. Processing can be stopped when 1000+ files have failed.
    • Confirm that multiple .Uex files have been created.
    • Confirm that all .Uex files are approximately 2 MB. (between 1.9 and 2.1 MB is acceptable)
    • Confirm that all .Uex files, except the most current, are stamped with both the date and the time in a zero-padded format.
    • Confirm that the most current .uex file is named ExtractException.uex.
    • Open the UEXViewer via View Exception Log.
      • Confirm that the most recent .uex file is loaded by default.
      • Confirm that the complete filename without path is shown on the title bar.
      • Confirm that the "Exception log files in the current folder" combo box list displays the complete filename without path.
      • Confirm that the above-mentioned combo box list contains a list of all the .Uex files in the same directory as the loaded .Uex file, listed in order by name.
      • Confirm that clicking on a .Uex filename in the combo box list loads that file.
    • Uninstall FLEX Index or ID Shield.
    • [FlexIDSCore #2458] Install FLEX Index or ID Shield and confirm that the previously existing ExtractException.uex has been renamed with the date and time as part of the filename in a zero-padded format.

    • Edit Menu

      • Find
        • Confirm that Edit - Find brings up a modeless Find dialog.
        • Confirm that CTRL + F performs the same function as Edit - Find.
      • Delete selection
        • Select a large subset of the loaded exceptions. Confirm that selecting Edit - Delete selection removes them from the display.
        • Confirm that a wait cursor is displayed during a lengthy delete operation.
        • Confirm that Edit - Delete selection is disabled when no exceptions are selected.
        • Confirm that Del performs the same function as Edit - Delete selection.
      • Invert Selection
        • Select a subset of the loaded exceptions. Confirm that clicking Invert Selection selects the other loaded exceptions.
        • Confirm that clicking Invert Selection again changes selection back to the original subset.
        • Confirm that CTRL + I performs the same function as Edit - Invert selection.
      • Clear
        • Confirm that the exceptions loaded above are cleared from the display.
        • Confirm that the title bar reverts to UEX Viewer.
        • Confirm that the Exception log files in the current folder combo box list, the < button, and the > button are disabled.

        • Confirm that double-clicking on an exception performs the same function as Exception - View details.

      • Show application traces

        • Confirm that this setting is not persistent.
      • Show displayed exceptions
        • Confirm that this setting is checked by default.

        • Confirm that this setting is not persistent.

  3. Find Dialog
    • User Interface
      • Confirm that an edit box is provided to specify search text.
      • Confirm that a message is displayed if no matches are found.
      • Confirm that a Select group box is provided allowing specification of all, next or previous matching exceptions.
      • Confirm that an error message is displayed if the search text is empty.
    • Find All
      • Confirm that each exception containing the search text is selected.
    • Find Next
      • Select next matching exception. Confirm that only the first exception (after the first currently selected exception) containing the search text is selected.
      • Confirm that subsequent Find operations advance forwards through the list.
      • Confirm that a message is displayed when no more matches are found.
    • Find Previous
      • Select previous matching exception. Confirm that only the first exception (before the first currently selected exception) containing the search text is selected.
      • Confirm that subsequent Find operations advance backwards through the list.
      • Confirm that a message is displayed when no more matches are found.

      • Confirm that focus remains with the Find dialog.
