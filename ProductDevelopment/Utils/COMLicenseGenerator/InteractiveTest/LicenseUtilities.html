Private Licensing Testing Checklist

NOTE: Items prefaced with an asterisk indicate functionality that is not yet complete.

  1. License Generator - Errors and Restrictions

    • Start with a clean test machine or virtual machine.
    • Create the usual drive mappings for I and K.
    • Confirm that COMLicenseGenerator.exe displays an error message that Mscal.ocx is not installed.
    • Install the calendar control via ocx-install.bat and confirm that COMLicenseGenerator.exe starts without error.

    • Remove the drive mappings for I and K.
    • Navigate to COMLicenseGenerator.exe via UNC path (not mapped drive). Confirm that an error message is displayed Unable to run License Generator Utility.
    • Copy the contents of a specific COMLicense folder to a local folder on the test machine. Confirm that an error message is displayed Unable to run License Generator Utility.

    • Confirm that the package group names such as FLEX Index, ID Shield, FLEX Index / ID Shield Utilities cannot be selected in the Packages list.
    • Paste a valid user license string into a temporary text file. Copy a modified version of the license string back into the Clipboard. Confirm that clicking Paste User String to paste the modified license string generates a The User License string you have provided is not a valid license string! error message.
    • Confirm that the Generate Unlock File... button is not enabled until a valid User License String has been provided.
    • Confirm that the Generate License File... button is not enabled until a valid User License String, a Licensee Name and an Organization Name have been provided and at least one package has been selected.

    • Confirm that Computer Name and Computer MAC Address options are not enabled until a valid User License String has been provided.
    • Confirm that Disk Serial Number option is both checked and disabled when the Fully Licensed mode is selected. This prevents creation of a non-expiring license file that is not locked to a specific machine.

    • Confirm that selection of the UCLID Password for end-user products option controls the enabled or disabled state of the associated checkbox and combo box controls.
    • Confirm that the Random Password for Input Funnel SDK option is not enabled unless the -sdk command-line switch is provided via sdklicense.bat.
    • Confirm that selection of the Specified Password for SDK option controls the enabled or disabled state of the associated Paste button.

    • [LegacyRC #5475] User License and MAC address
      • Use the installed UserLicense.exe to generate a license string.
      • Use the generated license string to generate a license file and an Unlock file.
      • Use COMLicenseEval.exe to evaluate the MAC address section of the files.
      • Confirm that the two addresses are identical.
      • Delete the two files.

      • Disconnect the network cable and use the installed UserLicense.exe to generate a new license string.
      • Use the new license string to generate a license file and an Unlock file.
      • Use COMLicenseEval.exe to evaluate the MAC address section of the files.
      • Confirm that the two addresses are Address N/A.

  2. License Generator - Functionality

    • Confirm that the Packages, Licensee Name and Organization Name controls are always enabled.
    • Confirm that clicking Generate User String updates the associated read-only edit box with a valid user license string.
    • Confirm that after providing a valid user license string, the Computer edit box is updated with the name of the computer that created the string.

    • Confirm that selecting Fully Licensed mode hides the calendar control.
    • Confirm that selecting Evaluation License mode displays the calendar control.
    • Confirm that selecting a date on the calendar control updates the Evaluation License edit box.
    • Confirm that an already expired evaluation license can be created.

    • Confirm that the Special Passwords combo box provides the following options: Default, IcoMap and Simple Rules.

    • Confirm that clicking Generate License File... for an evaluation license displays a File Save dialog with a default file name of OrganizationName_LicenseeName_ExpirationDate.lic.
    • Confirm that clicking Generate License File... for a non-expiring license displays a File Save dialog with a default file name of OrganizationName_LicenseeName_Full.lic.
    • Confirm that the default folder for saving a license file is I:\Common\Sales\SBL.
    • Confirm that the license file and folder edit boxes are updated with the name and location of the new license file.

    • Confirm that the License Password edit box contains <UCLID Password> and the OEM Password edit box contains <UCLID OEM> after creating a license file with default Extract Systems passwords.
    • Confirm that the License Password edit box and the OEM Password edit box contain updated values after creating a license file with Random passwords.
    • Confirm that the user is prompted to save a password file (*.pwd) after creating a license file with Random passwords.
    • Confirm that the License Password edit box contains desired values after creating a license file with Specified passwords.
    • Confirm that the user is prompted to save a password file after creating a license file with Specified passwords.
    • Create a license file using passwords copied from a previously saved password file. After creating a license file with these Specified passwords, confirm that the OEM Password edit box contains the same value as stored in the password file.

    • Confirm that selecting Fully Licensed mode hides the calendar control.
    • Confirm that selecting Evaluation License mode displays the calendar control.
    • Confirm that selecting a date on the calendar control updates the Evaluation License edit box.
    • Confirm that an already expired evaluation license can be created.

    • Confirm that the Special Passwords combo box provides the following options: Default, IcoMap and Simple Rules.

    • Confirm that clicking Generate License File... for an evaluation license displays a File Save dialog with a default file name of OrganizationName_LicenseeName_ExpirationDate.lic.
    • Confirm that clicking Generate License File... for a non-expiring license displays a File Save dialog with a default file name of OrganizationName_LicenseeName_Full.lic.
    • Confirm that the default folder for saving a license file is I:\Common\Sales\SBL.
    • Confirm that the license file and folder edit boxes are updated with the name and location of the new license file.

    • Confirm that the License Password edit box contains <UCLID Password> and the OEM Password edit box contains <UCLID OEM> after creating a license file with default Extract Systems passwords.
    • Confirm that the License Password edit box and the OEM Password edit box contain updated values after creating a license file with Random passwords.

  3. License Evaluator - Functionality

    • Confirm that the File Open dialog displayed via the browse button defaults to displaying license (*.lic) files.
    • Confirm that the File Open dialog displayed via the browse button can also display license (*.esl), text (*.txt) files or all (*.*) files.

    • Confirm that the default Extraction String is User String.
    • For User String selection, confirm that the Evaluate Code button is enabled only if the License File Password edit box is non-empty.
    • For UCLID String selection, confirm that the Evaluate Code button is enabled regardless of the License File Password contents.

    • Select a license file created with Default passwords and provide nonsense passwords (i.e. 1,2,3,4). For User String selection, confirm that Evaluate Code displays an error message suggesting that passwords are incorrect.
    • Select a license file created with Default passwords and provide nonsense passwords (i.e. 1,2,3,4). For UCLID String selection, confirm that Evaluate Code updates the Licensed Components list with meaningful information.
    • Select a license file created with Default passwords and provide nonsense passwords (i.e. 1,2,3,4). For UCLID String selection and special passwords of type 1, confirm that Evaluate Code displays a error message suggesting that passwords are incorrect.
    • Select a license file created with Default passwords and provide nonsense passwords (i.e. 1,2,3,4). For UCLID String selection and special passwords of type 2, confirm that Evaluate Code displays a error message suggesting that passwords are incorrect.

    • Select a license file created with special IcoMap passwords and provide nonsense passwords (i.e. 1,2,3,4). For User String selection, confirm that Evaluate Code displays an error message suggesting that passwords are incorrect.
    • Select a license file created with special IcoMap passwords. For UCLID String selection and special passwords of type 0, confirm that Evaluate Code displays an error message suggesting that passwords are incorrect.
    • Select a license file created with special IcoMap passwords. For UCLID String selection and special passwords of type 1, confirm that Evaluate Code updates the Licensed Components list with meaningful information.
    • Select a license file created with special IcoMap passwords. For UCLID String selection and special passwords of type 2, confirm that Evaluate Code displays an error message suggesting that passwords are incorrect.

    • Select a license file created with special Simple Rules passwords and provide nonsense passwords (i.e. 1,2,3,4). For User String selection, confirm that Evaluate Code displays an error message suggesting that passwords are incorrect.
    • Select a license file created with special Simple Rules passwords. For UCLID String selection and special passwords of type 0, confirm that Evaluate Code displays an error message suggesting that passwords are incorrect.
    • Select a license file created with special Simple Rules passwords. For UCLID String selection and special passwords of type 1, confirm that Evaluate Code displays an error message suggesting that passwords are incorrect.
    • Select a license file created with special Simple Rules passwords. For UCLID String selection and special passwords of type 2, confirm that Evaluate Code updates the Licensed Components list with meaningful information.

    • Create a license file with known passwords or with saved random passwords and use this license file for the following test cases.
      • *Meaningful information in the Licensed Components list includes one column of component IDs and one column of component names. The list shall be sorted by component ID.
      • For User String selection and nonsense passwords (i.e. 1,2,3,4), confirm that Evaluate Code displays a error message suggesting that passwords are incorrect.
      • For User String selection and the correct passwords, confirm that Evaluate Code updates the Licensed Components list with meaningful information.
      • For UCLID String selection and with or without passwords, confirm that Evaluate Code updates the Licensed Components list with meaningful information.

    • Test a non-expiring license file and confirm that Licensing Mode = Fully Licensed.
    • Test an expiring license file and confirm that Licensing Mode = Evaluation License and the Expires edit box is updated. Confirm that the date is correct and the time is 23:59:59.

    • For successful evaluation of a license file, confirm that License Version, Issue Date, Licensee Name, Organization Name, and Issuer Name fields are updated.
    • Create a license file locked to Computer Name, Disk Serial Number, and MAC Address. After successful evaluation of the license file, confirm that all three check boxes and all three edit boxes are updated properly.
