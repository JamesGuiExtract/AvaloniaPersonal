Dynamic Input Grid (DIG) - Enhancements and Bug Fixes

  1. Start ArcMap and IcoMap.

  2. [P10 #3091] Start editing.

    Confirm that IcoMap sets focus to the command prompt and that the command prompt has a yellow background.

  3. [P10 #3101] Start a new sketch with a starting point. Swipe a bearing from an image. Use the mouse to put focus on any cell where data does not exist - such as LDR cell or the empty cell where LDS would exist if a distance was entered.

    Confirm that IcoMap maintains focus in either the command prompt or in one of the DIG data cells.

  4. [P10 #3099] Start a new sketch and draw two lines. Use the mouse to click on the first bearing value. Press the Tab key.

    Confirm that the yellow background moves to the first distance value cell.

    Press the Tab key again.

    Confirm that the yellow background moves to the second bearing value cell.

  5. [P10 #3100] Continue using the new sketch from the previous test. Press the Tab key until the yellow background has moved to the last data cell. Press the Tab key one more time.

    Confirm that the yellow background moves to the command prompt and that additional Tab keystrokes have no effect.

  6. [P10 #3092] Continue using the new sketch from the previous test. Click one of the existing data cells and edit the value. Press Enter to accept the edits.

    Confirm that the yellow background moves to the command prompt and that the drawing is updated.

  7. [P10 #3095] Continue using the new sketch from the previous test. With focus in the command prompt, press Shift+Tab to navigate to the last editable cell in the DIG.

    Continue pressing Shift+Tab to navigate through the entire collection of editable cells.

    Confirm that Shift+Tab has no effect once navigation has reached the first editable cell.

  8. [P10 #3095] Continue using the new sketch from the previous test. With focus in the first editable cell in the DIG, press Tab to navigate to the next editable cell.

    Continue pressing Tab to navigate through the entire collection of editable cells back to the command prompt.

    Confirm that Tab has no effect once navigation has reached the command prompt.

  9. [P10 #3093] Continue using the new sketch from the previous test. Click one of the existing data cells and edit the value. Press Tab to accept the edits.

    Confirm that the yellow background moves to the next editable cell and that the drawing is updated. If the edits were made to the last editable cell in a row that the focus and yellow highlight should move to t he first editable cell in the next row.

    Repeat the test using the Right-Arrow key to navigate to the next cell.

  10. [P10 #3093] Continue using the new sketch from the previous test. Click one of the existing data cells and edit the value. Press Shift+Tab to accept the edits.

    Confirm that the yellow background moves to the previous editable cell and that the drawing is updated. If the edits were made to the first editable cell in a row that the focus and yellow highlight should move to the last editable cell in the previous row.

    Repeat the test using the Left-Arrow key to navigate to the previous cell.

  11. [P10 #3098] Continue using the new sketch from the previous test. Click in one of the DIG data cells such that the yellow background moves to that cell. Click the 'Delete sketch' button on the IcoMap toolbar.

    Confirm that the sketch is deleted and the grid is emptied. Also confirm that IcoMap prompts for a new starting point and that the yellow highlight is on the command prompt.

  12. Close IcoMap and ArcMap.

    IcoMap should close gracefully.
