FAM Tasks for NetDMS
Basic Testing Checklist

  • NetDMS: Advance work item

    • Confirm that this task is available from the FAM.
    • Confirm that clicking the Configure button displays the NetDMS: Advance work item settings dialog.
    • Confirm that the dialog includes a NetDMS dispatcher connection group box.
    • Confirm that the group box contains a Server edit box.
    • Confirm that the group box contains a Port edit box.
    • Confirm that the group box contains a User edit box.
    • Confirm that the group box contains a Password edit box.
    • Confirm that the group box contains a Load last successful connection button.
    • Confirm that the group box contains a Test connection button.
    • Confirm that the dialog contains another group box with text about the task not advancing a work item item until all contained documents have been processed.

    • Confirm that the Password edit box does not display clear text.
    • Confirm that a successful connection can be made. The following settings may be effective for Sumter.
      • Server = 1.2.3.59
      • Port = 22548
      • User = staff
      • Password = staff
    • Confirm that a menaingful exception is provided if an incorrect parameter is provided for a test connection.
    • Confirm that the Load last successful connection button successfully restores settings for the previous successful connection.
    • Confirm that settings are persistent.

  • NetDMS: Add redactions

    • Confirm that this task is available from the FAM.
    • Confirm that clicking the Configure button displays the NetDMS: Add redactions settings dialog.
    • Confirm that the dialog includes a NetDMS dispatcher connection group box.
    • Confirm that the group box contains a Server edit box.
    • Confirm that the group box contains a Port edit box.
    • Confirm that the group box contains a User edit box.
    • Confirm that the group box contains a Password edit box.
    • Confirm that the group box contains a Load last successful connection button.
    • Confirm that the group box contains a Test connection button.
    • Confirm that the dialog contains an ID Shield data file location group box.
    • Confirmt that the default contents include a You are using default settings label.
    • Confirm that the group box contains a Customize... button.
    • Confirm that clicking the Customize button displays a Specify ID Shield data file path dialog.
    • Confirm that the dialog contains an ID Shield data file edit box with associated tag selector and browse buttons.
    • Confirm that the edit box contents default to <SourceDocName>.voa.
    • Confirm proper behavior for the tag selector button.
    • Confirm proper behavior for the browse button.
    • Confirm that the Reset button resets the edit box back to the default string.

    • Confirm that the Password edit box does not display clear text.
    • Confirm that a successful connection can be made. The following settings may be effective for Sumter.
      • Server = 1.2.3.59
      • Port = 22548
      • User = staff
      • Password = staff
    • Confirm that a menaingful exception is provided if an incorrect parameter is provided for a test connection.
    • Confirm that the Load last successful connection button successfully restores settings for the previous successful connection.
    • Confirm that settings are persistent.
