ID Shield for NetDMS
Basic Testing Checklist

  • Testing Setup

    • NetDMS Server Setup
      • Confirm with the NetDMS administrator that the server software is installed.
      • Confirm that valid login information is available for connecting to the NetDMS server.
        • Confirm presence of Workflow Configuration Manager software under VistaSG in the Start menu.
      • If applying the NetDMS patch, run the Exporter Install batch file.

    • NetDMS Client Setup
      • Confirm installation of Mobile VPN with SSL client (wgsslvpnc.exe) software under the WatchGuard folder.
      • Install and license the appropriate version of ID Shield.
      • If applying the NetDMS patch, run the ID Shield Components Install batch file.

  • Workflow Configuration Manager

    • Start the Mobile VPN with SSL client application with the following connection settings.
      • Server = 71.46.218.130
      • User = ExtractSys
      • Password = ids60sumt
    • Open a remote desktop session
      • IP Address = 1.2.3.70 (or 1.2.3.71)
      • User = ExtractSys
      • Password = ids60sumt

    • Open the application and click the browse button for database selection.
    • Create a new database with the following settings
      • Data Source Name = C60NetdmsDev2
      • Server = 1.2.3.77\clericustest
      • Database = C60NetdmsDev2
      • UserID = staff
      • Password = staff
    • Test the connection settings and confirm that the tests completed successfully.

    • Login to the WorkFlow Configuration Manager with the following settings.
      • Database = C60NetdmsDev2
      • UserID = admin
      • Password = admin
    • Confirm that the Configuration Manager window appears.
    • Customers
      • Open the Customers node.
      • Open the C60NetdmsDev2 node.
      • Open the Projects node.
      • Open the CLERICUS_WORKFLOW node.
      • Select the WorkFlow node.
      • Select the Graph node.
      • Confirm that a workflow appears and that an Extract item is included.
      • Right-click on the Extract item and select Modify to set or confirm settings.
        • Confirm that nothing is Assigned for the Configurations tab.
        • Confirm that nothing is Assigned for the Tasks tab.
        • Select the Visual tab to change the item image to the ID Shield logo, if desired.
        • Retain the default settings in the Escalation tab.
        • Confirm that a PROGRESSDOCKET_STORAGE Rule is defined in the Send Rules tab.
          • Select PROGRESSDOCKET_STORAGE as Node.
          • Select staff user as User.
        • Confirm that staff user is Assigned for the Groups and Users tab.
        • Confirm that an Execute custom script event is defined for a Work Item In event trigger on the Events tab.
          • Select the CRIMINAL, PROBATE, PROGRESSDOCKET, CIVIL, and TRAFFIC Nodes.
          • Select the ID Shield Export script.
    • Scripts
      • Select the drop-down combo box and confirm presence of an ID Shield Export script.
      • The following steps will create an ID Shield Export script.
        • Click the Add button.
        • Enter ID Sheild Export as the Name.
        • Confirm that MethodBody is the Type.
        • Add Event as a Usage type.
        • Click the References button.
          • Click the Add Reference button.
          • Navigate to C:\Program Files (x86)\VistaSG\WorkFlow Configuration Manager.
          • Select Extract.NetDMSExporter.dll
        • Add the following string to the edit box: Extract.NetDMSExporter.NetDMSDocumentExporter.Export(taskClient, workItem, @“C:\IDShield”);. Note that a different path should be provided as the last parameter if the FAM file supplier will be listening to a different folder.
        • Click the Validate button and confirm that a success message is displayed.
        • Click the Save button.
      • Close the WorkFlow Configuration Manager window.
      • Open the Task Dispatcher Configuration utility.
        • Click the Start button.
        • Click the Stop button.
        • Close the Task Dispatcher Configuration utility.
      • Reopen the WorkFlow Configuration Manager window.

  • ID Shield Server Setup

    • Create a suitable database to exercise the desired ID Shield workflow.
    • Queuing
      • Create an FPS file to supply files from the C:\IDShield folder as defined above or another appropriate folder that will receive files exported from NetDMS.
      • Confirm that suitable image types will be supplied from subfolders.
      • Confirm that Added files will be continuously supplied.
    • Automatic Processing
      • Create an FPS file to process the supplied images. Include the following tasks for automatic processing.
        • Core: Execute rules - use rules appropriate for the test images
        • NetDMS: Add redactions - use settings for the NetDMS server
          • Server = 1.2.3.70 or 1.2.3.71
          • Port = 22548
          • User = staff
          • Password = staff
        • NetDMS: Advance work item - use settings for the NetDMS server
      • Configure the FPS file for 1 thread.
      • Configure the FPS file for continuous processing.
    • Start processing for each FPS file once work items have been provided via WorkFlow User Console. It is also appropriate to start the FPS files before work items are available to confirm that all added files are queued and processed.

  • WorkFlow User Console

    • Start the WorkFlow User Console application.
    • Expand the CLERICUS_WORKFLOW node.
    • Confirm that the list of Nodes corresponds to the collection of items defined in the Workflow graph.
    • Confirm that suitable test images are available for processing.

    • Use the Create WI toolbar button to add one or more work items to the node immediately preceding the ID Shield node. For basic testing, only one image should be used for each work item.
    • Select a new work item and click the Capture toolbar button.
    • Select File pickup... and navigate to a suitable image file.
    • Confirm that the image is displayed in a Vista Viewer - New Document window.
    • Confirm that File - Save associates the selected image with the work item.
    • Confirm that closing the viewer without saving displays a reminder prompt.

    • Select a new work item and click the Send Manual toolbar button.
      • Confirm that a Send Work Item window is displayed.
      • Select the Extract Node and click the Send button.
    • Confirm that the work item moves to the Extract node.
    • Confirm that the image is successfully queued and processed.
    • Confirm that a VOA and/or an XML output file is present in the appropriate folder under the working folder (C:\IDShield).
    • Confirm that the work item advances to the following node.
    • Confirm that the image shows redactions when viewed in the Vista Viewer window.

    • Select a new work item and click the Send Auto toolbar button.
    • Confirm that a Send Work Item(s) window is displayed indicating that the selected parcel was sent to Extract.
    • Confirm that the work item moves to the Extract node.
    • Confirm that the image is successfully queued and processed.
    • Confirm that a VOA and/or an XML output file is present in the appropriate folder under the working folder (C:\IDShield).
    • Confirm that the work item advances to the following node.
    • Confirm that the image shows redactions when viewed in the Vista Viewer window.
