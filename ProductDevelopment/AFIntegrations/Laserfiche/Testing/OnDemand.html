ID Shield for Laserfiche
On-Demand Redaction Testing Checklist

  • Testing Setup

    • Create a new virtual machine using the virtual machine disk ( .vhd file ) at: I:\Common\Engineering\VirtualMachines\Laserfiche_72 or I:\Common\Engineering\VirtualMachines\Laserfiche_8.
    • Close the Laserfiche Client.
    • Install ID Shield for Laserfiche on the virtual machine.
    • Install FKB update 8.2.
    • Generate a Laserfiche Integration license file. Copy the license to the usual CommonComponents folder.
    • Launch USB License Key Manager and point to a machine that contains a USB key (usually Lisa).
    • Launch Rule Tester and verify the About Rule Tester button displays the correct version and FKB numbers.
    • Configure the repository for use with ID Shield
      • Launch ID Shield Admin Console.
      • Verify ID Shield Demo or LF8 appears in the repository dropdown list of the Select Repository dialog.
      • Select ID Shield Demo or LF8 and log in with the username "admin" and password "a".
      • Click on the About and verify the ID Shield and FKB versions.
      • On the Repository tab, click the Create Tags button and verify the Admin Console reports that the tags were successfully created.
      • Click the Create Tags button and verify the Admin Console reports that the tags were successfully created.
      • In the Laserfiche client create a new top-level folder called ID Shield.
      • Use the Fields | Tags context menu to apply the ID Shield - Settings tag to the ID Shield folder.
      • In the ID Shield Admin Console, press the Refresh button.
      • Confirm \ID Shield is displayed as the settings folder following the refresh.
      • On the Redaction Settings tab enter "I:\Common\Testing\Product Testing\Laserfiche\Rules\ID Shield for LF.rsd.etf" in the Redaction Rules box
      • Press "OK" to accept the default value for all other settings.
    • OCR The test documents in the Laserfiche Client
      • Select all 10 documents in the Documents folder.
      • Select the Action | OCR/Extract Text/Index... menu option (or button)
      • Check OCR / Extract Text
      • Select All image pages
      • Under options, make sure all options are unchecked
      • Press OK to OCR the documents.

  • Test 1: Basic On-Demand redaction

    • Redact selected documents

      • Once the OCR process has completed, select documents 01, 02 and 03 (Ctrl or Shift + click).
      • Press the ID Shield On-Demand Redaction button on the Laserfiche Client Toolbar.
      • Confirm the appearance of a progress bar with 2 levels of progress and a Stop button.
      • While the documents are being redacted click on the Laserfiche Client window in the background and confirm that it is disabled. (No action can be taken in the Client, the System menu Minimize, Maximize and Close are non-functional, and the window can't be moved.)

    • Automatic document verification

      • Once the documents complete processing, verify document 01 is opened in a Laserfiche document window and that the ID Shield Verification Toolbar appears on top of the document window.
      • Ensure Verifying Document = 1.
      • Ensure Documents Remaining = 2.
      • In the image pane of the Laserfiche document window, at the top left of the image, confirm Social Security Number is highlighted in yellow.
      • Confirm the social security number below the highlight is covered with a Laserfiche redaction annotation (see-through because "user" has rights to see through redaction annotations)
      • If the Text pane is not visible select View | Views | Text.
      • In the Text pane, confirm that the text associated with the social security number in the image is contained in a Laserfiche redaction annotation.
      • Press the Verify button on the toolbar.
      • Confirm that the document window closes.
      • Confirm document 02 opens in a new Laserfiche document window and the verification toolbar re-appears.
      • Confirm Verifying Document = 2.
      • Confirm Documents Remaining = 1.
      • At the top left of the image, confirm Social Security Number is highlighted in yellow.
      • Beneath the Social Security Number clue, confirm most of the box below (and some surrounding area) is contained in a Laserfiche redaction.
      • Confirm the corresponding text is redacted in the text pane (the text in this case will not have OCR'd very well).
      • Press the Stop button on the verification toolbar.
      • Confirm document 02 closes.
      • Confirm the main Laserfiche client is enabled.

    • Check Document Status

      • Right-click on document 01 and select "Fields... | Tags".
      • Confirm ID Shield - Processed and ID Shield - Verified are checked.
      • Open document 01 with Laserfiche.
      • Confirm that the redaction annotation is still present in the document but that the highlight of Social Security Number has been removed.
      • Right-click on document 02 and select "Fields... | Tags".
      • Confirm ID Shield - Processed and ID Shield - Needs Verification are checked.
      • Right-click on document 03 and select "Fields... | Tags".
      • Confirm ID Shield - Processed and ID Shield - Needs Verification are checked.

    • Test 2: Verification Initiation Methods

      • Verify by document selection

        • Select documents 01, 02 and 03 (Ctrl or Shift + click).
        • Press the ID Shield Verify button .
        • Confirm that document 02 is opened in the document window.
        • Confirm the verification toolbar reports:
          • Verifying Document = 1.
          • Documents Remaining = 1.
        • [Integrations #71] Click in the verification window to add a manual redaction. Confirm that a single click is sufficient to begin a drag / highlight operation.
        • Press the Stop button .
        • Do not save changes.

      • Verify by folder

        • In the Folders pane of the Laserfiche Client, click on the root ID Shield Demo node and then on the Documents folder to ensure the Documents folder is highlighted (and nothing is highlighted in the document pane).
        • Press the ID Shield Verify button.
        • Confirm that document 02 is opened in document window.
        • Confirm the verification toolbar reports:
          • Verifying Document = 1.
          • Documents Remaining = 1.
        • Press the Stop button.

      • Verify by Repository

        • In the Folders pane of the Laserfiche Client, Click on the root ID Shield Demo node and ensure nothing else in the document pane is highlighted.
        • Press the ID Shield Verify button.
        • Confirm that document 02 is opened in document window.
        • Confirm the verification toolbar reports:
          • Verifying Document = 1.
          • Documents Remaining = 1.
        • Press the Stop button.

      • Verify by Search

        • In the Laserfiche client, press the Search button (or select menu "View | Explorer Bar | Search).
        • In the Text / Basic Text Search area, enter "social" in the Phrase box.
        • Press the Search button.
        • Confirm that 4 documents are found in the search results.
        • Select all search results by clicking anywhere in the document list and pressing Ctrl + A.
        • Press the ID Shield Verify button.
        • Confirm that document 02 is opened in document window.
        • Confirm the verification toolbar reports:
          • Verifying Document = 1.
          • Documents Remaining = 0.
        • Press the Stop button.

      • Attempt Verification of documents that don't need verification

        • Press the Folders button (or or select menu "View | Explorer Bar | Folders").
        • In the Documents folder, select documents 04 thru 10.
        • Press the ID Shield Verify button.
        • Confirm the message There are no documents in the current selection to verify!

    • Test 3: Verification

      • Verification Toolbar behavior

        • Open Windows Explorer in the background in order to indirectly test Laserfiche window foreground and focus concerns.
        • In the Laserfiche client, select menu "Tools | Options... | Prompts" and ensure Save Document is checked.
        • Click on the root ID Shield Demo node and ensure nothing else in the document pane is highlighted.
        • Press the ID Shield Verify button.
        • Confirm that document 02 is opened in document window.
        • Confirm the verification toolbar reports:
          • Verifying Document = 1.
          • Documents Remaining = 1.
        • Move the ID Shield verification toolbar to the far bottom right corner of the screen.
        • Minimize the Laserfiche document window and confirm the toolbar disappears with it.
        • Restore the Laserfiche document window and confirm the toolbar re-appears in the same bottom-right position.
        • Maximize & restore the Laserfiche document window and confirm the toolbar remains in the same place.
        • Press the Verify button on the toolbar.
        • Confirm that the document window closes.
        • Confirm document 03 opens in a new Laserfiche document window.
        • Confirm the ID Shield verification toolbar re-appears in the same bottom-right position.
        • Confirm the verification toolbar reports:
          • Verifying Document = 2.
          • Documents Remaining = 0.
          (keep the document window & verification open)

      • Stopping Verfication

        • System close button
          • Press the Close button on the document window's system toolbar (red X).
          • Ensure the document and verification toolbar both close.
          • Use the "Fields | Tags" context menu to verify document 03 contains the ID Shield - Processed and ID Shield - Needs Verification tags, but not the ID Shield - Verified tag.
        • File menu close
          • Select document 03 and press the ID Shield Verify button.
          • Select the menu option File | Close.
          • Ensure the document and verification toolbar close.
          • Use the "Fields | Tags" context menu to verify document 03 contains the ID Shield - Processed and ID Shield - Needs Verification tags, but not the ID Shield - Verified tag.

      • Save Document Prompt

        • Cancel
          • Select document 03 and press the ID Shield Verify button.
          • Right-click on the redaction annotation to the right of the Account Number clue and delete it.
          • Press the Stop button.
          • When prompted whether to save the changes to the document, press Cancel.
          • Ensure the document remains open and that the verification toolbar is visible.
        • Don't save
          • Press the Stop button.
          • When prompted whether to save the changes to the document, press No.
          • Confirm the document window & the verification toolbar both close.
          • Ensure document 03 is still selected and press the ID Shield Verify button.
          • Confirm the document 03 opens along with the verification window.
          • Confirm that the account number redaction is present.
        • Save
          • Right-click on the redaction annotation to the right of the Account Number clue and delete it.
          • Press the Stop button.
          • When prompted whether to save the changes to the document, press Yes.
          • Confirm the document window & the verification toolbar both close.
          • Ensure document 03 is still selected and press the ID Shield Verify button.
          • Confirm that document 03 opens along with the verification window.
          • Confirm that the account number redaction is missing.
        • Auto-save on Verify behavior
          • Delete the remaining redaction annotations on the document (Note: there are overlapping redactions. Remove them both.)
          • Press the Verify button on the toolbar.
          • Confirm that the document closes without a prompting whether to save the document.
          • Confirm the Verification Complete! prompt appears.
          • Open document 03 (with Laserfiche).
          • Confirm that the document does not contain any redaction annotations.

    • Test 4: Stop Processing

      • Stop but continue with verification

        • Select documents 05 and 06 in the On-Demand Test folder.
        • Press the ID Shield On-Demand Redaction button.
        • Once processing starts (and before document 05 is done processing), press the Stop button on the progress dialog.
        • Confirm that the progress information is replaced by Stopping...
        • Confirm that the progress dialog remains for a time (and document 05 completes processing before processing is stopped).
        • After processing is finished, confirm the prompt Do you wish to verify the documents that completed processing now? is displayed.
        • Press Yes.
        • Confirm that document 05 is launched opened for verification.
        • Confirm the Verification Toolbar reports:
          • Verifying Document = 1.
          • Documents Remaining = 0.
        • Press the Verify button on the verification toolbar to accept the found redactions.
        • Verify document 05 contains the tags ID Shield - Processed and ID Shield - Verified.
        • Verify document 06 doesn't have any ID Shield tags applied.

      • Stop and don't continue with verification

        • Select document 06 in the Documents folder.
        • Press the ID Shield On-Demand Redaction button.
        • Once processing starts (and before document 06 is done processing), press the Stop button on the progress dialog.
        • Confirm that the progress information is replaced by Stopping...
        • Confirm that the progress dialog remains for a time (and document 06 completes processing before processing is stopped).
        • After processing is finished, confirm the prompt Do you wish to verify the documents that completed processing now? is displayed.
        • Press No.
        • Confirm that processing ends without displaying document 06 for verification.
        • Verify document 06 contains the tags ID Shield - Processed and ID Shield - Needs Verification.

    • Test 5: Processing failure

      • In the folder C:\Program Files\Extract Systems\FlexIndexComponents\Bin, rename UCLIDAFValueFinders.dll to UCLIDAFValueFinders.dll.temp.
      • In the Laserfiche Client, highlight documents 08 - 10 in the Documents folder.
      • Press the ID Shield On-Demand Redaction button.
      • In the middle of processing, confirm the exception Failed to process document '08'! appears on top of the progress dialog.
      • Press OK on the exception.
      • Confirm the prompt Do you wish to continue processing the remaining documents?
      • Press Yes.
      • In the middle of processing, confirm the exception Failed to process document '09'! appears on top of the progress dialog.
      • Press OK on the exception.
      • Confirm the prompt Do you wish to continue processing the remaining documents?
      • Press No.
      • Confirm that processing stops and that verification does not launch.
      • Right-click on document 08 and select "Fields... | Tags".
      • Confirm that ID Shield - Failed Processing is checked and that the comment for the tag contains the exception text.
      • Right-click on document 09 and select "Fields... | Tags".
      • Confirm that ID Shield - Failed Processing is checked and that the comment for the tag contains the exception text.
      • Right-click on document 10 and select "Fields... | Tags".
      • Confirm that none of the ID Shield tags are checked.
      • Rename UCLIDAFValueFinders.dll.temp back to UCLIDAFValueFinders.dll.

    • Test 7: Login behavior for Laserfiche 8.0 and higher

      • Using Windows Authentication (1.0.0.46 Patch C)

        • Use Laserfiche's administration console to add your domain account under Users and Groups | Windows Accounts.
        • Log on to the machine using your domain account.
        • Log into the Laserfiche Client using Windows authentication.
        • With a document selected, press Submit for background redaction.
        • Ensure that the operation commences without displaying a log in prompt.

      • Using the default login registry value (1.0.0.46 Patch C)

        • Remove any keys or values under the HKEY_CURRENT_USER\Software\Extract Systems\Laserfiche\Client Logins registry key.
        • Use Laserfiche's administration console to add your domain account under Users and Groups | Windows Accounts.
        • Log on to the machine using your domain account.
        • Log into the Laserfiche Client using Windows authentication.
        • While still logged in, use the Laserfiche Administration Console to remove your domain username from Users and Groups | Windows Accounts.
        • With a document selected, press Submit for background redaction.
        • Ensure a login prompt is displayed with your domain user account in the username field.
        • Press Cancel on the login prompt.
        • For the registry value HKEY_CURRENT_USER\Software\Extract Systems\Laserfiche\Client Logins\(default) specify *[server]/[repository]/[username] where username is a valid Laserfiche username.
        • With a document selected, press Submit for background redaction.
        • Ensure a login prompt is displayed with the username you specified in the registry value.
        • Enter the correct password and ensure the document submission is processed.
        • With another document selected, press Submit for background redaction.
        • Ensure the submission is processed without displaying a login prompt.
        • In the Laserfiche Client log out of all repositories and log back in under a Laserfiche username (not Windows authentication)
        • For the registry value HKEY_CURRENT_USER\Software\Extract Systems\Laserfiche\Client Logins\(default) specify *[server]/[repository].
        • With another document selected, press Submit for background redaction.
        • Ensure a login prompt is displayed with a blank username field.
        • Press Cancel on the login prompt.
        • While still logged in, use the Laserfiche Administration Console to re-add your domain username to Users and Groups | Windows Accounts.
        • Press Submit for background redaction again.
        • Ensure the submission is processed without displaying a login prompt.

    • Test 8: Special cases

      • Document Window already open

        • Using the Laserfiche Client, open document 09
        • With document 09 still open go back to the main Client window.
        • Ensure document 09 is still selected.
        • Press Verify redaction
        • Confirm that verification launches using the existing document window.
        • Press Stop to cancel the verification.

      • Multiple instances of document prompt

        • Using the Laserfiche Client, open document 09
        • With document 09 still open go back to the main Client window.
        • Open another copy of document09 with the Laserfiche Client.
        • With both copies of document 09 open go back to the main Client window.
        • Ensure document 09 is still selected.
        • Press Verify redaction
        • Confirm the an error is displayed suggesting that multiple copies of the document are open.
        • Close the error and the document windows.

      • Already running prompt

        • Select document 09 in the Laserfiche Client and press the Verify redactions button.
        • Once the document window opens, select the bring up the main Laserfiche Client window again.
        • Select any document or folder and press On-demand redaction.
        • Confirm the message ID Shield for Laserfiche is already running appears.
        • Close the error message.
        • Select any document or folder and press Submt for background redaction.
        • Confirm the message ID Shield for Laserfiche is already running appears.
        • Close the error message.
        • Select any document or folder and press Verify redactions.
        • Confirm the message ID Shield for Laserfiche is already running appears.
        • Close the main client window, then the open document window
        • Confirm that the ID Shield Toolbar closes without error.

      • Multiple Client Windows open

        • Open multiple copies of the Laserfiche Client.
        • In one of the Client windows, select any document or folder.
        • Press On-demand redaction.
        • Confirm the error Multiple Laserfiche Clients are open! is displayed.
        • Press Submit for background redaction.
        • Confirm the error Multiple Laserfiche Clients are open! is displayed.
        • Press Verify redactions.
        • Confirm the error Multiple Laserfiche Clients are open! is displayed.

    • Final checks

      • Check C:\Documents and Settings\Administrator\Local Settings\Temp and ensure no .tif, .uss or .voa files are present that were created during testing.
      • View Exception Log and look for problems.
