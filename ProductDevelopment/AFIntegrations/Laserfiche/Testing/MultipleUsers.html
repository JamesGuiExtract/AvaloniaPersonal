ID Shield for Laserfiche
Multiple Users Testing Checklist

  • Testing Setup

    • Identify at least 3 test machines and testers for this checklist.
    • Confirm that the Large Scale Repository is available on HPQC7.
    • Within the Large Scale Repository,
      • Identify one folder that contains one fewer files than there are testers.
      • Identify a different folder that contains approximately 1000 documents to be used in shared testing between the multiple users.

    • For each test machine:
      • Install ID Shield for Laserfiche on the test machine.
      • Install FKB update 8.2.
      • Generate a Laserfiche Server Redaction license file. Copy the license to the usual CommonComponents folder.
      • Launch USB License Key Manager and point to a machine that contains a USB key (usually Lisa).
      • Launch Rule Tester and verify the About Rule Tester button displays the correct version and FKB numbers.

  • On-Demand Redaction - Direct Conflict

    • Using the Laserfiche Client, log into the Large Scale Repository repository using the "user" account.
    • Copy into a new directory one less documents than there are testers. (make sure to keep a clean copy of the documents in the source directory)
    • Modify the Redaction Settings in the Admin Console such that Automatically initiate document verification after on-demand redaction is unchecked.
    • Each user should select the folder.
    • Simultaneously have each tester press the On-demand redaction button on the Laserfiche Client Toolbar.

    • Confirm that most (or all) testers receive warning messages reporting that some or all of the selected files are locked.
    • Confirm that at least one of the test machines finds unlocked files to process.
    • Confirm that all files get redacted. They may or may not all be redacted by the same machine.

    • Delete the documents in the folder, and recopy the documents from the original location.
    • Repeat this test a 2nd time.

    • Delete the documents in the folder, and recopy the documents from the original location.
    • Repeat this test a 3rd time.

  • Background Redaction - Direct Conflict

    • Delete the documents in the folder, and recopy the documents from the original location.
    • Each user should start the ID Shield background service.
    • Each user should select the folder.
    • Simultaneously have each tester press the Submit for background redaction button on the Laserfiche Client Toolbar.

    • Confirm that most (or all) testers receive warning messages reporting that some or all of the selected files are locked.
    • Confirm that at least one of the test machines finds unlocked files to process.
    • Confirm that all files get redacted. They may or may not all be redacted by the same machine.

    • Delete the documents in the folder, and recopy the documents from the original location.
    • Repeat this test a second time.

    • Delete the documents in the folder, and recopy the documents from the original location.
    • Repeat this test a third time. (after the final test, keep the documents in this folder)

  • Verification - Direct Conflict

    • Using the Laserfiche Client, log into the Large Scale Repository repository using the "user" account.
    • Simultaneously have each tester press the Verify redactions button. on the Laserfiche Client Toolbar.

    • Confirm that all but one of the testers are presented a document for verification.
    • Confirm that the documents that have been presented for verification have the ID Shield - Verifying tag applied to them.
    • Confirm that the final test that did not receive a document receives a "Verification Complete" message
    • Confirm that all other verifiers recieve a "Verification Complete!" message after verifying the document they were presented with.
    • Confirm that all document are marked as verified after the test.

  • Background Redaction - Shared Testing

    • Make sure all testers have their background redaction services stopped.
    • Make sure there are no documents in the repository tagged as ID Shield - Needs Processing prior to starting this test.
    • Prepare a folder that contains approximately 200 times the number of documents as there are testers
    • Each user should select a portion of the documents in the folder that overlaps with the selections of at least one other tester. The goal is for some conflict to exist for each tester and also for each tester to have some unique documents selected. All of the documents in the folder should be selected by at least one user.
    • Simultaneously have each tester press the Submit for background redaction button on the Laserfiche Client Toolbar.

    • Confirm that most testers receive a warning message reporting that some or all of the selected files are locked.
    • Confirm that each of the test machines finds unlocked files to process.
    • Confirm that after submission completes on all machines that all files are tagged as ID Shield - Needs Processing.

    • Have each user start their background redactions services and use task manager to confirm that each begins processing the documents within a couple minutes.

  • On-Demand Redaction - Shared Testing

    • Similar to the previous test, have each tester select an overlapping set of documents such that all testers have some overlap in their selection with other testers yet also have documents that no other users have.
    • This time, only about half of the documents in the folder should be selected in total.
    • Simultaneously have all testers press the On-demand redaction button on the Laserfiche Client Toolbar.

    • Confirm that most or all testers receives a warning message reporting that some or all of the selected files are locked.
    • Confirm that each of the test machines finds unlocked files to process.
    • One or more testers may get a message reporting documents have already been processed. The tester(s) should choose to reprocess these documents.
    • Once on-demand redaction is complete on all machines, wait for background processing to complete if necessary.
    • Confirm that all documents have been tagged as ID Shield - Processed

  • Verification - Shared Testing

    • Have all testers select the folder that contains the shared testing documents
    • Simultaneously have all testers press the Verify redactions button on the Laserfiche Client Toolbar.

    • Confirm that each user continues to be supplied documents for verification
    • As the documents are verified manual image or text redactions should be randomly applied before marking the document as verified. When prompted to review changes to ensure corresponding redactions, alternate between responding Yes or No to review the document again.
    • Confirm that when the verification process completes for all testers that all documents have been verified.
