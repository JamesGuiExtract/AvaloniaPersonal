ID Shield for Laserfiche
Background Service UI Testing Checklist

  • Testing Setup

    • Create a new Windows XP virtual machine with 512 MB memory using either:
      I:\Common\Engineering\VirtualMachines\Laserfiche_72\Laserfiche_72.vhd
      -or-
      I:\Common\Engineering\VirtualMachines\Laserfiche_8\Laserfiche_8.vhd
    • Install ID Shield for Laserfiche.

  • General UI

    • Launch ID Shield Background Service configuration.
    • Confirm the configuration console appears and no exceptions are thrown.

    • Default settings:
      • Confirm Laserfiche Repository displays either ID Shield Demo or LF8.
      • Confirm that there are no other entries in the Laserfiche Repository dropdown.
      • Confirm the Laserfiche User edit box is empty and enabled.
      • Confirm the Laserfiche Password edit box is empty and enabled.
      • Confirm Use as many threads as needed is selected to achieve peak performance is selected by default.
      • Confirm Use [] threads is not selected and that the edit box is disabled by default.
      • Confirm the Status edit box is set to Stopped and that it is read-only (not editable).
      • Confirm the Start button is enabled.
      • Confirm the Stop button is disabled.
      • Confirm Run background service automatically on startup is unchecked.

    • UI Behavior
      • Select User [] threads.
      • Confirm the thread count edit box and spinner control become enabled.
      • Confirm that the edit box contains a default value of 1.
      • Confirm the thread count edit box accepts only digits and no letters or special characters.
      • Confirm the spinner control increments and decrements the thread count edit box.
      • Confirm the spinner control will not decrement the thread count below 1.
      • Set the thread count to 3.
      • Select Use as many threads as needed is selected to achieve peak performance.
      • Confirm that the thread count edit box and spinner control are disabled.
      • Select User [] threads.
      • Confirm the thread count edit box and spinner control are enabled.
      • Confirm the thread count edit box still contains 3.

  • Service Login

    • Failed Login
      • Enter "BadUser" in the Laserfiche User box.
      • Enter "a" in the Laserfiche Password box.
      • Press the Start button.
      • Confirm an exception is displayed reporting a failed login. Confirm that the exception details specify the repository name ID Shield Demo and user BadUser, but that the password is not displayed.
      • Close the exception and confirm "BadUser" is displayed in the Laserfiche User box, but that the Laserfiche Password box is empty.
      • Enter "Admin" in the Laserfiche User box.
      • Enter "badpassword" in the Laserfiche Password box.
      • Press the OK button.
      • Confirm an exception is displayed reporting a failed login.
      • Close the exception and confirm "Admin" is displayed in the Laserfiche User box, but that the Laserfiche Password box is empty.
      • Press Start without entering a password.
      • Confirm an exception is displayed reporting a failed login.
      • Close the exception.
      • Press Cancel.
      • Confirm the ID Shield Background Service configuration console closes without throwing an exception.

    • Login Validation Failure
      • From the Start menu, select ID Shield Background Service.
      • Confirm that the Laserfiche Repositories combo box is pre-populated with ID Shield Demo and that it is selected by default.
      • Confirm that Laserfiche User edit box is empty.
      • Enter "Peon" in the Laserfiche User box.
      • Enter "a" in the Laserfiche Password box.
      • Press the Start button.
      • Confirm the exception Failed to validate Laserfiche Connection! is displayed and that the exception details specify that a necessary privilege is missing.
      • Close the exception and confirm the Laserfiche User and Laserfiche Password boxs remain populated.

  • Applying Settings

    • Open Windows Service Management
      • Right-click on My Computer and select Manage.
      • Select Services under Services and Applications.
    • Find the Extract Systems ID Shield for Laserfiche service.
    • Confirm a startup type of Manual in ID Shield Background Service configuration.
    • Start the ID Shield Background Service configuration console.
    • Specify the appropriate repository.
    • For Laserfiche User enter Admin.
    • For Laserfiche Password enter a.
    • Ensure thread count is set to 3.
    • Check Run background service automatically on startup.
    • Press OK.
    • Confirm that the program closes without exception.
    • In Windows Service Managament, confirm the startup type of ID Shield Laserfiche Background Redaction has been changed to Automatic.

    • From the Start menu, select ID Shield Background Service.
    • Confirm Run background service automatically on startup is checked.
    • Confirm Laserfiche User is set to Admin.
    • Confirm Laserfiche Password is filled in with a hidden character.
    • Confirm the thread count is still set to 3 and that the thread count edit box and spinner are enabled.
    • Uncheck Run background service automatically on startup and press OK.
    • Confirm that the program closes without exception.
    • In Windows Service Managament, confirm the startup type of ID Shield Laserfiche Background Redaction has been changed to Manual.

  • Service Start/Stop

    • Use the ID Shield admin console to configure the repository for use with ID Shield if necessary.
    • From the Start menu, select ID Shield Background Service.
    • Confirm Laserfiche Repository is set to ID Shield Demo.
    • Confirm Laserfiche User is set to Admin.
    • Confirm Laserfiche Password is filled in with a hidden character.
    • Select Use as many threads as needed is selected to achieve peak performance.
    • Press Start.
      • Confirm the Status changes to Starting...
      • Confirm all controls are disabled except OK, Cancel and the Run background service automatically on startup checkbox.
      • Confirm after a couple seconds the Status changes to Running.
      • Confirm the Stop button becomes enabled.
    • Press OK.
    • In Windows Service Managament, confirm the status type of ID Shield Laserfiche Background Redaction has been changed to Started.

    • From the Start menu, select ID Shield Background Service.
    • Confirm Laserfiche Repository is set to ID Shield Demo.
    • Confirm Laserfiche User is set to Admin.
    • Confirm Laserfiche Password is filled in with a hidden character.
    • Confirm Status is Running.
    • Confirm all controls are disabled except Stop, OK, Cancel and the Run background service automatically on startup checkbox.
    • Press Stop.
      • Confirm the Status changes to Stopping...
      • Confirm all controls are disabled except OK, Cancel and the Run background service automatically on startup checkbox.
      • Confirm that within 1 minute the Status changes to Stopped.
      • Confirm the Start button becomes enabled.
      • Confirm all controls are now enabled except the Stop button and the thread count edit box and spinner.

  • Using Windows Authentication (1.0.0.46 Patch C)

    • Use Laserfiche's administration console to add your domain account under Users and Groups | Windows Accounts.
    • Log on to the machine under a different account than your domain account (such as local admin).
    • Open Windows Service Management
      • Right-click on My Computer and select Manage.
      • Select Services under Services and Applications.
    • Find the Extract Systems ID Shield for Laserfiche service.
    • Right-click on the entry and select Properties.
    • On the Log on tab, ensure the service is set to run under the Local System account.
    • Open the ID Shield Background Service console.
    • Enter garbage text in the Laserfiche User and Laserfiche Password fields.
    • Check the Use Windows Authentication option.
    • Ensure the Laserfiche User and Laserfiche Password fields become grayed-out.
    • Press the OK button.
    • Ensure the console closes without error (Note: When using Windows authentication it will not validate the login at this time which is unlike when Windows authentication is not being used).
    • Re-open the ID Shield Background Service console.
    • Ensure when the log in prompt is displayed the Laserfiche User and Laserfiche Password fields are grayed out and the Use Windows Authentication option is checked.
    • Press Start.
    • Confirm that the service fails to start.
    • Temporarily uncheck the Use Windows Authentication option and enter a valid laserfiche user and password, then recheck Use Windows Authentication.
    • Press Start.
    • Confirm that the service fails to start.
    • Go back to the Log on tab for the service and configure it to run under your domain account.
    • Attempt to start the service again.
    • Confirm that this time, the service starts successfully.
    • Stop the service.
    • Temporarily uncheck the Use Windows Authentication option clear the username and password fields, then recheck Use Windows Authentication. (FlexIDSIntegrations:222)
    • Attempt to start the service again.
    • Confirm the service starts successfully.
