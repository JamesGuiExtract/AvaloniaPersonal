ID Shield for Laserfiche
Miscellaneous Testing Checklist

  • PVCS

    • [Integrations #6] Repository list has empty entry
      • Delete HKCU\Software\Extract Systems\Laserfiche\Admin Console.
      • Delete HKLM\Software\Extract Systems\Laserfiche\Service.
      • Start the Admin Console.
      • Confirm that an available repository is displayed in the Repository field.
      • Press the down arrow to expand the repository selection box
      • Confirm that the first entry in the list is a repository name and not a blank entry.

    • [Integrations #17] Verification should bring document window to the front
      • Open the Laserfiche Client.
      • Open a document marked Needs verification.
      • Minimize the document.
      • With the document still highlighted in the client, press the verification button.
      • Confirm that the document is restored and open for verification.

    • [Integrations #23] Annotation exception with layered attributes
      • Open the Laserfiche Client.
      • Add TestImage006.tif from the SCR to the repository.
      • Process the document.
      • Confirm that the document is processed and verified without error.

    • [Integrations #90] Wait dialog for repository search
      • Open the ID Shield Admin Console.
      • Confirm that a brief wait message is presented before the application window appears.
      • Close the Admin Console.

      • Open the ID Shield Background Service.
      • Confirm that a brief wait message is presented before the application window appears.
      • Close the Background Service.

    • [Integrations #105] Redactions on multipage document
      • Open the Laserfiche Client.
      • Locate a multipage image with Needs Processing in the repository (i.e. 04).
      • Process the document with On-demand redaction.
      • Verify the document.
      • Add a manual redaction on page 2.
      • Confirm that the redactions are saved without error.
      • Open the document and confirm that redactions were saved properly.

    • [Integrations #106] Exceptions during verification not displayed
      • Open the Laserfiche Client.
      • Locate an image with Needs Verification in the repository.
      • Open two copies of the document.
      • Verify the document.
      • Confirm that an exception is displayed about multiple instances of the iamge already open.

  • Other

    • Process a multi-page document
      • Open the Laserfiche Client.
      • Add MultipageDocument.tif to the repository from I:\Common\Testing\Product Testing\Laserfiche.
      • OCR the document.
      • Process the document with On-demand redaction.
      • Verify the document.
      • Confirm that each redaction found beyond the first page has associated linked text.

    • Reprocess a document - no duplicated items
      • Open the Laserfiche Client.
      • Process document 01 with On-demand redaction.
      • Confirm that a highlighted clue and an redacted SSN item are found when the document is presented for verification.
      • Verify the document.

      • Process the document again. Accept the document for re-processing.
      • Confirm that a highlighted clue and an redacted SSN item are found when the document is presented for verification.
      • Right-click and delete each item.
      • Confirm that only one copy of each item is present.

    • Exceptions appear in the foreground
      • Rename the rules file.
      • Rename the AFValueFinders DLL.
      • Open the Laserfiche Client.
      • Attempt to process one document with On-demand redaction.
      • Confirm that displayed exception appears on top of all other windows.
      • Close the Laserfiche Client.

      • Correct the name of the rules file.
      • Open the Laserfiche Client.
      • Select a document and process it with On-demand redaction.
      • Confirm that the progress dialog appears and moves to approximately 40%.
      • Confirm that displayed exception appears on top of the progress dialog.
      • Close the Laserfiche Client.
      • Correct the name of the AFValueFinders DLL.

    • Laserfiche 8 : Meaningful exception when no selected repository
      • Open the Laserfiche Client.
      • Select the root LF Repositories item.
      • Click the On-demand redaction button.
      • Confirm that the displayed exception includes meaningful information suggesting that a repository may not be selected.

    • On-demand redaction from various selections
      • Open the Laserfiche Client.
      • Select a repository.
      • Begin processing the repository with On-demand redaction.
      • Stop the processing.

      • Select a folder.
      • Begin processing the folder with On-demand redaction.
      • Stop the processing.

      • Select a single document.
      • Process the document with On-demand redaction.
