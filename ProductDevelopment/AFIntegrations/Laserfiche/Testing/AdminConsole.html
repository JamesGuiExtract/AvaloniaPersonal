ID Shield for Laserfiche
Admin Console Testing Checklist

  • Testing Setup

    • Create a new Windows XP virtual machine with 512 MB memory using either:
      I:\Common\Engineering\VirtualMachines\Laserfiche_72\Laserfiche_72.vhd
      -or-
      I:\Common\Engineering\VirtualMachines\Laserfiche_8\Laserfiche_8.vhd
    • Install ID Shield for Laserfiche.

  • Select Repository Dialog

    • Failed Login
      • From the Start menu, select ID Shield Admin Console.
      • Confirm that the Select Repository dialog appears.
      • Confirm that the Laserfiche Repositories combo box is pre-populated with "ID Shield Demo" and that it is selected by default.
      • Enter "BadUser" in the Laserfiche User box.
      • Enter "a" in the Laserfiche Password box.
      • Press Login
      • Confirm an exception is displayed reporting a failed login. Confirm that the exception details specify the repository name ID Shield Demo and user BadUser, but that the password is not displayed.
      • Close the exception and confirm "BadUser" is displayed in the Laserfiche User box, but that the Laserfiche Password box is empty.
      • Enter "Admin" in the Laserfiche User box.
      • Enter "badpassword" in the Laserfiche Password box.
      • Press Login.
      • Confirm an exception is displayed reporting a failed login.
      • Close the exception and confirm "Admin" is displayed in the Laserfiche User box, but that the Laserfiche Password box is empty.
      • Press Login without entering a password.
      • Confirm an exception is displayed reporting a failed login.
      • Close the exception and confirm the application ends after the 3rd failed login attempt.

    • Cancelled Login
      • From the Start menu, select ID Shield Admin Console.
      • Confirm that the Laserfiche Repositories combo box is pre-populated with "ID Shield Demo" and that it is selected by default.
      • Confirm that Laserfiche User edit box is empty.
      • Enter "Admin" in the Laserfiche User box.
      • Enter "a" in the Laserfiche Password box.
      • Press the Cancel button and confirm the dialog closes and the program ends.

    • Login Validation Failure
      • From the Start menu, select ID Shield Admin Console.
      • Confirm that the Laserfiche Repositories combo box is pre-populated with "ID Shield Demo" and that it is selected by default.
      • Confirm that Laserfiche User edit box is empty.
      • Enter "Peon" in the Laserfiche User box.
      • Enter "a" in the Laserfiche Password box.
      • Press the Login button.
      • Confirm the exception "Failed to validate Laserfiche Connection!" is displayed and that the exception details specify that a necessary privilege is missing.
      • Close the exception and confirm that the application ends.

    • Successful Login
      • From the Start menu, select ID Shield Admin Console.
      • Confirm that the Laserfiche Repositories combo box is pre-populated with "ID Shield Demo" and that it is selected by default.
      • Confirm that Laserfiche User box is pre-populated with "Peon".
      • Confirm that Laserfiche Password box is empty
      • Enter "Admin" in the Laserfiche User box.
      • Enter "a" in the Laserfiche Password box.
      • Press the Login button.
      • Confirm that the login succeeds and the ID Shield Administration Console window is displayed.
      • Press the Cancel button and confirm the Admin Console closes.

    • Manually Specify Server
      • From the Start menu, select ID Shield Admin Console.
      • In the Laserfiche Repositories combo box, enter "Laserfiche-72/ID Shield Demo"
      • Enter "Admin" in the Laserfiche User box.
      • Enter "a" in the Laserfiche Password box.
      • Press the Login button.
      • Confirm that the login succeeds and the ID Shield Administration Console window is displayed.

    • Using Windows Authentication (1.0.0.46 Patch C)
      • Use Laserfiche's administration console to add your domain account under Users and Groups | Windows Accounts.
      • Log on to the machine using your domain account.
      • Open the ID Shield Admin Console.
      • Enter garbage text in the Laserfiche User and Laserfiche Password fields.
      • Check the Use Windows Authentication option.
      • Ensure the Laserfiche User and Laserfiche Password fields become grayed-out.
      • Ensure you are able to log in to the repository.
      • Close and re-open the admin console.
      • Ensure when the log in prompt is displayed the Laserfiche User and Laserfiche Password fields are grayed out and the Use Windows Authentication option is checked.

  • Repository Preparation

    • Confirm the Repository and About tab appear (but not the Redaction Settings tab).
    • Confirm that the About Tags button displays a meaningful message describing each of the following ID Shield tags:
      • ID Shield - Settings
      • ID Shield - Needs Processing
      • ID Shield - Processed
      • ID Shield - Failed Processing
      • ID Shield - Needs Verification
      • ID Shield - Verifying
      • ID Shield - Verified

    • Create Tags
      • Confirm that the Create Tags button is enabled.
      • Press Create Tags and confirm the Admin Console reports that the tags were successfully created.
      • View "MetaData Management | Tags" in Laserfiche's Administration Console to confirm all all of the required ID Shield tags were created.
      • Confirm that the Create Tags button is disabled once the repository contains all tags required for ID Shield.

    • ID Shield settings directory
      • Confirm that the edit box text is Could not find a folder tagged "ID Shield - Settings"!.
      • In the Laserfiche client create a new top-level folder called ID Shield.
      • Use the Fields | Tags context menu to apply the ID Shield - Settings tag to the ID Shield folder.
      • In the ID Shield Admin Console, press the Refresh button.
        • Confirm that a wait cursor appears while the refresh is in progress.
        • Confirm "\ID Shield" is displayed as the settings folder following the refresh.
        • Confirm that the Redaction Settings tab appears and that the About tab remains the the last tab in the tab order.
      • Tag an additional Laserfiche folder with ID Shield - Settings
      • Press Refresh.
        • Confirm that a wait cursor appears while the refresh is in progress.
        • Confirm that the ID Shield settings directory edit box reads Multiple folders are tagged "ID Shield - Settings"!.
        • Confirm that the Redaction Settings tab disappears.
      • Remove the ID Shield - Settings tag from the second folder.
      • Press the Refresh button.
        • Confirm that a wait cursor appears while the refresh is in progress.
        • Confirm the edit box once again displays "\ID Shield".
        • Confirm that the Redaction Settings tab re-appears.

  • Redaction Settings

    • Confirm the following default settings:
      • (empty): Redaction Rules.
      • Checked: Redact high confidence data.
      • Checked: Redact medium confidence data.
      • Checked: Redact low confidence data.
      • Checked: Automatically tag processed documents for verification.
      • Checked: Tag all documents that are processed.
      • Un-checked: Tag only documents that contain evidence of sensitive data or clues.
      • Checked: Automatically initiate document verification after on-demand processing if one or more documents were tagged for verification.
      • Checked: Upon verification of a document ensure a corresponding text or image redaction exists for all manually added redactions.

    • Confirm the Redaction rules edit box is blank and enabled by default.
    • Press the redaction rules Browse button ("...").
      • Confirm that a an Open - File dialog is displayed.
      • Confirm Ruleset definition files (*.rsd;*.rsd.etf) is the default filetype.
      • Confirm Ruleset definition files (*.rsd) is available as a filetype.
      • Confirm Encrypted ruleset files (*rsd.etf) is available as a filetype.
      • Confirm All Files (*.*) is available as a filetype.
    • Browse to and select the file:
      \\fnp\internal\Common\Testing\Product Testing\Laserfiche\Rules\ID Shield for LF.rsd.etf
    • Confirm that the path to the selected file is displayed in the Redaction rules edit box.

    • On the Redaction Settings tab, enter "BadPath" in the Redaction Rules box.
    • Press Apply.
    • Confirm an exception reports that the rules file cannot be found.
    • Press OK.
    • Confirm an exception reports that the rules file cannot be found and that the Admin Console does not close.
    • In the Redaction Rules box, enter: \\fnp\internal\Common\Testing\Product Testing\Laserfiche\Rules\ID Shield for LF.rsd.etf
    • Press Apply and confirm that no exception is displayed.

    • Uncheck Automatically tag processed documents for verification and verify all other settings in the Verification group box except the ensure corresponding redactions box are disabled.
    • Check Automatically tag processed documents for verification again and verify all controls in the Verification group box are re-enabled and retain the setting they had prior to being disabled.
    • Press OK and confirm the Admin Console closes without error.

  • Startup warning message

    • Open the ID Shield Admin Console.
    • After logging in, confirm an OK/Cancel prompt is displayed warning against making settings changes or performing Admin operations while ID Shield is processing.
    • Press Cancel and confirm the Admin Console closes.
    • Open the ID Shield Admin Console again.
    • This time, when the prompt is displayed, press OK.
    • Confirm the Admin Console is displayed.

  • About tab

    • Confirm the tab displays the correct product version.
    • Confirm the tab displays the correct FKB version.
    • Confirm the tab displays up-to-date copyright info.
    • [Integration #100] Confirm the tab displays the licensing level.
      • Not licensed
      • Verification license
      • Desktop redaction license
      • Server redaction license

  • Change Settings Directory

    • Open the ID Shield Admin Console.
    • Press the Change button.
    • Confirm a wait cursor is displayed prior to the Change ID Shield Settings Directory dialog appearing.
    • Confirm that the dialog is pre-populated with the existing path of \ID Shield.
    • Enter \BadPath in the dialog.
    • Press OK to close the dialog.
    • Confirm that an exception reports Invalid Laserfiche Directory!

    • Press the Change button.
    • Enter \BadPath in the dialog.
    • Press Cancel and confirm that the dialog closes without exception.

    • In the Laserfiche Client create a new top-level folder called New Settings Folder.
    • In the ID Shield Admin Console's Repository tab, press the Change button.
    • Enter \New Settings Folder in the dialog.
    • Press OK.
    • Confirm a wait cursor is displayed.
    • Confirm once the wait cursor ends, the ID Shield settings directory edit box reads: \New Settings Folder.
    • On the "Redaction Settings" tab, de-select Redact low confidence data.
    • Press OK to close the ID Shield Admin Console.
    • Reopen the Admin Console and go to the "Redaction Settings" tab.
    • Confirm that the Redact low confidence data checkbox remains un-checked.
    • Use the Change button to change the settings directory back to "\ID Shield".
    • On the Redaction Settings tab, confirm that Redact low confidence data is checked once again.

  • Repository Maintenance

    • In the Laserfiche Client, open the Documents folder.
    • Select documents 01 through 03
    • Using the "Action | View Tags..." menu, mark all three documents as ID Shield - Needs Processing.
    • Select documents 04 through 06.
    • Using the "Action | View Tags..." menu, mark all three documents as ID Shield - Processed and ID Shield - Needs Verification.
    • Right-click on document 07 use the "Fields..." menu to apply only the ID Shield - Processed tag.
    • Select documents 08 through 10.
    • Using the "Action | View Tags..." menu, tag all documents as ID Shield - Failed Processing.

    • Remove "Needs Processing" tag from all documents.

      • Open document 01 and make an edit, but don't save the document
      • In the ID Shield Admin Console's Repository tab, press Remove "Needs Processing" tag from all documents.
      • Confirm a Yes/No prompt confirming the action is displayed.
      • Press No to the prompt and confirm the documents are not processed.

      • Press Remove "Needs Processing" tag from all documents again.
      • Press Yes on the first prompt
      • Confirm a second Yes/No prompt is displayed warning that the action cannot be undone
      • Press No to the prompt and confirm the documents are not processed.

      • Press Remove "Needs Processing" tag from all documents again.
      • Press Yes on both confirmation prompts.
      • Verify that a dialog reading Removing "Needs Processing" tag from all documents... is displayed while the operation in progress.
      • Verify the message Failed to remove \"Needs Processing\" tag from 1 document(s)! See log for details.
      • View the exception log and verify that it reports that is failed to process document 01 because the entry was locked.
      • Close document 01 (discard the edit if prompted)

      • Press Remove "Needs Processing" tag from all documents again.
      • Press Yes on both confirmation prompts.
      • Verify the operation completes without error.
      • Verify document 01 and 03 are no longer tagged ID Shield - Needs Processing.
      • Verify document 04 through 06 remain tagged as ID Shield - Processed and ID Shield - Needs Verification.
      • Verify document 07 remains tagged as ID Shield - Processed
      • Verify document 08 thru 10 remain tagged ID Shield - Failed Processing

    • Tag all "Processed" documents as "Verified".

      • Open document 04 and make an edit, but don't save the document
      • In the ID Shield Admin Console's Repository tab, press Tag all "Processed" documents as "Verified".
      • Confirm a Yes/No prompt confirming the action is displayed.
      • Press No to the prompt and confirm the documents are not processed.

      • Press Tag all "Processed" documents as "Verified" again.
      • Press Yes on the first prompt
      • Confirm a second Yes/No prompt is displayed warning that the action cannot be undone
      • Press No to the prompt and confirm the documents are not processed.

      • Press Tag all "Processed" documents as "Verified" again.
      • Press Yes on both confirmation prompts.
      • Verify that a dialog reading Marking \"Processed\" documents as \"Verified\"... is displayed while the operation in progress.
      • Verify the message Failed to tag 1 "Processed" document(s) as "Verified"! See log for details. is displayed.
      • View the exception log and verify that it reports that is failed to process document 04 because the entry was locked.
      • Close document 04 (discard the edit if prompted)

      • Press Tag all "Processed" documents as "Verified" again.
      • Press Yes on both confirmation prompts.
      • Verify the operation completes without error.
      • Verify documents 01 thru 03 are untagged.
      • Verify documents 04 through 07 are now tagged with both ID Shield - Processed and ID Shield - Verified but not ID Shield - Needs Verification.
      • Verify document 08 thru 10 are tagged ID Shield - Failed Processing

    • Tag all "Failed Processing" documents as "Needs Processing".

      • Open document 08 and make an edit, but don't save the document
      • In the ID Shield Admin Console's Repository tab, press Tag all "Failed Processing" documents as "Needs Processing".
      • Confirm a Yes/No prompt confirming the action is displayed.
      • Press No to the prompt and confirm the documents are not processed.

      • Press Tag all "Failed Processing" documents as "Needs Processing" again.
      • Press Yes on the first prompt
      • Confirm a second Yes/No prompt is displayed warning that the action cannot be undone
      • Press No to the prompt and confirm the documents are not processed.

      • Press Tag all "Failed Processing" documents as "Needs Processing" again.
      • Press Yes on both confirmation prompts.
      • Verify that a dialog reading Marking "Failed Processing" documents as"Needs Processing"... is displayed while the operation in progress.
      • Verify the message Failed to tag 1 "Failed Processing" document(s) as "Needs Processing"! See log for details. is displayed.
      • View the exception log and verify that it reports that is failed to process document 08 because the entry was locked.
      • Close document 08 (discard the edit if prompted)

      • Press Tag all "Failed Processing" documents as "Needs Processing" again.
      • Press Yes on both confirmation prompts.
      • Verify the operation completes without error.
      • Verify documents 01 thru 03 are untagged.
      • Verify documents 04 through 07 are tagged with both ID Shield - Processed and ID Shield - Verified.
      • Verify document 08 thru 10 are tagged ID Shield - Needs Processing
