ID Shield for Laserfiche
Endurance Testing Checklist

  • Testing Setup

    • Machine 1: Server
      • Use a machine with a server OS that supports 4+ threads.
      • Install Laserfiche server 7.2, Laserfiche Client 7.2 and the Laserfiche Administration Console.
      • See Large Scale Repository Configuration to set up a repository with 200,000 documents and about 400,000 pages.
      • Create a directory to store statistics and copy the file I:\Common\Testing\Product Testing\Laserfiche\LF_Stats.bat to it.

    • Machine 2: Submission & Supplying Machine
      • A faster machine will be helpful, but only one core is needed as the work done on this machine will be single-threaded. If a multi-core machine is used, the Laserfiche 7.2 VM and Laserfiche 8 VM described below can be run on the same machine.
      • Install Laserfiche Client 7.2
      • Install ID Shield for Laserfiche.
      • Move the file I:\Common\Testing\Product Testing\Laserfiche\Rules\ID Shield for LF.rsd.etf to a directory on the network that is readable without any authenticaion.
      • Use ID Shield Admin Console to configure the Redaction Settings as follows:
        • Redaction Rules: Specify the rules file relocated in the previous step.
        • Checked: Redact high confidence data.
        • Checked: Redact medium confidence data.
        • Checked: Redact low confidence data.
        • Checked: Automatically tag processed documents for verification.
        • Un-Checked: Tag all documents that are processed.
        • Checked: Tag only documents that contain evidence of sensitive data or clues.
        • Un-Checked: Automatically initiate document verification after on-demand processing if one or more documents were tagged for verification.
        • Checked: Upon verification of a document ensure a corresponding text or image redaction exists for all manually added redactions.
      • Create a directory to store statistics and copy the file I:\Common\Testing\Product Testing\Laserfiche\LF_Stats.bat to it.

    • Machine 3 - 6: Background Processing Machines
      • Use a machines that supports 4+ threads including one that supports 8+ threads
      • Ensure no Laserfiche software is installed.
      • Install ID Shield for Laserfiche.
      • Use ID Shield Background Service configuration to ensure the Large Scale Repository is visible (but do not start the service yet)
      • Create a directory to store statistics and copy the file I:\Common\Testing\Product Testing\Laserfiche\LF_Stats.bat to it.

    • Laserfiche 7.2 VM
      • Create a new Windows XP virtual machine with 512KB memory using I:\Common\Engineering\VirtualMachines\Laserfiche_72\Laserfiche_72.vhd
      • Stop and disable the Laserfiche Server 7.2.1 service.
      • Install ID Shield for Laserfiche.
      • Open the Laserfiche Client and ensure it can connect to the Large Scale Repository.
      • Add the ID Shield buttons to the toolbar. (see Installation and Licensing for details)
      • Create a directory to store statistics and copy the file I:\Common\Testing\Product Testing\Laserfiche\LF_Stats.bat to it.

  • Starting the Test

    • Machine 3 - 6 :Start Background Processing
      • Open ID Shield Background Service configuration.
      • Select the Large Scale Repository
      • Specify username "User", password "a" or a user with appropriate privileges to process files for redactions.
      • Run the LF_Stats.bat file to start logging
      • Select Use as many threads as needed to achieve peak performance
      • Start the service and ensure the status changes to Running
      • Close ID Shield Background Service configuration.

    • Machine 2: Submit files for background processing
      • Run the LF_Stats.bat file to start logging
      • Open the Laserfiche Client and select the root of the repository.
      • Press the Process in background with ID Shield button.
      • Ensure ID Shield prompts that more that 50,000 documents have been selected and that only 50,000 can be processed at one time.
      • Ensure any files ID Shield says it can't process are a small fraction (10% or less) of those selected and that the reasons specified seem reasonable.
      • For all but the first submission, ensure the prompts with the number of processed and failed documents seem reasonable and select No to avoid reprocessing them.
      • Once the first batch is submitted, repeat until all 200,000 documents are submitted. NOTE: As documents continue to be processed, processed documents will make up a progressively larger amount of each 50,000 document batch. If more than half the documents or so are processed before the first 200,000 documents are submitted, you can use the following advanced search with the Laserfiche Client to first find all unprocessed documents: {LF:Name="*", Type=D} - {LF:Tags="ID Shield - Processed"}. Press Ctrl + A to select all search results and press the submit button.

    • Laserfiche 7.2 VM:
      • Run the LF_Stats.bat file to start logging
      • Select about 500 documents that were not submitted for background processing on other machines
      • Press the Process with ID Shield button to immediately redact the documents in the current Laserfiche Client session.
      • Ensure processing starts successfully

  • On a daily basis:

    • Machine 1
      • Day 1
        • Via a Laserfiche Client, create folder 02 in the documents folder
        • Open folder 02 and select the File | Import... menu option
        • Click Options... in the lower right and ensure the following settings:
          • Unchecked: Always show Document Metadata dialog
          • Unchecked: OCR/Extract text
          • Select: (Save images as) one page documents
          • Unchecked: New text page every [x] lines
          In the I:\Common\Testing\Product Testing\Laserfiche\LargeScaleDatabase directory select Large Scale Repository Part2.lfb and press Import. (The imported documents will already be tagged ID Shield - Needs Processing.)
      • Day 2
        • Via a Laserfiche Client, create folder 03 in the documents folder
        • Open folder 03 and select the File | Import... menu option
        • Import Large Scale Repository Part1.lfb.
      • Day 3
        • In the Laserfiche client open folder 03 and select the File | Import... menu option
        • Import Large Scale Repository Part2.lfb. (The imported documents will already be tagged ID Shield - Needs Processing.)
      • Day 4+
          The repository now contains more than 1 million pages and does not need any more images.

    • Machine 2
      • Day 1
        • In the Laserfiche Client, select Tools | Options | New Documents...
        • Ensure always show metadata dialog is unchecked and OCR/Extract Text is checked.
        • Open folder 02 in the documents folder
        • Open the folder I:\Common\Testing\Product Testing\Laserfiche\LargeScaleDatabase\Files\Part1
        • Select all the folders contained in Part1 (in Windows explorer) and drag them into the Laserfiche Client.
      • Day 2 +
        • If the document OCR'ing process is still going, let it run
        • If the document OCR'ing is complete or if every document page is reporting an error:
          • Cancel the process if it is still running
          • Select or search for documents needing submission
          • Press the Process in background with ID Shield button.
          • Ensure any files ID Shield says it can't process are a small fraction (10% or less) of those selected and that the reasons specified seem reasonable.
          • Submit the documents & repeat as needed.

    • Machine 3 - 6:
      • Once Daily (starting on day 2)
        • Open ID Shield Background Service configuration and ensure the service is still running.
        • Close ID Shield Background Service configuration.
        • Check the statistics generated by LF_States.bat and look for any obvious problems.
        • View Exception Log and look problems. NOTE: The pre-assembled Large Scale Repository has many problems including missing image files and missing spatial info for OCR'd test. The logs will likely report many problems. Spot check these problems to ensure they make sense given the document being processed. (ie, "Cannot find file" if the Laserfiche document is missing the image file or a warning about inaccurate/missing text redactions if the document's text is missing spatial info. The later case can be verified by attempting to create a linked redaction using the Laserfiche client-- it will fail if the text is missing spatial information)

    • Laserfiche 7.2 VM:
      • Once Daily (starting on day 2)
        • Ensure the "Processing Complete!" message box is displayed.
        • If processing has not finished, determine if it is reasonable that processing has not finished (ie, documents with a large number of pages), and press Stop to interrupt processing.
        • In the Laserfiche Client, select the root of the repository and press the Verify ID Shield Redactions button. Verify ID Shield Redactions button and confirm that the verification process is started.
        • Verify documents
          • Ensure each document displayed for verification either has automatically added redaction zones or highlighted clues.
          • On every other document either add a new redaction zone to the image or text or delete an existing one.
          • Press the Verify button on the toolbar to mark the current document as verified.
          • When manual redactions are added, confirm ID Shield prompts that the redactions were updated to properly link image and text redactions.
          • After a couple dozen documents have been verified, press the stop button.
        • If there are a significant number documents in the repository that have not yet been submitted and machine 6 is busy supplying documents
          • Select or search for documents needing submission
          • Press the Process in background with ID Shield button.
          • Ensure any files ID Shield says it can't process are a small fraction (10% or less) of those selected and that the reasons specified seem reasonable.
          • Submit the documents & repeat as needed.
        • Select ~500 documents
        • Press the Process with ID Shield button to immediately redact the documents in the current Laserfiche Client session. (make sure most have not yet been processed)
        • Ensure processing starts successfully

  • After the entire repository is processed

    • 2 of the 4 background service machines as well as the virtual machines can be released. Be sure to keep the background service running uninterrupted on the background machines.
    • Reset the repository
      • Use the utility I:\Common\Testing\Product Testing\Laserfiche\LFTool.exe to remove ID Shield tags, redactions and annotations from all documents in the repository.
    • Use Machine 2 to start re-submitting all documents for processing.
      • The first 2 times, do so by selecting the repository as whole ensuring that the documents are processed in 50,000 document batches.
      • Subsequent submissions can be done by selecting specific subfolder or search results to avoid getting too many processed documents in the batches.
      • Continue submitting until all the documents have once again submitted for processing.
    • Let the 2 background service machines process the second round of documents for a minimum of 72 hours brining the total "up" time to a total of at least one week.
