SharePoint Testing - Installation and Setup

  1. Installation

    • Install and license ID Shield or Flex Index.
    • Note: The current FKB update is installed as part of Flex/IDS 9.0. A manual installation is only required when using ID Shield 8.0.

    • Note: To install SharePoint, you must be using a machine running 64-bit Server 2008 R2
    • Install SharePoint Foundation 2010. The installer can be found at \\Jake\Applications\Microsoft SharePoint\2010 Foundation. The associated software prerequisites must be installed first.

    • Extract Exception service:
      • For FLEX Index or ID Shield 9.0
        • The Extract Exception service is installed.
        • Configure the service to run as a privileged user. Note that the default Network Service account is sufficient.
      • For FLEX Index or ID Shield 8.0
        • Install .NET 4.0.
        • Put ExceptionHelper.exe in the CommonComponents folder.
        • Install the exception service by running InstallUtil.exe (part of the .NET 4.0 framework) on the service .EXE.
        • Configure the service to run as a privileged user. Note that the default Network Service account is sufficient.

    • Enable multiple-document uploads and the Windows Explorer view:
      • Install a trial version of Microsoft Office 2010 or install the SharePoint Designer. The Sharepoint Designer can be found at \\Jake\Applications\Microsoft SharePoint.
      • In the server management console, enable the Desktop Experience feature.
        • Open Server Manager MMC.
        • Select Features.
        • Click Add Features.
        • Check Desktop Experience.
        • Click the Next button and finish the installation.

    • Install Microsoft Silverlight to enhance the appearance of the SharePoint UI.

    • To install ID Shield for SharePoint:
      • Make a local copy of the appropriate ID Shield for SharePoint build folder.
      • Confirm that the SharePoint Administration service has started.
      • Open the SharePoint Management Shell from the start menu.
      • In the management shell, run the iisreset command.
      • In the management shell, run the appropriate install script(s) from the ID Shield for SharePoint build folder.
        • InstallDataCaptureForSharePoint.ps1 and/or
        • InstallIDShieldForSharePoint.ps1
      • If appropriate, run the .bat file to install the ID Shield for SharePoint Client.
        • The batch file copies 5 DLLs and IDShieldForSPClient.exe to the CommonComponents folder.

  2. SharePoint Setup

    • Open SharePoint Central Administration from the Start menu. Note that you will need to enter appropriate credentials.
    • Click Application Management
      • Configure Extract Systems ID Shield
        • the SharePoint working folder,
        • the Location for ID Shield for SharePoint Client FPS file,
        • the Extract exception service IP address
        • the Delay in minutes before exporting,
        • the Random folder name length
      • Click Save.
      • [Integrations #258] Confirm that the ID Shield logo and the current version number are each presented on the settings dialog.
      • Confirm that settings are persistent.

      • Configure Extract Systems Data Capture
        • the SharePoint working folder,
        • the Extract exception service IP address
        • the Delay in minutes before exporting,
        • the Random folder name length
      • Click Save.
      • Confirm that settings are persistent.

    • Navigate to the appropriate SharePoint site (i.e. http://hpqc3) as a user with admin privileges.
    • Click Site Actions > Site Settings
    • Go to Site Collection Features.
    • Activate the Extract Systems ID Shield feature.
    • Activate the Extract Systems Data Capture feature.
    • Create a new document library.
    • Navigate to the document library and confirm that appropriate ribbons are available under Library Tools.
      • ID Shield
      • Data Capture

  3. Site Permissions

    • General
      • Confirm that the following SharePoint Groups are defined.
        • Extract Data Capture Administrators
        • Extract ID Shield Administrators
        • Team Site Members
        • Team Site Owners
        • Team Site Visitors
      • Confirm that Extract Data Capture Administrators contains the following entries.
        • <machine_name>\administrator
      • Confirm that Extract ID Shield Administrators contains the following entries.
        • <machine_name>\administrator
      • Confirm that Team Site Members contains the following entries.
        • <none>
      • Confirm that Team Site Owners contains the following entries.
        • <machine_name>\administrator
      • Confirm that Team Site Visitors contains the following entries.
        • <none>

    • Basic Site Access
      • Confirm that with the default group entries as listed above, only the local administrator can connect to the SharePoint site.
      • Confirm that adding a user or group to the Team Site Members group allows a user to access the SharePoint site.
      • Confirm that a user must be a member of the Team Site Owners group in order to view and modify Site Permissions.
