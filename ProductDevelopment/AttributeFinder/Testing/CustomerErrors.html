Customer Errors Testing Checklist

  1. [LegacyRC #6242, #6249] - Old Republic Title : Verification Navigation

    • Goal
      • Confirm that the PendingActionStatusChange table works properly.
      • Confirm that processing of ID Shield files after verification is complete works properly.
      • Confirm that verification changes to previously verified files are correctly handled.

    • Test Setup
      • Prepare a database and a collection of images for ID Shield processing and verification.
        • Use Process for processing
        • Use Verify for verification
        • Use Redact for creating an output image
        • Use Cleanup for deleting intermediate files
      • Create suitable FPS files for operation with each action defined above.
      • Verify some images and confirm that appropriate output images are created.

    • Test 1
      • Verify an image, adding one or more manual redactions and confirm that an appropriate output image is created.
      • Navigate back to a previous document and remove one or more redactions.
      • Confirm that a new output image is created that contains the updated collection of redactions.

    • Test 2
      • While processing files, run another FAM with a different action that sets files from the first FAM to
        • Unattempted
        • Pending
        • Completed
        • Failed
        • Skipped
      • Also confirm that this exercise works if the processing file from FAM #1 is cancelled.
      • Also confirm that this exercise works if FAM #1 was processing Skipped files and the processing file is cancelled.
      • Also confirm that this exercise works if the processing file from FAM #1 fails.

    • Test 3
      • Process an image for verification.
      • Process the image in a different FAM
        • Set the status of the file first to one status and then to a second before processing completes in the first FAM.
        • Confirm that the second status is applied.

      • Repeat the above test and stop processing in the first FAM before all files retrieved for processing are completed.
      • Confirm that the second status is applied to the files that were not actually processed by the first FAM.

      • Repeat the above test but kill the first FAM before processing is complete.
      • Confirm that the second status is applied after auto-revert.

    • Test 4
      • Confirm that the ChangeStatus column in the QueuedActionStatusChange database table is set to I for each place where queued events are not respected.
        • Set file action status from DB Admin
        • Copy action status from DB Admin
        • When files are deleted from the database
