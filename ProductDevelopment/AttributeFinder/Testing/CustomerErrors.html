Customer Errors Testing Checklist

  1. [LegacyRC #4434] - FACC - Exits Without Completing Processing

    • Test Setup - Follow instructions in \\fnp\internal\Common\Testing\PVCS_Testing\P13\4434\Readme.txt

    • Exercise the test at least 50 times on SuperTester or UltraTester and review results to see if processing failed.

  2. [FlexIDSCore #2482, #2485, #2487, #2488] - Cott - Various Errors During Processing

    • Test Setup
      • Use the latest rules delivery from PA - Adams - Cott.
      • Copy several thousand PA - Adams images to the test machine.
      • Create an FPS file to supply and process these images.
      • Exercise the FPS file for several hours - at least overnight.

    • Test Results - the following errors should NOT be seen
      • ELI14973 - not enough storage is available to complete this operation
      • ELI06970 - unable to create file storage object
      • ELI03837 - COM Exception caught
      • ELI04367 - COM Exception caught

  3. [LegacyRC #4937] - Cott - Delayed-Write Errors During Processing

    • Test Setup
      • Create a DelayedWrite_Verify database on UltraTester or another server-class machine. Add two actions: Compute and Verify.
      • Confirm that the test machine has delayed-write caching enabled. This setting is displayed via the Policies tab on the disk drive Properties dialog and can be accessed from the Device Manager window.
      • Copy at least 100,000 redaction samples images to the test machine.
      • Copy a current Redaction customer ruleset to the test machine.
      • Create an FPS file to supply and process these images. Two processing tasks should be defined: Execute Rules and Conditionally execute task(s). The conditional task should use the ID Shield Data File Contents condition with a Set File Action Status task to set the Verify action to Pending if sensitive data is found.

    • Test Execution
      • Exercise the FPS file for several days and confirm that no processing failures due to a sharing violation were seen. During initial pre-fix testing, M failures were seen during a test of N images.

  4. [LegacyRC #5050, #5078, #5088, #5092] - Internal - File Access Errors During Processing

    • Test Setup
      • Locate a collection of several thousand sample images on FNP2.
      • Create an FPS file to queue the images.
      • Create an FPS file to process the images. Include an OCR document task, an Execute rules task and a Conditionally evaluate VOA file task. If the file is sensitive, set the Verify action to Pending. If the file is not sensitive, set the Export action to Pending.
      • Create an FPS file to Export the images. The FPS file should include several Delete tasks to remove the image file, the USS file, the VOA file, and the XML file.

    • Test Execution
      • Exercise the FPS file for several days and confirm that no file access exceptions are seen in the UEX logs.
      • Note that occasional ELI24062 exceptions noting retries for file reading are expected. Other related ELI codes may also be seen.
