Customer Errors Testing Checklist

  1. [FlexIDSCore #3609, LegacyRC #5778] - CA - Santa Barbara - DFM : Various out-of-memory errors during processing

    • Test Setup
      • Prepare a server-class machine such as UltraTester for this test.
      • Both Indexing and Redaction rules will be exercised on the Santa Barbara images.
      • Note that separate databases are used for DayForward and for BackFile processing.
      • Note that the testing scenerio does not include any verification since the out-of-memory errors were already reproducible.
      • Review the multiple FPS files found at I:\Common\Testing\PVCS_Testing\FlexIDSCore\3609
        • Queue-Process.fps - watches a UNC share to supply images for processing against Indexing rules
        • Process.fps - OCR's images, runs Indexing rules, copies the image to another folder, copies the XML output from the rules to yet another folder, and sets the Redaction action to Pending
        • Process-Redaction.fps - runs the Redaction rules, has a conditional task to set Verify to Pending if this is a sensitive document, otherwise sets the Cleanup action to Pending.
        • CleanUp.fps - sleeps for 5 minutes to give verifiers an opportunity to navigate the historical documents; deletes the metadata XML, USS, VOA, and source documents
        • Queue-BackRedaction.fps - watches a UNC share
        • Process-BackRedaction.fps - OCR's images, runs the Redaction rules
        • CleanUp-BackRedaction.fps - sleeps, deletes unnecessary files
      • Copy a set of CA - Santa Barbara images to a testing folder. Prepare other sets for copying into the listening folders.
      • Exercise the FPS files for at least a few days.

    • Test Results - the following errors should NOT be seen
      • ELI04217 - Exception of type 'System.OutOfMemoryException' was thrown
      • ELI05716 - Exception of type 'System.OutOfMemoryException' was thrown
      • ELI06432 - Out of memory
      • ELI08206 - Out of memory
      • ELI12967 - COM Exception caught
      • ELI15668 - Exception of type 'System.OutOfMemoryException' was thrown

      • [LegacyRC #5778] There also should not be any situations where the exception log is stuck with multiple processes trying to write at the same time.
