FLEX Index / ID Shield Installation Testing
For Each Operating System

Each operating system shall be tested from a clean Clonezilla image or Virtual Machine.

The .NET and SQL Server installations can be exercised from either the LabDE, FLEX Index or the ID Shield DemoShield application.

  • .NET 3.5 and 4.0 Installation
    • Confirm that the Demo Shield text fields do not overlap when selecting other items.
    • Confirm that the installation finishes without error.
    • [JIRA #7540] Silent Installs Should Install .NET
      • If software installation is via Silent Install, confirm that the required .NET elements are also included in the installation.
      • Confirm that if .NET cannot be installed, the script reports an error.

  • SQL Server Installation
    • Confirm that each of the following items are available and can be installed from the Demo Shield install.
      • SQL Server Applications
        • Windows Installer 4.5
        • SQL Server 2014
        • SQL Management Studio
    • Note that SQL Server 2014 cannot be installed on the following operating systems.
      • Windows XP
      • Windows Vista
      • Server 2003

  • Create a license file for LabDE, FLEX Index and ID Shield.
  • Create a local user on the test machine. Do not add the user to the Administrators group.
  • Add the new user to the Remote Desktop Users group.

  • Confirm that elevated permissions are requested during installation of Extract Systems software if administrator is not the logged in user.
  • With a clean installation, confirm that FPS, USS, and VOA files are all correctly associated with appropriate icons and Extract Systems programs.
  • [JIRA #12034] Confirm that all appropriate file suppliers and FAM tasks are available for selection and configuration without an extra RegisterAll.bat.

  • FLEX Index Installation
    • Confirm that the Demo Shield text fields do not overlap when selecting other items.
    • FKB installer
      • Confirm that the ComponentData rules are installed to a named subfolder such as Extract Systems\FlexIndexComponents\ComponentData\11.1.0.123.
      • Confirm that the FKB version and install directory includes a major version, minor version, a patch number, and build number.

    • Confirm that the following items are available from the Demo Shield install. Also confirm that an appropriate message is provided in the right-hand pane after a mouse-over event on the product name.
      • Microsoft .NET 3.5 and 4.0
      • SQL Server Applications
        • Windows Installer 4.5
        • SQL Server 2014
        • SQL Management Studio
      • Install FLEX Index
      • Extract Systems License Manager

    • Confirm that copyright dates include the appropriate year for the following
      • FLEX Index installation splash screen
      • FLEX Index installation licensing page
    • Confirm that FLEX Index is installed without error.

    • Install FLEX Index in Custom mode and install redistributable components. Confirm that the following merge modules are installed.
      • DataEntry.msm
      • ExtractCommonMM.msm
      • ExtractFlexCommonMM.msm
      • UCLIDFlexIndex.msm
    • Confirm via the Control Panel that the following external programs have been installed along with FLEX Index.
      • ClearImage 7 PDK
      • Crystal Reports 2008 Runtime SP1
      • Microsoft SQL Server Compact 3.5 SP2
      • Microsoft SQL Server Compact 4.0
    • Confirm that the following files have been installed in the CommonComponents folder.
      • DataEntryApplication.FlexIndex.resources
    • Confirm via the Control Panel that the following services programs have been installed with FLEX Index.
      • Extract Systems Exception Service
      • Extract Systems FAM Service
      • Extract Systems FDRS Service
      • Extract Systems IPC Service
    • Confirm that Extract Systems Application Monitor Service has not been installed.
    • Run Install ESAppMonitorService.bat from the CommonComponents folder.
      • Confirm that Extract Systems Application Monitor Service has been installed.

  • Uninstalls
    • Login as a user that is a member of the Administrators group.
    • Confirm that RDT, LabDE, FLEX Index, ID Shield and Extract Systems LM are installed.
    • Confirm that the appropriate FKB version is also installed.
    • Confirm that FLEX Index cannot be uninstalled.
    • Confirm that ID Shield cannot be uninstalled.
    • Confirm that LabDE cannot be uninstalled.
    • Confirm that the FKB update uninstalls without error.
    • Confirm that RDT uninstalls without error.
    • Uninstall LabDE.
    • With FLEX Index, ID Shield and Extract Systems LM installed, confirm that Extract Systems LM uninstalls without error.
      • Confirm that DB Administration still runs.
      • Confirm that File Action Manager still runs.
      • Confirm that USB License Key Manager still runs.

    • If necessary, create and use an evaluation license for ID Shield. Any non-expiring licenses should be renamed or deleted.
    • Confirm that DB Administration still runs.
    • Confirm that ExtractTRP2.exe is running.
    • Close DB Administration.
    • Uninstall ID Shield.

    • Confirm that FLEX Index Help is still accessible from the Start menu.
    • Confirm that File Action Manager still runs.
    • Uninstall FLEX Index.
      • Confirm that a prompt is presented about uninstalling Crystal Reports.
        • Confirm that Crystal Reports is uninstalled if selected.
      • Confirm that a prompt is presented about uninstalling SQL Compact Edition.
        • Confirm that SQL Compact Edition is uninstalled if selected.
      • Confirm that ClearImage 7 PDK is uninstalled without error.
    • Confirm that when you uninstall all Extract Systems software, all Extract Systems services (FAM service, FDRS service, IPC, exception service) are uninstalled.

  • Miscellaneous
    • [LegacyRC #6272] Confirm that OCR functionality remains licensed after uninstalling Extract Systems software and then reinstalling it without a reboot. Previous problems were resolved by stopping and then restarting the Nuance Licensing service.
