ID Shield Benchmark Tests

  1. Testing Goals

    1. To compute a processing rate for ID Shield using a server-class machine (UltraTester).
    2. To compare processing rates for a broad representation of customers.

  2. Software Setup

    • Start with a clean installation of the relevant ID Shield build.
    • Install SQL Server 2008 Express Edition.
    • Install Extract Systems LM.
    • Create an ID Shield Server license file with FLEX Index / ID Shield Utilities items and include Disable USB Key Serial Numbers.

  3. Database, Image and Rules Preparation

    • Use the FAM DB Admin utility to create an IDSHIELD_RATE database. Add Test as an action name.
    • Copy relevant image files to a working folder. Note that copies of image sets can be used if necessary. Enough images for at least 12 hours of processing should be available.
    • The following ID Shield Analysis projects are recommended for Benchmark testing
      • CA - Districts - ACS
      • CO - Secretary of State
      • IL - Lake - Cott
      • IN - Districts - CSI
      • LA - State Rules - Cott
      • MA - Executive Office of Transportation
      • NY - Secretary of State - eBiz
      • OH - Montgomery Clerk
      • PA - Mercer - Cott
      • St Johns
      • UT - Department of Commerce - UCC
      • VA - Judgments - Logan
    • Create a Queue.fps file to supply the images.
    • Create a local folder for the rule set to be exercised and select one of the following:
      • Use a relevant rules delivery _________________
      • Use a labeled rule set from source control _________________

  4. Test Execution

    • Delete any USS files that exist in the Images folder.
    • Create a multiple-thread Process.fps file with one Redact images (no verification) task.
    • Open a command window and run LogProcessStats.exe ProcessFiles.exe,SSOCR2.exe,XOCR32b.exe,CleanupImage.exe 5m \.Stats /el to watch memory and handle usage. It may be necessary to update the system path to include I:\Common\Engineering\Tools\Utils.
    • Open Queue.fps and start supplying.
    • Open Process.fps and start processing.
    • Continue processing for at least 12 hours. If a different test duration is desired, the results may not be accurate.

  5. Reviewing and Saving Test Results

    • Data will be stored in I:\Common\Testing\Product Testing\FLEX_Index_80\BenchmarkTests.
    • Make sure to document the following:
      • Rules delivery or source control label
      • Number of pages queued
      • Number of pages processed
      • Processing start time
      • Processing end time
      • Any unusual situation or error noticed during processing
      • LogProcessStats results with or without memory leaks
