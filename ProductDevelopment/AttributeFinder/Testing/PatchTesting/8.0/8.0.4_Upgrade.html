Patch/Upgrade Testing

Purpose and Details

  • SCRs: DotNetRC #763
  • Issue: The GetAbsolutePath() method is used to expand a relative path into a fully-qualified absolute path. The root path used in the code is Application.GetExecutablePath. Everywhere in the Extract code base that GetAbsolutePath is called, CommonComponents is the desired result. This causes a problem when our software is being used via API, especially in a web form when the Application object is a web browser
  • Solution: Modify the code in GetAbsolutePath() to return the path to the CommonComponents folder.
