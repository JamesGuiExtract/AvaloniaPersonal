FLEX Index / ID Shield 9.5 Patch Testing

Purpose and Details

  • Customer: UW Health
  • SCRs: DotNetRC #1078, #1079, #1081, #1082
  • Issue: Add support for printing a specified image to a desired printer. Also ensure that the entire image is printed without unwanted offset.

Patch Tests

  1. Test Setup
    • Prepare a test machine with FLEX Index 9.5.0.35 or ID Shield 9.5.0.35 or LabDE 9.5.0.35.

  2. Tests
    • Installation
      • Confirm that the patch installs properly by adding PrintDocument.exe to CommonComponents.

    • Functionality
      • Confirm that PrintDocument.exe will print a specified image to the default printer.
      • Confirm that PrintDocument.exe will print a specified image to a specified printer.
      • Confirm that PrintDocument.exe will print a specified image and that retaining annotations is the default behavior.
      • Confirm that PrintDocument.exe will print a specified image and discard annotations.
      • Confirm that PrintDocument.exe will log exceptions to the specified UEX file.
      • Confirm that a usage message is displayed for PrintDocument /?.
      • Confirm that the usage message specifies Do not print annotations for the /a- switch.
      • Print TestImage004.tif from Demo_IDShield.
      • Confirm that the entire visible image area is printed.
