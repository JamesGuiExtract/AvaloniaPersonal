FLEX Index / ID Shield 9.0 Hotfix 01 Testing

Purpose and Details

  • Customer: Health Network Labs
  • SCRs: DotNetRC #805, #810-#819, #821, #823-#829
  • Issue: Editing a SQL Compact database would be easier if the modifications were made to a temporary copy of the file. More functionality for customers would be available if the utility had an MDI interface to display tables and query results. Also, adding the capability to create and edit queries would facilitate updates to customers' AKA databases.

Hotfix Tests

  1. Test Setup
    • Prepare a test machine with FLEX Index 9.0.0.200 and/or ID Shield 9.0.0.200.
    • Prepare another test machine with LabDE 9.0.0.200.

  2. Tests
    • Installation
      • Confirm that the hotfix installs properly on FLEX Index and/or ID Shield build 200 by exercising Configure FAM Service.
        • An MDI interface is presented.
        • The Settings table can be modified.
        • The FPSFile table can be modified.
        • A query can be modified and saved.
      • Confirm that the hotfix installs properly on LabDE build 200 by exercising Configure FAM Service.
        • An MDI interface is presented.
        • The Settings table can be modified.
        • The FPSFile table can be modified.
        • A query can be modified and saved.

    • Functionality
      • Exercise the SQLCDBEditor portion of the SQL Compact Utilities checklist.
      • Exercise the DE_DB_SimultaneousUsage test from the .\IndustrySpecific\LabResults\AutomatedText folder.
