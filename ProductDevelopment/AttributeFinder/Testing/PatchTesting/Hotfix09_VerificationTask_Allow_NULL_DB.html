FLEX Index / ID Shield 7.0 Hotfix 09 Testing

Purpose and Details

  • Customer: Priton Group
  • SCRs: FlexIDSCore #3627
  • Issue: This customer would like to use our Redaction Verification user interface without a back-end database. The current verification task requires a database pointer so that the ID Shield data table can be updated appropriately. Modify the relevant code so that for a NULL database pointer, the database updates are skipped.

Hotfix Tests

  1. Test Setup
    • Install 7.0.0.39 on a test machine.
    • Install Visual Studio 2008 Express C# on the same test machine.
    • Install Hotfix #09 on the same test machine.

  2. Tests
    • SDK Testing
      • Create an SDK license file with saved passwords for the test machine.
      • Copy the IDShieldIntegrationDemo.zip solution to the test machine from I:\Users\Jeff_Shergalis\Public\Hotfix-9.
      • Modify Form1.cs to use the appropriate license file and passwords.
      • Build the test solution.

      • Exercise IDShieldIntegrationDemo.exe with Demo_IDShield rules and images
        • Click the Run rules button.
        • Confirm that (for TestImage001.tif) two message boxes containing position information for a redaction rectangle appear.

        • Click the Verify results button.
        • Confirm that the verification UI appears.
        • Click the Save redacted image button.
        • Confirm that an ELI25082 - Invalid metadata root node name. error is displayed.
        • After the exception is closed, confirm that the verification UI prompts the user about being unable to save and asks whether or not the image should be failed.
        • Confirm that the verification UI closes gracefully.
        • Confirm that no redacted image is created.
        • Confirm that no metadata XML file is created for the test image.

        • Delete the XML file created by the rules.
        • Click the Verify results button.
        • Confirm that the verification UI appears.
        • Click the Save redacted image button.
        • Confirm that a redacted image is created.
        • Confirm that a metadata XML file is created for the redacted image.
        • Delete the redacted image.
        • Confirm that RedactFromXML.exe properly creates a redacted image from the metadata XML file.

        • Click the Verify results button again.
        • Confirm that the verification UI appears.
        • Add one or more manual redactions.
        • Click the Save redacted image button.
        • Confirm that a redacted image is created that includes the manual redactions.
        • Confirm that a metadata XML file is created for the redacted image.
        • Delete the redacted image.
        • Confirm that RedactFromXML.exe properly creates a redacted image from the metadata XML file.

    • Normal Testing
      • Exercise the ID Shield Testing section of the Sanity Test checklist.
      • Confirm that RedactFromXML.exe properly creates redacted images from metadata XML files.

      • Exercise the ID Shield List of Documents Verified By Specific User report. Confirm that data was provided to the database so that a meaningful report could be created.
      • Exercise the ID Shield Verification Times And Rates (By User) report. Confirm that data was provided to the database so that a meaningful report could be created.
