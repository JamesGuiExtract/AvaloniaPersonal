FLEX Index / ID Shield / LabDE 10.0.1 Patch Testing

Purpose and Details

  • Customer: UW Transplant
  • SCRs: ISSUE-12787
  • Issues:
    • While exercising the Duplicate Documents feature in LabDE, files were prevented from being checked out.

Patch Tests

  • Problem Replication
    • Begin by queuing and processing the Extended images from Demo_LabDE.
    • Use SQL Management Studio to add Domain_User access privileges to the Demo_LabDE database.
    • Use SQL Management Studio to add and then remove a row from the ActiveFAM table.
    • Verify a document and click the Duplicate Documents button to display the FAM File Inspector.
      • Confirm that the current document is displayed as In Use and that no actions are available on the file.
      • Confirm that an image being verified by another user on another machine is displayed as In Use but with the actual user not properly identified.
    • Close verification.

  • Patch Installation
    • Unzip LabDE Patch 10.0.1.zip.
    • Run Install.bat to update the Extract Systems files.
    • [TESTING ONLY] Replace Extract.DataEntry.DEP.Demo_LabDE.dll in the C:\Demo_LabDE\Solution\Bin folder.

  • Testing
    • Verify a document via Duplicate Documents.
    • Confirm that the current document is displayed as In Use.
    • Confirm that other actions are available.
    • Stop verification.
    • Verify an image on another workstation.
    • Restart verification on the first workstation.
    • Confirm that when the two documents being verified pass the tests for being potential duplicates that the remote document is properly identified as in use by the remote user.
