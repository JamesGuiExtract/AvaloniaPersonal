FLEX Index / ID Shield Post-Release Activities

  1. Create Master CDs

    • FLEX Index
      • Create an autorun.inf file for the root of the CD with the following lines:
        • [autorun]
        • open=FlexIndexInstall\Launch.exe
        • icon=FlexIndexInstall\FlexIndex.ico
      • Add the following folders and their contents to the root of the CD:
        • DotNet 3.5 Framework
        • Extract Systems LM
        • FlexIndex
        • FlexIndexInstall
        • InliteKeyInstall
        • SQLServerExpress2008
      • Create an appropriate ISO image to facilitate creation of copied CDs.

    • ID Shield
      • Create an autorun.inf file for the root of the CD with the following lines.
        • [autorun]
        • open=IDShieldInstall\Launch.exe
        • icon=IDShieldInstall\IDShield.ico
      • Add the following folders and their contents to the root of the CD:
        • DotNet 3.5 Framework
        • Extract Systems LM
        • IDShield
        • IDShieldInstall
        • InliteKeyInstall
        • SQLServerExpress2008
      • Create an appropriate ISO image to facilitate creation of copied CDs.

    • Exercise
      • Test the FLEX Index CD by installing each application.
      • Test the ID Shield CD by installing each application.

  2. Source Code Backup - On the Build Machine

    • Rename the D:\temp folder on the build machine to temp a.b.c.d.
    • As Vault and/or domain administrator.
      • Undo current checkouts.
      • Create a branch.
      • Create a backup of the new branch.

  3. Documentation Backup - From I:\Common\Tech Comm

    • Zip the I:\Common\Tech Comm\RoboHELP\File Action Manager folder and store it I:\Common\Tech Comm\RoboHELP\Archives.

  4. Release Notes and Licensing

    • Create an email to company employees describing in fair detail each of the new features and enhancements associated with the new release.
    • Update the Notable FlexIndex Versions.rtf file located at I:\Common\Engineering\ProductReleases\FlexIndex.

    • Update the I:\Common\Engineering\Tools\SecureClients\COMLicense folder used to create license files for the new release.
    • Confirm that the new licensing folder cannot be used to create the following license packages: Disable USB Key Serial Numbers, Rule Development Toolkit, FLEX Index Rule Writing, ID Shield Rule Writing.

  5. Post-Release Development / Testing Activities

    • Finish exercising any remaining testing checklists.
    • Finish creating and exercising any missing automated tests.
    • Finish creating and exercising any missing Memory Leak tests.
    • Update testing checklists per comments and notes from testers.

    • Review and test the Untested SCRs in Data Entry.
    • Review and test the Untested SCRs in DotNetRCAndUtils.
    • Review and test the Untested SCRs in FlexIDSCore.
    • Review and test the Untested SCRs in LegacyRCAndUtils.

  6. Post-Release Rule Writer / Tester Activities

    • Run Customer_Max_Server_Rates_80 tests for the specified customers and update the appropriate spreadsheet.
    • Run IDShield_Customer_Rates_80 tests for the specified customers and update the appropriate spreadsheet.
    • Run FLEXIndex_Customer_Rates_80 tests for the specified customers and update the appropriate spreadsheet.
    • Run Maximum_Server_Rate_Tests_80 tests and update the appropriate spreadsheet.
    • Run Thread_Scalability_80 tests and update the appropriate spreadsheet.

  7. Optional Post-Release Rule Writer Activities

    • Run FLEX Index Capture Rate tests for each current customer and update the appropriate Performance Results spreadsheet.
    • Run ID Shield Capture Rate tests for each current customer and update the appropriate Performance Results spreadsheet.
