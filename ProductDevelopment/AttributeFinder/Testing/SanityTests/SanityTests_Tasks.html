FAM Sanity Test - File Processing Tasks

*ID Shield, LabDE and FLEX Index should be installed and licensed for this test.

Goals

  • Exercise each defined task.
  • Confirm that various configurations work as expected.
  • Confirm that no unwanted exceptions are logged.
  • Confirm that each task works properly on various operating systems.

File Processing Tasks

  1. Core: Add watermark
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that a watermark image can be positioned in the (default) top corner of the first page.
    • Confirm that a watermark image can be positioned in a specified position on a specified page.

  2. Core: Apply Bates number
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that a Bates number with default settings can be applied to documents.
    • Confirm that a Bates number with non-default settings can be applied to documents.

  3. Core: Archive or restore associated file
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that files can be archived.
    • Confirm that files can be restored.

  4. Core: Clean up image - requires USB dongle
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that images can be cleaned.
    • Confirm that, by default, cleaned images are used for OCR.

  5. Core: Conditionally execute task(s)
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that appropriate tasks are executed when the condition is met.
    • Confirm that appropriate tasks are executed when the condition is not met.

  6. Core: Convert to searchable PDF
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that a TIF image can be converted to a searchable PDF image.
    • Confirm that a TIF image can be converted to a PDF/A compatible image.
    • Confirm that PDF security can be enabled.

  7. Core: Convert VOA to XML
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that a VOA file can be converted to a Version 1 XML file.
    • Confirm that a VOA file can be converted to a Version 2 XML file.

  8. Core: Copy, move or delete file
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that a file can be copied to a non-existent destination folder.
    • Confirm that a file can be moved and that SourceDocName can be modified.
    • Confirm that a read-only file can be deleted.
    • Confirm that a file can be securely deleted.

  9. Core: Create file
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that a file can be created with specified contents.

  10. Core: Delete empty folder
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that an empty folder can be deleted along with empty parent folders.

  11. Core: Encrypt/decrypt file
    • Confirm that encrypted files can be created.
    • Exercise decryption.
    • Confirm that the decrypted files are the same size as the original files.

  12. Core: Enhance OCR
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that processing works for each of the following settings.
      • General enhancement - Medium.
      • Focused enhancement - Shaded.
      • Focused enhancement - Diffuse.
      • Focused enhancement - Smudged.

  13. Core: Execute rules
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that rules can be executed from a file specified via expandable tags.

  14. Core: Extract image area
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that an image area can be extracted for a specified attribute.
    • Confirm that multiple image areas can be extracted for all attributes from a particular VOA file.

  15. Core: Launch application
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that an application without parameters can be launched in non-blocking mode.
    • Confirm that an application with parameters can be launched in blocking mode.

  16. Core: Manage tags
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that the specified tag can be applied.
    • Confirm that the specified tag can be removed.

  17. Core: Modify PDF file
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that annotations can be removed from the specified PDF image.
    • Confirm that hyperlinks can be added to the specified PDF image.
    • Confirm that highlights can be added to the specified PDF image.

  18. Core: Modify source document name in database
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that SourceDocName can be modified.

  19. Core: OCR document
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that the specified image can be OCR'd.

  20. Core: Rasterize PDF
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that the specified PDF can be rasterized with SourceDocName modified in the database.
    • Confirm that the specified PDF can be rasterized with the original PDF being deleted after rasterization.

  21. Core: Send email
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that an email can be sent with one or more attachments.
    • Confirm that an email can be sent using one or more expandable tags.
    • Confirm that an email can be sent including one or more attributes from a VOA file.

  22. Core: Set file priority
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that file priority can be changed for the specified file.

  23. Core: Set file action status in database
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that file action status can be set for SourceDocName.

  24. Core: Sleep
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that the FAM will sleep for a specified period.
    • Confirm that the FAM will sleep for a random period.

  25. Core: Translate, rename or delete via FTP/SFTP
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that each of the following operations succeed.
      • Upload file
      • Download file
      • Rename remote file
      • Delete remote file

  26. Core: Validate XML
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that XML can be validated against a hard-coded XSD file.
    • Confirm that XML syntax can be validated without XSD.

  27. Core: View image
    • Confirm that no settings are available.
    • Confirm that images can be viewed.
    • Confirm that the thumbnails window can be shown or hidden.
    • Confirm that the magnifier window can be shown or hidden.

  28. Data Entry: Verify extracted data
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that verification results can be saved.

  29. Redaction: Create metadata XML
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that metadata can be saved in an XML file.

  30. Redaction: Create redacted image
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that redacted output images can be created with the following settings.
      • Basic black redactions
      • Redactions as annotations
      • White redactions with black border, with REDACTED as redaction text

  31. Redaction: Create redacted text
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that sensitive text can be replaced by specified characters.
    • Confirm that sensitive text can be surrounded with a named XML element.

  32. Redaction: Extend redactions to surround context
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that surrounding context can be redacted.
      • This task operates on an existing VOA file.
      • Results can be viewed by subsequently running the Redaction: Create redacted image task.
    • Confirm that redaction height can be adjusted to further obscure context.

  33. Redaction: Filter ID Shield data file
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that one or more default data types can be filtered and written to the specified output file.
    • Confirm that one or more custom data types can be filtered and written to the specified output file.

  34. Redaction: Merge ID Shield data files
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that two VOA files can be merged.
    • Confirm that equal redactions are not duplicated in the output VOA file.

  35. Redaction: Verify sensitive data
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that default settings are effective:
      • Users must visit all pages
      • Users must visit all data items
      • Users must specify type
      • Users can seamlessly navigate between documents
      • Slideshow is enabled
    • Confirm that other settings are effective:
      • Users must specify exemption codes
      • Feedback collection is enabled
      • Input event tracking is enabled
      • Verification window opens in full-screen mode
      • User is prompted to save changes before navigating away
    • Confirm that file action status can be set after file commitment.
