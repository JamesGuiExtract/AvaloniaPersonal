FAM Sanity Test - New Employee Training

Goals

  • Help employee learn about installing and licensing Extract software.
  • Teach employee a little about DB Admin and FAM.
  • Teach employee a little about each standard demo.
  • Prepare employee for further training and/or testing using mainstream checklists.

Installation

  • Start from one of the following installation folders. Note that version numbers change with each release.
    • I:\Common\Engineering\ProductReleases\FlexIndex\External\Current
    • M:\ProductReleases\FlexIndex\Internal\BleedingEdge
  • Select an appropriate version number and navigate to the FlexIndexInstall folder.
  • Double-click on Launch.exe.
  • Find and install Microsoft .NET 3.5 and 4.0.
    • Note that it may or may not be installable depending on the operating system.
  • Find and install SQL Server 2014 and SQL Management Studio.
    • Note that it may or may not be installable depending on the operating system.
    • SQL Server 2008 R2 and the associated Management Studio can be found in any pre-10.0 software release. SQL 2008 R2 is compatible with later releases of Extract Systems software.
  • Install FLEX Index.
  • Back up to the ProductReleases folder and change to ID Shield.
  • Install ID Shield.
  • Back up to the ProductReleases folder and change to LabDE.
  • Install LabDE.
  • Install Extract Systems License Manager.

  • If you expect to use Rule Development Toolkit, the corresponding version of the RDT must be installed. Start from one of the following installation folders.
    • M:\ProductReleases_InternalUseOnly\FlexIndex\External\Current
    • M:\ProductReleases_InternalUseOnly\FlexIndex\Internal\BleedingEdge
  • Confirm that RDT installs correctly.

Licensing

  • Start from the I:\Common\Engineering\Tools\SecureClients licensing folder.
    • COMLicense - used for customer licenses
    • COMLicense_RuleWriter - used for internal rule-writer licenses
    • COMLicense_Developer - used for internal development and testing licenses
  • Select the approriate software version. Note that FlexIndex9_x works for any 9.0, 9.2, ..., 9.9 version as well as 10.0, 10.1 and 10.2.
  • If you are testing on a new or clean machine, run ocx-install.bat as local administrator. A confirmation message should be displayed.
  • Run COMLicenseGenerator.exe
    • One or more Package items must be selected
    • Some text must be entered in the Licensee Name text box.
    • Some text must be entered in the Organization Name text box.
    • A valid User License String must be entered.
      • The Generate User License String button provides the string for the machine that is currently running the utility. This is useful for licensing your own workstation or an internal test machine.
      • A string can be generated via UserLicense.exe. This is how a customer provides the string for a server or workstation that is to be licensed.
    • Set the Licensing Mode.
      • A Fully Licensed license is almost always used for internal use.
      • An Evaluation License license is sometimes used for customers or testing.
      • The default period for an evaluation license is 30 days.
    • Run UserLicense.exe and click the Open License Folder button to create and open the folder to which the license files (for the computer to be licensed) should be saved.
    • The following separate license files are typically created for use in testing.
      • One or more of FLEX Index Server, ID Shield Server, LabDE Server.
      • All of the FLEX Index / ID Shield Utilities items EXCEPT Disable USB Key Serial Numbers .
      • Rule Development Toolkit - only used by Rule Writers and Testers.
    • Contact Wayne or another member of the Development team to create a Disable USB Key license file. This is for internal use only except in rare circumstances.

Demos

  • Start from one of the following installation folders. Note that version numbers change with each release.
    • I:\Common\Engineering\ProductReleases\FlexIndex\External\Current
    • M:\ProductReleases\FlexIndex\Internal\BleedingEdge
  • Select an appropriate version number and navigate to each SetupFiles folder.
  • Copy Demo_FLEXIndex, Demo_IDShield, and Demo_LabDE folders to the C:\ drive on the test machine.
  • Install each demo by exercising the associated CleanAll batch files.
  • Use SQL Server Management Server to exercise I:\Common\Testing \Product Testing\IDShield_Sanity\CreateDomainLogin_AddForDemoDBs.sql to allow Domain Users to access the demo databases.

Database Administration (DB Admin)

  • Exercise DB Admin for one of the demo databases noted above.
  • Review the various menu items and the Summary tab.
  • Exercise the Manage Actions dialog and add a new TEST action that you can use for other experimentation. Note that clearing the database via CleanAll.bat will clean up any inadvertent messes.
  • The Tools - Export file list menu item will be used repeatedly in later FAM testing.

File Action Manager (FAM)

  • Use Tools - File Action Manager from DB Admin to open a new FAM instance to use for testing.
  • On the Action tab, select the new action that was created in the Manage Actions dialog.
  • Check the Queue files checkbox to allow adding files to be processed.
  • Click the Add... button on the Queue Setup tab.
  • Select Files from folder.
    • Choose some settings and click OK.
    • Click the Start processing button and watch the Queue Log tab to see what happens.
    • Export a file list and use Notepad to review the files that were queued.
    • Experiment with other settings such as Recursively supply
  • Click the lower Commands - Select Condition button on the Queue Setup tab.
  • Select the File name pattern condition.
  • Specify a simple expression that will pass for some but not all of the files being tested.
  • Export a file list and use Notepad to review the files that were queued and also to see that certain files were not queued.
  • Clear the database via Database - Clear from DB Admin.
  • Create a new FAM and select Files from dynamic list on the Queue Setup tab.
  • Browse to and select an exported file list.
  • Start processing and confirm that the expected files have been queued.

  • EXERCISES:
    • Confirm that settings can be selected such that files can be dropped into a specific folder and then queued.
    • Confirm that a condition can be defined to exclude multi-page images from being queued.
    • Confirm that Mathematical condition can be used to randomly queue files from a folder.
    • Confirm that Mathematical condition can be used to randomly queue files from a list.
