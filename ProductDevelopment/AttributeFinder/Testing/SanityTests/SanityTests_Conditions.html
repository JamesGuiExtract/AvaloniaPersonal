FAM Sanity Test - FAM Conditions

* ID Shield should be installed and licensed for this test.

Goals

  • Exercise each defined condition object.
  • Confirm that various configurations work as expected.
  • Confirm that no unwanted exceptions are logged.
  • Confirm that each condition works properly on various operating systems.

FAM Conditions

  1. Compare ID Shield data files condition
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Exercise redaction verification for a few files.
      • Configure verification to save the VOA files to <SourceDocName>.session1.voa.
      • Resize each redaction to be close to perfect.
    • Exercise redaction verification again for the same few files.
      • Configure verification to save the VOA files to <SourceDocName>.session2.voa.
      • Adjust redactions in one file to be too small.
      • Retain the ideal redactions in another file.
      • Adjust redactions in another file to be too large.
      • Remove redactions in another file.
      • Add redactions in another file.
    • Exercise the condition object.
      • Confirm that the condition matches for the image where each VOA file contains ideal redactions.
      • Confirm that the condition can match for the specified overlap percentage.
      • Confirm that the condition does not match for files where redactions were added or removed.

  2. Database contents condition.
    • Create a FAM_Condition database.
    • Add a DB_Condition action to the database.
    • Run Exercise_DB_Condition.bat from .\Engineering\ReusableComponents\COMComponents\UCLIDFileProcessing \ESSkipConditions\InteractiveTest\FPS and review the results in the parallel Output folder.
      • Each TXT file should be named with two or three characters, followed by an underscore, followed by PASS.
      • Review TXT contents of any FAIL files.
      • Confirm presence of D:\DBCondition.uex. This file should be created or updated each time 07c_SDF.fps is executed in the batch file.

  3. Document type condition
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Exercise the Demo_FlexIndex images and rules.
    • Confirm that some VOA files contain DocumentType attributes.
    • Confirm that the condition can match for a document type.
    • Confirm that the condition does not match if the VOA file does not contain a DocumentType attribute.

  4. File existence condition
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that the condition can match if a file does exist.
      • File specified via absolute path.
      • File specified via expandable tags.
    • Confirm that the condition can match if a file does not exist.
      • File specified via absolute path.
      • File specified via expandable tags.

  5. File name pattern condition
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that the condition can match if a file does contain an expression.
      • Text specified via regular expression.
    • Confirm that the condition can match if a file does not contain an expression.
      • Text specified via case-sensitive regular expression.
    • Confirm that the condition can match if a file does exactly match an expression.
      • Text specified via regular expression from file.

  6. File timestamp condition
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that the condition can match for a modified timestamp that uses a date range.
    • Confirm that the condition can match for a modified timestamp that is exactly equal to a specified time.
    • Confirm that the condition can match for a modified timestamp that is before a specified time.
    • Confirm that the condition can match for a modified timestamp that is after a specified number of days ago.
    • Confirm that the condition can match for a created timestamp that is from or before this week.
    • Confirm that the condition can match for a created timestamp that is equal to or after the created timestamp of a specified file.

  7. ID Shield data file condition
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that the condition can match for finding any of HCData and MCData.
    • Confirm that the condition can match for finding only any of Clues.
    • Confirm that the condition can match for finding Mortgage as the document type.
    • Confirm that the condition can match for finding any of Clues OR Warranty Deed as the document type.
    • Confirm that a non-default setting can be used for the ID Shield Data File Location.

  8. Mathematical condition
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that the condition can be met randomly.
    • Confirm that the condition can be met once every specified number of times.
    • Confirm that the condition can be met using FileID modulus.

  9. Page count condition
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Exercise the Demo_IDShield images.
    • Confirm that the condition can match for greater than or equal to a page count.
    • Confirm that the condition can match for less than a page count.
    • Confirm that the condition can match when not using the page count from the database.

  10. Tag condition
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that the condition can be met using all of multiple specified tags.
    • Confirm that the condition can be met using any of the specified tags.

  11. Task condition
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm that the condition is met when the specified task does not throw an exception.
    • Confirm that the condition is not met when the specified task does throw an exception.

  12. VOA file contents condition
    • Confirm that the configuration dialog can be displayed and that settings are persistent.
    • Confirm proper behavior for the options in the contains combo box.
    • Confirm proper behavior for the options in the name/value/type combo box.
    • Confirm proper behavior for each of the attribute radio button options.
      • equals
      • is in the range
      • contains a match for
      • matches one of the entries in the following list
    • Confirm proper behavior of the case sensitive check box.
    • Confirm proper behavior of the regular expression check box.

LOGICAL Conditions

  • Exercise the following steps - taken from the FAM Conditions checklist to test functionality of LOGICAL Conditions.
  • FPS and Image files referenced below can be found in Vault under appropriate subfolders of Engineering\ReusableComponents\COMComponents\UCLIDFileProcessing\ESSkipConditions\InteractiveTest.
  • Create a FAM_Condition database.

  • AND multiple conditions
    • Clear the database.
    • Add a skip_basedonand action.
    • Open Skip_AND.fps.
    • Press the Start processing button.
    • Confirm that TestImage.tif, TestImage015.tif, TestImage020.tif, and TestImageB.tif are queued.
    • Confirm that TestImage05.tif and TestImage09.tif are not queued.

  • Exactly ONE in multiple conditions
    • Clear the database.
    • Add action skip_basedonexactlyone to the database.
    • Open Skip_ExactlyOne.fps.
    • Press the Start processing button.
    • Confirm that TestImage.tif, TestImage005.tif, TestImage009.tif, and TestImageB.tif are queued.
    • Confirm that TestImage015.tif and TestImage020.tif are not queued.

  • NONE in multiple conditions
    • Clear the database.
    • Add action skip_basedonnoneinmultiple to the database.
    • Open Skip_NoneInMultiple.fps.
    • Press the Start processing button.
    • Confirm that TestImage005.tif, TestImage009.tif, TestImage015.tif. and TestImage020.tif are queued.
    • Confirm that TestImage.tif and TestImageB.tif are not queued.

  • OR multiple conditions
    • Clear the database.
    • Add action skip_basedonor to the database.
    • Open Skip_OR.fps.
    • Press the Start processing button.
    • Confirm that TestImage.tif and TestImageB.tif are queued.
    • Confirm that TestImage005.tif, TestImage009.tif, TestImage015.tif, and TestImage020.tif are not queued.
