Simple Rule Writing Testing

Goals

  • Exercise installation and licensing for Simple Rule Writing.
  • Confirm that every simple rule object is available.
  • Confirm that every advanced rule object is unavailable.
  • Confirm that the current customer delivery rules can be read, if that is intended for a specific customer.
  • Confirm that complex rules cannot be read.
  • Confirm that complex rules can still be opened via Rule Tester and exercised via File Action Manager.

    • Confirm that the rule set cannot be a swiping rule set.

  • Using Rule Set Editor, open a complex rule set like one of the Demo rule sets.
    • Confirm that an Unable to load RuleSet exception is displayed.
    • Confirm that the rule set can be tested using the built-in Rule Tester.
  • Special handling for RSD-based objects
    • Confirm that the following rule objects will only operate on encrypted input files.
      • Find from RSD file value finder

    • Confirm that a path tag expression cannot be created that literally ends with .etf but evaluates to an expression ending with .rsd.
    • Confirm that the filename property of the RSDFinder COM object cannot be programmatically changed unless an RDT or simple rules license is present.
