FKB Updates Testing Checklist

  1. General Instructions

    • Run one instance of LogProcessStats to collect information on memory and handle usage. One instance will collect statistics for all TestHarness instances. Run the following command string from the folder containing the TCL file.

      LogProcessStats TestHarness 30s .\Statistics /el

    • Close the LogProcessStats command window when the last TestHarness instance has finished.

    • For each Test Harness instance:
      • Review the results and add one or more SCRs to PVCS - P16 if needed.
      • Review the Exception log and add SCRs if needed. The following error items and some others are benign.
        • ELI09263 - No Rules Files Found.
        • ELI15464 - Exception from ISubMatches GetItem!
        • ELI15935 - Application trace information.
      • Update the statistics in the appropriate spreadsheet.

    • Copy the Statistics folder to a YYYY-MM-DD named subfolder under the appropriate FKB version number under I:\Testing\Product Testing\FKB_Updates.
    • Add a TXT file inside the YYYY-MM-DD subfolder describing which rules and images were tested for each of the CSV files.

  2. Indexing Projects

    • Print a copy of FLEX Index - Analysis Results to retrieve an up-to-date list of active Indexing projects and desired fields.
    • Select at least three projects for which to check the capture rate, execution time, memory usage and handle usage.

  3. Redaction Projects

    • Print a copy of ID Shield - Analysis Results to retrieve an up-to-date list of active Redaction projects.
    • Select at least five projects for which to check the capture rate, execution time, memory usage and handle usage.
