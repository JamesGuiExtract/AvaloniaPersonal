  1. Recently fixed SCRs

    • [FlexIDSCore #2686] Select Document Type dialog
      • Create a ruleset using a Document classifier.

    • [FlexIDSCore #4707] Automatic OCR for ID Shield Verification
      • Confirm that a checkbox exists under Tools - Options - General to indicate if automatic background OCR should occur.
      • Confirm that the default state is checked.
      • Confirm existence of an associated combo box to select OCR quality.
        • Accurate
        • Balanced (the default)
        • Fast
      • Confirm that for files requiring automatic OCR, no USS files are saved.
      • Confirm that when performing automatic OCR in FAM mode, the OCR results are saved as part of the history information so that navigation to a previous document does not require additional OCR.
      • Confirm that for files requiring automatic OCR, the Word redaction tool is initially disabled.
      • Confirm that for files requiring automatic OCR, the Word redaction tool is enabled after the OCR is finished.
      • Test automatic OCR for a multi-page image.
        • Confirm that background OCR starts for a new page as soon as the user navigates to that page even if OCR has not finished for the previous page.
      • Confirm that background OCR will pause during the auto-fit mode for the Word redaction tool.
      • Open the Find dialog for a multi-page document that requires automatic OCR.
        • Confirm that a dialog with a progress bar is displayed notifying the user that they will need to wait until OCR is complete for the entire document.
        • Confirm that if the user waits, the progress dialog will automatically close before the Find dialog appears.
        • Confirm that if the user closes the progress dialog, the Find dialog will not appear.
      • Confirm that a new status bar label is provided to display:
        • Whether or not OCR is available.
        • OCR progress for the current page.
        • That word zones are currently being loaded (if the page already has OCR data).

    • [FlexIDSCore #4715] FAM VOA File Contents Condition
      • Confirm that an edit box is available to specify the VOA file to be evaluated.
      • Confirm that <SourceDocName> is the default.
      • Confirm that an associated tag selector is available.
      • Confirm that the following requirements are available for considering the condition to be satisfied:
        • Contains exactly
        • Contains at least
        • Contains at most
        • Does not contain exactly
      • Confirm that the requirement includes an edit box for specifying how many attributes that meet the specification.
      • Confirm that an attribute query field is provided.
      • Confirm that the attribute can be evaluated based on:
        • Name
        • Value
        • Type
      • Confirm that evaluation of each attribute can be done in any one of the following ways:
        • Equals
        • Is not equal to
        • Is less than
        • Is less than or equal to
        • Is greater than
        • Is greater than or equal to

        • Whether or not it falls within a specified range
        • Whether or not a search term is found in the specified attribute field
          • Fully matches
          • Contains a match for
        • Whether or not the attribute fully matches one of the values in a specified list

        • Where appropriate, the following options should be available
          • Case sensitive searches
          • Search as a regular expression
          • Numeric comparison
          • Condition is met or not met

    • [DotNetRC #632, #675, #677, #678] FTP File Supplier
      • Confirm that the file supplier will retry an operation if a failure occurs.
      • Confirm that the Connection Settings tab includes a checkbox to display Advanced settings.
      • Confirm that after the file supplier downloads a file from FTP, a .info file is created in the same folder that received the downloaded file.
        • Confirm that the .info file is in XML format.
        • Confirm that the .info file contains a RemoteSourceDocName element.
        • Confirm that the .info file contains a RemoteFileSize element.
        • Confirm that the .info file contains a RemoteLastModified element.

        • Confirm that if the .info file exists in the local folder it is used to determine if the file should be re-downloaded.
          • Downloaded if file is not in the database.
          • Downloaded if Force processing is set.
          • Downloaded if file size has changed.
          • Not downloaded if last modified information is unchanged.
          • Confirm that the after download action is also performed if the file is re-downloaded.

    • [DotNetRC #676, #679] FTP File Transfer Task
      • Confirm that the task will retry an operation if a failure occurs.
      • Confirm that the task allows the use of the <RemoteSourceDocName> tag whose value is obtained from the .info file.
      • Confirm that if the tag is used and the .info file does not exist then an exception is thrown.
