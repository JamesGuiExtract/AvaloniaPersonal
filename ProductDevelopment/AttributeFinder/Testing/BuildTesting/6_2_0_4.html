FLEX Index / ID Shield 6.2.0.4 Testing

  1. Descriptions of Changes

    • [FlexIDSCore #3073] Running rule set causes FAM to crash
      • Prepare a test database
      • Modify the associated Crash.fps
      • Cue the associated MTG image
      • Exercise the associated RSD file
      • Confirm that the FAM does not crash.

    • [FlexIDSCore #3065] Remove RDT Help file
      • Confirm that an RDT Help file is not installed.

    • [FlexIDSCore #3062] Rulesets with Internal-use only flag
      • Using an RDT license, create and test an encrypted RSD file that has the Internal-use only flag checked.
      • Using a non-RDT license, test the encrypted Internal-use RSD file.
        • Confirm that the Rule Tester will not exercise the rule set.
        • Confirm that Run Rules will not exercise the rule set.

    • [FlexIDSCore #3061] Rulesets without a USB counter
      • Using an RDT license, create and test an encrypted RSD file that has the Internal-use only flag unchecked and does not require a USB counter.
      • Using a non-RDT license, test the encrypted counter-less RSD file.
        • Confirm that the Rule Tester will not exercise the rule set.
        • Confirm that Run Rules will not exercise the rule set.

    • [FlexIDSCore #3059] Simple Rule Writing and USB counters
      • Using an ID Shield Rule Writing license, create and test an RSD file.
      • Confirm that the rule set will not run without checking the Redaction counter.

    • [FlexIDSCore #3046] Retaining existing annotations
      • Confirm that auto-redaction successfully retains existing annotations.

    • [LegacyRC #5029] Max Files Per SSOCR2 instance registry item
      • Check the HKLM/Software/ReusableComponents/COMComponents/UCLIDRasterAndOCRMgmt/OCREngines/SSOCR/MaxFilesPerInstance registry key.
        • Confirm that the key is not present on a clean system.
        • Confirm that the key is present after ID Shield installation.
        • Confirm that the key is not present after ID Shield is uninstalled.

    • [LegacyRC #5028] Location of UEX file
      • Confirm that the file is not present at Program Files\Extract Systems\LogFiles\Misc on a clean system.
      • Confirm that the file is not present at Documents and Settings\All Users\Application Data\Extract Systems\LogFiles\Misc on a clean system.
      • Install ID Shield and create a database.
      • Confirm that the file is present at Documents and Settings\All Users\Application Data\Extract Systems\LogFiles\Misc.
      • Confirm that the View Exception Log item in the Start menu displays the exception log.

    • [LegacyRC #5020] Creation of license files using either JAKE or FNP2
      • Log in to a test machine that is connected to the extract.local domain.
      • Copy the I:\Common\Engineering\Tools\SecureClients\COMLicense_InternalTesting\Current folder to the test machine.
      • Disconnect the drive mapping to \\fnp2\internal.
      • Disconnect the drive mapping to \\jake\internal.
      • Confirm that COMLicenseGenerator.exe does not run.
      • Re-connect the drive mapping to \\fnp2\internal.
      • Confirm that COMLicenseGenerator.exe does run.
      • Disconnect the drive mapping to \\fnp2\internal.
      • Re-connect the drive mapping to \\jake\internal.
      • Confirm that COMLicenseGenerator.exe does run.
