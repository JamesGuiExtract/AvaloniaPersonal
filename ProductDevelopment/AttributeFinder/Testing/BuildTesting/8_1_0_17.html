FLEX Index / ID Shield 8.1.0.17 Testing

  1. Recently Fixed SCRs

    • [LegacyRC #5608, #5637, #5801] DB Admin
      • Confirm that each of the following DB Admin dialogs can be dismissed by pressing the Esc key:
        • Manage counters
        • Manage tags
        • Manage actions
        • Manage users

      • Confirm that when the Add new users button is clicked in the Manage users dialog, a dialog box is displayed that is prepopulated with the name of the user who is currently logged in.

      • Open the Manage tags dialog.
      • Add a tag.
      • Press enter.
      • Confirm that the tag is created.

    • [LegacyRC #5610] FAM processing threads must grab file before stopping
      • Create a database and enqueue a small (~20) number of files.
      • Configure an .fps file with a verification task. Set it up to use a large number of threads to processing (at least twice as many threads as there are files to process).
      • Start processing, then immediately stop.
      • Confirm that processing stops immediately and that the initially idle threads do not have to grab files first.

    • [LegacyRC #5830, #5845, #5846] CreateMultiPageImage.exe
      • Get some many-paged files. Examples can be found at "I:\Common\Engineering\ ProductTesting\ManyPage Images". Split them into single-page images with SplitMultiPageImage.exe
      • Run CreateMultiPageImage.exe on the newly-split images while simultaneously using the computer for other tasks (e.g., processing documents, copying files, etc).
      • Confirm that the output images have the correct number of pages.

      • Confirm that CreateMultiPageImage works correctly to combine documents with four- and five-digit extensions (e.g., a.0001, a.0002).

      • Confirm that the utility correctly reports the number of files that process successfully and the number that fail.

    • [LegacyRC #5840] FAM switchs /fc and /c
      • Create an .fps file configured to stop processing once the queue is empty.
      • Run the command "Processfiles.exe <fpsfile> /s /fc".
      • Confirm that when the files are finished processing, the FAM exits after stopping.

    • [FlexIDSCore #1073] Loading lists dynamically from files
      • Confirm that the following rule objects support dynamic reading from lists:
        • ValueFromList value finder
        • ReplaceStrings value modifier
        • TranslateToClosestValueInList value modifier
        • TranslateValueOrTypes value modifier
        • RemoveEntriesFromList output handler
      • Confirm that both .dat and .dat.etf files can be loaded.
      • Confirm that list files can be autoencrypted as appropriate.

    • [FlexIDSCore #2363] DocTypeCondition should allow multiply-classified documents
      • Exercise the document type condition on multiply-classified documents.
      • Confirm that works as expected with documents that have been classified as multiple document types.

    • [FlexIDSCore #3080] Copying an attributes value to another attribute
      • Execute LegalDescription.rsd from I:\Common\Testing\PVCS_Testing\FlexIDSCore\3080 on the image in that folder.
      • Confirm that any existing sibling subattributes Section and XSection have identical spatial information.

    • [FlexIDSCore #3296] ID Shield verification
      • Open verification.
      • Draw a large redaction that covers some text and the whitespace surrounding it.
      • Select the new redaction, then right-click and select Block Fit from the context menu.
      • Confirm that the redaction shrinks to cover all of the black pixels in the area.

    • [FlexIDSCore #4112] Autoencrypt
      • Exercise the ruleset at I:\Common\Testing\PVCS_Testing\FlexIDSCore\4112 on sampleText.txt.
      • Confirm that each of the related .txt files is correctly autoencrypted.

    • [FlexIDSCore #4234] ID Shield metadata
      • Run the following tasks on a sample image:
        • Execute rules (with .voa output)
        • Create redacted image
        • Surround context
        • Create redacted image
        • Create metadata xml
      • Open the .xml file
      • Confirm that the .xml file nodes appear in the order that the tasks were performed (i.e., it's possible to tell which tasks were performed when).

    • [FlexIDSCore #4263] Merge attributes output handler
      • Configure a Merge attributes output handler, specifying a blank type for the result.
      • Click okay.
      • Confirm that no exception is seen.
