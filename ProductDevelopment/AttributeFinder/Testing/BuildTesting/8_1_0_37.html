    • [DotNetRC #457, #465, #473, #474, #475, #476, #477, #480, #481, #493] ExtractImageViewer - command-line options
      • Confirm that ExtractImageViewer.exe supports the following command-line options:
        • /? - displays usage message
        • /r - register the .tif extension
        • /u - unregister the .tif extension
        • /o <ocrfile> - OCRed text is written to the specified file
        • /c - OCRed text is written to the clipboard
        • /xml - must be used with either /c or /o; writes out swipe in XML format including full text as well as zone coordinates.
        • /s - display the search window
        • /l - reuse the current image viewer window if one exists
        • /closeall - close all open instances of the image viewer
        • /e <scriptfile> - execute the script
        • /script? - display help for scripting
        • /ctrlid? - lists toolbar control ID numbers
        • /reset - resets the positions of all toolstrips and the thumbnail viewer.

      • Open an image in the image viewer.
      • Minimize the image viewer.
      • On the command line, try to open another image using the /l switch.
      • Confirm that the image opens and no exceptions are displayed.

      • Open an image using the /c flag.
      • Swipe a blank area of the image.
      • Confirm that no exception is displayed.

      • Open an image viewer.
      • Open another instance of the image viewer.
      • Rearrange the toolstrips in the second instance.
      • Use the /closeall flag on the command line to close all instances of the image viewer.
      • Confirm that no exception is displayed.

      • On the command line, try to run the image viewer with an invalid argument (e.g., /z).
      • Confirm that an "Invalid Argument!" exception is displayed.

      • Use the /e flag on the command line and specify a script file that does not exist.
      • Confirm that an exception is displayed and the image viewer does not open.

      • Open an image viewer window.
      • From the command line, open another image in the image viewer using the /l switch.
      • Confirm that the thumbnail viewer is correctly displaying thumbnails.
