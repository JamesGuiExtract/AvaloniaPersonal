    • [FlexIDSCore #4684, 4687] IDShieldOnDemand
      • Confirm that IDShieldOnDemand is installed.
      • Confirm that there is a start menu shortcut for IDShieldOnDemand.
      • Confirm the following behavior:
        • Application title is "ID Shield on Demand"
        • The application will optionally accept a single command-line parameter as the document to open when it launches.
        • does not connect to a database.
        • has an Open file button and menu option.
        • Document navigation buttons and shortcuts are not available.
        • Tab and the Next item button do not attempt to save and close the current document once the last item is reached.
        • the "Save and commit" button is just a save button.
        • When a document is saved, it will prompt for a location to save redacted output.
        • PDF files can be opened if licensed
        • If the opened file is a PDF, when saving redacted output, it should default to a PDF extension.
        • The default output location is $InsertBeforeExtension(<SourceDocName>, .redacted)
        • The user cannot overwrite the source image with the redacted image.
        • The settings cannot be configured
          • VerifyAllPages is false
          • VerifyAllItems is false
          • Require types is false
          • Seamless mode is false
          • Slideshow is disabled
          • Redactions are created for all enabled items
          • Redaction text will be <Exemption Codes>
        • There is an "ID Shield On Demand" verification options tab.
        • That tab provides the option to create a VOA file every time, after a prompt, or never.
        • If a VOA file already exists, changes will be saved regardless.
        • VOA files will be saved with OnDemandSession metadata.
