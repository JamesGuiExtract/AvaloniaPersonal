Bidirectional Workflow

  1. Goals

    • Use Demo_LabDE as a platform for demonstrating bidirectional workflow.
    • Exercise basic features of bidirectional workflow.
    • Exercise special cases and error conditions associated with bidirectional workflow.

  2. Setup

    • Install LabDE as local admin.
    • Copy Demo_LabDE from install folder to test machine.
      • Run Clean.bat to install a clean Demo_LabDE database.
      • Open Demo_LabDE with DB Admin and confirm that the database is using the current schema.
    • Confirm that the following tables contain meaningful data
      • LabDEOrders
      • LabDEPatients
    • IF PREVIOUS TABLES NEED DATA
      • Exercise A01_InsertPatients.sql and A01_InsertOrders.sql from M:\PVCS_JIRA\JIRA\ISSUE-12783.
    • Run A01_ExtractData.fps to prepare VOA files.
