  2. Double-Blind Verification Scenarios (#9)

    • SCRs: TBD
    • Details: Make it easier for Legislation Guard to find new sensitive data and identify it for review or redaction.
    • Main Checklists Updated: TBD
    • Tested By:
      • TBD, 9.0.0.xx, MM/DD/YYYY
    • Testing:

      • Confirm that performance is as expected.
      • Exercise the Match based upon compare ID Shield data files condition.
      • Confirm that performance is as expected.

      • Scenario 1 - Double Blind Verification
        • Process the Demo_IDShield images.
        • Verify the Demo_IDShield images.
          • Make sure that one or more sensitive items are missed.
          • Make sure that one or more redactions are too small.
          • Make sure that one or more redactions are too large.
        • Verify the Demo_IDShield images again, making sure that the output VOA files are named differently than the first set.
          • Make sure that one or more sensitive items are missed.
          • Make sure that one or more redactions are too small.
          • Make sure that one or more redactions are too large.
        • Compare the two sets of VOA files several different ways
          • Make sure that the missed items are identified.
          • Make sure that the small redactions are identified.
          • Make sure that the large redactions are identified.
          • Make sure that properly sized redactions are handled correctly.

      • Scenario 2 - Legislation Guard
        • Process the Demo_IDShield images for SSN.
        • Process the Demo_IDShield images for DOB, making sure that the output VOA files are named differently than the first set.
        • Merge the two sets of VOA files.
        • Verify the Demo_IDShield images using the merged VOA files as input.
          • Confirm that the SSN items are found and displayed correctly.
          • Confirm that the DOB items are found and displayed correctly.
