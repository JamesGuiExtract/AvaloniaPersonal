  1. Tag Files During Verification (#4)

    • Details: This feature is designed to allow the user to apply their own informative tag strings to documents during verification. A Manage Tags task allows handling of files based on tags. A Task Condition object allows for additional handling options.

    • Tested By:

        • Updated reports are not yet implemented.

        • Confirm that the toolbar button provides a means to define a new tag.
        • Confirm that the ability to define a new tag is controlled by an AllDynamicTagCreation item in the DB Info table.
        • Confirm that AllDynamicTagCreation has a default value of 0.
        • Confirm that the drop-down menu provides checkboxes to indicate that one or more tags are selected.
      • DB Admin
        • Confirm that the Select Files dialog includes an option for any or all of the following tags.
        • Confirm that the list of tags supports selection of one of more tags.
        • Confirm that the tag-related controls are disabled unless the appropriate option is selected.

        • Confirm that the Manage Tags dialog includes a button to Delete an existing tag.

        • Confirm that the Manage Tags dialog includes a button to Refresh the list of tags.
        • Confirm that the Manage Tags dialog includes a button to Close the dialog.

      • Tag Condition object

        • Tagged files - shows a listing of files for each tag, along with a count per tag and a total count.
        • Files tagged with a certain tag - prompts user for a tag name and displays the Tagged files report narrowed down to the specified tag.

        • ID Shield detailed report for specific document - to include a section listing all tags applied on the document.

  2. Skip Files During Verification (#5)

    • SCRs: LegacyRC #5362, #5363, #5364, #5365, #5366, #5367, #5368, #5371, #5372, #5373, #5374, #5375, #5376, #5377, #5378, #5383, #5384, #5385, #5386, #5413, #5414, #5415, #5416, #5417, #5418
    • Details: Verification will support the ability to skip files during processing. Statistics and reports for skipped files must be supported. The DB Admin utility must support skipping files and processing skipped files - either by the current user or by any user. Specifying a comment when skipping a file is supported.
    • Main Checklists Updated: TBD
    • Tested By:
      • Doug, 7.1.1.55, 09/??/2009
    • Testing:
      • DB Admin
        • Confirm that the Summary tab correctly reports the number of files that have been skipped. Include skipping files by multiple users in this test.
        • Confirm that the Select Files dialog includes an option for selecting files skipped by the current user or any user.
      • File Action Manager
        • Confirm that the Status bar includes a pane to report skipped files.
        • Confirm that the Processing setup tab includes a File processing scope group box that allows the user to either process all files in the queue or process skipped files.
        • Confirm that processing skipped files can be either files that the current user skipped or that anyone skipped.
        • Confirm presence of a DB Info setting for RequirePasswordToProcessAllSkippedFiles and that the default setting is 1.
        • Confirm that processing files that anyone skipped requires providing the DB Admin password if the associated setting is on.
        • Confirm that processing files that anyone skipped does not require the DB Admin password if the associated setting is off.

        • Confirm that if processing is stopped during processing of skipped files, then the remaining unprocessed files revert to Skipped status.
        • Confirm that the user can skip a previously skipped file and that it remains in Skipped status.
      • Verification UI
        • Confirm that the user interface contains menus for File, Edit and Help.
        • Confirm that the File menu contains items for Save, Save and commit Ctrl+S, Print, Skip processing and Stop processing.
        • Confirm that the Edit menu contains a Discard changes item.
        • Confirm that the Help menu contains items for ID Shield Help and About ID Shield.
        • Confirm that the user interface contains a multi-line edit box for Comments.

        • Confirm that File - Save saves any pending changes to the VOA file but does not advance to the next document.
        • Confirm that File - Save and commit saves any pending changes to the VOA file and does advance to the next document.
        • Confirm that File - Save is disabled when the user has navigated to a previous document if the previous document was processed and not skipped.
        • Confirm that File - Save is enabled when the user has navigated to a previous document if the previous document was skipped and not processed.
        • Confirm that File - Skip processing prompts the user to save any pending changes to the VOA file and does advance to the next document.
      • New Reports
        • Files currently in skipped status - shows timestamp, user name, file ID, filename, action name, comment and time since skipped for all files currently in skipped status.
        • Skipped files - shows timestamp, user name, file ID, filename, action name, and comment for all rows in the FileActionComment table. Note that contents of this table are dependent on the AutoDeleteFileActionCommentsOnComplete setting in the DB Info table.
      • Updated Reports
        • Detailed report for specific document - to include a dump of the FileActionComment table for the document.
        • Summary of actions and associated document counts - to include a column for skipped files.
        • Summary of actions and associated document page counts - to include a column for skipped files.
