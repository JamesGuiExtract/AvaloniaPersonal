Large-Scale Verification Testing

NOTE: Test notes should be archived under I:\Common\Testing\Product Testing\FLEX_Index_80\VerificationTests

  1. Server Preparation

    • Select a server test machine ( i.e. SuperTester or UltraTester ).
    • Copy suitable image files for redaction from K: to the local machine. The Endurance Test checklist can provide example image folders.
    • Confirm that SQL Server is installed on the server. Either the Standard edition or the Express edition of SQL 2005 or SQL 2008 are acceptable.
    • Create a V: drive mapped to the root folder containing the local images for redaction.

  2. Database Preparation

    • Use FAM DB Admin to create a local LargeVerificationTest database. The server name should be (local).
    • Log in and add actions: Compute and Verify to the database.

  3. File Preparation

    • Create a Compute.fps file with a file supplier to supply the images from drive V:.
    • Include an Execute rules task using an up-to-date set of redaction rules.
    • Include a Conditionally execute task(s) task using the ID Shield data file contents condition object.
    • Execute a Set file-action status task if the condition is True to set the Verify action to Pending.

    • Note that for an average verification rate of 10 seconds per document, 10 verifiers require 1800 sensitive documents in order to exercise the verification task for 30 minutes.
    • Exercise Compute.fps long enough to accumulate an adequate number of sensitive documents.

    • Create Verify.fps stored on drive V:.
    • Use default settings for verification and enable collection of redaction accuracy feedback.

  4. Test Execution - Physical Machines

    • Each test machine to be used in the verification test should be configured:
      • Install the chosen build of ID Shield.
      • Add a license file for ID Shield Client.
      • Add a drive mapping for V:.

    • Open Verify.fps from drive V:.
    • Start processing to confirm that images and VOA files can be read.
    • Stop processing without saving any redacted images until all of the testers are ready.

    • Start processing and make suitable corrections as needed to the redacted images.
    • Stop processing at the end of the testing time or when the sensitive images have all been verified.

  5. Test Execution - Virtual Machines

    • A virtual machine to be used in the verification test should be configured:
      • From a clean operating system VHD file template.
      • Apply the required Windows Updates.
      • Install the chosen build of ID Shield.
      • Add a license file for ID Shield Client.
      • The virtual machine does not need to be added to the domain at this point.

    • Open Verify.fps from drive V:.
    • Start processing to confirm that images and VOA files can be read.
    • Stop processing without saving any redacted images.

    • Each tester should:
      • Receive a copy of the configured VHD file.
      • Create a new virtual machine ( and .vmc file ) from the Virtual PC console and use the new VHD file.
      • Modify the computer name of the new virtual machine to avoid IP address conflicts.
      • After modifying the computer name the virtual machine needs to be added to the extract.local domain.
      • Restart the virtual machine and log in with your domain user account.
      • Add a drive mapping for V:.
      • Open Verify.fps from drive V:.
      • Start processing to confirm that images and VOA files can be read.
      • Stop processing without saving any redacted images until all of the testers are ready.

    • Start processing and make suitable corrections as needed to the redacted images.
    • Stop processing at the end of the testing time or when the sensitive images have all been verified.

  6. Test Review and Data Analysis

    • The following items should be reviewed after each test execution exercise.
      • Confirm that no unexpected errors were seen by any tester.
      • Confirm that the expected number of redacted images are present.
      • Run the ID Shield Report and confirm that rational numbers are present for the verification exercise.
