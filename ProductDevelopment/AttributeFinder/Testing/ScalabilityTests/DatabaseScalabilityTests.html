File Action Manager Database Scalability Tests

* ID Shield and Extract Systems LM should be installed and licensed for these tests. It is not necessary to install FLEX Index.

  1. Test #1 - Machine Scalability

    • Purpose & Machine Setup
      • Measure the maximum database throughput as more processing machines are added.
      • Select a test machine to serve as the image and database host.
      • Collect a large folder of files to process; typically we use the Utah images.
      • Queue all of the images; note that the files must be queued via UNC path or mapped drive. Save a copy of the database with all files queued.
      • Configure an FPS file to run a Set File Action Status task. In the advanced processing settings, set it to grab 1000 files from the database at a time.
      • Select a group of processing machines or a group of virtual machines for use in the test.

    • Test Execution
      • Before each test:
        • Restore the unprocessed database from SQL Management Studio.
        • Reboot all test machines from the previous test.
        • Add one more machine or VM to the pool of processing machines.
      • On each processing machine, start the test FPS file.

    • Test Results
      • Record the starting and ending processing times in an appropriate spreadsheet; see I:\Common\Testing\Product Testing\FLEX_Index_90\ScalabilityTests\DatabaseScalability for an example.
      • Create a chart to display the rate of documents processed per day for each test configuration.
        • 1 test machine or VM
        • 2 test machines or VMs
        • 3 test machines or VMs
        • 4 test machines or VMs
        • 5 test machines or VMs
        • 6 test machines or VMs
        • 7 test machines or VMs
        • 8 test machines or VMs
