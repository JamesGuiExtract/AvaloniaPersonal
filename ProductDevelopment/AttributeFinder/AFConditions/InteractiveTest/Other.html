Ruleset Editor - AF Conditions

  1. Character confidence condition

    • Confirm that the property page includes a combo box for treating the condition as met or not met.
    • Confirm that the combo box defaults to met.
    • Confirm that the property page includes a combo box for the type of character confidence as average, minimum or maximum.
    • Confirm that the combo box defaults to average.
    • Confirm that the property page includes a mathematical operator combo box for <, >, <=, >=, !=, or =.
    • Confirm that the property page includes an associated edit box for the actual character confidence.
    • Confirm that the edit box is 0 by default.
    • Confirm that the edit box accepts values from 0 to 100 inclusive.
    • Confirm that the property page includes a check box for allowing specification of a second character confidence.
    • Confirm that the associated controls (two combo boxes and an edit box) are enabled only when the check box is checked.
    • Confirm that the first combo box describes how the two confidence levels should be combined as and or or.
    • Confirm that the second combo box is for mathematical operators and includes <, >, <=, >=, !=, or =.
    • Confirm that the associated edit box is for the actual character confidence.
    • Confirm that the edit box is 0 by default.
    • Confirm that the edit box accepts values from 0 to 100 inclusive.
    • Confirm that the object settings are persistent.

    • Confirm that the following scenarios can be successfully tested
      • Single-sided condition with average confidence met.
      • Single-sided condition with average confidence not met.
      • Single-sided condition with maximum confidence met.
      • Single-sided condition with maximum confidence not met.
      • Single-sided condition with minimum confidence met.
      • Single-sided condition with minimum confidence not met.
      • Double-sided condition with average confidence met.
      • Double-sided condition with average confidence not met.
      • Double-sided condition with maximum confidence met.
      • Double-sided condition with maximum confidence not met.
      • Double-sided condition with minimum confidence met.
      • Double-sided condition with minimum confidence not met.

    • Exercise various double-sided conditions and confirm that invalid settings cannot be saved
      • < 50 and > 90.
      • = 50 and < 40.

  2. Document type condition

    • Confirm that the property page includes a combo box for the document type to match or not match one of the following types.
    • Confirm that the property page includes a list box for the selected document types.
    • Confirm that the property page includes a Select... button.
    • Confirm that the Select... button is always enabled.
    • Confirm that clicking the Select... button displays a Select Document Types dialog.

    • Confirm that the Select Document Types dialog includes a disabled Document Category combo box.
    • Confirm that the Document Category combo box has County Document selected by default.
    • Confirm that the Select Document Types dialog includes a Select one or more of the following types: list box.
    • Confirm that the list box includes Any Unique and Unclassified at the top of the list.
    • Confirm that the list box includes each document type as included in the .\ComponentData\DocumentClassifiers\County Document\DocTypes.idx file.
    • Confirm that the list box supports selection of multiple document types.
    • Confirm that the selected document types replace the document types currently specified in the Document type condition property page.

    • Confirm that the property page includes a Remove button.
    • Confirm that the Remove button is enabled only when one or more document types are selected.
    • Confirm that the selected document types are removed without confirmation.

    • Confirm that the property page includes a Clear button.
    • Confirm that the Clear button is enabled only when one or more document types are defined.
    • Confirm that the defined document types are removed without confirmation.

    • Confirm that the property page includes a combo box to specify the minimum probability of the classification.
    • Confirm that the combo box includes entries for maybe, probable and sure.
    • Confirm that the default selection in the combo box is maybe.

    • Confirm that settings are persistent.

  3. Finding rule condition

    • Confirm that the property page includes a combo box for selection of an attribute finding rule.
    • Confirm that the property page includes a Configure... button.
    • Confirm that the Configure... button is disabled if the selected finding rule does not have a property page.
    • Confirm that the Configure... button is enabled if the selected finding rule does have a property page.
    • Confirm that The rule must be configured also appears if the selected finding rule has a property page.
    • Confirm that settings are persistent.

  4. RSD file condition

    • Confirm that the property page includes an edit box for specification of a rule set.
    • Confirm that the property page includes an associated tag selector button.
    • Confirm that clicking the tag selector button displays the following items
      • <RSDFileDir>
      • <DocType>
      • <ComponentDataDir>
      • <SourceDocName>
      • <RuleExecutionID>
      • various $ functions
    • Confirm that the property page includes an associated browse ... button.
    • Confirm that clicking the browse button displays a File - Open dialog.
    • Confirm that the fully qualified path to the selected file is displayed in the edit box.
    • Confirm that settings are persistent.
