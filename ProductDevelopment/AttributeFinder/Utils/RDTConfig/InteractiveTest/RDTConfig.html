RDT Config Testing Checklist

  1. Licensing

    • Confirm that the application displays an error message without an RDT license.
    • Confirm that the application starts properly with an RDT license.

  2. Default Settings

    • Clear the AutoEncrypt key under HKCU\Software\Extract Systems\AttributeFinder\Settings.
    • [LegacyRC #4926] Confirm that Automatically re-encrypt out-of-date ETF files is checked by default.

    • Clear the LoadFilePerSession key under HKCU\Software\Extract Systems\AttributeFinder\Settings.
    • Confirm that Load .dcc and .dat files only once per session is checked by default.

    • Clear the LoggingEnabled key under HKCU\Software\Extract Systems\AttributeFinder\AFUtils\EntityFinder.
    • Confirm that Enable logging in the Entity Finder object is unchecked by default.

    • Clear the LoggingEnabled key under HKCU\Software\Extract Systems\AttributeFinder\AFSplitters\EntityNameDataScorer.
    • Confirm that Enable logging in the Entity Name Data Scorer object is unchecked by default.

    • Clear the StoreRulesWorked key under HKCU\Software\Extract Systems\AttributeFinder\IndustrySpecific\County\CountyCustomComponents\GrantorGranteeFinderV2.
    • Confirm that Add document tag with RuleID for rule that captured data when using the Grantor-Grantee Finder is unchecked by default.

    • Clear the ScrollLogger key under HKCU\Software\Extract Systems\TestingFramework\Settings.
    • Confirm that Enable Auto Scrolling of the Test Logger window as test runs is unchecked by default.

    • Clear the DisplayPercentageEnabled key under HKCU\Software\Extract Systems\InputFunnel\InputReceivers\SpotRecIR.
    • Confirm that Display X, Y percentages in Image Viewer is unchecked by default.

    • Clear the AutoOpenImage key under HKCU\Software\Extract Systems\AttributeFinder\Utils\EAVGenerator.
    • [FlexIDSCore #3799] Confirm that Automatically open image in VOA File Viewer is checked by default.

    • Clear the AutoExpandAttributes key under HKCU\Software\Extract Systems\AttributeFinder\AFCore\RuleTester.
    • Confirm that Automatically expand attributes in Rule Tester is unchecked by default.

    • Clear the Prefix key under HKCU\Software\Extract Systems\AttributeFinder\Settings.
    • Confirm that Optional prefix for .dcc and .dat files combo box is cleared by default.

    • Clear the RFolder_1 key under HKCU\Software\Extract Systems\AttributeFinder\Utils\RDTConfig\RootFolderList.
    • Confirm that Root folder for automated testing files combo box is cleared by default.

    • Clear the ComponentDataFolder key under HKCU\Software\Extract Systems\AttributeFinder\Settings.
    • Confirm that Full path to component data folder combo box is cleared by default.

    • Clear the DiffCommandLine key under HKCU\Software\Extract Systems\TestingFramework\Settings.
    • Confirm that Diff tool command line combo box is cleared by default.

    • Make a change to each item and click the Restore Defaults button.
    • Confirm that each item has been restored to the default settings.

  3. Functionality

    • Automatically re-encrypt out-of-date ETF files
      • Confirm that the checkbox is checked.
      • Modify a .dat file associated with a specific .dat.etf file.
      • Exercise an automated test that uses the .dat file.
      • Confirm that the timestamp for the .dat.etf file changes to reflect re-encryption.

      • Confirm that the checkbox is unchecked.
      • Modify a .dat file associated with a specific .dat.etf file.
      • Exercise an automated test that uses the .dat file.
      • Confirm that the timestamp for the .dat.etf file does not change (i.e. the file is not re-encrypted).

    • Load .dcc and .dat files only once per session
      • Confirm that the checkbox is checked.
      • Exercise the DocumentClassification National Rule Set test.
      • Do not close the Test Harness.
      • Retrieve an unencrypted copy of Mortgage.dcc
      • Modify the DCC file.
      • Exercise the DocumentClassification National Rule Set test again.
      • Confirm that the classification results for Mortgage are unchanged.

      • Confirm that the checkbox is unchecked.
      • Exercise the DocumentClassification National Rule Set test.
      • Do not close the Test Harness.
      • Retrieve an unencrypted copy of Mortgage.dcc
      • Modify the DCC file.
      • Exercise the DocumentClassification National Rule Set test again.
      • Confirm that the classification results for Mortgage are changed.

    • Enable logging in the Entity Finder object
      • Modify C:\Program Files\Extract Systems\TestingFiles\ProductDevelopment\AttributeFinder\AFUtils\AFUtilsTest\TestFiles\TestEntityFinder.dat to comment out all of the <FILE> lines except for one.
      • Modify C:\Program Files\Extract Systems\TestingFiles\ProductDevelopment\AttributeFinder\AFCore\AutomatedTest\All\AFCore.tcl to comment out all lines except for the line starting with UCLIDAFUtilsTest.EntityFinderTester.1;;.

      • Confirm that the checkbox is checked.
      • Exercise Run All Tests from the RDT Start menu.
      • Confirm that EFALog.dat is updated in the CommonComponents folder.

      • Confirm that the checkbox is unchecked.
      • Exercise Run All Tests from the RDT Start menu.
      • Confirm that EFALog.dat is not updated in the CommonComponents folder.

    • Enable logging in the Entity Name Data Scorer object
      • Modify C:\Program Files\Extract Systems\TestingFiles\ProductDevelopment\AttributeFinder\AFCore\AutomatedTest\All\AFCore.tcl to comment out all lines except for the line starting with UCLIDAFDataScorersTest.EntityNameDataScorerTester.1;;.

      • Confirm that the checkbox is checked.
      • Exercise Run All Tests from the RDT Start menu.
      • Confirm that gstrENDSLog.dat is updated in the CommonComponents folder.

      • Confirm that the checkbox is unchecked.
      • Exercise Run All Tests from the RDT Start menu.
      • Confirm that gstrENDSLog.dat is not updated in the CommonComponents folder.

    • Add document tag with RuleID for rule that captured data when using the Grantor-Grantee Finder
      • This option is no longer since Grantor-Grantee Finder is obsolete.

    • Enable Auto Scrolling of the Test Logger window as test runs
      • Confirm that the checkbox is checked.
      • Execute a long-running test in the Test Harness.
      • Confirm that the Test Result Logger window automatically scrolls up as the window fills.

      • Confirm that the checkbox is unchecked.
      • Execute a long-running test in the Test Harness.
      • Confirm that the Test Result Logger window does not automatically scroll up as the test runs.

    • Display X, Y percentages in Image Viewer
      • Confirm that the checkbox is checked.
      • Open an image file in the Image Viewer.
      • Move the mouse around the image and confirm that the status bar displays position information in both pixels and percentages.

      • Confirm that the checkbox is unchecked.
      • Open an image file in the Image Viewer.
      • Move the mouse around the image and confirm that the status bar displays position information in just pixels.

    • Automatically open image in VOA File Viewer
      • Confirm that the checkbox is checked.
      • Double-click on a VOA file to open it in the VOA Viewer.
      • Confirm that the associated image is also opened in the Image Viewer.
      • Close the VOA Viewer.
      • Confirm that the Image Viewer also closes automatically.

      • Confirm that the checkbox is unchecked.
      • Double-click on a VOA file to open it in the VOA Viewer.
      • Confirm that the associated image is not automatically opened in the Image Viewer.

    • Automatically expand attributes in Rule Tester
      • Confirm that the checkbox is unchecked.
      • Exercise some rules in the Rule Tester that provide attributes and subattributes.
      • Confirm that the attributes are not expanded.

      • Confirm that the checkbox is checked.
      • Exercise some rules in the Rule Tester that provide attributes and subattributes.
      • Confirm that the attributes are expanded.

    • Optional prefix for .dcc and .dat files
      • TBD.

    • Root folder for automated testing files
      • Copy the installed TestingFiles folder installed with the RDT to TestingFiles2.
      • Make some changes to TCL or DAT files, perhaps similar to those described above for logging in the Entity Finder object.
      • Use RDTConfig.exe to modify the root folder setting.
      • Exercise the changed file(s) and confirm that the expected results are seen.
      • Restore the previous root folder.
      • Exercise the same file(s) and confirm that the changed results are not seen.
