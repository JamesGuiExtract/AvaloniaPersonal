Demo_FLEXIndex Testing Checklist

NOTE: Items prefaced with an asterisk indicate functionality that is not yet complete.

  1. Instructions and Installation

    • Confirm that the latest Demo_FlexIndex folder can be copied from the current build folder to C:\.
    • [FlexIDSCore #4137] Confirm that all files under the Rules folder are encrypted.
    • Confirm that DataEntry: Verify extracted data requires only a FLEX Index Client license.
    • [FlexIDSCore #4068] Confirm that the blank database associated with the demo has the current schema version.

  2. Demo Images

    • Exercise the installed rules and images using the Demo_FLEXIndex.xls spreadsheet.
    • Review the following items after finishing the verification
      • Confirm that VOA files were updated for each file in which changes were made.
      • Confirm that XML files were created for each file.

  3. User Interface

    • General
      • Confirm that unviewed text is displayed in Bold font.
      • Confirm that viewed text is displayed in normal font.
      • Confirm that presence and of an Undo command. Test its behavior:
        • in different controls (party table, address table, text boxes...)
        • after saving
        • editing multiple attributes at once
        • using the button, keyboard shortcut, and menu item
        • any other situations that come to mind

    • Parcel Number
      • Confirm that an edit box exists for Parcel Number.
      • Confirm that text can be swiped into the Parcel Number edit box.

    • Consideration Amount
      • Confirm that an edit box exists for Consideration Amount.
      • Confirm that text can be swiped into the Consideration Amount edit box.

    • Document Type
      • Confirm that an edit box exists for Document Type.
      • Confirm that text can be swiped into the Document Type edit box.

    • Return Address
      • Confirm that rows exist for
        • Recipient
        • Street Address
        • City
        • State
        • Zip
      • Confirm that text can be swiped for each individual row.
      • Confirm that an entire return address can be swiped.
      • [DataEntry #890] Confirm that the rows cannot be resized.
      • [DataEntry #892] Confirm that multiple rows can be selected and deleted.
      • [DataEntry #911]
        • Open an image with a Return Address.
        • Click on the City field and type DEL.
        • Click the Down arrow and confirm that the City stays deleted.
        • Click on the State field and type DEL.
        • Click the Down arrow and confirm that the State stays deleted.
        • Click on the ZipCode field and type DEL.
        • Click the Up arrow and confirm that the Zip Code stays deleted.

    • Parties
      • Confirm that columns exist for
        • First
        • Middle
        • Last / Company
        • Party Type
        • MappedTo
      • Confirm that text can be swiped for each individual cell.
      • Confirm that text can be swiped to add a new party.
      • Confirm that multiple parties can be added with a single swipe.
      • [DataEntry #898] Confirm that text in the Type column does not include spatial information.
      • [DataEntry #899] Confirm that an entire row can be selected and deleted.

    • Legal Description
      • Confirm that columns exist for
        • Lot
        • Block
        • Subdivision
      • Confirm that a multi-line edit box is provided for the full-text Legal Description.
      • Confirm that a scrollbar is available if enough text is provided.
      • Confirm that text can be swiped for each individual cell in the split Legal Description.
      • Confirm that a full Legal Description can be swiped to add a new split Legal Description.
      • Confirm that text can be swiped to replace selected text in the full-text Legal Description field.
      • [DataEntry #899] Confirm that an entire row can be selected and deleted.
      • [DataEntry #927] Confirm that the Legal description table supports tabbing by row.
