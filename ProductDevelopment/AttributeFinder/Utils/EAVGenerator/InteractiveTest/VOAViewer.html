VOA Viewer Testing Checklist

  1. Toolbar

    • Confirm that the Create new EAV file button clears the current data and creates a new EAV file.
      • Click the Open a new Image Viewer toolbar button.
      • Open an image file.
      • Click the Recognize text and process button.
      • [FlexIDSCore #2651] Confirm that no drop-down menu appears.
    • Confirm that the Open an EAV file button displays a standard Windows open file dialog box.
      • Confirm that the default file extensions are .eav and .voa.
      • Confirm that the contents of the opened file are displayed in the list box.
      • Confirm that the window caption shows Filename - VOA File Viewer.
      • [FlexIDSCore #2669] Drop extension before VOA for image files
        • Open a VOA file.
        • Confirm that double-clicking a sensitive data item opens the associated image and highlights the appropriate text.
        • Close the image window and the VOA file.
        • Rename the VOA file to Filename.tif.test.voa.
        • Open the renamed VOA file.
        • Confirm that double-clicking a sensitive data item opens the associated image and highlights the appropriate text.
      • [FlexIDSCore #2648] Exception raised when not necessary
        • Open a VOA file containing two or more sensitive data items.
        • Confirm that double-clicking a sensitive data item highlights the item in the Image Viewer.
        • Rename the image file.
        • Double-click on a different sensitive data item.
        • Confirm that the sensitive data item is still highlighted in the Image Viewer.
      • [FlexIDSCore #2649] Exception raised for missing image file
        • Open a VOA file.
        • Rename or delete the associated image file.
        • Confirm that double-clicking a sensitive data item displays a Cannot locate source image error message.
        • Confirm that the error includes the filename as part of the debug information.
    • Confirm that the Save file button is inactive by default.
    • Confirm that the Save file button becomes active when the attributes have been modified.
      • Confirm that the Save file button displays a Windows Save as dialog if the application was started with a new file.
      • Confirm that the Save file button just saves changes and overwrites the existing file if the application was started with a previously existing file.
      • [FlexIDSCore #2861] Open a VOA file that contains sub-attributes. Save the file to a different VOA file. Confirm that the new file has the same contents as the original file.
      • [FlexIDSCore #2860] Open an EAV file containing C++ escaped characters (i.e. backslash r backslash n). Save it to another temporary EAV file. Open the new file and confirm that the escaped characters have not been modified.
      • [FlexIDSCore #2848] Open an EAV file from the National Ruleset that contains sub-attributes. I:\Common\Engineering\ProductDevelopment\AttributeFinder\NationalRuleSet\GrantorGranteeFinder\Images\Abstract of Support Judgment\Grantee\0002.tif.eav
        Confirm that the file opens without an Invalid Identifier exception.
    • Confirm that the Save file as... button displays a Windows Save as dialog.
      • Confirm that the Save as dialog allows a file to be saved in text format as an EAV file.
      • Confirm that the Save as dialog allows a file to be saved in binary format as a VOA file.
      • [FlexIDSCore #2711] Confirm that an empty EAV file can be saved and that the file size is 0 bytes.
      • [FlexIDSCore #2711] Confirm that an empty VOA file can be saved and that the file size is 2560 bytes.
    • Confirm that the Open a new Image Viewer button runs the Image Viewer application.
    • Confirm that the Highlight the current attribute in the image window button will open corresponding image and highlight text in the Image Viewer.

  2. Drag and Drop

    • Confirm that dropping a .voa file on the main dialog opens the .voa file.
    • Confirm that dropping an .eav file on the main dialog opens the .eav file.
    • [FlexIDSCore #2887] Confirm that double-clicking on an .eav file opens the file in the VOA Viewer.
    • [FlexIDSCore #3111] Confirm that dropping (or double clicking) an .evoa file on the main dialog opens the .evoa file.

    • [FlexIDSCore #2727] Create a new attribute and add an Attribute Name. Tab to the Attribute Value field. Confirm that text can be added for the value.

    • Confirm that the Copy Attribute button is inactive by default.
    • Confirm that the Copy Attribute button becomes active when a non-blank attribute is selected.
    • Confirm that Copy Attribute places a copy of the selected attribute at the bottom of the list.

    • Confirm that the Delete Attribute button is inactive by default.
    • Confirm that the Delete Attribute button becomes active when an attribute is selected.
    • Confirm that Delete Attribute removes the currently selected attribute from the list.
    • [FlexIDSCore #3512] Confirm that the confirmation dialog is modal.

    • Confirm that the Split Attribute... button is inactive by default.
    • Confirm that the Split Attribute..." button becomes active when an attribute is selected.
    • Confirm that Split Attribute... displays the Select Attribute Splitter dialog.

    • Confirm that the Merge Attributes button is inactive by default.
    • Confirm that the Merge Attributes button becomes active when there are at least two attributes selected in the list at the same level.
    • Confirm that the new attribute Name is a combination of the original Names.
    • Confirm that the new attribute Value is a combination of the original Values.
    • Confirm that merging attributes prompts a Save File? message when the application is closed.

    • Confirm that the Name edit box is enabled when an attribute is selected.
    • Confirm that the Name edit box contains the current name data from the list control when an attribute is selected.

        • Confirm that the text from the new swipe is appended to the text from the old swipe.

        • Select the same attribute and make a swipe from the new image.
        • Confirm that an error is shown explaining that you cannot append swipes from different files.
