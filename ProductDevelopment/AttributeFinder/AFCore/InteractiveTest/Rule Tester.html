Rule Tester Testing Checklist

  1. Toolbar

    • Confirm that Open a new Image Viewer button runs the Extract Image Viewer application.
    • Confirm that Clear output window button clears results in the output pane.
    • Confirm that Execute rules button executes rule file on the selected input and displays results in the output pane.
    • Save attributes to VOA file
      • Confirm that the button is inactive by default.
      • Confirm that the button becomes active when results are displayed in the output pane.
      • Confirm that the button displays the Windows Save As dialog.

    • [FlexIDSCore #4861] Confirm that multiple FKB versions are separated with a comma and space in the About box.

    • [ISSUE-13422] Rule Tester
      • Confirm that the Run button is disabled unless both of the following licenses are present.
        • RDT
        • Disable Rule Execution Counters <Dev Only>
      • Confirm that the combination of Disable Rule Execution Counters and either license for simple rule writing does not enable the run button.

  2. General Settings Tab

    • Confirm that this tab is visible only when run from Ruleset Editor application.
    • Confirm that All Attributes radio button is selectable.
    • Confirm that Current Attribute radio button is disabled if no attribute is defined.
    • Confirm that Current Attribute radio button is selectable.
    • Confirm that current selected attribute name appears in parentheses next to button text.

  3. Ruleset Tab

    • Confirm that this tab is visible only when the Rule Tester application is run directly.
    • Confirm that the Ruleset edit box is editable.
    • Confirm that the ... button displays the standard Windows Open dialog.
    • Confirm that the selected file is displayed in the Ruleset edit box.
    • Confirm that a Ruleset Properties... button is provided.
    • Confirm that the button is enabled if and only if a rule set has been specified.
    • Confirm that the dialog includes a Properties tab.
    • Confirm that the tab includes the Counters list box, the Key Serial #'s edit box, and the FKB Version edit box.
    • Confirm that the internal use only check box is only present if the user has an RDT license.
    • Confirm that the swiping rule check box is only present if the user has an RDT license.
    • Confirm that the dialog includes a Comments tab.
    • Confirm that the tab includes a multi-line read-only edit box.

    • Confirm that the Rule Set Editor can modify the Comments edit box.

  4. Input Tab

    • Confirm that the Input combo box is selectable.
    • Confirm presence of a Make the contents of the following data file available to the rules checkbox.
      • Confirm presence of an associated edit box.
      • Confirm presence of an associated browse button.
      • Confirm that the associated elements are enabled only when the checkbox is checked.
      • Confirm proper behavior of the browse button.
    • Confirm that Most recently OCR'd text from image window is listed.
    • Confirm that the main edit box is editable.

    • Confirm that Most recently processed text from file is listed.
    • Confirm that selection of this option displays a Filename: edit box and a ... button.
    • Confirm that the ... button displays the standard Windows Open dialog.
      • Confirm that the selected file is displayed in the Filename: edit box.
      • Confirm that file selection displays file contents in the main edit box.
      • Confirm that the main edit box becomes read-only.

    • Confirm that Text from manual input is listed.
    • Confirm that the main edit box is editable.

    • Rule Tester Support For Empty USS Files
      • Confirm that the Input tab contains a Perform OCR on input file if it is an image checkbox.
      • Confirm that the checkbox is checked by default.
      • Confirm that rules run as expected when the checkbox is unchecked.

  5. Found Attributes

    • Confirm that Name column tree control displays sub-attributes.
    • Confirm that attribute trees are auto-expanded or not as specified by the Automatically expand attributes in Rule Tester setting in RDT Configuration.
    • Confirm that selection of an attribute will open the corresponding image and highlight text in the Image Viewer.
    • Confirm that double-click on an attribute displays the Edit Attribute dialog.
      • Confirm that the Name edit box is editable.
      • Confirm that the Value edit box is editable.
      • Confirm that the Type edit box is editable.
      • Confirm that changes made are displayed in the Found Attributes.
      • Confirm that the OK button saves changes and exits the dialog.
      • Confirm that the Cancel button ignores changes and exits the dialog.
      • Confirm that the X button ignores changes and exits the dialog.

  6. Miscellaneous

    • [FlexIDSCore #4456]
      • Open the rule tester.
      • Confirm that the Perform OCR button is checked.
      • Paste the path to an image into the filename box.
      • Add a space at the end.
      • Hit the play button or click in the text box that displays the input text.
      • Confirm that the image is appropriately OCRed.

    • [ISSUE-12325, ISSUE-12535] Rule Tester
      • Exercise some rules via Rule Tester.
      • Confirm that total rules execution time is displayed in the status bar.
      • Confirm that execution time is displayed with millisecond precision.

    • [ISSUE-12523, ISSUE-12529] Rule Tester
      • Open and modify an RSD file with Rule Set Editor.
      • Kill Rule Set Editor.
      • Reopen Rule Set Editor and confirm that the previously unsaved changes are available in the Recovery File.
      • Save the RSD file.
      • Open the Rule Tester from withint Rule Set Editor.
      • Confirm that the saved path to the RSD file is displayed in the title bar.
      • Confirm that the Rule Tester window can be aero-snapped on a Windows 7 or later operating system.

    • [ISSUE-13151] Rule Tester
      • Exercise the rules and images found in M:\PVCS_JIRA\JIRA\ISSUE-13151.
      • Confirm that each image processes in reasonable time.

    • [ISSUE-13458] Confirm that RuleTester.exe can display ruleset properties including counter definitions without an RDT license.
