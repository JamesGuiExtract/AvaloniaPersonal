Rule Set Editor - Menu Items

Window Form:

  • RuleSet Editor text appears in title bar.
  • RSD filename appears in title bar when loaded.
  • _ - Minimizes window and child windows (including Ruleset Tester).
  • X - Closes window.
  • Click on X displays Save Changes dialog when changes have not been saved.
  • ALT+F4 closes window and child windows (including Ruleset Tester)
  • ALT+F4 displays Save Changes dialog when changes have not been saved.

Menu Items:

  1. File Menu

    New

    • Click on New clears current rules.
    • Click displays Save Changes dialog when changes have not been saved.
    • CTRL+N clears current rules.

    Open

    • Click on Open displays File - Open dialog.
    • Click displays Save Changes dialog when changes have not been saved.
    • CTRL + O displays dialog.
    • Opens RSD files.
    • Opens RSD.ETF files.
    • When opened, RSD.ETF files are not editable or viewable.

    Save

    • Click on Save saves file if has previously been saved.
    • Click on Save displays Save As dialog, if file has never been saved.
    • CTRL+S saves changes to the file.
    • CTRL+S opens Save As dialog if file has never been saved.

    • [FlexIDSCore #4865] Rule Set Profiling
      • Create an RSD file Test.rsd that will find some data.
      • Create a Master.rsd file that calls Test.rsd.
      • Copy and paste Test.rsd and edit Master.rsd to also call the copy.
      • Run Master.rsd while profiling and confirm that the profiler is unable to distinguish the rules in the original and the copy.
      • Open the copy in the ruleset editor and save without making any changes.
      • Run Master.rsd again while profiling and confirm that the profiler is able to distinguish between them.

    • [FlexIDSCore #4872]
      • Open a new instance of the ruleset editor and set up a rule that will find something. Save the file.
      • Open the rule tester and confirm that the rule works.
      • Uncheck the box for the rule and run again. Confirm that the rule doesn't run and no data are found.
      • Save again and reopen the RSD file. Confirm that the rule is still unchecked.

    • [FlexIDSCore #4974] Ruleset Auto-save
      • Open an 8.0 ruleset that contains deprecated rule objects (IN - Marion - Trimin).
      • Confirm that the ruleset cannot be saved in its present form.
      • Manually remove some of the deprecated objects.
      • Confirm that no errors from the auto-save process are seen.

    • [FlexIDSCore #4994] Rule Set Editor
      • Open an 8.0 RSD file that contains a deprecated rule object such as Grantor-Grantee Finder V2.
      • Attempt to save it using File - Save As.
      • Confirm that an error is displayed about needing an FKB version.
      • Update the FKB version and remove any deprecated objects. Add another rule, if needed in order to save.
      • Attempt to save the modified ruleset using File - Save As.
      • Confirm that the save is successful.

    • [LegacyRC #6255] COM Object Data Files
      • Open a version 8.0 master ruleset.
      • Attempt to save it and confirm that an exception is displayed complaining about a missing FKB version.
      • Close the exception and the RuleSet Editor.
      • Confirm that the master ruleset can still be opened.

    Save As

    • Click on Save As displays Save As dialog.

    Recent Files

    • Selecting Recent Files displays submenu file list of recently opened files.
    • Click on Recent Files -> Submenu Item opens the file.
    • Click on Recent Files -> Submenu Item displays Save Changes. dialog when changes have not been saved.
    • Confirm that files opened via File - Open get added to this list.

    Properties

    • Selecting Properties opens the Ruleset Properties dialog.
    • Ruleset Properties Dialog
      • Confirm that a Properties tab is available.
      • FlexIndex - Indexing check box is checkable.
        • When checked, Indexing appears on status bar.
      • FlexIndex - Pagination check box is checkable.
        • When checked, Pagination appears on status bar.
      • IDShield - Redaction (By Page) check box is checkable.
        • When checked, Redaction (pages) appears on status bar.
        • When checked using the mouse, Redaction (By Document) is automatically unchecked.
      • IDShield - Redaction (By Document) check box is checkable.
        • When checked, Redaction (docs) appears on status bar.
        • When checked using the mouse, Redaction (By Page) is automatically unchecked.
      • Only one of the check boxes: IDShield - Redaction (By Page) or IDShield - Redaction (By Document) may be checked at one time.
      • Key Serial #s: edit box is editable.
        • When entered, the value entered is displayed on status bar.
        • Confirm that hyphenated ranges (e.g., "7 - 11") can be specified.
      • FKB Version edit box is editable
        • Confirm that ComponentData is installed in an FKB-version-specific subdirectory of the ComponentData folder.
        • Confirm that an FKB version must be specified to save a ruleset with a counter or a swiping rule.
        • Confirm that nested rulesets do not require that an FKB version be specified.
        • Confirm that if a nested ruleset does specify an FKB version, it must be the same as the master ruleset's FKB version.
        • Confirm that old rules with no FKB version specified use the root of ComponentData.
        • Confirm that a ruleset saved with a legacy FKB version installed to the root of ComponentData will confirm the version number in FKBVersion.txt.
        • Confirm that if a non-default ComponentDataDir is specified in RDTConfig, it is assumed to include the full path (including any needed version number).
        • Confirm that about boxes for AttributeFinder-specific applications list all installed FKB versions.
      • Ruleset can be executed only internally check box is checkable.
        • When checked, InternalUseOnly = Yes is displayed on status bar.
        • When unchecked, InternalUseOnly = No is displayed on status bar.
      • This is a swiping rule check box is checkable.
        • [FlexIDSCore #3714] NOT FIXED When checked, SwipingRule = Yes is displayed on the status bar.
        • [FlexIDSCore #3714] NOT FIXED When unchecked, SwipingRule = No is displayed on the status bar.
      • Confirm that a Comments tab is available.
        • Confirm that a multi-line edit box is provided.
        • Confirm that comments are persistent.

      • OK - Save changes and exits dialog.
      • Cancel - Cancels changes and exits
      • X button ignores changes and exits dialog.

      • [FlexIDSCore #4744] Rule Set Editor
        • Open a version 8.0 rule set that specifies a counter but not an FKB version.
        • Leave the RSD file open for at least one minute.
        • Confirm that no Unable to save Ruleset exceptions are logged or displayed.
        • Confirm that no File cannot be deleted exceptions are logged or displayed.

      • [FlexIDSCore #4775, #4828, #4830, #4831, #4832, #4836, #4837] FKB Updates
        • Install a legacy FKB update to the root of the ComponentData folder.
        • Create a rule set that specifies use of this FKB update.
        • Confirm that the rule set executes properly via Rule Tester.
        • Confirm that an older FKB update can be installed after a newer version has been installed.
        • Confirm that the Document Classifier works by exercising the Demo_IDShield rules.
        • Exercise the automated tests without a legacy FKB update and confirm that the following tests pass
          • Entity Name Data Scorer
          • Entity Name Splitter
          • MERS Modifier
          • Entity Finder
        • Confirm that .etf files are included in the FKB update.
        • Confirm that specifying the Full path to component data via RDT Config overrides the setting in an RSD file.
        • Confirm that FKB update version numbers are in the form MM.mm.0.###.
          • MM = Major version or last two digits of year
          • mm = Minor version or numbered update for the year
          • ### = Build number
        • Confirm that the FKB update build number deployed with a build, especially a released build, matches the software build number.

    Exit

    • Click on Exit - exits program successfully (without an exception or error message).
    • Click displays Save Changes dialog when changes have not been saved.

      Save Changes Dialog

      • Yes - Saves changes and exits program.
      • No - Does not save changes and exits program.
      • Cancel - Does not exit program, but does exit menu and does not save changes.
      • Clicking X - Does not exit program, but does exit menu and does not save changes.

  2. Tools Menu

    • Click on Open Rule Tester opens Rule Tester dialog.
      • Works on RSD files.
      • Works on RSD.ETF files.
      • [FlexIDSCore #4266] Unexpected exception
        • Open the RuleSet Editor.
        • Press ctrl+t to open the rule tester.
        • Type something into the filename edit box of the rule tester.
        • Close the RuleSet Editor.
        • Confirm that no exceptions are seen.
    • CTRL + T opens Rule Tester dialog.
    • Click on Execute Test Harness starts an instance of the Test Harness.
    • Click on Check for New Components works.
      • Confirm that a list of object types and counts found is provided.

  3. Help - Menu

    • Click on Attribute Finder Help opens Flex Index RDT help file.
    • Click on About FLEX Index brings up about About FLEX Index dialog box.
      • Dialog displays the correct version number - including Patch if appropriate.
      • Dialog displays the current FKB Update version number.
