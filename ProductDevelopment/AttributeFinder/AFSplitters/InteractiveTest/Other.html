  4. Split Attributes Using Tokens

    • Split string into parts using the following delimiting character edit box is editable.
    • Balloon help is functional.
    • Create each part as a sub-attribute radio button is selectable.
      • Selection activates the Sub-attribute name expression edit box.
        • Sub-attribute name expression edit box is editable.
      • Balloon help is functional.
    • Create following sub-attributes radio button is selectable.
      • Selection activates corresponding list box and buttons.
      • Balloon help is functional.
      • Add button displays Specify Sub-Attribute name & token expression dialog.

        Specify Sub-Attribute name & token expression

        • Sub-Attribute Name edit box is editable.
        • An exception is raised when using invalid characters in the name of the sub-attribute. ("a-b" for example)
        • An exception is raised when using invalid characters in the token edit box. ("%First%" for example)
        • Token Expression edit box is editable.
        • OK button saves changes and exits the dialog.
        • Changes made are persistent.
        • Cancel button ignores changes and exits the dialog.
        • X button ignores changes and exits the dialog.
      • Highlighting an existing replace string list box entry activates Remove and Modify buttons.
      • Remove button displays Confirm Delete dialog.

        Confirm Delete Dialog

        • YES button deletes the replacement string and exits the dialog.
        • NO button ignores delete request and exits dialog.
      • Modify button displays a pre-populated Specify Sub-Attribute name & token expression dialog.
        • Changes made are persistent.
      • Graphical UP arrow button moves the selected item up in the list box.
      • Graphical DOWN arrow button moves the selected item down in the list box.
      • Graphical UP and DOWN arrow buttons become inactive when reaching the top or bottom of the list.
    • OK button saves changes and exits dialog.
      • An error message is displayed if an invalid delimiter value is specified.
    • Cancel button ignores changes and exits the dialog.
    • X button ignores changes and exits the dialog.

  5. Split Date and/or Time

    • Configure button displays Configure Split Date and/or Time dialog.
    • Number (i.e. 8) radio button is selectable.
    • Name (i.e. August) radio button is selectable.
    • Four Digits (i.e. 2004) radio button is selectable.
    • Two Digits (i.e. 04) radio button is selectable.
    • Normal (i.e. Hour=8, AMPM=PM) radio button is selectable.
    • Military (i.e. Hour=20, AMPM not shown) radio button is selectable.
    • Generate default sub-attributes (i.e. Month, Day, Year, Time) check box is checkable.
    • Generate Day of the Week sub-attribute (i.e. Monday) check box is checkable.
    • Generate formatted sub-attribute from specified format string check box is checkable.
      • Selection activates the corresponding edit box.
        • The edit box is editable.
      • Test button displays the formatted date/time for the current GMT date/time in a grayed out edit box.
      • Test button displays <Format text is empty> message if edit box is empty.
    • Interpret Two-Digit Year as:
      • On or after the year radio button is selectable.
      • The associated edit box is enabled only when the radio button is checked.
      • The edit box is editable.
      • The edit box accepts only digit characters.
      • [FlexIDSCore #3540] The edit box requires exactly four digit characters.
      • On or before the current year radio button is selectable.
    • Settings are persistent.
    • OK button saves changes and exits the dialog.
    • Cancel button ignores changes and exits dialog.
    • X button ignores changes and exits dialog.

  8. Split region into content areas

    • Configure button displays Configure Split region into content areas dialog.
    • Filter results
      • Minimum width of areas to be returned edit box is editable.
        • Default value is 3.00.
      • Minimum height of areas to be returned edit box is editable.
        • Default value is 0.50.
      • Include areas based on well OCR'd text with this attribute type: check box is enabled.
        • Associated edit box is editable when the check box is checked.
        • [FlexIDSCore #3171] Associated edit box is not editable when the check box is unchecked.
        • Default value is ProbableText.
      • Include areas based on poorly OCR'd text or un-OCR'd markings with this type: check box is selectable.
        • Associated edit box is editable when the check box is checked.
        • [FlexIDSCore #3171] Associated edit box is not editable when the check box is unchecked.
        • Default value is ProbableHandwriting.
      • Treat text as well OCRed if the character confidence is at least: edit box is editable.
        • Default value is 60.
    • Result properties
      • Exclude pixel content of lines from content areas check box is enabled.
      • If the text in the final area is not OCR'd well, attempt to re-OCR the are with handwriting recognition check box is enabled.
      • Include the area's OCR'd text as a true spatial string in a grandchild attribute check box is enabled.
      • Sub-attribute name: edit box is editable.
        • Default value is Content.
      • Default content text value: edit box is editable.
        • Default value is [unrecognized].

    • OK button saves changes and exits the dialog.
    • Cancel button ignores changes and exits the dialog.
    • X button ignores changes and exits the dialog.
