ID Shield On Demand Testing Checklist

  1. Licensing et cetera

    • Confirm that an ID Shield [TBD] license is needed to run this application.
    • Confirm that licensing is checked when the application opens.

    • Confirm that there is a Start menu shortcut for this application.

  2. Basic functionality
    • Confirm that the application can be opened from the command line.
    • Confirm that a usage message is presented if it is run with the /? flag.
    • Confirm that the application can be run with the /ef flag to specify where to log exceptions.
    • Confirm that the application can be run with a single argument specifying a file to open.

    • Confirm that the File menu contains an Open item.
    • Confirm the presence of an Open toolstrip button.
    • Confirm that the toolstrip button has an associated MRU list and that it works as expected.
    • Confirm that the Save and Commit option is now just a Save option.
    • Confirm that document navigation options are not available.
    • Confirm that the Skip and Tag options are not available.
    • Confirm that the Tab key and the Next item command do not attempt to save and move on when you reach the end of the document.
    • Confirm that when you save, you are prompted to save a redacted output file.
    • Confirm that the default output filename is the original filename with .redacted inserted before the extension.
    • Confirm that you are not able to overwrite the original image.

    • Confirm that the verification and redaction behave like their default configurations except:
      • VerifyAllPages is false
      • VerifyAllItems is false
      • Require types is false
      • Seamless mode is false
      • Slideshow is disabled
      • Redactions will be created for all items except clues
      • Redaction text will be <ExemptionCodes>

    • Confirm that there is an ID Shield On Demand tab in the verification options.
    • Confirm that it contains the following options that determine whether or not to create a VOA file:
      • Do not create data file (default)
      • Prompt for whether to create data file
      • Create a data file
    • Confirm that if there is a pre-existing VOA file, changes will be saved regardless of which option is selected.
    • Confirm that metadata is saved under an OnDemandSession node in the VOA file.

    • Confirm that TIF images can be verified and saved.
    • Confirm that PDF images can be verified and saved.
    • Confirm that multipage images can be verified and saved.
    • With each redaction tool, confirm that redactions can be drawn and rotated.
    • Confirm that auto-OCR works as expected.
    • Confirm that all the toolstrip buttons work as expected.

  3. Bug fixes
    • [FlexIDSCore #4778] Background OCR
      • Prepare some images for verification.
      • Delete any existing USS files to make background OCR work.
      • Start verifying documents and confirm that background OCR is taking place.
      • Select the Word redaction tool. Confirm that background OCR does not restart.
      • Repeatedly click (without dragging the mouse) in the document with the Word redaction tool. Confirm that background OCR does not restart.

    • [FlexIDSCore #4826]
      • Select an image with an associated VOA file.
      • Open the image in IDSOD via command line or Open with... from an Explorer Window context menu.
      • Confirm that the VOA file data are displayed when the image opens.
