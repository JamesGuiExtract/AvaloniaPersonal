ID Shield Testing Checklist
Verification - UI

Redaction: Verify sensitive data

  • Data Window
    • User Interface
      • Title bar
        • Confirm that there is a Data window bar across the top of the data window.
        • Confirm that at the right end of the bar, there are two buttons:
          • A small triangle that detaches the window if double-clicked.
          • A pushpin

      • Menu bar
        • Confirm that the File menu contains the following items:
          • Save
          • Save and Commit (Ctrl+S)
          • Print... (Ctrl+P)
          • Print view...
          • Skip processing
          • Stop processing
        • Confirm that the Edit menu contains the following items:
          • Discard changes
          • Find or redact... (Ctrl+F)
        • Confirm that the View menu contains the following items:
          • Zoom
            • Fit to page (P)
            • Fit to width (W)
            • Zoom in (F7, Ctrl++)
            • Zoom out (F8, Ctrl+-)
            • Zoom previous (R, Alt+Left)
            • Zoom next (Alt+Right)
          • Rotate
            • Rotate counterclockwise (Ctrl+Shift+R)
            • Rotate clockwise (Ctrl+R)
          • Goto page
            • First page (Ctrl+Home)
            • Previous page (PgUp)
            • Page number...
            • Next page (PgDn)
            • Last page (Ctrl+End)
          • Full screen (F11)
        • Confirm that the Tools menu contains the following items:
          • Zoom window (Z)
          • Pan (A)
          • Select redactions and other objects (Esc)
          • Angular redaction (H)
          • Rectangular redaction (H)
          • Word redaction (H)
          • Slideshow
          • Options
        • Confirm that the Help menu contains the following items:
          • ID Shield help
          • About ID Shield

      • Toolstrips
        • Save toolstrip - confirm that all buttons are present with appropriate tooltip text.
          • Save and commit (Ctrl+S)
          • Previous document (Ctrl+Shift+Tab)
          • Next document (Ctrl+Tab)
          • Previous item or page (Shift+Tab)
          • Next item or page (Tab)
        • Exemption code toolstrip - confirm that all buttons are present with appropriate tooltip text.
          • Apply exemption codes (E)
          • Apply last exemption codes (Ctrl+E)
          • Apply tags
            • Confirm that this button is available if there are tags defined in the database.
            • Confirm that this button is available if dynamic tag creation is enabled in the database.
            • Confirm that if neither of the above conditions is true, this button is not available.
        • Slideshow toolstrip - confirm that all buttons are present with appropriate tooltip text.
          • Slideshow play (F5)

      • Confirm the presence of a read-only Document type edit box.
      • Confirm the presence of a read-only Current document edit box.
      • Confirm the presence of a writeable multi-line Comments edit box.

      • Data grid
        • Confirm that there are six columns in the grid view:
          • An unlabeled column - This column contains a checkbox indicating the enabled/disabled state of the data item.
          • Text - This column contains the text of the attribute's value.
          • Category - This column indicates the name of the attribute.
          • Type - This column contains a drop-down list indicating the type of the attribute.
          • Page - This column indicates which page of the document the attribute appears on.
          • Exemptions - This column indicates what exemption codes (if any) have been applied to the data item.
        • Confirm that the rows in the grid view cannot be resized for height.

      • Confirm the presence of a page summary grid with one labeled cell for each page of the current document.

    • Functionality
      • Title bar
        • Confirm that clicking on the pin at the right end of the Data Window title bar toggles the auto-hide feature of the data window pane.
        • Confirm that double-clicking in the Data Window title bar pops it out into a separate window.
        • Confirm that double-clicking in the title bar again returns it to its original location.
        • Confirm that by clicking and dragging, the data window can be docked in the top, bottom, left, or right of the ID Shield window. Confirm that its position is correctly persisted.

      • Menu bar
        • Confirm that all keyboard shortcuts work as expected.
        • File menu
          • Confirm that the Save menu option saves changes without setting the current document to complete in the database.
          • Confirm that Save does not require that commit conditions (e.g., setting exemption codes or viewing all pages) be satisfied.

          • Confirm that Save and commit saves changes and sets the current document to complete in the database.
          • Confirm that it does require that commit conditions be satisfied.
          • Confirm that Save is disabled when viewing a document in the verification history.

          • Confirm that Print... brings up the standard print dialog.
          • Confirm that on printed documents:
            • Enabled data items are redacted with black boxes.
            • Disabled data items are not redacted.

          • Confirm that Print view... prints only the portion of the document currently visible in the image viewer.

          • Confirm that Skip processing closes the current document, sets its file action status to Skipped, and brings up the next document (if any).
          • Confirm that commit conditions are not checked when skipping documents.
          • Confirm that if there are unsaved changes when skipping a document, a save prompt appears and functions as expected.
          • Confirm that Skip processing is disabled when viewing a document in the verification history.

          • Confirm that Stop processing closes the current document, sets its file action status back to Pending, and stops processing.
          • Confirm that commit conditions are not checked when stopping processing.
          • Confirm that if there are unsaved changes when stopping, a save prompt appears and functions as expected.
          • Confirm that if no document is currently open, Stop processing brings up a prompt advising the user to use the FAM to stop processing.

        • Edit menu
          • Confirm that Discard changes discards all unsaved changes.
          • Confirm that it does not undo changes that were saved but not committed.

          • Confirm that Find or redact... brings up the Find or redact dialog.
          • Confirm that Ctrl+F also brings up the Find or redact dialog.
          • The Find or redact dialog is further tested in the General section of the verification checklist.

        • View menu
          • For each menu item with an analogous image viewer function
            • Confirm that the tool or operation selected functions correctly.
            • Confirm that the listed shortcut keys function correctly, regardless of the pinned/unpinned state of the data window, and regardless of which ID Shield window (e.g., image viewer or data window) has focus.
          • Confirm that Full screen toggles fullscreen mode.
          • Confirm that F11 performs the same function.

        • Tools menu
          • For each menu item with an analogous image viewer function
            • Confirm that the tool or operation selected functions correctly.
            • Confirm that the listed shortcut keys function correctly, regardless of the pinned/unpinned state of the data window, and regardless of which ID Shield window (e.g., image viewer or data window) has focus.
          • Confirm that the Slideshow play item works as expected.
            • If the verification task is set up to require the run key, you should not be able to start the slideshow with the menu item.
            • Confirm that PgDn does not stop the slideshow.
            • Confirm that all other keypresses/mouse clicks (except for F5) do stop the slideshow.
            • Confirm that the slideshow options are not displayed if the slideshow is disabled in the task configuration.
          • Confirm that Options... brings up the Options dialog.
          • Confirm that verification options are honored even for the first verified image.

          • General tab
            • Confirm that the AutoPan settings are read from and written to Extract.Redaction.Verification.dll.config located in the appropriate user's AppData folder.
            • Confirm that the options can be configured to select the pan tool, the selection tool, or the zoom tool after creating a manual redaction, or (by unchecking the related box) to disable the automatic tool change. Confirm that each option works properly.
            • Confirm that the initial configuration is to select the selection tool after creating a manual redaction.

            • Confirm that there is a checkbox to OCR text in the background.
            • Confirm that the box is checked by default and that when it is checked, an associated method dropdown list is enabled.
            • Confirm that the dropdown list options are:
              • Accurate
              • Balanced (the default)
              • Fast
            • Confirm no USS files are saved when files are automatically OCRed.
            • Confirm that OCR data are retained for documents in the navigation history.
            • Confirm that preview highlights for the word redaction tool are disabled for documents with no OCR data.
            • Confirm that once the current page has been OCRed, word redaction tool preview highlights become available.
            • On a multipage image, confirm that background OCR starts for a new page as soon as the user navigates to that page even if OCR has not finished for the previous page.
            • Open the Find dialog for a multipage document that requires automatic OCR.
              • Confirm that a dialog with a progress bar is displayed notifying the user that they will need to wait until OCR is complete for the entire document.
              • Confirm that if the user waits, the progress dialog will automatically close before the Find dialog appears.
              • Confirm that if the user closes the progress dialog, the Find dialog will not appear.
            • Confirm that the status bar indicates
              • Whether or not OCR data are available.
              • OCR progress for the current page.
              • That word zones are currently being loaded (if the page already has OCR data).
          • Slideshow tab
            • Confirm that there is a spin control for the number of seconds the slideshow should display each page.
            • Confirm that the slideshow does, in fact, display each page for the specified number of seconds.
            • Confirm that the spin control does not allow non-integer input.
            • Confirm that the Automatically start slideshow checkbox works as expected.
            • Confirm that the dropdown box to specify the run key works as expected.
            • Confirm that the slideshow tab is not displayed if the slideshow is disabled in the task configuration.
          • Zoom tab
            • Confirm that the Automatic Zoom settings are read from and written to Extract.Redaction.Verification.dll.config located in the appropriate user's AppData folder.
            • Confirm that with Auto Zoom enabled, all selected redactions on the current page are within view.
            • Confirm that with a higher Auto Zoom setting, the view is further out from the redactions.
            • Confirm that selecting a single redaction in the Image Viewer does not change the zoom.
            • Confirm that drawing a manual redaction does not change the zoom.
            • Confirm that with Auto Zoom disabled, the zoom does not change when redactions are selected.
            • Confirm that the zoom level slider matches the rest of the window regardless of which Windows theme you are using.

        • Help menu
          • Confirm that ID Shield help brings up the ID Shield help file.
          • Confirm that About ID Shield brings up the about box with the appropriate copyright and version information.

      • Toolstrips
        • Confirm that toolstrips can be clicked and dragged and rearranged. Confirm that they can be docked in different areas of the ID Shield window, and that their positions are correctly persisted.
        • Confirm that all keyboard shortcuts function as expected.

        • Save toolstrip
          • Confirm that the Save and commit toolstrip button works as expected.
          • Confirm that the Save and commit button is disabled when viewing a document in the verification history.

          • Confirm that the Previous document button is disabled when viewing the first document or the earliest document in the verification history.
          • Confirm that the Previous document button navigates to previously saved and committed documents.
          • Confirm that the Next document button is enabled even when viewing the current document.
          • Confirm that the Next document button navigates forward through the history queue.
          • Confirm that the Next document button prompts you to save before navigating forward from the current document if the verification task is configured to do so.
          • Confirm that the default size of the history queue is 20 documents.
          • Confirm that the size of the history is controlled by the registry setting at HKCU/Software/ExtractSystems/ AttributeFinder/IndustrySpecific/Redaction/ RedactionCustomComponents/IDShield/NumPreviousDocsToQueue.
          • Confirm that the user is prompted to save when navigating away from a document with unsaved changes.
          • Confirm that information about viewed pages is saved during history navigation.
          • Confirm that information about viewed redactions is saved during history navigation.

          • Confirm that the Previous item button is disabled when viewing the first redaction on the first document.
          • Confirm that the Previous item button navigates to the previous redaction in the data grid.
          • Confirm that if Review All Pages is selected and the user is viewing the first redaction on the current page, the Previous item button navigates to the previous unviewed page.
          • Confirm that the Next item button is disabled only when no document is open.
          • Confirm that the Next item button navigates to the next redaction in the data grid.
          • Confirm that if Review All Pages is selected and the user is viewing the last redaction on the current page, the Next item button navigates to the next unviewed page.
          • Confirm that if the user is viewing the last redaction or page (depending on settings) of the current document, the Next redaction button will prompt the user to save and move on to the next document if the verification task is configured to do so.
          • Confirm that the Next/Previous document and Next/Previous item buttons function symmetrically.

        • Exemption code toolstrip
          • Confirm that the Apply exemption codes button is disabled if no redaction is selected.
          • Confirm that the Apply exemption codes button brings up the exemption codes dialog.
          • Confirm that the Apply last exemption codes button is disabled if no redaction is selected or no exemption codes have been applied.
          • Confirm that the Apply last exemption codes button applies the most recently applied exemption code(s) to the currently selected redaction(s).

          • Exemption code dialog
            • Confirm that the exemption code dialog contains a drop-down box with options for different exemption code categories.
            • Confirm that the options in the drop-down list come from the filenames of the .xml files located in C:\Program Files\Extract Systems\IDShield\ExemptionCodes.
            • Confirm that the dialog contains a multi-line read-only edit box with a detailed explanation of the currently selected exemption code.
            • Confirm that there is a check-box which enables an edit box for a user-specified exemption code.
            • Confirm that there is a read-only edit box that correctly displays the selected exemption code(s).
            • Confirm that multiple exemption codes can be selected and applied.
            • Confirm that the dialog is keyboard-navigable.
            • Confirm that the Clear All button works as expected.
            • Confirm that the Apply Last button works as expected.
            • Confirm that the OK, Cancel, and X buttons each work as expected.
            • Confirm that the exemption code dialog is modal to the verification UI.
            • Confirm that exemption codes can be both added and removed from redactions that are toggled off.

          • Confirm that the Apply tags button brings up a list of available tags.
          • Confirm that the list is populated from a query of the database at the time of the click (i.e., if the list of available tags has changed since the document was loaded, that change should be reflected in the tag list).
          • Confirm that hovering over a tag in the list displays the tag's description as a tooltip.
          • Confirm that multiple tags can be applied to a single document.
          • Confirm that the list of tags remains open after selecting a tag.
          • Confirm that tags are applied to the document immediately after they are selected (not when the document is saved).
          • Confirm that if and only if dynamic tag creation is enabled in the database, the list of tags includes an Apply new tag... option.
          • Confirm that selecting the Apply new tag option brings up the Add and apply tag dialog.
          • Confirm that new tag names are correctly validated.
          • Confirm that tag descriptions are not validated.
          • Confirm that the OK, Cancel, and X buttons each work as expected.
          • Confirm that new tags are correctly created and applied.

      • Slideshow toolstrip
        • Confirm that the Start and Stop toolstrip buttons work as expected.
        • Confirm that the Stop button is disabled when the slideshow is not running.
        • Confirm that the Start button is disabled when the slideshow is running.
        • Confirm that when the slideshow runs, a blue progress bar is displayed at the bottom of the Image Viewer window.
        • Confirm that when you stop the slideshow by interacting with the UI (e.g., by pressing a key or clicking the mouse), the text SLIDESHOW STOPPED is displayed prominently.

        • Confirm that the slideshow toolstrip is not displayed if the slideshow is disabled in the task configuration.

      • Confirm that the Current document edit box correctly displays the full path to the current document.

      • Comments

          • Confirm that changes made while viewing documents in the history are correctly saved.
          • Confirm that ID Shield keyboard shortcuts are disabled while editing comments.
          • Confirm that standard text-editing keyboard shortcuts (e.g., Ctrl+X, Ctrl+V, etc.) work as expected while editing comments.

      • Data Grid

        • Confirm that viewed redactions are displayed in normal font.
        • Confirm that redactions are listed in the Data Grid in sorted order from top to bottom and left to right.

        • Confirm that the Delete key works as expected after selecting an item in the grid view.
        • Confirm that a single Tab is always sufficient to navigate to the next row in the Data Grid.

        • Toggling redactions
          • Confirm that double-clicking the redaction checkbox in the Data Grid updates the fill color appropriately.
          • Confirm that selecting a redaction and pressing SPACE toggles the redaction state and sets the dirty flag.

          • Confirm that all of the selected redactions are now toggled off.

          • Confirm that if types are not required, the Type drop-down list does include a blank entry.
          • With an attribute selected, confirm that T is a supported shortcut to activate the Type drop-down list.
          • Confirm that ID Shield keyboard shortcuts are disabled while the Type combo box is dropped down.

        • New redactions
          • Confirm that new manual redactions appear in the data grid based on page location.
          • Confirm that new manual redactions are selected by default.
          • Confirm that the default type of a new manual redaction is read from the .ini file.

      • Confirm that the current page has a blue background.
      • Confirm that a vertical scrollbar is presented for a document with many pages and that it autoscrolls as the user tabs through the pages.

      • Confirm that the data window status bar displays the Seconds verifying this document and Average pages/hour.
      • Confirm that the displayed numbers are correct for the current verification session.
