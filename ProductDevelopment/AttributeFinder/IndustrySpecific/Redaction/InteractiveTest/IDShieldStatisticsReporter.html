ID Shield Statistics Reporter Testing Checklist

  1. User Interface

    • Confirm that the ID Shield Statistics Reporter includes tabs for Feedback data, Analyze and Review.
    • Confirm that the utility has a non-trivial icon.
    • Confirm that the utility can be resized using the resize grip at the bottom right of the window.
    • Confirm that its size and location are persisted.

    • Feedback data tab
      • Confirm that the Feedback data tab includes a group box for File and folder options.
      • Confirm that the group box includes an edit box for Select feedback data folder from which to analyze results.
      • Confirm that the edit box can be modified manually.
      • Confirm that the group box includes a ... (browse) button.
      • Confirm that clicking the browse button displays a Folder browser dialog.
      • Confirm that the folder selected in the dialog is displayed in the edit box.
      • Confirm that cancelling the folder browser dialog retains the previous text in the edit box.
      • Confirm that the feedback folder is persisted across sessions.

      • Confirm that the group box contains an Advanced options... button.
      • Confirm that the button brings up the Advanced Options dialog.
      • Confirm that the dialog contains an ID Shield data file with found redactions edit box with a default value of <SourceDocName>.found.voa.
      • Confirm that the dialog contains an ID Shield data file with expected redactions edit box with a default value of <SourceDocName>.expected.voa.
      • Confirm that each edit box has an associated Tag selector and ... (browse) button, and that the buttons work as expected.
      • Confirm that settings are correctly persisted.

    • Analyze tab
      • Confirm that the Analyze tab includes a group box for Analysis options.
      • Confirm that the group box includes a combo box for Select analysis type.
      • Confirm that the following combo box options are supported:
        • Automated redaction
        • Hybrid
        • Standard verification
      • Confirm that the group box includes a check box for Limit data types to be tested to.
      • Confirm that the group box includes an associated edit box which is enabled only when the box is checked.
      • Confirm that if the box is checked, a value must be entered in the text box in order to run the analysis.
      • Confirm that if the RDT is licensed, the label indicates that types may be separated by either a comma or a pipe, and if the RDT is not license, only the comma is mentioned.
      • Confirm that the group box includes the following Default to redacting the following check boxes:
        • High confidence sensitive data
        • Medium confidence sensitive data
        • Low confidence sensitive data
        • Manually identified sensitive data
      • Confirm that the first three are checked by default.
      • Confirm that the group box includes an Automated file selection condition... button which is enabled only with a Hybrid or Automated analysis type.
      • Confirm that clicking the Automated file selection condition... button displays a Configure automated condition dialog.
        • Confirm that the dialog contains a The ID Shield data file contains checkbox.
        • Confirm that there is an associated dropdown box with options any, at least one of each, none, and only any.
        • Confirm that only when the box is checked, five checkboxes are enabled below:
          • High confidence sensitive data
          • Medium confidence sensitive data
          • Low confidence sensitive data
          • Manually identified sensitive data
          • Clues potentially identifying sensitive data
        • Confirm the presence of an Indicates one of the following as the document type checkbox.
        • Confirm that only when that box is checked, an associated Select... button is enabled.
        • Confirm that the Select button brings up the Select Document Types dialog and that the dialog works as expected.
        • Confirm that the document types selected are displayed in a list box in the conditional dialog.
        • Confirm that condition settings are persistent.
      • Confirm that the group box includes an Verification file selection condition... button which is enabled only with a Hybrid or Standard analysis type.
      • Confirm that clicking the Verification file selection condition... button displays a Configure verification condition dialog.
        • Confirm that the dialog is functionally the same as the automated condition dialog
        • Confirm that by default, the verification condition is configured to verify HCData, MCData, LCData, and Clues.
      • Confirm that the Analyze tab includes an Analyze button.
      • Confirm that an hourglass cursor is displayed while the analysis process is executing.
      • Confirm that a Test Result dialog with the number of successfully completed test cases appears when analysis is complete.
      • Confirm that the application automatically moves to the Review tab when analysis is complete.

    • Review tab
      • Confirm that the Review tab includes a combo box for Review analysis from.
      • Confirm that the combo box contains one entry for each previously executed analysis.
      • Confirm that the analysis entries are separate folders under the previously defined feedback data folder.
      • Confirm that the format of the folder names is Analysis - MM-DD-YYYY - HH.MM.SS.
      • Confirm that the combo box entries have removed the Analysis - text.
      • Confirm that the Review tab includes a Statistics tab.

      • Confirm that the Statistics tab includes a multi-line edit box that includes the following information:
        • Number of files tested
        • Number of pages tested
        • Number of files containing sensitive data
        • Number of pages containing sensitive data
        • Number of sensitive data items
        • Number of documents with overlapping sensitive data items
        • Analysis type
        • Manually review... (Standard or Hybrid only)
        • Automatically redact... (Automated or Hybrid only)
        • Default to redacting the following...
        • Number of sensitive data items redacted after processing
        • Number of false positives in redacted images (Automated or Hybrid only)
        • Ratio of correctly redacted items to false positives (Automated or Hybrid only)
        • Automated redaction efficiency:
          • Number of automatically redacted files
          • Number of sensitive data items in automatically redacted files
          • Number of correct redactions
          • Number of over-redactions
          • Number of under-redactions
          • Number of false positives
          • Ratio of correctly redacted items to false positives
        • OR Verification efficiency: (If standard work-flow is selected)
          • Number of files selected for review
          • Number of correct redactions presented to reviewers
          • Number of over-redactions presented to reviewers
          • Number of under-redactions presented to reviewers
          • Number of false positives presented to reviewers
          • Ratio of correctly redacted items to false positives
        • OR (If hybrid work-flow is selected) Verification efficiency:
          • Number of files selected for review
          • Number of sensitive data items in files presented for review
          • Number of correct redactions presented to reviewers
          • Number of over-redactions presented to reviewers
          • Number of under-redactions presented to reviewers
          • Number of false positives presented to reviewers
          • Ratio of correctly redacted items to false positives
        • AND Automated redaction efficiency:
          • Number of automatically redacted files
          • Number of sensitive data items in automatically redacted files
          • Number of correct redactions
          • Number of over-redactions
          • Number of under-redactions
          • Number of false positives
          • Ratio of correctly redacted items to false positives
        • Counts of documents for each found document type

      • Confirm that the Review tab includes a File lists tab.
      • Confirm that the File lists tab includes a Select file list combo box with the following choices:
        • Various DocType_*.txt
        • FailedTestCases.txt
        • Files_AtLeastOneFalsePositive.txt
        • Files_AtLeastOneFoundClue.txt
        • Files_AtLeastOneFoundHCData.txt
        • Files_AtLeastOneFoundLCData.txt
        • Files_AtLeastOneFoundMCData.txt
        • Files_AtLeastOneFoundRedaction.txt
        • Files_NoFoundRedactions.txt
        • InsensitiveFiles_All.txt
        • InsensitiveFiles_SelectedForReview.txt (Standard or Hybrid only)
        • SensitiveFiles_All.txt
        • SensitiveFiles_AtLeastOneCorrectRedaction.txt
        • SensitiveFiles_AtLeastOneMissedRedaction.txt
        • SensitiveFiles_AtLeastOneOverRedaction.txt
        • SensitiveFiles_AtLeastOneUnderRedaction.txt
        • SensitiveFiles_SelectedForReview.txt (Standard or Hybrid only)
      • Confirm that the File lists tab includes a multi-line edit box for Entries in selected file list.
      • Confirm that double-clicking on a line in the edit box displays the selected image in the Image Viewer plus the associated TestOutput VOA file in the VOA Viewer.

  2. Basic functionality
    • Prepare a set of images for verification.
    • Collect feedback during verification.
    • Confirm that an ID Shield Client license is sufficient to run the IDSSR utility.
    • In all of the following cases, confirm that the correct number of successful test cases is reported.
    • Confirm that Automated redaction analysis works as expected when limiting the test to either a single data type or multiple data types.
    • Confirm that Hybrid analysis works as expected when redacting only HC and MC data.
    • Confirm that Standard verification analysis works as expected.
    • Confirm that previous results can be checked via the Review tab.
    • Confirm that appropriate files are provided in the File lists tab.
    • Examine some test output .voas:
      • Confirm that the source VOA file is indicated in the attribute type
        • Expected - Misses and Correct
        • Found - Over-redactions, Under-redactions and False positives
      • Confirm that a single found redaction is not counted as both an over-redaction and an under-redaction for the same expected attribute.
      • Confirm that the following statistics are reported for a single sensitive data item:
        • 1 redaction, too large = 1 correct, 1 over-redaction
        • 2 redactions, 1 too large = 1 correct, 1 over-redaction
        • 2 redactions, both too large = 1 correct, 2 over-redactions
        • 2 redactions, both correct = 1 correct
        • 2 redactions, 1 too little = 1 correct, 1 under-redaction
      • Confirm that the following statistics are reported for a page with two sensitive data items and one redaction covering the entire page:
        • 2 correct, 2 over-redactions
