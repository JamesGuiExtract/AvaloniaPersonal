FAM Network Manager Testing Checklist

  1. Licensing

    • Confirm that a FLEX Index or ID Shield server license is required to run this utility.
    • Confirm that licensing is checked when the utility opens.

  2. UI

    • Window
      • Confirm that the title bar includes the name of the application and the name of the currently open file, if any.
      • Confirm that there are reasonable limits on resizing the window.
      • Confirm that the title bar reflects the dirty/non-dirty state of the currently open file.

    • File menu
      • Confirm that the File menu contains the following items:
        • New - Ctrl+N
        • Open... - Ctrl+O
        • Save... - Ctrl+S
        • Save As
        • Exit - Alt+F4
      • Confirm that only Open and Exit are initially enabled.

      • Confirm that New clears any currently-open configuration.
      • Confirm that Ctrl+N performs the same function.

      • Confirm that Open brings up the Open dialog.
      • Confirm that the Open dialog defaults to an .fnm file type.
      • Confirm that the Open dialog works as expected.
      • Confirm that Ctrl+O performs the same function.

      • Confirm that Save works as expected.
      • Confirm that Ctrl+S performs the same function.

      • Confirm that Save As works as expected.

      • Confirm that Exit works as expected.
      • Confirm that Alt+F4 performs the same function.

    • Tools menu
      • Confirm that the Tools menu contains the following item:
        • Options...
      • Confirm that it is initially enabled.

      • Confirm that Options brings up the Refresh Time dialog.
      • Confirm that the dialog contains a text box, an OK button, and a Cancel button.
      • Confirm that the buttons work as expected.
      • Confirm that the dialog validates the text box for positive integers.

    • Help menu
      • Confirm that the Help menu contains the following item:
        • About...
      • Confirm that it is initially enabled.

      • Confirm that About brings up the FAM Network Manager About box.
      • Confirm that the About box has the expected version number and copyright information.

    • Toolstrips
      • Confirm that both of the following toolstrips exist, that they contain the listed buttons, and that the tooltip text for each button contains the name of the tool and the appropriate keyboard shortcut:
        • Open toolstrip:
          • Open file (Ctrl+O)
          • Save file (Ctrl+S)
          • Add machine (F8)
          • Remove machine (Del)
          • Edit machine(s) group and name (F2)
          • Group dropdown list

        • Start toolstrip:
          • Start service
          • Stop service
          • Service dropdown list
          • Modify service database
          • Auto refresh status
          • Refresh status (F5)

      • Confirm that the various buttons are enabled only at appropriate times.
      • Confirm that when buttons/menu items are disabled, their associated keyboard shortcuts are also disabled.
      • Confirm that toolstrip items with analogs in the File menu function as expected.

      • Confirm that clicking on the Add machine button brings up the Add Machine dialog.
      • Confirm that the Add Machine dialog contains:
        • A Machine name text box with an input limit of 256 characters.
        • An associated, disabled browse button.
        • A Group name combo box with an input limit of 40 characters.
        • An OK button.
        • A Cancel button.
      • Confirm that all controls work as expected.
      • Confirm that pressing F8 performs the same function as clicking the Add machine button.

      • Confirm that the Remove machine button is enabled when one or more machines is selected in the data grid.
      • Confirm that pressing the Remove machine button when one or more machines is selected removes the machine(s) from the grid.

      • Confirm that the Edit machine(s) group and name button is enabled when one or more machines is selected in the data grid.
      • Confirm that pressing the Edit button with one or more machines selected brings up the Modify Machine Name Or Group dialog.
      • Confirm that the Edit dialog contains:
        • A Machine name text box with an input limit of 256 characters.
        • An associated, disabled browse button.
        • A Group name combo box with an input limit of 40 characters.
        • An OK button.
        • A Cancel button.
      • Confirm that if one machine is selected, the Machine name and Group name text boxes are writeable and prepopulated with the current values for the selected machine.
      • Confirm that if multiple machines are selected, the Machine name text box contains a comma-delimited list of machines and is read-only.
      • Confirm that if multiple machines are selected and they are all part of the same group, the group name combo box is writeable and populated with the current value.
      • Confirm that if multiple machines are selected and they are part of different groups, the group name combo box is writeable and initially blank.
      • Confirm that pressing F2 performs the same function as hitting the Edit button.

      • Confirm that the Group name drop down list contains a list of all group names currently in use.

      • Confirm that the Start service button is always enabled.
      • Confirm that the Stop service button is always enabled.
      • Confirm that the Service dropdown list includes entries for FAM, FDRS, and Both.

      • Confirm that the Modify service database button is enabled only when exactly one machine is selected in the data grid.
      • Confirm that the Modify service database button brings up the SQLCEDBEditor application with the Open and Close commands hidden.
      • Confirm that the SQLCEDBEditor otherwise appears as expected.

      • Confirm that the Auto refresh button is always enabled.
      • Confirm that Auto refresh functions as a toggle.

      • Confirm that the Refresh button is disabled when the Auto refresh feature is turned on.
      • Confirm that the refresh button is otherwise always enabled.
      • Confirm that the refresh button refreshes the data from each of the machines in the data grid.

      • Confirm that toolstrips can be rearranged by clicking and dragging them.
      • Confirm that toolstrip locations are persisted.
      • Confirm that toolstrip locations can be reset using the /reset flag on the command line.

    • Grid
      • Confirm that the data grid includes the following columns:
        • Machine
        • Group
        • FAM Status
        • FDRS Status
        • CPU
      • Confirm that the data grid can be sorted by any of those columns.
      • Confirm that the grid columns and rows can be resized in a reasonable way.

  3. Functionality

    • New, open, save, etc
      • Confirm that Open allows you to open saved .fnm files.
      • Confirm that all rows immediately refresh on Open.
      • Confirm that an .fnm file can be dropped onto the network manager to open it.

      • Confirm that New clears the current grid.
      • Confirm that you are prompted to save if changes have been made.

      • Confirm that Save saves the current state or brings up the Save As dialog if the current file hasn't been saved yet.

      • Confirm that save prompts are provided when closing the network manager if changes have been made.
      • Confirm that a Please Wait dialog appears when closing the network manager.
      • Confirm that the wait dialog is modal.
      • Confirm that the wait dialog cannot be closed prematurely.

    • Adding machines
      • Confirm that the Add machines dialog requires that a machine name be specified.
      • Confirm that leading and trailing whitespace is trimmed from the group name.

      • Add a machine that does not exist with a group name Zombies. Confirm that:
        • A red error icon appears in the row header.
        • Double-clicking in the row causes the exception to be displayed.
        • Double-clicking elsewhere does not cause the exception to be displayed.
        • A plausible error is given as the reason why you are not connected.
        • Confirm that the FAM and FDRS status are both "Not Available" and the CPU usage is "Counter N/A".

      • Add Lisa or another machine that exists, but does not have the service installed. Confirm that:
        • "Zombies" is included in the group name drop-down in the Add Machine dialog.
        • The FAM status and FDRS status are both "Not Available"
        • The CPU column correctly reports the current CPU usage.

      • Add a machine that does exist and that has the FAM service installed. Confirm that:
        • FAM status is either "Stopped" or "Running".
        • FDRS status is "Not Available"
        • The CPU column correctly reports the current CPU usage.

      • Add a machine that does exist and that has the FDRS service installed. Confirm that:
        • FAM status is "Not Available"
        • FDRS status is either "Stopped" or "Running".
        • The CPU column correctly reports the current CPU usage.

      • Add a machine that does exist and that has both services installed. Confirm that:
        • FAM and FDRS status are both either "Stopped" or "Running".
        • The CPU column correctly reports the current CPU usage.

      • Add many machines and confirm that the data grid scrolls down to ensure that the newly-added row is always visible.

    • Modifying machine or group names
      • Confirm that machine names are required to be defined.
      • Confirm that leading and trailing whitespace is trimmed from group names.

      • Select one machine and open the Modify dialog.
        • Confirm that the machine name and group name are prepopulated.
        • Modify the machine name to point to an existing machine.
        • Modify the group name.
        • Confirm that the data grid row is refreshed immediately when you hit OK in the dialog.

      • Select two machines and open the Modify dialog.
        • Confirm that modifying the group name changes the group for both machines.

    • Group names and filtering
      • Add several machines. Put some in group Robots, some in group Monkeys, and one in group Cthulu.
      • Confirm that the group name dropdown list contains entries for Monkeys, Robots, Cthulu, and a blank entry.
      • Confirm that selecting one of those entries filters the data grid so that only members of the relevant group are displayed.
      • Select the machine in group Cthulu and delete it from the grid.
      • Confirm that the dropdown list has been updated to reflect that change.

      • Select the blank row in the dropdown to display all entries.
      • Note the current CPU percentages for the machines in the group Monkeys.
      • Filter to show only Robots.
      • Refresh.
      • Remove the filter.
      • Confirm that the Monkeys were not refreshed.

      • Filter to show only Monkeys.
      • Select all rows.
      • Modify them to be in group Team Sunshine.
      • Remove the filter.
      • Confirm that Robots were not edited.

    • Starting and stopping the services
      • Confirm that you are prompted before starting and stopping the services.
      • Confirm that you can start/stop either or both of the services as specified.
      • Confirm that attempting to start a service that is not installed does nothing.
      • Confirm that the selected rows are refreshed when the start or stop command is sent.
      • Confirm that the services are started/stopped on all currently-selected machines.

    • Editing the database
      • In the following situations, confirm that a reasonable error message is displayed and that the editor appears or doesn't as expected:
        • Select a nonexistant machine and hit the Modify database button.
        • Select a machine on which the FAM service is not installed and hit the Modify database button.
        • Select a machine with a read-only database file and hit the Modify database button.
        • Select a machine with a corrupt FAMService.sdf file and hit the Modify database button.
        • While monitoring machine A, log on to A and open the service database, then try to edit the database from the network manager.

      • Confirm that the database editor works as expected.

    • Refresh and auto-refresh
      • Confirm that the auto-refresh refreshes all currently visible rows with a frequency specified under Tools > Options.
      • Confirm that the manual refresh refreshes all currently visible rows.

  4. Bug fixes

    • [DotNetRC #545] Behavior with failed open
      • Configure the network manager to monitor some machines.
      • Try to open a file that is not an .fnm file.
      • After dismissing the error message, confirm that your current configuration has not been cleared.

    • [DotNetRC #546] Group list updating
      • Configure the manager to monitor a machine in group Zombies.
      • Open an .fnm file that has a list of machines, none of which are in the Zombies group.
      • Confirm that Zombies is no longer included in the group name dropdown list.

    • [DotNetRC #549, #557] Reconnecting after reboot
      • Open the network manager on machine A. Configure it to monitor machine B and set it to autorefresh.
      • Log on to B and reboot.
      • When B starts up again, start the FAM service.
      • On A, confirm that the network manager successfully reconnected and is displaying appropriate information about B.
