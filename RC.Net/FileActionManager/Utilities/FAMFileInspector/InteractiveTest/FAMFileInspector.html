FAM File Inspector Utility
Testing Checklist

  1. Basic Operation

    • Confirm that FAM File Inspector is available as a Start Menu item under FLEX Index - Utilities.
    • Confirm that FAM File Inspector is available as a Start Menu item under ID Shield - Utilities.
    • Confirm that FAM File Inspector is available as a Start Menu item under LabDE - Utilities.
    • Confirm that FAM File Inspector requires only a Client license.
      • Confirm that a meaningful exception is displayed if an appropriate license is not present.
    • Confirm that double-clicking on FAMFileInspector.exe displays a the usual Select database dialog.
      • Confirm that the FAM DB Admin password is not required to connect to the selected database.

  2. Command-Line Operation

    • Confirm that FAM File Inspector.exe /? displays a usage message.
    • [DotNetRC #1097, #1101, #1103, JIRA #12134]Confirm that parameters and options are available as follows.
      • ServerName
      • DatabaseName
      • /action <actionName>
      • /status <statusName>
      • /query <queryFileName>
      • /directory <directory>
      • /filefilter <filefilter>
      • /r
      • /filecount [top|bottom|random] <count>

    • Confirm that ServerName and DatabaseName are required if not using the /directory option.
    • Confirm that a meaningful error message is presented if an incorrect action name is used.
    • Confirm that a meaningful error message is presented if an incorrect status is used.
    • Confirm that a /query argument can be used without /action and /status.
    • Confirm that the parameter following /query is a fully-qualified path to a text file containing a SQL query.
    • Confirm that a meaningful error message is presented if the query is not fully formed.
    • Confirm that /filecount can be used with each of the following options.
      • <none>
      • top
      • bottom
      • random
    • Confirm that /directory can be used with /filefilter.
    • Confirm that /directory can be used without /filefilter.
    • Confirm that /directory can be used with /r.
    • Confirm that /directory can be used without /r.
