DataEntryPrompt - Testing Checklist

NOTE: This utility uses DataEntryPrompt.exe.config.

  1. Operation and Basic Errors

    • [DotNet #844] Confirm that DataEntryPrompt.exe requires a license file.
    • Confirm that a meaningful error is presented if DataEntryPrompt.exe.config is not present.

  2. Config File Usage

    • Confirm that the OK button label depends on the OKButtonLabel field.
    • Confirm that the Cancel button label depends on the CancelButtonLabel field.

    • Confirm that the Field1 field specifies the label for the corresponding edit box.
      • Confirm that the field is not presented if the config file definition is not present.
    • Confirm that the Field1Validation field specifies the validation requirement for the corresponding edit box.
      • Confirm that no validation is performed if the config file definition is not present.
    • Repeat the above tests for the following fields:
      • Field2
      • Field3
      • Field4
      • Field5
      • Field6
      • Field7
      • Field8
      • Field9
      • Field10

    • Confirm that the Title field specifies the window caption.
    • Confirm that the KeepOpen field specifies whether or not the utility stays open after the user clicks the OK or Cancel button.
    • Confirm that the DataFileName field specifies the name of the output file that stores the entered data in a comma-separated format.
    • Confirm that the PostEntryCommandLine field specifies the name of an application to be run after the user clicks the OK button.
    • Confirm that the PostEntryCommandLineArguments field specifies the command-line arguments associated with the PostEntryCommandLine application.
