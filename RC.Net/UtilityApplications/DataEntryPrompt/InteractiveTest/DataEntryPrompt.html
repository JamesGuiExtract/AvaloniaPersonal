DataEntryPrompt - Testing Checklist

NOTE: This utility uses DataEntryPrompt.exe.config from Demo_RedactionGame.

  1. Operation and Basic Errors

    • [DotNet #844] Confirm that DataEntryPrompt.exe requires a license file.
    • Confirm that a meaningful error is presented if DataEntryPrompt.exe.config is not present.

  2. Redaction Game

    • User Interface
      • Copy DataEntryPrompt.exe.config from I:\Common\Engineering\ProductReleases\FlexIndex\External\Current\9.0.0.200\OtherDemos\Demo_RedactionGame\Misc to the CommonComponents folder.
      • Confirm that the window caption is ID Shield Redaction Game.
      • Confirm that the following fields are defined:
        • FirstName
        • LastName
        • Company
        • Title
        • Email
        • PhoneNumber
        • Comments
      • Confirm that a red error icon is present next to each empty field (except for Comments).
    • Field Validation
      • Confirm that FirstName, LastName, Company and Title require any alphanumeric text to be entered.
      • Confirm that the Email field requires a text string configured as an email address to be entered.
      • Confirm that the following strings are valid for PhoneNumber.
        • (608) 216-7950
        • 608 216 7950
        • 608.216.7950
        • 608-216-7950
        • 6082167950
      • Confirm that the following strings are invalid for PhoneNumber.
        • 608 216 79501
        • (608 216-7950
        • 011.216.7950
    • Other Configuration
      • Confirm that the Title field specifies the window caption.
      • Confirm that the OKButtonLabel field specifies the text on the OK button.
      • Confirm that the CancelButtonLabel field specifies the text on the Cancel button.
      • Confirm that the DataFileName field specifies the name of the output file that stores the entered data in a comma-separated format.
      • Confirm that the PostEntryCommandLine field specifies the name of an application to be run after the user clicks the OK button.
      • Confirm that the PostEntryCommandLineArguments field specifies the command-line arguments associated with the PostEntryCommandLine application.
      • Confirm that the KeepOpen field specifies whether or not the utility stays open after the user clicks the OK or Cancel button.
        • Confirm that the fields are cleared after OK or Cancel.
        • Confirm that the output file is updated only after OK.

  3. Miscellaneous

    • Confirm that multiple results can be written to the DataFileName file.
    • Add field names and optional validation for Field8, Field9 and Field10.
    • Confirm that appropriate output is seen in the CSV file.
    • Confirm that any additional fields defined in the config file are ignored.
