SQL Compact Database Utility Applications
Testing Checklist

  1. SQLCDBEditor

    • Command-line options
      • Confirm that sqlcdbeditor.exe <DatabaseFile> opens the specified database file.
      • Confirm that with no database file defined, sqlcdbeditor.exe brings up the application with no database file open.
      • Confirm that sqlcdbeditor.exe /? brings up a usage message.

      • [DotNetRC #437] Confirm that SQLCDBEditor is automatically associated with *.sdf files.

    • Toolbar and Menus
      • Confirm that an Open Database button is provided for opening a different database.
      • Confirm that a Save Database button is provided for saving database changes.
      • Confirm that a Create New Query button is provided.

      • Confirm that File - Open displays an Open dialog.
      • Confirm that the Open dialog defaults to choosing Database files (*.sdf).
      • Confirm that the Open dialog also allows All files (*.*).
      • Confirm that the selected file is opened.
      • Confirm that the user is prompted to save changes (if necessary) before the Open dialog is displayed.
      • Confirm that File - Save and the associated toolbar button are both disabled if no changes have been made to the database.
      • Confirm that a Database was saved successfully message is displayed after saving changes.
      • Confirm that File - Close closes the open database.
      • Confirm that a confirmation message is presented if unsaved changes have been made.
      • Confirm that the application stays open if Cancel is selected.
      • Confirm that File - Exit closes the open database and exits the application.
      • Confirm that a confirmation message is presented if unsaved changes have been made.

      • Confirm that Tools - Update to current schema... is disabled when connected to a database with the current schema version.
      • Confirm that it is enabled when connected to a database with an out-of-date schema.
      • Confirm that Update to current schema works as expected.
      • Confirm that Tools - Create new query creates a numbered New Query tab.

      • Confirm that Help - About displays the About SQLCDBEditor dialog.
      • Confirm that the dialog includes the correct version number and copyright date.

    • Windows
      • Tables
        • Open an existing SQL Compact Database file and confirm that the Tables window includes one entry for each defined table.
        • Confirm that selecting a Tables entry displays the contents of the table in the Primary tab.
        • Confirm that the context menu contains the following
          • Open in separate tab
          • Copy to new query
        • Confirm that the context menu entries have the same functionality as the corresponding toolbar buttons.
        • [DotNetRC #821] Confirm that the Table list items are visible.
      • Queries
        • Open an existing SQL Compact Database file and confirm that the Queries window includes one entry for each .sqlce file found in the same folder.
        • Confirm that selecting a Queries entry displays the contents of the query in the Primary tab.
        • Confirm that the Context menu contains the following
          • Open in separate tab
          • Copy to new query
          • Rename query
          • Delete
        • Confirm that the context menu entries have the same functionality as the corresponding toolbar buttons.
        • Confirm that Delete removes the entry from the Queries window and also deletes the .sqlce file.
        • [DotNetRC #821] Confirm that the Query list items are visible.
      • Primary tab - Table
        • Open a Tables entry into the Primary tab.
        • Confirm that the Primary tab includes a yellow star icon before the Table name.
        • Confirm that selecting a different table replaces the existing contents of the Primary tab.
        • Confirm that the Primary tab includes toolbar buttons.
          • Send to separate tab
          • Copy to new query

        • Confirm that clicking the Send to separate tab button moves the results to a different tab and clears the Primary tab.
        • Confirm that clicking the Copy to new query button adds a new numbered entry to the Queries window.
          • Confirm that a .sqlce file is created and saved in the same folder as the original .SDF file.
        • [DotNetRC #829] Table Re-Ordering
          • Sort a table by the second column (ascending).
          • Add a value to the first column of the new row.
          • Click in the second column.
          • Confirm that the new row is not re-ordered until the user navigates out of the row.
      • Primary tab - Query
        • Open a Queries entry into the Primary tab.
        • Confirm that the Primary tab includes a yellow star icon before the Query name.
        • Confirm that selecting a different query replaces the existing contents of the Primary tab.
        • [DotNetRC #823] Confirm that editing query text retains focus in the edit box.
        • Confirm that the Primary tab includes toolbar buttons.
          • Hide query or Show query
          • Send to separate tab
          • Copy to new query
          • Save
          • Rename
          • Execute

        • Confirm that clicking the Show query button opens a multi-line edit box containing the formatted SQL text.
          • Confirm that the button label changes to Hide query.
          • Confirm that the query results, if any, remain displayed.
        • Confirm that clicking the Hide query button closes the multi-line edit box containing the formatted SQL text.
          • Confirm that the button label changes to Show query.
          • Confirm that the query results, if any, remain displayed.
        • Confirm that clicking the Send to separate tab button moves the results to a different tab and clears the Primary tab.
        • Confirm that clicking the Copy to new query button adds a new numbered entry to the Queries window.
          • Confirm that a .sqlce file is created and saved in the same folder as the original .SDF file after a query is saved.
        • [DotNetRC #824, #832] Re-evaluating Queries
          • Open and edit a query such that the query is not valid.
          • Click on another table or query and then back to the editing query.
          • Confirm that an error is not displayed as a result of re-evaluating the invalid query.
          • Also confirm that if a query fails to evaluate when first opened (whether it is because the query is bad, or the DB schema is no longer in sync with the query), the tab for the query will now open so that a user can correct it.
          • Modify the field order of a query and confirm that the results are updated properly after executing the query. Test this situation several times.
        • [DotNetRC #830] Confirm that a row cannot be deleted from the results grid of a query.
      • Tab Window
        • Confirm that tabs can be reordered via drag-and-drop.
        • Confirm that tabs can be docked side by side.
        • Confirm that tabs can be docked up and down.
        • Confirm that more than two tabs can be redocked.
        • Confirm that the Save and Execute buttons are disabled if the loaded query has not been modified.
        • Confirm that the tab label for a modified query includes a trailing *.
        • Confirm that the Tab Window includes left-pointing and right-pointing triangle buttons allowing scrolling between tabs when many tabs are open.
        • Confirm that Ctrl+Tab navigates between tabs.
        • Confirm that the Tab Window includes an X button allowing the active tab to be closed.

      • [DotNetRC #819] Concurrency Error
        • Open a database with tables such that a value in table A is a foreign key in table B.
        • Modify a foreign key in table B such that it is not valid.
        • In table A, change the value that had been present before the edit in table B.
        • In table B, change the invalid value to match the new value of the foreign key.
        • Confirm that the database changes can be saved.
      • [DotNetRC #825, #833] Post-Update Actions

          • Name.sdf.vbs
      • [DotNetRC #826] Validation Error

        • Enter a new tag where the TagName matches an existing entry.

      • [DotNetRC #827] Inconsistent Behavior
        • Open an HNL OrderMapping database.
        • Open the LabOrder and LabOrderTest tables side by side.
        • Modify the STALB Code in the LabOrder table.

        • Revert the OrderCode for one of the STALB items in LabOrderTest.
        • Confirm that an error icon is displayed.
        • Revert the STALB Code in the LabOrder table.
        • Confirm that the two LabOrderTest items change immediately.
        • Modify the HPVSB Code in the LabOrder table.
        • Confirm that the one LabOrderTest item changes immediately.
        • Modify the OrderCode for the HPVSB item in LabOrderTest.
        • Confirm that an error icon is displayed.
        • Adjust the HPVSB Code in the LabOrder table to match the changed value in the LabOrderTest table.
        • Confirm that the error icon goes away.
      • [DotNetRC #828] Invalid Data State
        • Open a table with at least one foreign key.
        • Sort by one of the columns.
        • In the last row, change the foreign key value to an invalid value. Confirm that the row has been marked invalid.

        • Tab out of the row to cause the row to be sorted.
        • Confirm that after the sort has completed the error icons are still properly displayed on the invalid rows.
        • Delete the two invalid rows.
        • Confirm that the table as a whole is no longer marked invalid.

      • [FlexIDSCore #4139] Editing tables with no unique key.

        • Confirm that a table with no unique key cannot be edited.

  2. SQLCompactExporter

    • Database Creation
      • Make a working copy of the ESFAMService.sdf database.
      • Add entries to the FPSFile table.

    • Options
      • Confirm that DatabaseFile is a required parameter.
      • Confirm that Query is a required parameter.
      • Confirm that OutputFileName is a required parameter.
      • Confirm that /rd <RowDelimiter> is an optional parameter for specifying a delimiter between rows in the result set.
      • Confirm that the default is \r\n.
      • Confirm that /cd <ColumnDelimiter> is an optional parameter for specifying a delimiter between each field in the result set.
      • Confirm that the default is \t.
      • Confirm that /fp <FilePrefix> is an optional parameter for specifying text to be written at the beginning of the output file.
      • Confirm that the default is for no text to be provided.
        • /fp "ALT+0176\r\n" can be used to provide a degree symbol on a separate line at the beginning of the output file.
      • Confirm that /fs <FileSuffix> is an optional parameter for specifying text to be written at the end of the output file.
      • Confirm that the default is for no text to be provided.
        • /fs "\r\nALT+0176" can be used to provide a degree symbol on a separate line at the end of the output file.
      • Confirm that /enc <CodePage> is an optional parameter for specifying the code page to use to encode the output.
        • Use the /fp example given above to exercise various code pages.
        • Confirm that no /enc option provides a degree symbol.
        • Confirm that /enc Windows-1252 provides a degree symbol.
        • Confirm that /enc 437 provides a slashed zero character.
        • Confirm that /enc utf-8 provides a degree symbol.
        • Confirm that /enc unicode provides a degree symbol. Also confirm that the file size is approximately twice the size of an earlier example.
      • Confirm that the default is ansi.
