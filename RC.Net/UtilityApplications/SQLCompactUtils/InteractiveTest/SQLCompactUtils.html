SQL Compact Database Utility Applications
Testing Checklist

  1. SQLCDBEditor

    • Command-line options
      • Confirm that sqlcdbeditor.exe <DatabaseFile> opens the specified database file.
      • Confirm that with no database file defined, sqlcdbeditor.exe brings up the application with no database file open.
      • Confirm that sqlcdbeditor.exe /? brings up a usage message.

    • Toolbar and Menus
      • Confirm that an Open Database button is provided for opening a different database.
      • Confirm that a Save Database button is provided for saving database changes.

      • Confirm that File - Open displays an Open dialog.
      • Confirm that the Open dialog defaults to choosing Database files (*.sdf).
      • Confirm that the Open dialog also allows All files (*.*).
      • Confirm that the selected file is opened.
      • Confirm that the user is prompted to save changes (if necessary) before the Open dialog is displayed.
      • Confirm that File - Save and the associated toolbar button are both disabled if no changes have been made to the database.
      • Confirm that a Database was saved successfully message is displayed after saving changes.
      • Confirm that File - Close closes the open database.
      • Confirm that a confirmation message is presented if unsaved changes have been made.
      • Confirm that the application stays open if Cancel is selected.
      • Confirm that File - Exit closes the open database and exits the application.
      • Confirm that a confirmation message is presented if unsaved changes have been made.

      • Confirm that Tools - Update to current schema... is disabled when connected to a database with the current schema version.
      • Confirm that it is enabled when connected to a database with an out-of-date schema.
      • Confirm that Update to current schema works as expected.

      • Confirm that Help - About displays the About SQLCDBEditor dialog.
      • Confirm that the dialog includes the correct version number and copyright date.

    • Miscellaneous
      • [DotNetRC #400] Confirm that the application retains focus after closing the Database save message.
      • [DotNetRC #401] Active edits are saved.
        • Edit a row without moving focus from the row.
        • Confirm that the pencil icon is still visible in the row selector.
        • Save the database.
        • Confirm that the active changes are saved.
      • [DotNetRC #409] Confirm that the application does not close if after prompting to save changes after attempting to close the editor, the save operation fails.
      • [FlexIDSCore #4139] Editing tables with no unique key.
        • Open a database that contains a table that has no unique key. One example would be the Units table of the LabDE order mapping database.
        • Confirm that a table with no unique key cannot be edited.

  2. SQLCompactExporter

    • Database Creation
      • Make a working copy of the ESFAMService.sdf database.
      • Add entries to the FPSFile table.

    • Options
      • Confirm that DatabaseFile is a required parameter.
      • Confirm that Query is a required parameter.
      • Confirm that OutputFileName is a required parameter.
      • Confirm that /rd <RowDelimiter> is an optional parameter for specifying a delimiter between rows in the result set.
      • Confirm that the default is \r\n.
      • Confirm that /cd <ColumnDelimiter> is an optional parameter for specifying a delimiter between each field in the result set.
      • Confirm that the default is \t.
      • Confirm that /fp <FilePrefix> is an optional parameter for specifying text to be written at the beginning of the output file.
      • Confirm that the default is for no text to be provided.
        • /fp "ALT+0176\r\n" can be used to provide a degree symbol on a separate line at the beginning of the output file.
      • Confirm that /fs <FileSuffix> is an optional parameter for specifying text to be written at the end of the output file.
      • Confirm that the default is for no text to be provided.
        • /fs "\r\nALT+0176" can be used to provide a degree symbol on a separate line at the end of the output file.
      • Confirm that /enc <CodePage> is an optional parameter for specifying the code page to use to encode the output.
        • Use the /fp example given above to exercise various code pages.
        • Confirm that no /enc option provides a degree symbol.
        • Confirm that /enc Windows-1252 provides a degree symbol.
        • Confirm that /enc ansi provides a degree symbol.
        • Confirm that /enc 437 provides a slashed zero character.
        • Confirm that /enc utf-8 provides a degree symbol.
        • Confirm that /enc unicode provides a degree symbol. Also confirm that the file size is approximately twice the size of an earlier example.
