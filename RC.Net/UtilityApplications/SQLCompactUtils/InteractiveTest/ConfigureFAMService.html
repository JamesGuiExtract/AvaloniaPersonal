Configure FAM Service
Testing Checklist

  • Start menu

    • Confirm that a Configure FAM Service shortcut is available under the Extract Systems FLEX Index start menu.
    • Confirm that the shortcut opens the ESFAMService.sdf database.
    • Confirm that a Configure FAM Service shortcut is available under the Extract Systems ID Shield start menu.
    • Confirm that the shortcut opens the ESFAMService.sdf database.

    • Confirm that the SQLCDBEditor recognizes the database schema version and the status bar correctly indicates the following situations:
      • current schema
      • schema needs to be updated
      • schema unrecognized
      • schema is newer than current
    • Connect to a database with an out-of-date schema version and confirm that you are prompted to update the schema.
      • Select No and confirm that you can still edit the database normally.
      • Select Yes and confirm that the database is updated correctly.

  • FPSFile table

    • Confirm that the table contains an ID column.
      • Confirm that each value in the ID column is unique.
      • Confirm that this column is auto-populated when a new row is added to the table.
      • Confirm that the column is read-only.
    • Confirm that the table contains a FileName column.
      • This column specified which FPS files will be run by the FAM service; confirm that it works as expected.
      • Confirm that the service correctly interprets paths enclosed in quotation marks.
      • Confirm that this column defaults to empty when a new row is added to the table.
    • Confirm that the table contains a NumberOfInstances column.
      • This column specifies the number of instances of each FPS file the FAM service will run; confirm that it works as expected.
      • Confirm that the column does not allow null values.
      • Confirm that this column defaults to 1 when a new row is added to the table.
    • Confirm that the table contains a NumberOfFilesToProcess column.
      • This column specifies the number of files to process before respawning.
      • Confirm that a value of 0 means that FAMProcess.exe will not respawn.
      • Confirm that a value of -1 means that the default value from the Settings table will be used.
      • Confirm that the setting works as expected.
      • Confirm that this column defaults to -1 when a new row is added to the table.

  • Settings table

    • Confirm that the table contains settings for SleepTimeOnStart, DependentServices, NumberOfFilesToProcessPerFAMInstance, ServiceDBSchemaVersion, and DatabaseSchemaManager.
    • Confirm that the default value for SleepTimeOnStart is 120000.
    • Confirm that the default value for DependentServices is empty.
    • Confirm that the default value for NumberOfFilesToProcess is 0.

    • [DotNet #425] Checking for DBNull
      • Add some dependent services and save the changes.
      • Exit the application.
      • Edit the database again and clear the dependent services.
      • Save the changes and exit the application.
      • Confirm that the service will run without logging an ELI29151 exception about a DBNull value.

  • Miscellaneous
    • [DotNetRC #442] Database pointing to nonexistent FPS file
      • Configure the ESFAMService database to point to a FPS file that does not exist.
      • Start the service and confirm that you don't see access violation errors from FAMProcess.exe.
    • [DotNetRC #511] Sorting by filename with no filename specified
      • Open a service database with an empty FPSFile table.
      • In the FPSFile table, click in the NumberOfFilesToProcess cell to add a new row.
      • Click in the FileName column header to sort the table by filename.
      • Confirm that no exceptions are seen.
    • [DotNetRC #810] Edit temporary file
      • Configure the ESFAMService database to point to a valid FPS file.
      • Save the edits but do not close the SDF file.
      • Start the service and confirm that the FAM Service can process files while SQLCDBEditor edits the ESFAMService database.
